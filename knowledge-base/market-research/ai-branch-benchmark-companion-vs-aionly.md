# [Round 4.1 Step 1] AI 갈래 리서치 — Companion AI vs AI-Only 관광 서비스 벤치마크

> 목적: 관광 앱의 AI 활용 방식을 **Companion AI**와 **AI-Only** 두 갈래로 분류하고, 각 모델의 대표 서비스·강약점·한국 관광 공모전 적합성을 벤치마크.

---

## 1. 두 갈래 정의

| 구분 | 정의 | 핵심 가치 |
|---|---|---|
| **Companion AI** | AI가 인간 여행자의 의사결정을 보조. 사람이 최종 선택권 유지 | 편의성 + 신뢰(인간 판단 보완) |
| **AI-Only** | AI(또는 로봇)가 가이드·추천·예약 등 서비스 전 과정을 자동 처리. 인간 개입 최소/없음 | 24/7 운영 + 비용 절감 + 일관성 |

| 서비스 | 국가 | 유형 | 핵심 기능 | 출시 | 차별점 | 한계 |
|---|---|---|---|---|---|---|
| **MIZKAN MUSEUM 로봇가이드** | 일본 | 로봇 가이드 투어 | 인적 개입 없이 종일 자동 운행 로봇 투어 | 2025.09 | 일본 최초 완전 무인 가이드 투어 | 고정 장소·루트, 확장성 제한 |
| **OriHime 일본교 가이드투어** | 일본 | 원격 조종 로봇 (분신 로봇) | 어깨 위 소형 로봇 + 원격 파일럿 실시간 통신 | 2025.09 | D&I(장애인 파일럿) + 비대면 가이드 | 로봇 = AI보다 텔레프레전스 (AI-only 아님) |
| **TravelMorph** | 글로벌 | AI 드리븐 여행가이드 앱 | 한국→일본 여행 특화 AI 가이드 (다국어) | 2025.12 | 한국인 대상 일본 AI 가이드 특화 | 방향 역전(한국→해외), 국내 관광 부적합 |
| **Destinations AI** | 미국 | AI 추천 엔진 | 관심사·예산·기간 입력→자동 여행 플랜 생성 | 2025 | 완전 자동화 여행 계획 | 현지 실시간 연동 약함 |

> **참고**: 순수 AI-Only 관광 서비스는 **로봇 가이드** 형태(일본 중심)와 **완전 자동화 챗봇** 형태로 양분. 로봇 투어는 고비용·고정 인프라가 필요해 앱 기반 공모전 서비스에는 부적합.

## 4. 두 갈래 비교 벤치마크

| 평가 기준 | Companion AI | AI-Only |
|---|---|---|
| **사용자 신뢰도** | 높음 (인간 통제권 유지) | 낮음~중간 (AI 오류·할루시네이션 불신) |
| **개인화 깊이** | 중~고 (대화·질문으로 정제) | 중 (입력 파라미터 기반, 피드백 루프 제한) |
| **24/7 운영** | 가능 (AI 백엔드 + 유저 주도) | 가능 (완전 자동) |
| **감성·공감** | 높음 (AI가 감성 언어 사용) | 낮음 (트랜잭션 중심) |
| **운영비용** | 낮음 (AI 인프라만) | 중~고 (로봇: 매우 높음) |
| **외국인 대응** | 높음 (다국어 LLM) | 중 (사전 세팅된 언어만) |
| **한국 관광 API 연계** | 용이 (챗봇 + API 응답 통합) | 가능하나 UX 연계 복잡 |
| **20-40대 수용성** | 높음 (ChatGPT 사용 경험 80%+) | 낮음 (신기하지만 불안) |
| **공모전 구현 난이도** | 낮음~중간 | 중~높음 (로봇 불가, 앱 자동화만) |
| **경쟁 앱 포화도** | 높음 (글로벌 앱 다수) | 낮음 (국내 사례 거의 없음) |
| **주요 수익 모델** | Freemium + 예약 커미션 + 구독 | 라이선스 / B2B SaaS (관광지 위탁) |

## 5. 한국 관광 공모전 관점 — 갈래 선택 시사점

### 5-1. Companion AI 선택 시
**강점**:
- Visit Korea OpenAPI 27개 API를 챗봇 응답에 직접 연동 가능
- 20-40대 한국인 + 외국인 모두 ChatGPT류 인터페이스에 친숙 (Accenture: 80% 생성AI 여행 활용)
- 기존 앱(다님, 인천e지)의 공백: **로컬 특화 + 감성 큐레이션** 미흡 → 차별화 가능
- 구현 난이도 낮음, 빠른 MVP 가능

**핵심 차별화 방향**: "AI가 단순 추천이 아닌 *여행 감정·맥락*을 이해하고 동반하는 동반자"

### 5-2. AI-Only 선택 시
**강점**:
- 국내 경쟁 앱 거의 없음 → 레드오션 회피
- 24시간 운영, 인건비 0, 확장성 높음
- 일본 MIZKAN Museum 사례: 지속 가능 운영 모델 입증

**리스크**:
- 순수 AI-Only는 사용자 신뢰 구축이 관건
- 앱 기반 공모전에서 로봇 서비스 구현 불가 → **완전 자동화 챗봇·에이전트** 형태만 가능
- 한국 관광지 데이터 품질에 AI 정확성 의존 → 오류 시 신뢰 붕괴

## 6. 결론 및 추천

| 항목 | 추천 |
|---|---|
| **1차 갈래** | **Companion AI** — 신뢰도·구현성·사용자 수용성 모두 우위 |
| **2차 갈래** | AI-Only 요소를 *백오피스 자동화*에 활용 (e.g., 실시간 혼잡도·API 집계는 AI-Only 처리, 프론트엔드는 Companion 방식) |
| **포지셔닝** | "AI 동반자 (AI Companion)" — 여행 전·중·후 전 단계에서 사용자 옆에서 보조, 최종 결정은 사용자 |
| **핵심 Gap** | 글로벌 앱은 언어 일반화 → 한국 로컬 특화 콘텐츠 부재. 국내 앱은 UI/UX 구식. **한국 전통·지역 감성 + 최신 AI UX** 조합이 공백 |

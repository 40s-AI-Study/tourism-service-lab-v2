# SIM-157: PhotoHunt Korea × 이재호 (전동휠체어 개발자)

## 시나리오

재호는 부산 1박 2일 여행을 파트너와 계획 중이다. 개발자 커뮤니티 슬랙에서 PhotoHunt Korea가 언급됐고, "공식 DB 기반 포토스팟 큐레이션"이라는 설명에 관심을 가졌다. 재호는 전동휠체어 사용자로서 포토스팟을 발견해도 "실제로 내가 접근 가능한가"가 항상 첫 번째 질문이다. 앱을 설치하고 부산 감천문화마을을 검색했지만, 예상했던 접근성 정보는 보이지 않았다.

---

## 사용자 여정

### 1단계: 진입
- 개발자 슬랙 채널 링크 → 설치
- 앱 실행: 개발자답게 기능 구조 빠르게 파악 (탭 구성, 데이터 출처 확인)
- 부산 검색 → 감천문화마을·해운대·흰여울문화마을 포토스팟 지도 표시

### 2단계: 행동
- 감천문화마을 선택 → 공모전 수상작 사진: 계단 많은 골목길 구도 — "저기 나는 못 올라가는데"
- 접근성 필터 검색 → 해당 기능 없음 확인
- `visitor-concentration-forecast`: 감천 토요일 오전 9시 방문자 32% — 혼잡도 정보 자체는 유용
- 해운대 해변 포토스팟: 평지 위주 스팟 — 접근 가능성 높음 추측, 실제 확인 불가
- API 구조 분석 모드: "tourism-photos 데이터에 barrier-free 필드 없나" 스스로 추측
- 흰여울문화마을: 계단 경사 사진 보고 스스로 패스 결정

### 3단계: 결과
- 혼잡도 정보로 해운대 방문 시간 최적화 — 이 기능만 유용하게 활용
- 포토스팟 대부분이 접근성 미정보라 불안한 상태로 현장 방문 → 절반은 진입 불가
- 앱 유지하나 낮은 신뢰도 — "좋은 스팟 큐레이션인데 나한테는 절반만 작동"
- 개발자 관점 피드백: "barrier-free-travel API 연동 PR 올리고 싶다"

| 구현 타당성 | **7** | 핵심 기능 구현 타당, barrier-free API 연동 추가로 빠른 개선 가능 |

## Pass/Fail 판정

> **⚠️ CONDITIONAL PASS**

이재호는 PhotoHunt Korea의 잠재적 타깃(사진 취미 여행자)이지만 전동휠체어 사용자라는 맥락이 앱 활용도를 절반으로 제한했다. 혼잡도 예측 기능은 실질적 가치를 제공했으나, 접근성 정보 부재로 현장에서 반복적인 진입 불가 경험이 발생했다. `barrier-free-travel` API 연동으로 포토스팟별 접근성 정보를 추가하면 장애인 여행자 시장까지 포괄하는 PASS 전환이 가능하다.

## 개선 제안

1. 포토스팟 접근성 정보 레이어 추가: `barrier-free-travel` API 연동으로 각 포토스팟의 경사도·계단 여부·휠체어 접근 가능 거리를 지도 위에 아이콘으로 표시
2. 접근 가능 포토스팟 전용 필터: "휠체어 접근 가능 스팟만 보기" 필터 옵션 추가로 장애인 여행자가 사전 계획 단계에서 안심하고 탐색 가능하도록 지원

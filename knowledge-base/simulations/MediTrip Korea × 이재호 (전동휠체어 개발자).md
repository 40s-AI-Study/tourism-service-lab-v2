# SIM-117: MediTrip Korea × 이재호 (전동휠체어 개발자)

## 시나리오

재호는 앱스토어에서 새로운 한국 관광 앱을 탐색하다 MediTrip Korea를 발견한다. "의료관광"이라는 단어에 척수 장애인으로서 순간 관심이 생긴다 — 혹시 장애인 관련 의료·재활 시설 정보가 있을까 하는 기대였다. 앱을 설치하고 살펴보니 외국인 의료관광객을 위한 성형·피부·치과·검진 중심의 서비스임을 파악한다. 재활 의학이나 장애인 관련 의료 정보는 없다. 개발자 시각으로 API 구조를 분석하면서 `area-tourism-demand-density`와 `related-attractions` API 활용은 흥미롭게 평가하지만, 무장애 경로 데이터(`barrier-free-travel`)가 없다는 점은 큰 실망이다. 한국인 장애인 내국인으로서 비자 안내는 완전 무관하다.

---

## 사용자 여정

### 1단계: 진입
- 앱스토어에서 "의료관광" 키워드로 발견 — 장애 관련 정보 기대
- 설치 후 실행. 한국어 UI, 직관적 구성 확인
- 홈: 성형/피부과, 치과, 척추, 한방, 검진 카테고리 — 재활·장애 카테고리 없음

### 2단계: 행동
- 척추 카테고리 탭: 외국인 환자 대상 척추 클리닉 목록 — 한국인 내국인 입장에서 불필요
- "관광 코스" 탭: 강남·신촌 주변 코스 확인. 무장애 필터 없음
- `area-tourism-demand-density` 기반 혼잡 회피는 전동휠체어 이동 공간 확보 관점에서 흥미
- 그러나 경사도·단차·바닥재질 등 재호에게 필수인 무장애 세부 데이터 전무
- 비자 안내 섹션: 내국인에게 완전 무관

### 3단계: 결과
- 기술 참고용으로 1주일 보관 후 삭제
- "혼잡도 API 활용 방식은 참고할 만하나 내 여행에는 쓸 수 없음" 내부 평가
- 장애인 여행 카페 네이버로 복귀

| 항목 | 점수 (1-10) | 근거 |

| 사용 편의 | **6** | 한국어 UI 명확, 개발자로서 앱 구조 파악 용이. 내용과 맞지 않을 뿐 |
| 구현 타당성 | **7** | 외국인 의료관광객 대상으로는 잘 설계된 서비스 |

**종합 점수: 15/30**

## Pass/Fail 판정

> **⚠️ CONDITIONAL PASS**

이재호는 MediTrip Korea의 타깃 외 사용자(내국인 장애인 여행자)지만, 혼잡도 회피 기능이 전동휠체어 이동 공간 확보 측면에서 부분적 가치를 가진다. 그러나 무장애 세부 데이터 부재로 실제 여행 계획에 활용하기 어렵다. 종합 15점, CONDITIONAL PASS — 단 핵심 가치는 미전달.

## 개선 제안

1. `barrier-free-travel` API를 관광 코스 탭에 연동하면 의료관광 목적의 이동 제약 환자(수술 후 재활 중 방문자 포함)에게 즉각적인 가치를 제공할 수 있음
2. 척추·재활 관련 병원 카테고리에 입원·외래 환자를 위한 병원 내 이동 편의시설(엘리베이터, 휠체어 대여) 정보를 추가하면 의료관광 맥락 내 접근성 니즈를 충족 가능

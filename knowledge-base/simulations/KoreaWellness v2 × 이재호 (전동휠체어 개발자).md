# SIM-R3-028: KoreaWellness v2 × 이재호

## 시나리오

이재호는 38세 IT 개발자다. 전동휠체어를 사용한다. 서울 성수동 스타트업에서 백엔드를 담당하고, 코드에 관해서는 누구보다 꼼꼼하다. 그 꼼꼼함이 여행에서도 발휘된다 — 어딘가를 가기 전에 반드시 무장애 접근성 정보를 검증한다. "휠체어 이용 가능"이라고 쓰여 있어도 실제로 가보면 입구에 턱이 있거나, 엘리베이터가 공사 중이거나. 그래서 재호에게 여행 앱의 신뢰도는 단 하나로 결정된다: 데이터가 얼마나 정확한가.

자연을 보고 싶다. 나무, 물소리, 바람. 몸을 움직이기 어렵다고 해서 자연을 포기할 이유는 없다. 조용한 카페도 좋다. 하지만 계단이 없어야 한다.

개발자 커뮤니티 슬랙 채널에서 "이 앱 접근성 데이터 제법 신뢰할 만하다"는 동료의 메시지를 보고 KoreaWellness v2를 설치했다. 개발자 특성상 앱 첫 진입 시 데이터 출처와 업데이트 주기를 먼저 확인했다.

첫 화면: "지금 가장 필요한 게 뭐예요?" 재호는 카드를 훑었다. 🌿 **고요 (아무것도 안 하고 싶어)**. 그 카드를 눌렀다. 그리고 설정 아이콘을 눌러 "무장애 코스 필터"가 있는지 확인했다.

---

## 사용자 여정

### 1단계: 진입
- 개발자 커뮤니티 슬랙 → 접근성 데이터 신뢰도 언급에 반응하여 설치
- 🌿 고요 선택 → 설정에서 "무장애 접근 필터" ON, 전동휠체어 유형 선택
- 데이터 출처 페이지 확인: 한국관광공사 무장애 관광정보 API + 자체 현장 검증 배지 시스템

### 2단계: 행동
- 추천 목록에 "현장 검증 완료" 배지가 붙은 장소만 표시 — 재호가 가장 먼저 보는 필터
- 추천 1순위: 경기 양평 세미원 수생식물원 — 전동휠체어 전 구간 포장 경로, 단차 없음, 무장애 화장실 위치 3곳 핀 표시
- 상세 페이지: 주차장에서 입구까지 거리(87m), 경사도(2.1%), 노면 재질(콘크리트 포장), 마지막 현장 검증일(2026-03-14) 명시
- 인근 카페 목록: 입구 단차 0cm, 테이블 높이 74cm, 실내 회전 반경 150cm 이상 — 전동휠체어 기준으로 필터링된 데이터
- 두루누비 트레일 API 연동: 세미원 주변 휠체어 가능 트레일 구간 별도 표시
- 앱 내 교통 정보: 저상버스 정류장 위치, 장애인 콜택시 호출 연동

### 3단계: 결과
- 세미원 현장 도착. 앱 정보와 실제가 일치했다 — "드디어 믿을 수 있는 앱을 찾았다"
- 연꽃 정원 옆 벤치에서 한 시간. 물소리, 바람, 아무 생각 없이
- 인근 카페: 입구에서 직원이 먼저 문을 열어줬다. 테이블 높이도 딱 맞았다
- 귀가 후 앱에 상세 리뷰: "데이터 정확도 최고. 개발자로서 이 API 연동 어떻게 했는지 궁금할 정도."
- 장애인 여행 커뮤니티 카페에 앱 정보 공유 — "저도 써봤는데 진짜 맞아요" 댓글 12개

### 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **10** | 무장애 접근성 데이터의 정확도와 세밀함(경사도, 노면, 마지막 검증일)이 핵심 니즈를 완전히 충족 |
| 사용 편의 | **9** | 개발자 수준의 데이터 투명성 제공. 필터 설정이 직관적이고 결과가 즉각적 |
| 구현 타당성 | **7** | 현장 검증 배지 시스템 유지 비용이 높음. 데이터 업데이트 주기 관리가 핵심 과제 |

**종합 점수: 26/30**

## Pass/Fail 판정

> **✅ PASS**

무장애 여행자에게 "데이터 정확도"는 신뢰의 전부다. 현장 검증 배지, 경사도·노면 상세 정보, 마지막 검증일 표시가 개발자 페르소나의 높은 검증 기준을 통과했다. 장애인 커뮤니티로의 자연 확산은 이 앱의 가장 강력한 바이럴 채널이 될 수 있다.

## 리텐션 평가: 이 페르소나가 앱을 다시 열까?

**재사용 확률**: 🔥매우 높음

**감정적 이유**:
> "여행 전에 열두 곳을 검색해서 결국 못 가는 게 익숙했는데, 처음으로 앱이 알려준 대로 갔더니 그대로였다. 이 신뢰를 다른 데서 찾기 어렵다."

**재사용 트리거**:
- 자연을 보고 싶을 때마다 — 계절마다 달라지는 추천 확인
- 장애인 여행 커뮤니티에서 "어디 가봤어요?" 질문받을 때 앱 기반으로 추천
- 새로운 지역 여행 계획 시 접근성 검증 1순위 도구로 사용

**이탈 리스크**:
- 한 번이라도 현장과 데이터가 불일치하면 (예: 검증됐다는 경로에 공사로 단차 생김) 즉시 이탈 결정
- 현장 검증 배지의 업데이트가 6개월 이상 지연되면 신뢰도 하락

## 개선 제안

1. **사용자 참여형 실시간 검증**: 방문한 사용자가 앱 내에서 접근성 정보를 업데이트할 수 있는 "현장 리포트" 기능 — 개발자 커뮤니티에서 자발적 기여자 확보 가능
2. **전동휠체어 배터리 범위 필터**: 현재 배터리 충전량 입력 시 도달 가능한 거리 내 장소만 필터링하고, 경로 중 충전소 위치 표시

# SIM-190: KoreaDemand Atlas × 타나카 하루카 (일본 직장인)

## 시나리오

하루카는 한국 여행을 3주 앞두고 일본어 한국 여행 유튜브 채널 @koreatabi_jp에서 KoreaDemand Atlas 소개 영상을 본다. 영상에서 "데이터로 뜨기 전 명소를 찾는 앱"이라고 소개하는데, K-드라마 촬영지나 덜 알려진 포토스팟을 원하는 하루카에게 직관적으로 매력적이다. 설치 후 일본어 UI를 찾아보지만 없고, 영어로 전환해 사용한다. 전주를 검색해 히트맵을 확인하는데, 데이터 지표의 의미를 정확히 파악하기 어렵다.

---

## 사용자 여정

### 1단계: 진입
- 일본어 유튜브 채널 추천 → 앱스토어 설치 (SNS 추천에 즉각 반응하는 ENFP)
- 일본어 UI 없음 확인 → 영어로 전환해 시도 (영어 중급 수준이라 가능하나 불편)
- 히트맵 첫 화면 → "Hot", "Rising", "Hidden" 뱃지는 직관적으로 이해

### 2단계: 행동
- 전주 검색 → "Rising" 뱃지 확인 → 전주 한옥마을 탭
- 영어 수요 지표 읽기: "Tourism Diversity Index: High" → 의미 파악에 시간 소요
- 방문자 집중률 예측: 연휴 기간 78% 집중 → "붐빔 주의" 메시지 이해 (영어 단순 문구)
- K-드라마 촬영지 관련 정보 탐색 → 일반 관광 명소 정보만 있고 촬영지 연결 없음
- 코스 생성: 영어 코스 자동 생성 → 일본어 설명 없어 내용 파악에 집중력 소모

### 3단계: 결과
- 방문 전 혼잡도 예측 확인 → 연휴 피해 평일 방문 결정 (이 기능은 실질 도움)
- 전주 방문, 한옥마을 비교적 한산 → 인증샷 촬영 성공
- 앱 전반 평가: "혼잡도 확인용으로만 씀, 나머지는 @koreatabi_jp 유튜브 영상이 더 유용"
- 일본어 부재로 재사용 의향 낮음

---

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **5** | 혼잡도 예측 기능은 유용. 그러나 일본 여행자 핵심 관심사(K-drama 성지, 포토스팟, 일본어 설명)는 미지원 |
| 사용 편의 | **4** | 일본어 UI 없어 영어 대체 사용 — 중급 영어 수준으로 지표 해석에 인지 부담. 즉흥적 성향의 하루카에게 데이터 분석 앱은 맞지 않음 |
| 구현 타당성 | **7** | 일본어 지원은 `tourism-info-japanese` API로 가능 — 미구현이 아쉬운 부분 |

**종합 점수: 16/30**

## Pass/Fail 판정

> **⚠️ CONDITIONAL PASS**

종합 점수 16점, 유용성 5점으로 CONDITIONAL PASS 구간에 해당한다. 혼잡도 예측이 최소한의 실용성을 제공했으나, 일본어 UI 부재와 K-drama 성지 정보 미지원으로 하루카의 핵심 여행 동기와 앱 기능 간의 정렬이 약하다. KoreaDemand Atlas는 데이터 분석을 즐기는 사용자에게 적합하며, 감성·즉흥형 여행자인 하루카에게는 방향이 맞지 않는다.

## 개선 제안

1. `tourism-info-japanese` API를 활용한 일본어 UI 지원 추가 — 방한 외국인 1위 국적인 일본 여행자 접근성 확보는 서비스 성장에 핵심
2. 히트맵 "Rising" 지역에 "지금 뜨는 이유" 한 줄 설명 추가 (예: "K-드라마 촬영지 포함") — 감성형 여행자에게 데이터 지표를 여행 스토리로 번역

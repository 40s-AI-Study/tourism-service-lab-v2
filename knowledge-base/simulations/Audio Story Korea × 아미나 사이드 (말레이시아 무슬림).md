# SIM-69: Audio Story Korea × 아미나 사이드 (말레이시아 무슬림)

## 시나리오

아미나 사이드(30세, 말레이시아 쿠알라룸푸르 인플루언서)는 무슬림 여행자를 위한 한국 여행 콘텐츠를 제작 중이다. 인스타그램 팔로워 15만 명을 대상으로 "무슬림 프렌들리 서울 가이드"를 연재하고 있다. 경복궁 방문 중 QR 코드를 발견하고 영어를 선택했다. 오디오 가이드의 문화재 역사 해설이 자신의 팔로워들에게 소개할 "배경 지식 콘텐츠"로 적합한지 평가하며 청취했다.

---

## 사용자 여정

### 1단계: 진입
- 경복궁 향원정 인근 QR 스캔 → 영어 선택
- 앱 실행 직후 공모전 수상작 갤러리 먼저 확인 (비주얼 콘텐츠 평가 습관)
- 오디오 챕터 1 "경복궁의 물과 정원 — 풍수지리의 철학" 시작

### 2단계: 행동
- 챕터 1 청취 중 히잡 착용 상태에서 이어폰 사용 불편 → 스피커 모드 전환
- 배경 소음으로 인해 내용 일부 놓침 → 챕터 재생 버튼으로 다시 듣기
- 챕터 2 "조선 왕실의 하루" 청취 → 팔로워 콘텐츠용 메모 작성
- 공모전 수상작 중 히잡 착용 무슬림 여행자가 등장하는 사진 없음 → 대표성 부재 인식
- 오디오 내용 중 음식·생활 문화 관련 내용은 할랄 맥락에서 추가 설명 필요 느낌
- 챕터 2개 완주 후 다음 일정(점심 할랄 식당)으로 이동

### 3단계: 결과
- "역사 해설 내용은 좋았고 팔로워들에게 유용할 것 같다"
- 단, 무슬림 여행자 특화 정보 부재로 앱을 "무슬림 여행 가이드"로 소개하기는 어려움
- 인스타그램 스토리에 앱 화면 + "경복궁 역사 공부 중" 캡션으로 부분 소개
- 재사용 의향: 역사 배경 지식 수집 목적으로 긍정적, 단 전문 무슬림 여행 앱 대체 불가

| 사용 편의 | **7** | 스피커 모드 전환, 다시 듣기 기능이 이어폰 제약 상황 대응, 전반적 UX 무난 |
| 구현 타당성 | **7** | tourism-info-english API 기반 표준 구현, 무슬림 여행자 특화 기능은 별도 콘텐츠 레이어 필요 |

## 개선 제안

1. **무슬림 여행자 레이어 추가**: 각 문화재 인근 할랄 식당, 기도 공간(무살라) 정보를 "실용 정보" 탭으로 제공하여 무슬림 FIT 여행자의 원스톱 사용 가능성 강화
2. **갤러리 다양성 확대**: photo-contest-winners API 큐레이션 시 다양한 문화권·외모의 여행자가 등장하는 수상작을 포함하여 글로벌 무슬림 여행자의 대표성 인식 개선

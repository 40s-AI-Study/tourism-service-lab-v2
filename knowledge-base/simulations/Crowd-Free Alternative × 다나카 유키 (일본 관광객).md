# SIM-CFA-02: Crowd-Free Alternative × 다나카 유키 (일본 관광객)

## 시나리오

다나카 유키(田中ゆき, 27세, 도쿄 패션 브랜드 직원)는 한국 3번째 방문이다. 이번엔 "진짜 한국인만 아는 숨은 명소"를 목표로 한다. 인스타그램에서 한국 여행 계정을 팔로우하다 "CrowdFree 앱으로 북촌에서 사람 없는 골목 찾았어요" 게시물을 보고 앱을 설치했다. 일본어 지원이 있는지 확인이 우선이다.

---

## 사용자 여정

### 1단계: 진입 (앱 설치 → 온보딩)
- 일본 앱스토어에서 "クラウドフリー 韓国旅行" 검색 → Crowd-Free Alternative 발견
- 앱 실행 → 언어 설정 화면: "日本語" 선택 → UI 전환 완료
- 온보딩: "どの観光地が混雑しているか知りたいですか?" → "はい" 선택
- 관심 카테고리: "街並み/골목", "宮殿/궁궐", "カフェ" 선택
- 서울 도착 예정일 입력 (3일 후) → "여행 전 미리 계획하기" 모드 활성화

### 2단계: 혼잡도 확인
- 일본어로 "경복궁" 검색: "景福宮" 입력 → 자동 인식 및 한국 관광지 매칭
- 방문 예정일(토요일) 혼잡도: 🔴 매우 혼잡 (76%)
- 하단 그래프: 평일 수요일 오전이 가장 낮음 (혼잡도 22%)
- "이 관광지를 방문한 일본인 여행자 72%가 혼잡으로 불만족" 통계 카드 표시
- 유키: "やっぱり混んでるか… 대안 찾아보자"

### 3단계: 대안 탐색
- "景福宮の代わりに" 탭 → 일본어 대안 명소 카드 표시
  1. **창경궁 (昌慶宮)** — 🟢 혼잡도 19%, 경복궁에서 1.5km, 야간 특별 개장 있음
  2. **경희궁 (慶熙宮)** — 🟢 혼잡도 8%, 현지인도 잘 모르는 궁궐, 사진 명소
  3. **덕수궁 돌담길 (德寿宮石垣通り)** — 🟡 혼잡도 41%, 드라마 촬영지 분위기
- 경희궁 카드: "일본인 여행자 추천 포인트: 관광객 거의 없어 조용한 사진 촬영 가능"
- 유키: "경희궁은 몰랐어! 사진도 잘 나올 것 같고…"

### 4단계: 결정 & 계획
- 경희궁 선택 → "日本語ガイド対応" 뱃지 확인 → 안심하고 선택
- 주변 맛집: "경희궁 근처 현지인 카페 3곳" (영업시간, 웨이팅 예측 포함)
- 3일 코스 자동 생성: "혼잡도 낮은 순으로 3일 일정 짜드릴까요?" → "はい" → 코스 생성
  - Day 1: 경희궁(오전) → 서촌 카페거리(오후, 혼잡도 31%)
  - Day 2: 북촌 조용한 골목(오전, 혼잡도 14%) → 창경궁 야간(저녁)
  - Day 3: 서울 숲(오전) → 성수동 카페(오후)
- LINE으로 친구에게 공유: "これ使ったら穴場見つかった✨ 경희궁激おすすめ"

### 5단계: 현장 + 피드백
- 경희궁 방문 당일: 앱 알림 "慶熙宮 현재 혼잡도 9% — 거의 비어 있습니다"
- 실제 방문: 관광객 10명 미만, 넓은 정원에서 한복 대여 사진 촬영 → SNS 퀄리티 완성
- 방문 후 피드백: "混んでなかった！" 최저 혼잡 선택 → +50 포인트
- 인스타그램: "韓国旅行で穴場発見🏯 경희궁 最高すぎる！#CrowdFreeKorea #숨은명소"

## Pass/Fail 판정

> **✅ PASS**

다나카 유키는 혼잡 회피 + 숨은 명소 발견이라는 이중 니즈를 가진 고가치 타겟이다. 일본어 지원과 한자 명칭 자동 매칭이 핵심 UX 요소로 작동한다. 인스타그램·LINE 공유로 일본인 관광객 커뮤니티 내 바이럴 잠재력이 매우 높다. 다만 일본어 UI 품질과 일본 앱스토어 등록이 전제 조건이므로 개발 우선순위에 반영 필요.

## 개선 제안

1. **일본어 번역 품질**: tourism-info-japanese API의 관광지 설명 텍스트가 딱딱한 공식 번역인 경우 자연스러운 여행 문체로 리라이팅 필요 (AI 후처리 권장)
2. **일본인 특화 필터**: "K-드라마 촬영지", "한복 대여 가능", "일본어 안내판 있음" 필터 추가 시 유키 같은 타겟에게 더 강력한 차별화
3. **오프라인 저장**: 해외 로밍 데이터 절약을 위해 방문 예정 지역 혼잡도 데이터 사전 다운로드 기능 필요

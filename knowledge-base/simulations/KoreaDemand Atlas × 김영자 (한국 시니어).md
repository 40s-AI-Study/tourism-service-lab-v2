# SIM-186: KoreaDemand Atlas × 김영자 (한국 시니어)

## 시나리오

영자의 딸이 "엄마, 이번 불국사 여행 전에 이 앱으로 언제 덜 붐비는지 확인해봐요"라며 KoreaDemand Atlas를 설치해준다. 딸이 경주를 검색해 방문자 집중률 그래프를 보여주며 "이번 주 화요일 오전이 제일 한산하대요"라고 설명해준다. 영자는 그래프가 무슨 뜻인지 완전히 이해하지 못하지만, 딸의 말을 믿고 화요일 오전 출발을 결정한다. 앱 자체를 스스로 조작하기는 어렵지만, 딸이 옆에서 핵심 정보를 짚어주는 방식으로 간접적으로 서비스 가치를 경험한다.

---

## 사용자 여정

### 1단계: 진입
- 딸이 대리 설치 및 초기 설정 완료
- 영자 본인은 앱을 직접 여는 것조차 어려움 — 홈 화면에서 아이콘 찾기도 도움 필요
- 딸이 경주 지역 선택 → 방문자 집중률 예측 화면을 보여주며 구두 설명

### 2단계: 행동
- 딸이 화요일 오전 9-11시 혼잡도 낮음 확인 → 출발 시각 결정
- 영자는 화면을 보며 "많이 복잡하네"라고 코멘트 — 히트맵·지수 UI가 시니어에게 과도하게 복잡
- 딸이 불국사 명소 정보 페이지 열어 설명 → 영자는 텍스트 읽기에 어려움(글씨 크기 작음)
- 무장애 정보, 경사로 안내는 앱 내에 없음 → 딸이 별도로 전화 확인

### 3단계: 결과
- 화요일 오전 방문 → 예측대로 비교적 한산, 불국사 여유롭게 관람
- 영자 본인은 앱을 사용했다기보다 딸이 앱을 사용해 얻은 정보의 수혜자로 경험
- 영자 직접 재사용 가능성 거의 없음 — 딸 동반 없이는 앱 가치 접근 불가

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **4** | 방문 타이밍 정보는 유용하나, 시니어가 직접 접근 불가 — 딸이라는 중간자 없이는 가치 전달 불가 |
| 사용 편의 | **2** | 히트맵·수요 지수 UI는 디지털 리터러시 낮은 시니어에게 진입 장벽이 극도로 높음. 큰 글씨, 단순 UI 전혀 미지원 |
| 구현 타당성 | **7** | 플랫폼 구현은 가능하나 시니어 접근성은 설계 범위 외 |

**종합 점수: 13/30**

---

## Pass/Fail 판정

> **❌ FAIL**

종합 점수 13점, 유용성 4점으로 FAIL 기준(종합 14 이하)에 해당한다. KoreaDemand Atlas는 데이터 리터러시가 높은 파워 여행자를 위해 설계된 앱으로, 스마트폰 기초 수준의 시니어 김영자에게는 직접적 가치 전달이 불가능하다. 방문 타이밍 정보 자체는 유용하지만, 본인이 앱을 사용할 수 없다는 근본적 접근성 문제가 모든 가치를 차단한다.

## 개선 제안

1. "가족 공유 모드" 도입 — 자녀가 부모 일정을 대리 설정하고, 부모 폰에는 최종 결과(날짜·장소·시간)만 큰 글씨 카드 형태로 표시
2. 시니어 특화 진입점이 아니더라도, 접근성 최소 기준(글씨 크기 조절, 고대비 모드)은 앱 전반에 적용 권장

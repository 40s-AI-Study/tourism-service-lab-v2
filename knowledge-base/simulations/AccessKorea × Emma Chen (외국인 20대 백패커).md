# SIM-14: AccessKorea × Emma Chen (외국인 20대 백패커)

## 시나리오

Emma Chen(23세, 대만 여행 크리에이터)은 이번 여행에 할머니(74세, 걷기 불편)를 대만에서 데려오게 됐다. 할머니가 한국 여행을 꿈꿔왔다고 해서 함께 오게 됐는데, 한국어를 모르는 상태에서 무장애 정보를 얻어야 한다. 구글 검색에서 AccessKorea를 찾아 설치한다.

---

## 사용자 여정

### 1단계: 진입
- 앱 실행 → 언어 선택 화면 없음, 한국어 전용 UI
- 구글 번역기로 앱 전체 번역 시도 → 4분 소요
- 온보딩 텍스트 번역 오류로 "지팡이 사용자" 대신 엉뚱한 옵션 선택

### 2단계: 행동
- "전주" 검색 → 무장애 관광지 목록이 한국어로 표시
- 장소별 접근성 정보(계단 유무, 경사로, 화장실)는 있으나 전부 한국어
- 번역기로 하나씩 번역 → 할머니에게 설명하려 하나 정확성 불확실
- 오디오 가이드: 한국어 전용 → 할머니도 이해 불가
- 포기하고 구글 맵으로 복귀

### 3단계: 결과
- 앱의 접근성 정보는 분명히 가치 있지만 언어 장벽으로 완전히 활용 불가
- 할머니와 함께 무장애 정보 없이 전주 한옥마을 방문 → 몇몇 구간에서 계단 만나 난감
- "이 앱 중국어 지원됐으면 정말 좋았을 텐데"라는 리뷰를 Dcard에 남김

---

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **4** | 접근성 데이터 자체는 P4+할머니에게 정확히 필요한 정보지만 언어 장벽으로 접근 불가 |
| 사용 편의 | **2** | 한국어 전용 UI는 외국인에게 심각한 진입 장벽. 번역기 병행으로도 사용 어려움 |
| 구현 타당성 | **8** | 다국어 지원 추가는 기술적으로 단순 i18n 작업 |

**종합 점수: 14/30**

---

## Pass/Fail 판정

> **❌ FAIL**

AccessKorea의 무장애 정보는 P4가 할머니를 동반한 이번 상황에서 정확히 필요한 서비스다. 그러나 한국어 전용 UI가 결정적 장벽이다. 다국어 지원만 추가되면 즉시 PASS로 전환 가능한 케이스.

---

## 개선 제안

1. **다국어 지원 (최우선)**: 영어·중국어(번체/간체) UI 및 접근성 정보 번역 → 외국인 방문 시니어 동반 여행자 세그먼트 확보
2. **자동 언어 감지**: 기기 언어 기반 자동 전환으로 외국인 마찰 제거
3. **외국인 무장애 여행 마케팅**: 한국 방문 외국인 시니어 동반 여행자(일본·중국·동남아 가족 여행)는 신흥 고성장 세그먼트 — 다국어 지원 시 블루오션
4. **오디오 가이드 다국어 확장**: 무장애 관광지에서 중국어·영어 음성 해설 제공 시 외국인 시니어에게도 완결형 경험

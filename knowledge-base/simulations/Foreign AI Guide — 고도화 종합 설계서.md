## 2. 핵심 사용자 시나리오 — 5단계 플로우

> 대표 시나리오: Emma Chen (외국인 AI 가이드의 핵심 가치 집약)

```
[1단계: 언어 선택 & 위치 감지]
  └─ 앱 실행 → 언어 선택 (5개국어) → GPS 자동 감지 → 혼잡도 미리보기

[2단계: AI 채팅 탐색]
  └─ 자연어 질문 → 관광지 카드 (사진 + 혼잡도 + 오디오 버튼) → 연관 관광지 추천

[3단계: AI 일정 최적화]
  └─ 방문지 목록 입력 → 집중률 기반 시간대 재정렬 → 최적 루트 생성 → 저장

[4단계: 현장 실시간 가이드]
  └─ GPS 도착 감지 → 오디오 가이드 자동 실행 → AR 포토스팟 / 무장애 경로 오버레이

[5단계: 콘텐츠 & 공유]
  └─ 방문 완료 → 다국어 캡션 자동 생성 → SNS 공유 → 커뮤니티 피드백 루프
```

### 언어별 사용자 흐름 차이

| 언어 | 1단계 | 3단계 특이사항 | 4단계 특이사항 |
|---|---|---|---|
| 繁體中文 | 즉시 전환, 화권 커뮤니티 딥링크 | 크리에이터 일정 최적화 | 릴스 캡션 자동생성 |
| 한국어 | 기본값, 동행자 초대 QR | 그룹 투표 일정 합의 | 국문 오디오 고품질 |
| English | 접근성 필터 우선 | 무장애 루트 필터링 | AR 경로 오버레이 |
| 日本語 | 하루카 시나리오 동기화 | 실시간 번역 메시지 | 일본어 오디오 가이드 |
| 简体中文 | 대륙 여행자 타겟 | WeChat 공유 연동 | 간체 오디오 가이드 |

## 3. Visit Korea OpenAPI 매핑

### 3-1. 사용 API 명세 (7개)

| # | API명 | 카테고리 | 주요 활용 |
|---|---|---|---|
| ① | 영문 관광정보서비스 | i18n | 영어 관광지 정보, 위치 기반 검색 |
| ② | 일문 관광정보서비스 | i18n | 일본어 관광지 정보, 키워드 검색 |
| ③ | 중문 번체 관광정보서비스 | i18n | 번체 중국어 정보 (대만·홍콩) |
| ④ | 중문 간체 관광정보서비스 | i18n | 간체 중국어 정보 (대륙) |
| ⑤ | 관광지 오디오 가이드정보 | content | 한·영·중·일 4개국어 음성+스크립트 |
| ⑥ | 관광지 집중률 방문자 추이 예측 | big-data | 30일 혼잡도 예측, 최적 방문시간 |
| ⑦ | 무장애 여행 정보 | thematic | 휠체어 접근 가능 경로, 편의시설 |

### 3-2. 호출 시퀀스 (Emma Chen 시나리오 기준)

```
사용자: "홍대 포토스팟 추천해줘" (번체 중국어 입력)
  │
  ├─ [1] locationBasedList2 (중문번체)
  │     mapX=126.924, mapY=37.557, radius=3000, contentTypeId=12
  │     → 홍대 근처 관광지 목록 반환
  │
  ├─ [2] tatsCnctrRateList (병렬)
  │     각 관광지 ID → 내일 혼잡도 예측값 조회
  │     → 시간대별 집중률 배열 반환
  │
  ├─ [3] storyLocationBasedList (병렬)
  │     mapX, mapY → 오디오 가이드 보유 여부 확인
  │     → 번체 중국어 스크립트 URL 반환
  │
  └─ AI 응답 카드 생성:
        {
          name: "경의선 숲길",
          lang: "繁體中文",
          concentration_tomorrow_9am: "18%",
          audio_guide: true,
          audio_lang: ["ko","en","zh","ja"],
          photo_url: "...(관광사진정보 API)"
        }
```

### 3-3. 언어별 API 라우팅

```
사용자 언어 설정 → API 선택 분기
  ko  → 국문 관광정보서비스 (areaBasedList2)
  en  → 영문 관광정보서비스 (areaBasedList2)
  zh-TW → 중문 번체 관광정보서비스 (areaBasedList2)
  zh-CN → 중문 간체 관광정보서비스 (areaBasedList2)
  ja  → 일문 관광정보서비스 (areaBasedList2)
  fr  → 불어 관광정보서비스 (areaBasedList2)
  de  → 독어 관광정보서비스 (areaBasedList2)
  es  → 서어 관광정보서비스 (areaBasedList2)
  ru  → 노어 관광정보서비스 (areaBasedList2)
  
  * 오디오 가이드: ko/en/zh/ja 4개국어만 제공 (API 제약)
  * 혼잡도/무장애: 언어 무관 동일 API (한국어 응답 → 앱 내 번역)
```

## 5. 차별화 포인트 (경쟁 서비스 대비 3개+)

| # | 차별화 포인트 | 경쟁 서비스 현황 | Foreign AI Guide 우위 |

## 6. MVP 기능 우선순위

### Must Have (출시 필수)
- [ ] 9개국어 언어 선택 및 UI 전환
- [ ] Visit Korea 다국어 관광정보 API 연동 (영·일·중번·중간)
- [ ] 관광지 혼잡도 예측 표시 (30일)
- [ ] AI 채팅 가이드 (자연어 질문 → 관광지 추천)
- [ ] 오디오 가이드 재생 (한·영·중·일 4개국어)
- [ ] 위치 기반 근처 관광지 탐색

### Should Have (초기 릴리스 후 1-2개월)
- [ ] AI 일정 최적화 (혼잡도 기반 루트 생성)
- [ ] 무장애 여행 필터 및 AR 경로 오버레이
- [ ] 그룹 여행 모드 (다국어 플랜 공유)
- [ ] SNS 캡션 자동 생성 (언어별)
- [ ] 연관 관광지 추천

### Could Have (v2 로드맵)
- [ ] AR 포토스팟 마커 (최적 촬영 각도 가이드)
- [ ] 크라우드소싱 무장애 경로 업데이트
- [ ] 오프라인 캐시 (오디오 가이드 사전 다운로드)
- [ ] 크리에이터 배지 & 인플루언서 파트너십
- [ ] 불어·독어·서어·러시아어 확장

## 7. 4주 개발 일정

# SIM-R3-008: Audio Story Korea v2 × 이재호

## 시나리오

이재호(38세)는 서울 강남구에 거주하는 시니어 백엔드 개발자다. 척수 손상으로 전동휠체어를 사용하며, 이동 가능한 문화 체험에 대한 갈증이 크다. INTJ 성향으로 데이터 정확도와 논리적 일관성을 최우선으로 여기며, 앱의 신뢰성을 기술적 관점에서 냉정하게 평가한다. 역사와 IT 기술의 교차점에 관심이 많고, 부정확한 정보나 마케팅 과장에 강한 거부감을 보인다.

앱을 발견한 건 개발자 커뮤니티 슬랙 채널이었다. 누군가 "관광 앱 중에 API 설계가 탄탄한 게 있더라"며 Audio Story Korea v2를 언급했다. 기술적 호기심과 함께 접근성 기능(무장애 루트 필터)이 있다는 정보가 결정적이었다. 앱을 다운로드한 후 설정 화면부터 꼼꼼히 훑었다. "무장애 동선 필터", "경사도 표시", "턱 없는 진입로 확인" 옵션들을 하나하나 확인하며 실제 데이터 정확도에 대한 의구심과 기대를 동시에 품었다.

창덕궁 인근 원서동에서 앱을 켰다. 무장애 필터를 활성화하자 접근 불가 스팟들은 회색으로 흐려지고, 휠체어 접근 가능 별 아이콘만 밝게 남았다. 전체 별의 약 40%였다. "솔직한 데이터네." 이재호가 고개를 끄덕였다.

## 사용자 여정

### 1단계: 진입
- 개발자 슬랙 채널 → 앱 다운로드 → 설정 화면 전체 검토 → 무장애 필터 활성화
- 첫 반응: "접근 불가 스팟을 숨기지 않고 회색으로 표시한 게 좋다. 억지로 '모두 가능'이라고 거짓말하는 앱들과 다르다. 40% 접근 가능이면 서울 역사 구역에서 현실적인 수치다."

### 2단계: 행동
- **경사도 데이터 검증**: 앱이 표시한 경사도 정보와 실제 현장을 비교 검증. 원서동 골목 입구 경사도 "3% (휠체어 가능)" 표시 → 실제 확인 결과 정확. "데이터 신뢰할 수 있다."
- **첫 번째 별 스팟**: 원서동 한옥 밀집 구역 내 "조선시대 역관(譯官) 거주지" 터. 오디오 재생. 조선의 통역관 제도와 외교 문서 번역 역사 — "역관이 사실상 조선의 IT 인프라였구나. 정보 유통의 핵심."
- **개발자 연결 사고**: "역관들이 다국어 데이터를 중개한 것처럼, 이 앱도 문화유산 데이터를 중개하는 플랫폼이다." 메모 앱에 생각을 기록
- **턱 없는 진입로 실시간 보고**: 앱이 표시한 "턱 없는 진입로" 중 하나가 실제로는 3cm 턱이 있음을 발견. 앱 내 "정보 수정 제안" 기능으로 즉시 신고. "UGC 수정 시스템이 있군. 데이터 품질 관리 방식 적절."
- **오디오 배속 + 스크립트 병행**: 1.25x 속도로 오디오 청취하며 자막 스크립트를 함께 확인. "청각 장애 사용자 배려 포인트이기도 하고, 나는 텍스트로 다시 확인하는 게 더 정확하다."
- **접근 불가 스팟 아쉬움**: 창덕궁 후원은 접근 불가(자갈길 + 계단). 앱이 대체 루트로 "낙선재 앞 평지 구간 접근 가능" 안내. "완벽하진 않지만, 대안을 알려주는 게 낫다."
- **데이터 API 확인 욕구**: 앱 소개 페이지에서 "어떤 API를 사용하는지" 확인 시도. 공개 정보 없음 → 개발자 이메일로 기술 스택 문의 발송

### 3단계: 결과
- 원서동·창덕궁 인근 3시간 탐방. 접근 가능 별 스팟 3개 방문
- 개발자 커뮤니티 슬랙에 앱 리뷰 포스팅: "무장애 데이터 정확도 85% 수준. 나머지 15%는 UGC로 보완 중인 것으로 보임. 역사 콘텐츠 품질은 기대 이상. 접근성 앱 중에서 현재 최선."
- 데이터 수정 제안 3건 제출
- 앱 개발팀으로부터 48시간 내 회신 기대 (기술 스택 질문)

## Pass/Fail 판정

> **✅ PASS**

이재호가 가장 중요시하는 "데이터 정확도"에서 신뢰를 획득했다. 접근 불가 스팟을 회색 처리하는 솔직한 UX, UGC 수정 시스템, 대체 루트 안내의 조합이 개발자 출신 회의론자의 신뢰를 얻는 데 성공했다. 역사 콘텐츠의 깊이가 단순 관광 앱을 넘어서는 품질임을 확인시켜줬다.

## 🔄 리텐션 평가: 이 페르소나가 앱을 다시 열까?

**재사용 확률**: ⬆️높음

**감정적 이유**:
> "데이터가 정직한 앱은 드물다. 내가 갈 수 없는 곳을 '갈 수 있다'고 속이는 대신 회색으로 표시하고, 내가 간 길에서 오류를 발견하면 수정 제안을 받는 앱 — 이건 만드는 사람이 접근성을 진심으로 생각하고 만든 거다. 그 철학을 믿는다."

**재사용 트리거**:
- 개발팀 기술 스택 회신 수신 후 앱에 대한 신뢰 재확인
- 데이터 수정 제안이 실제 반영됐다는 알림 수신 시 강한 충성도 형성
- 전주·경주 방문 시 해당 지역 무장애 별 아이콘 밀도 사전 확인

**이탈 리스크**:
- 경사도 데이터 오류율이 지속적으로 높을 경우 신뢰 철회
- 기술 스택 질문 무응답 시 "블랙박스 앱"으로 분류하고 이탈 가능성

## 개선 제안

1. **무장애 데이터 신뢰도 지표 공개**: 각 스팟의 무장애 정보 출처(실측/UGC/공공데이터)와 최근 검증 날짜를 명시. "이 정보는 3개월 전 이재호 님이 검증했습니다" 표시. 개발자 출신 사용자의 신뢰와 기여 동기 동시 자극
2. **개발자 API 공개 문서**: 앱 내 "For Developers" 섹션에서 사용 API 목록, 데이터 갱신 주기, 오픈 데이터셋 출처 공개. 개발자 커뮤니티 신뢰 획득 + 데이터 기여자 커뮤니티 형성 기반 마련

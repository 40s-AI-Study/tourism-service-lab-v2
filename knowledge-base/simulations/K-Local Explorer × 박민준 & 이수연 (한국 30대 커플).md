# SIM-52: K-Local Explorer × 박민준 & 이수연 (한국 30대 커플)

## 시나리오

박민준(33세, 회사원)과 이수연(31세, 간호사)은 부산에서 2박 3일 여행을 계획 중이다. 광안리는 이미 두 번 가봤고, 이번에는 "부산 사람들이 즐겨 가는 숨은 오마카세나 로컬 식당"을 찾고 싶다. 블로그 검색은 광고성 글이 많아 신뢰하기 어렵고, SNS는 이미 유명한 곳만 나온다. 민준이 출장 중 동료에게서 K-Local Explorer를 소개받았다.

---

## 사용자 여정

### 1단계: 진입

여행 일주일 전, 민준이 앱을 설치하고 목적지를 "부산 수영구·남구"로 설정한다. 온보딩에서 관심사 태그를 `#로컬맛집`, `#오마카세`, `#조용한식당`, `#데이트`로 선택한다. 앱이 `central-attractions-by-municipality` API로 수영구 데이터를 불러오고, 혼잡도 하(下)인 식당·명소 위주로 필터링한다. 메인 화면에 "광안리 주변 현지인 추천 숨은 맛집 5곳" 섹션이 노출된다.

### 2단계: 행동

수연이 사진 공모전 갤러리를 훑다가 광안대교가 배경인 아늑한 오마카세 내부 사진에 반응한다. 해당 명소 페이지로 진입하니 `photo-contest-winners` API 메타데이터에 "광안리 이면도로 7분 거리, 예약 필수, 점심 타임 혼잡도 낮음" 정보가 표시된다.

`related-attractions` API가 오마카세와 연계된 "광안리 뒷골목 반나절 코스"를 제안한다: 오전 전통시장 → 점심 오마카세 → 오후 수변공원 포토스팟. 혼잡도 예측 기능이 "토요일 오전 11시 30분 예약이 최적"임을 알려준다.

두 사람이 코스를 함께 보며 일정을 조율하고 공유 링크로 코스를 저장한다. 오마카세 예약은 앱 내 외부 링크(네이버 예약)로 연결되어 처리한다.

### 3단계: 결과

현장 방문에서 오마카세는 기대 이상이었고, 뒷골목 탐방에서 우연히 발견한 빈티지 소품숍이 이번 여행의 숨은 하이라이트가 되었다(앱이 연관 명소로 추천한 곳). 수연이 리뷰를 앱에 남기고 별점 5점을 준다. 두 사람 모두 "블로그 검색보다 훨씬 신뢰할 수 있다"는 평가를 내린다. 단, 오마카세 예약이 앱 내에서 완결되지 않고 외부 링크로 넘어가는 점이 아쉬움으로 남는다.

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **8** | 숨은 오마카세 발굴 니즈를 직접 충족. 혼잡도 기반 타이밍과 연계 코스가 여행 계획에 실질적 도움. 광고 없는 큐레이션이 신뢰도 차별화 |
| 사용 편의 | **7** | 커플 공유 기능이 동선 조율에 유용. 외부 예약 연결이 불편한 단절감 유발. 지역 필터링은 직관적 |
| 구현 타당성 | **8** | 4개 API 활용 구조가 탄탄. 맛집 데이터 포함 여부가 핵심 변수 — 관광지 API 기반이라 식당 데이터 커버리지가 불완전할 가능성 있음 |

**종합 점수: 23/30**

## Pass/Fail 판정

> **✅ PASS**

유용성 8점, 종합 23점으로 기준을 충족한다. 숨은 맛집 발굴이라는 구체적 니즈를 앱의 핵심 기능이 커버하며, 커플 동선 공유 기능이 추가 가치를 제공한다. 관광지 API 기반 서비스에서 식당 데이터 품질이 실 사용 만족도를 좌우할 핵심 리스크로 식별된다.

## 개선 제안

1. **예약 통합 인앱 처리**: 네이버 예약·카카오 예약 API를 앱 내에 임베드하여 탐색-예약을 단절 없이 처리. 이탈률을 낮추고 전환율을 높이는 핵심 UX 개선.
2. **커플/그룹 코스 공동 편집**: 두 사람이 동시에 코스를 수정하고 의견을 달 수 있는 협업 기능 추가. 공유 링크 수준을 넘어서 실시간 협업 경험을 제공하면 30대 커플 타겟의 재사용률을 높일 수 있음.

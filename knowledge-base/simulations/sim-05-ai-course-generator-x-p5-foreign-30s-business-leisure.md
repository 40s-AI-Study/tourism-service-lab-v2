# SIM-05: KoreaPath AI × David Park (외국인 30대 블레저)

## 시나리오

David Park(35세, 미국 테크 기업 PM)은 서울 출장 중 수요일 오후가 갑자기 비었다. 호텔에서 KoreaPath AI를 설치하고 "지금 위치(강남) 기준으로 3시간 코스, K-역사, 혼자"를 입력하려 한다. 그러나 앱이 한국어 전용임을 확인.

---

## 사용자 여정

### 1단계: 진입
- 앱 실행 → 한국어 전용 UI 확인
- 재미교포로 기초 한국어 이해 → 간신히 탐색 가능
- 조건 입력: **"강남, 3시간, 역사, 혼자"** (한국어로 간신히 입력)
- 입력 소요 시간: 약 3분

### 2단계: 행동
- AI가 `related-attractions` + `visitor-concentration-forecast` 조합
- 생성된 코스 (3시간):
  ```
  14:00 경복궁 (지하철 3호선, 혼잡도 보통)
  15:30 북촌 한옥마을 (도보 이동, 혼잡도 낮음)
  16:30 인사동 전통 공예 거리
  17:00 귀환
  ```
- 코스 내용은 영어로 읽고 싶지만 전부 한국어 → 장소명은 Google Maps 병행 확인
- "3시간짜리 코스" 시간 제약 입력이 명확하지 않아 결과가 4시간 코스로 생성됨

### 3단계: 결과
- 코스 구성 자체는 훌륭 (경복궁→북촌→인사동은 David의 K-역사 관심과 딱 맞음)
- 한국어 설명 읽기가 불편해 Google Maps를 병행 사용
- 시간 초과 (3시간 입력 → 4시간 결과) 로 출장 저녁 미팅 전에 복귀 촉박
- "시간 제약 기능만 정확했어도 훨씬 좋았을 것"이라는 피드백

## Pass/Fail 판정

> **⚠️ CONDITIONAL PASS**

P5 David는 KoreaPath AI의 핵심 기능(시간 최적화 코스)이 절실히 필요한 사용자다. 영어 지원과 시간 제약 기능만 추가되면 강력한 PASS. 현재는 언어+기능 gap으로 조건부 통과.

## 개선 제안

1. **영어 UI 지원 (필수)**: David처럼 영어권 블레저 여행자는 방한 외래객 최대 세그먼트 중 하나
2. **시간 제약 파라미터 명확화**: "지금부터 N시간" 입력 시 이동 포함 실제 소요시간 계산해 코스 생성
3. **현재 위치 기반 즉시 실행**: GPS 기반 "지금 주변 3시간 코스" 버튼 → 바쁜 블레저 여행자 핵심 기능
4. **출장자 모드**: 반나절/저녁 시간대별 코스 프리셋 → 일정 없는 시간 즉각 활용

# SIM-50: KoreaTrend Radar × 레이첼 김 (교포 2세)

## 시나리오

2026년 4월 17일 토요일 오전 9시, 서울 종로구 게스트하우스. LA 출신 건축가 레이첼 김(42)은 2주일 한국 방문 중이다. 교포 2세로서 한국어는 일상 대화 수준이고, 이번 여행의 목적은 "트렌디한 핫플" 보다 "한국의 역사·건축·문화 심층 탐방"이다. 조선시대 한옥 밀집 지역, 근현대 건축물, 역사 유적지를 우선한다.

동생이 "이 앱 써봐, 요즘 뜨는 데 알려준대"라며 KoreaTrend Radar를 추천했다. 레이첼은 건축가 특유의 분석적 시각으로 앱을 살펴본다. 트렌드 히트맵을 보니 지금 뜨는 곳은 성수동·연남동·을지로 등 MZ세대 핫플 위주다. 그녀가 관심 있는 창덕궁·북촌·인왕산 자락 같은 역사 지역은 트렌드 지표가 낮게 표시된다 — "덜 뜨는 곳" 취급이다. 앱의 가치 기준(유동인구 급등, 카드 소비 증가)이 본인의 여행 목적과 정반대임을 확인하고 앱을 닫는다.

"빅데이터가 보여주는 트렌드와 내가 원하는 깊이 있는 여행은 다른 종류의 것"이라는 결론을 내린다.

## 사용자 여정

### 1단계: 진입
- 앱 실행 (영어/한국어 혼용 사용 가능)
- 홈 레이더 확인 → 성수·연남·을지로 핫스팟 인식
- 역사/건축 카테고리 탐색

### 2단계: 행동
- '관광지' 카테고리 필터 → 역사 유적 특화 필터 부재 확인
- 창덕궁 검색 → 트렌드 지수 낮음 (평일 방문객 안정적이나 급등 없음)
- 북촌 데이터 확인 → 트렌드 지표 중간, 역사·건축 정보 없음
- 앱의 가치 기준(유동인구·카드 소비 급등)이 본인 니즈와 불일치 인식
- 앱 이탈, 문화재청 앱 및 건축 전문 가이드북으로 전환

### 3단계: 결과
- KoreaTrend Radar 활용 포기
- 역사·건축 탐방 일정은 기존 자료로 직접 구성
- 앱 평가: "트렌드 앱이지 역사 탐방 앱이 아님 — 나한텐 맞지 않아"

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **3** | 역사·건축 심층 탐방 니즈와 트렌드 급등 중심 앱 가치 기준이 근본적으로 불일치. 앱이 제공하는 정보 유형 자체가 이 페르소나에게 무관함 |
| 사용 편의 | **6** | UI 자체는 직관적이고 건축가 특유의 분석적 시각으로 기능 파악은 빠름. 다만 원하는 정보 카테고리 부재 |
| 구현 타당성 | **6** | 앱 기능 구현은 타당하나, 이 페르소나는 앱의 설계 철학과 근본적 미스매치. 역사 문화 레이어 추가 시 가치 확장 가능 |

**종합 점수: 15/30**

## Pass/Fail 판정

> **❌ FAIL**

유용성 3점, 종합 15/30으로 기준 미달. 트렌드 급등 중심의 앱 패러다임이 역사·문화 심층 탐방을 원하는 페르소나와 구조적으로 불일치. 이 세그먼트 포용을 위해서는 앱의 데이터 해석 축 자체를 확장해야 함.

## 개선 제안

1. **역사·문화 깊이 레이어**: 트렌드 급등 지표 외에 "역사적 가치 지수" 또는 "문화 다양성 심층 스코어"를 별도 레이어로 제공해 역사 탐방 여행자도 앱을 활용할 수 있는 축 추가
2. **건축·역사 테마 큐레이션**: area-tourism-diversity API 데이터를 활용해 "문화유산 밀집 지역" 테마 코스를 트렌드 데이터와 독립적으로 제공하는 '깊이 탐방 모드' 신설

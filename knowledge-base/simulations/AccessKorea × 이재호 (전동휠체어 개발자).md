# SIM-27: AccessKorea × 이재호 (전동휠체어 개발자)

## 시나리오

38세 IT 개발자 이재호 씨는 척수 장애로 전동휠체어를 사용한다. 연 3~4회 여행을 다닐 만큼 활동적이지만, 매번 여행 전 접근성 정보를 수집하는 데 하루 이상을 쏟아붓는다. 블로그 후기는 날짜가 오래됐고, 관광지 공식 사이트 접근성 안내는 "경사로 있음" 한 줄이 전부다. 제주 2박 여행을 앞두고 AccessKorea를 발견한 이재호 씨는 "데이터가 얼마나 정밀한지, 얼마나 최신인지"를 냉정하게 검증하기로 한다.

---

## 사용자 여정

### 1단계: 진입

이재호 씨는 앱스토어에서 AccessKorea를 내려받기 전에 리뷰 수(847개)와 평점(4.6)을 먼저 확인하고, 리뷰 탭에서 "실제 휠체어 사용자 후기"를 키워드로 스크롤한다. "전동휠체어 40kg, 성산일출봉 매표소까지 진입 확인"이라는 구체적 후기를 보고 설치를 결정한다. 앱 실행 후 프로필 설정에서 '전동휠체어', '최대 경사도 15° 이하', '바닥 재질 민감(자갈·모래 제외'를 입력하자 이후 검색 결과가 자동으로 필터링된다.

### 2단계: 행동

**기능 1 - 상세 접근성 필터**
제주 검색 결과에서 이재호 씨가 직접 설정한 필터 조건("경사도 15° 이하 + 포장 노면 + 단차 2cm 이하")을 적용한다. 일반 검색 결과 42개 관광지 중 조건을 통과한 곳은 17개. 이재호 씨는 "절반도 안 된다는 게 현실이지"라며 쓴웃음을 짓지만, 이 정직한 필터링 결과 자체를 신뢰의 근거로 받아들인다. 성산일출봉 상세 페이지에는 "매표소~1구간 전망 데크: 경사 8°, 콘크리트 포장, 단차 없음 / 1구간 이후: 경사 22°, 비포장, 전동휠체어 진입 불가"라고 구간별로 명확히 구분돼 있다.

**기능 2 - 크라우드소싱 최신 리뷰 검증**
이재호 씨가 가장 중점적으로 보는 항목은 '최근 90일 이내 리뷰'. 성산일출봉 리뷰 중 "3주 전, 전동휠체어 사용자 - 1구간 데크 난간 보수 공사 중, 우회로 안내 직원 상주"라는 최신 글이 있다. 그는 리뷰어의 프로필("전동휠체어 6년 경력")과 과거 리뷰 이력을 확인해 신뢰도를 교차 검증한다. 이어 '리뷰 신고' 기능을 테스트하며 허위 정보 필터링 구조가 있는지도 확인한다.

**기능 3 - 충전소 위치 및 숙소 접근성**
전동휠체어 배터리는 하루 약 25km. 이재호 씨가 '전동휠체어 충전소' 레이어를 켜자 제주 전역에 충전 가능 지점 23개가 표시된다. 각 지점에 "충전 규격(C타입/DC)", "운영 시간", "예약 필요 여부"까지 표시돼 있다. 숙소 검색에서 accommodation-info API 연동으로 "출입문 폭 90cm 이상 + 롤인 샤워 + 침대 높이 45~55cm" 조건에 맞는 숙소 4곳이 필터링된다. 이재호 씨는 그중 한 곳의 '접근성 인증서 사진'을 원본 이미지로 확인한다.

**기능 4 - 무장애 동선 자동 추천**
2박 3일 일정을 입력하자 AccessKorea가 "전동휠체어 적합 동선"으로 하루 평균 이동 거리 8km 이내의 일정을 자동 구성한다. 이재호 씨는 추천 경로의 근거 데이터("각 구간 경사도, 노면 재질, 예상 배터리 소모")를 CSV로 내보내기해 스프레드시트에서 직접 검토한다. "이 정도면 사전 조사 시간이 6시간에서 1시간으로 줄겠다"고 속으로 평가한다.

### 3단계: 결과

이재호 씨는 출발 전날 AccessKorea로 모든 동선을 검증 완료하고, 여행 중 단 한 번의 예상치 못한 접근성 장벽도 만나지 않는다. 성산일출봉 1구간 데크에서 제주 바다를 바라보며 "정확한 데이터가 이 풍경을 가능하게 했다"고 생각한다. 귀경 후 앱에 리뷰 3건을 직접 작성하고, 개발자 커뮤니티 슬랙에 AccessKorea를 공유하며 "휠체어 사용자 필수 앱"이라고 소개한다.

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **10** | 경사도·노면 재질·단차 등 전동휠체어 사용자에게 생존 수준의 정보를 정밀하게 제공. 여행 사전 조사 시간을 6분의 1로 단축 |
| 사용 편의 | **9** | 상세 필터·CSV 내보내기·리뷰 신뢰도 검증 등 고급 사용자 흐름이 잘 설계됨. 프로필 기반 자동 필터링이 특히 유용 |
| 구현 타당성 | **9** | barrier-free-travel + accommodation-info API 조합으로 구현 가능. 크라우드소싱 리뷰 신뢰도 시스템과 충전소 데이터는 초기 구축 비용이 크지만 기술적으로 현실적 |

## Pass/Fail 판정

> **✅ PASS**

AccessKorea는 이재호 씨가 매번 겪어온 정보 공백을 정면으로 해결한다. 정밀한 수치 기반 필터, 최신 크라우드소싱 리뷰, 배터리를 고려한 동선 추천이라는 세 가지 핵심 기능이 IT 개발자 특유의 데이터 검증 습관과 완벽하게 맞아 떨어진다. 이 페르소나의 자발적 리뷰 기여와 커뮤니티 공유는 서비스의 선순환 생태계를 만드는 가장 중요한 동력이다.

## 개선 제안

1. **전동휠체어 모델별 호환성 데이터**: 사용자가 본인 휠체어 모델(제조사·폭·무게)을 등록하면 해당 모델 통과 가능 여부를 자동 계산해주는 기능 추가.
2. **실시간 현장 상태 알림**: 공사·임시 폐쇄·엘리베이터 고장 등 돌발 변수를 방문 당일 푸시 알림으로 제공 (관광지 공식 API 또는 크라우드소싱 연동).
3. **오프라인 상세 지도 캐싱**: 제주처럼 일부 구간에서 통신이 불안정한 지역을 위해 무장애 경로 + 편의시설 지도를 오프라인으로 저장하는 기능.
4. **리뷰어 신뢰도 레벨 시스템 공개**: 리뷰어의 장애 유형·경력·누적 기여도를 투명하게 표시해 정보 신뢰성을 높이고 고품질 기여자를 시각적으로 구분.

# SIM-34: PetKorea × Emma Chen (외국인 20대 백패커)

## 시나리오

2026년 3월, Emma Chen(23세, 대만 사진작가)은 한국 10일 백패킹 여행 중이다. 반려동물은 대만에 있어 동반 여행 니즈는 전혀 없다. 부산 게스트하우스에서 숙소 동료가 PetKorea 앱을 언급하며 "예쁜 동물 카페 찾을 때 좋다"고 말한다. Emma는 SNS 크리에이터로서 독특한 콘텐츠를 원하고, 고양이 카페나 강아지 카페처럼 한국 특유의 반려동물 문화 체험에 관심이 생긴다.

앱을 설치하고 언어를 영어 또는 번체 중문으로 설정하려 하지만 한국어만 지원된다는 것을 알게 된다. 구글 번역에 의존해 검색을 시도하지만 인터페이스 탐색이 불편하다. "dog cafe busan" 영문 검색이 작동하지 않아 한국어로 "부산 강아지 카페"를 복사 붙여넣기해야 한다. 결국 펫 카페 2곳의 정보를 얻는 데 성공하지만, 전반적인 경험은 마찰이 많았다.

Emma는 SNS용 사진 촬영 목적으로 펫 카페 1곳을 방문하고, 콘텐츠로서의 한국 펫 문화는 흥미롭다고 평가하지만 앱 자체는 재사용 의향이 낮다고 판단한다.

---

## 사용자 여정

### 1단계: 진입
- 게스트하우스 숙소 동료의 구두 추천으로 앱 설치
- 언어 설정에서 영어/중문 옵션 없음 확인 → 한국어 강제 사용
- 반려동물 프로필 등록 없이 탐색 모드 진입
- 영문 검색어 입력 시 결과 없음 → 구글 번역으로 한국어 변환 후 재검색

### 2단계: 행동
- `pet-friendly-travel` API (카테고리: 카페) → "부산 강아지 카페" 검색, 결과 수신
- 결과 카드에서 사진 위주로 탐색 (텍스트 내용은 번역 의존)
- `visitor-concentration-forecast` API → 혼잡도 확인 시도 (UI 라벨 이해 어려움으로 수동 추측)
- 카페 상세 페이지 영업시간·주소 확인 (주소는 지도 앱 연동으로 해결)
- 펫 카페 1곳 방문 결정, 외부 지도 앱으로 이동

### 3단계: 결과
- 해운대 강아지 카페 방문, 콘텐츠 촬영 성공 (팔로워 반응 좋음)
- 앱 자체 경험: 언어 장벽으로 전반적 불편. 재설치 의향 없음
- 반려동물 동반 여행 니즈가 없어 앱의 핵심 가치(AI 코스·혼잡 예측·맞춤 추천)를 전혀 활용하지 못함
- 앱의 주요 타겟 사용자가 아님이 명확히 드러남

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **3** | 반려동물 동반 니즈 없음. 펫 카페 검색 1회성 활용만 가능 |
| 사용 편의 | **2** | 한국어 전용 인터페이스로 외국인 사용자에게 심각한 언어 장벽 |
| 구현 타당성 | **7** | API 자체는 작동. 다국어 미지원이 UX 문제일 뿐 기술적 한계는 아님 |

## 개선 제안

1. **다국어 지원 우선 구현**: 영어 최소 지원, 중국어 번체·일본어 순차 추가. 외국인 펫 카페 체험 수요는 실재하며 인바운드 관광 트래픽 확보 가능
2. **비회원·비펫 사용자 탐색 모드**: 반려동물 없이도 "펫 카페 체험" 카테고리를 별도 제공해 외국인 관광객의 한국 펫 문화 체험 수요 수용
3. **앱스토어 설명 타겟 명확화**: "반려동물 동반 여행" 특화 앱임을 설명에서 명시해 비타겟 사용자의 잘못된 설치 기대치 관리

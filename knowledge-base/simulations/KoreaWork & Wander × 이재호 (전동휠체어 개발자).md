# SIM-177: KoreaWork & Wander × 이재호 (전동휠체어 개발자)

## 시나리오

재호는 스타트업 백엔드 개발자로 100% 재택근무다. 파트너와 함께 제주에서 한 달 살기를 계획 중인데, 일하면서 지내는 만큼 작업 환경이 잘 갖춰진 숙소가 필수다. 동시에 전동휠체어 사용자로서 숙소 접근성 — 경사로, 화장실 바닥 턱, 엘리베이터 유무 — 이 모두 사전에 확인돼야 한다. KoreaWork & Wander를 설치하고 제주를 검색하니 gocamping 기반 글램핑 목록과 업무 환경 필터가 잘 갖춰져 있다는 것에 만족한다. 그러나 "전동휠체어 접근 가능" 필터가 없고, 숙소별 무장애 정보가 전혀 표시되지 않는다는 것을 발견하는 순간 심각한 한계를 인식한다.

---

## 사용자 여정

### 1단계: 진입
- 개발자 커뮤니티 슬랙에서 앱 추천 → 설치 → 영어 UI 자연스럽게 사용
- 제주 검색 → "한 달 살기" 장기 체류 플로우 선택
- area-tourism-resource-demand 대시보드 확인: 제주 서부 추천 — 데이터 기반 지역 추천에 만족

### 2단계: 행동
- gocamping API 글램핑·장기 숙박 목록 확인 — 와이파이·콘센트 필터 유용
- "전동휠체어 접근 가능" 필터 탐색 → 없음. 무장애 관련 정보 일절 없음
- 숙소 카드에서 접근성 정보(경사로, 화장실 단차, 엘리베이터) 확인 불가
- 개발자 특유의 꼼꼼함으로 숙소 목록 중 전화 문의 필요 여부를 하나씩 체크하다 포기
- related-attractions API 주변 관광 추천은 유용하나 역시 무장애 여부 필터 없음

### 3단계: 결과
- 워케이션 업무 환경 탐색 목적으로는 앱이 유용했으나, 최종 결정은 별도 전화 확인 후 진행
- "업무 환경 데이터는 훌륭한데 장애인 접근성 데이터가 전무한 게 치명적"이라는 5점 만점 중 3점 리뷰 작성
- 앱 GitHub 이슈 트래커에 "Barrier-free filter for wheelchair users" 기능 요청 등록

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **6** | 워케이션 업무 환경 탐색에는 실질적으로 유용. 그러나 전동휠체어 사용자에게 필수인 무장애 접근성 데이터 부재가 최종 의사결정 단계에서 앱 가치를 절반으로 감소 |
| 사용 편의 | **7** | 영어 UI, 데이터 기반 대시보드는 개발자 배경의 재호에게 친숙하고 직관적. UX 품질 자체는 높은 평가 |
| 구현 타당성 | **7** | 서비스 구현 타당성은 높으나 장애인 사용자 니즈 미충족이 명백한 설계 공백 |

**종합 점수: 20/30**

## Pass/Fail 판정

> **✅ PASS**

KoreaWork & Wander는 이재호의 "원격근무 가능 숙소 탐색"이라는 주요 니즈를 충족하며, 재호는 실제로 앱을 워케이션 장소 탐색 도구로 활용한다. 다만 전동휠체어 사용자로서 무장애 접근성 데이터 부재로 최종 결정 단계에서 앱을 이탈해야 하는 구조적 한계가 있다. 워케이션 업무 환경과 무장애 정보가 통합된다면 재호에게 가장 완벽한 앱이 될 수 있다.

## 개선 제안

1. 숙박 시설 무장애 접근성 필터 추가: "전동휠체어 접근 가능", "화장실 단차 없음", "엘리베이터 있음" 태그를 gocamping 및 숙박 데이터에 결합하여 표시 — 재호 같은 장애인 원격근무자에게 이 앱이 유일한 워케이션 솔루션이 되는 결정적 기능
2. 장애인 사용자 크라우드소싱 리뷰 시스템 도입: "전동휠체어 실제 진입 가능 여부" 구조화 리뷰를 별도 수집하여 실시간 신뢰 데이터로 활용 — 재호가 직접 기여하고 싶어 하는 커뮤니티 참여 욕구와도 일치

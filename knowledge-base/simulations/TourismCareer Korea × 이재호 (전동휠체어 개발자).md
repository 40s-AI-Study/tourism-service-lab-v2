# SIM-147: TourismCareer Korea × 이재호 (전동휠체어 개발자)

## 시나리오

재호는 스타트업 백엔드 개발자로 재택근무 중이며, 한국관광공사 API에 대한 개발자 관점의 관심이 있다. 개발자 커뮤니티 슬랙 채널에서 "tourism-jobs API를 활용한 앱이 출시됐다"는 소식을 듣고 API 활용 방식을 살펴볼 겸 설치했다. 취업 전환 의사는 없지만, 관광 접근성 스타트업 창업 아이디어를 갖고 있어 관광 산업 채용 구조와 지역 데이터에 관심을 보였다. 다만 현장 탐방 코스에 무장애 필터가 없어 실제 여행 계획에는 활용하지 못했다.

---

## 사용자 여정

### 1단계: 진입
- 개발자 슬랙 채널 공유 링크 → 앱스토어 설치
- 개발자 답게 온보딩을 빠르게 건너뛰고 메인 기능 직접 탐색
- tourism-jobs API 활용 공고 탐색 구조를 코드 관점에서 분석 ("API 응답을 어떻게 파싱하고 있을까" 추측)

### 2단계: 행동
- 관광 스타트업 개발자 포지션 공고 검색 → 백엔드 엔지니어 공고 2건 발견
- 기업 지역 데이터 클릭 → `area-tourism-demand-density` 시각화 확인 — "데이터 출처가 명시되어 있어 신뢰도 높다"고 평가
- 현장 탐방 코스 생성 시도 → 무장애 필터 없음 확인 — 즉시 실망
- 탐방 코스 내 이동 경로의 계단·경사도 정보 전무 — 전동휠체어 사용자로서 이 코스는 사용 불가

### 3단계: 결과
- 취업 기능: 관심 공고 2건 북마크, 실제 지원 의사 없음
- 탐방 기능: 접근성 정보 없어 미사용
- 개발자 관점 피드백: "barrier-free-travel API 연동하면 훨씬 나아질 텐데"
- 앱 유지하나 낮은 활성 사용 (월 1-2회 공고 확인 정도)

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **5** | 관광 스타트업 개발자 공고 존재 자체는 유의미, 탐방 기능은 접근성 미지원으로 무용 |
| 사용 편의 | **7** | 개발자 친화적 구조, 데이터 출처 명시가 신뢰도 높임 |
| 구현 타당성 | **8** | tourism-jobs API 기반 공고 탐색은 잘 구현됨, 접근성 레이어 추가가 과제 |

**종합 점수: 20/30**

## Pass/Fail 판정

> **⚠️ CONDITIONAL PASS**

재호는 부분적 타깃(관광 스타트업 취업 관심 개발자)에 해당하며, 취업 공고 탐색 기능에서 제한적 가치를 찾았다. 그러나 전동휠체어 사용자로서 현장 탐방 기능의 접근성 부재는 앱 활용도를 절반으로 제한한다. 무장애 탐방 코스 필터 추가 시 PASS로 전환 가능하다. 개발자 관점의 API 품질 평가는 긍정적이며, 기술력 측면에서 신뢰 가능한 제품이다.

## 개선 제안

1. 현장 탐방 코스에 무장애 필터 추가: `barrier-free-travel` API 연동으로 전동휠체어·고령자 접근 가능 탐방 코스 생성 옵션 제공
2. 관광 테크 스타트업 개발자 포지션 공고 확충: IT 직군(백엔드·프론트엔드·데이터) 관광 기업 공고를 별도 트랙으로 분류하여 개발자 사용층 공략

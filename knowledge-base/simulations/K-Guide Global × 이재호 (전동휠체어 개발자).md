# SIM-17: K-Guide Global × 이재호 (전동휠체어 개발자)

## 시나리오

38세 IT 개발자 이재호 씨는 척수 장애로 전동휠체어를 사용하며 무장애 관광 정보 플랫폼에 깊은 관심을 가지고 있다. 사내 해커톤 주제로 "외국인 장애인 관광객을 위한 무장애 정보 서비스"를 검토하던 중 K-Guide Global이 Visit Korea 공공 API 27개를 연동했다는 소식을 기술 블로그에서 접했다. 개발자 관점에서 API 구조와 무장애 데이터 커버리지를 직접 검토해보기로 했다.

---

## 사용자 여정

### 1단계: 진입

이재호 씨는 앱을 설치한 후 영어로 언어를 설정한다. 전동휠체어 사용자답게 앱의 접근성(VoiceOver/TalkBack 호환성)부터 테스트한다. 버튼 레이블, 화면 읽기 순서는 준수한 편이지만 일부 이미지에 alt 텍스트가 없고, 혼잡도 색상 인디케이터는 색상만으로 구분해 색각 이상 사용자에게 문제가 될 수 있음을 즉시 인지한다. 개발자 시각으로 앱 구조를 훑으며 메모를 시작한다.

### 2단계: 행동 (핵심 기능 3회 이상 사용 장면)

**시도 1 — 무장애 필터 탐색**
이재호 씨는 관광지 검색에서 "Wheelchair Accessible" 필터를 찾는다. 필터 항목에 Accessibility 탭이 있지만 세분화가 부족하다. 경사로 존재 여부, 화장실 접근성, 주차 구역 정보가 단순 "Accessible: Yes/No" 이진 값으로만 표시된다. 실제 전동휠체어 사용자가 필요로 하는 경사도(%), 바닥 재질, 엘리베이터 위치 등 심층 데이터는 없다. 이재호 씨는 "API에 데이터가 없는 건지, 앱이 활용 안 하는 건지" 궁금해하며 다음 단계로 넘어간다.

**시도 2 — API 연동 구조 추정 (네트워크 로그 분석)**
개발자 도구를 통해 앱의 네트워크 요청을 살펴본다(Charles Proxy 사용). cultural-facility-info API 호출은 확인되지만 무장애 전용 파라미터가 별도로 없고, visitor-concentration-forecast는 시간대별 예측 데이터를 잘 활용하고 있음을 발견한다. Visit Korea 공공 API 중 장애인 편의시설 데이터(areaCode별 편의 정보)가 있음에도 앱이 해당 엔드포인트를 호출하지 않는다는 것을 확인한다. API는 있는데 앱이 미활용 중이라는 결론을 내린다.

**시도 3 — 오디오 가이드 접근성 테스트**
audio-guide API로 구동되는 오디오 가이드를 재생해본다. 자막(transcript) 기능이 없어 청각 장애인 외국인 관광객에게는 사용 불가 수준임을 메모한다. 재생 속도 조절 기능도 없어 언어 이해가 느린 사용자(고령 관광객 포함)에게도 불리하다. 이재호 씨는 이 문제를 해커톤 제안서에 "개선 필요 사례"로 기록한다.

**시도 4 — 한국인 사용자로서의 실용성 평가**
이재호 씨는 실제 자신이 서울에서 주말 외출 시 이 앱을 쓸 수 있는지 상상해본다. 앱 전체가 외국인 방문객을 전제로 구성돼 있어, 한국어 콘텐츠, 한국 국내 장애인 편의시설 법적 기준 기반 정보, 지하철 휠체어 리프트 실시간 고장 알림 등은 전혀 없다. "외국인 장애인 관광객"이라는 세그먼트 자체는 가치 있지만, 자신(한국인 장애인)에게는 맞지 않는다는 결론을 내린다.

### 3단계: 결과

이재호 씨는 K-Guide Global의 기술 스택과 API 연동 방식을 20분간 분석하고 앱을 닫는다. 해커톤 제안서에는 "K-Guide Global은 외국인 전용으로 설계된 올바른 결정이나, 무장애 API 활용이 표면적 수준에 그침. AccessKorea 또는 유사 서비스가 한국인 장애인에게 더 적합한 대안"이라고 작성한다. 앱은 참고 사례로 저장하되 개인적으로 사용하지는 않는다.

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **2** | 한국인 장애인 개발자에게 실질적 관광 가치 없음. 기술 참고용으로만 의미 있음 |
| 사용 편의 | **4** | 앱 자체 접근성은 평균 이상이나 무장애 정보의 세분화 부족, 오디오 자막 없음 |
| 구현 타당성 | **6** | 27개 API 연동 구조는 탄탄하나 장애인 관련 API 엔드포인트를 충분히 활용하지 못함 |

**종합 점수: 12/30**

## Pass/Fail 판정

> **❌ FAIL**

K-Guide Global은 외국인 방문객을 위한 서비스로, 한국인 이재호 씨의 일상적 장애인 관광 니즈와 근본적으로 맞지 않는다. 개발자 관점에서 API 구조를 탐색하는 데에는 부분적 가치가 있었으나, 이는 앱의 본래 목적과 무관한 활용이다. 한국인 장애인을 위한 서비스는 AccessKorea 또는 행정안전부 무장애 관광 플랫폼이 더 적합하다.

## 개선 제안

1. **장애인 편의시설 API 심층 활용**: Visit Korea 공공 API 중 미활용 중인 장애인 편의시설 엔드포인트(경사로 세부 규격, 화장실 접근성, 안내견 동반 가능 여부)를 앱에 연동하면 외국인 장애인 관광객(외국인 휠체어 사용자 등) 커버리지가 크게 향상됨.
2. **오디오 가이드 자막 및 속도 조절 추가**: 청각 장애인 외국인 관광객을 위한 자막 기능, 고령 방문객을 위한 재생 속도 0.75×/1.25× 조절 기능을 오디오 가이드에 추가.
3. **한국인 타겟 분리 안내**: 앱 진입 시 한국어 UI를 선택하려는 한국인 사용자에게 "이 앱은 외국인 방문객 전용입니다. 한국인 장애인 여행자는 [AccessKorea]를 이용해주세요"와 같은 명확한 안내 제공 — 잘못 접근한 사용자 이탈을 즉시 최소화.
4. **색각 접근성 개선**: 혼잡도 인디케이터를 색상 + 아이콘 + 수치(예: "혼잡 78%")로 병행 표시해 색각 이상 사용자 포용.

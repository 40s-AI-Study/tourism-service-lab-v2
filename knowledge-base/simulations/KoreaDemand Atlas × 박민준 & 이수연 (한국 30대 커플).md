# SIM-182: KoreaDemand Atlas × 박민준 & 이수연 (한국 30대 커플)

## 시나리오

결혼 1주년 기념 여행을 2주 앞두고 민준이 KoreaDemand Atlas를 발견한다. IT PM답게 "데이터 기반"이라는 키워드에 즉시 반응해 앱을 설치한다. 강원도 히트맵을 살펴보던 중 평창 인근 한 지역이 "수요 상승, 관광 다양성 지수 높음, 아직 혼잡 미도달"로 표시된 것을 발견한다. 수연은 감성 숙소 사진을 핀터레스트에서 찾고 있었는데, 민준이 "여기 뜨기 전이래, 데이터가 증명함"이라며 앱을 보여주자 설득된다. 두 사람은 앱에서 지역 잠재력 점수와 방문 최적 타이밍을 확인하고 주말 예약을 서두른다.

---

## 사용자 여정

### 1단계: 진입
- 민준이 Product Hunt 스타일 앱 소개 글을 LinkedIn에서 발견 → 설치
- "데이터 기반 여행지 발굴"이라는 포지셔닝이 PM 직업 특성상 즉각적 공감 유발
- 첫 화면 히트맵에서 강원도 선택 → 지역별 잠재력 점수 비교 화면 탐색

### 2단계: 행동
- 평창 인근 A지역 탭: 관광 잠재력 종합 점수 68점, 소비 다양성 지수 상위 15%, 방문자 집중률 "2주 후 급상승 예측" 알림 확인
- "지금 예약하면 붐비기 전 방문 가능" 메시지 → 즉각적 행동 유발
- 지역 핵심 명소 목록 확인 → 풀빌라 숙소 형태의 지역이 포함되어 있어 수연도 만족
- 코스 생성: 토요일 오전 도착 → 명소 2곳 → 저녁 지역 식당 → 일요일 스파 동선 자동 생성
- 단, 앱 내 예약 연동이 없어 숙소는 야놀자, 식당은 카카오맵으로 별도 이탈하여 예약

### 3단계: 결과
- 실제 방문 시 예측대로 주중보다 한산한 주말 경험 — 만족도 높음
- 앱을 "여행지 발굴 도구"로는 인정하나, 예약·결제 미연동으로 최종 플래닝은 3개 앱을 전전
- 수연: "어디 갈지 결정하는 데만 유용하고, 나머지는 결국 딴 앱"이라는 평

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **7** | 여행지 발굴 및 타이밍 결정에서 명확한 가치 제공. 민준의 데이터 선호 성향과 잘 맞음. 그러나 예약 연동 부재로 실용성 반쪽 |
| 사용 편의 | **7** | PM 배경의 민준에게는 충분히 직관적. 수연은 비주얼 탐색(핀터레스트식) 기능 없어 다소 건조하다고 느낌 |
| 구현 타당성 | **8** | 빅데이터 API 4종 + 방문 예측 조합은 기술적으로 실현 가능하며 차별성 명확 |

**종합 점수: 22/30**

## Pass/Fail 판정

> **✅ PASS**

KoreaDemand Atlas는 데이터 지향적인 민준의 여행지 선택 과정에서 명확한 역할을 수행하며, "뜨기 전에 가는" 동기 부여 프레이밍이 커플 의사결정을 효과적으로 촉진한다. 유용성 7점으로 기준 충족. 단, 예약 연동 부재로 인한 앱 파편화가 이 커플의 가장 큰 pain point인 "코스 기획 부담"을 완전히 해소하지 못해 추가 기능 개발이 필요하다.

## 개선 제안

1. 야놀자·에어비앤비·카카오맵 딥링크 연동으로 앱 내 발굴 → 외부 예약 원스텝 전환 흐름 구성
2. 히트맵 외에 "감성 미리보기" 사진 갤러리 섹션 추가 — 수연처럼 비주얼 탐색 선호 여성 사용자 포용

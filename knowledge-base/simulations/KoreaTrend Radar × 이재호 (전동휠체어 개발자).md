# SIM-47: KoreaTrend Radar × 이재호 (전동휠체어 개발자)

## 시나리오

2026년 4월 17일 일요일 오후 2시. IT 개발자 이재호(38)는 전동휠체어를 사용하며 서울 서초구에 거주한다. 주말 외출 장소를 고르면서 KoreaTrend Radar를 실행했다 — 데이터 기반 앱에 대한 직업적 호기심도 있고, "요즘 뜨는 곳이 실제로 어디인지" 빅데이터로 확인하고 싶었다. 그는 트렌드 데이터를 신뢰하고 적극 활용할 의향이 있는 파워 사용자다.

앱의 트렌드 레이더 기능 자체는 인상적이다. API 구조도 머릿속에서 그려지고, 히트맵과 지표 그래프도 직관적으로 읽힌다. 하지만 결정적 문제가 생긴다 — 앱에 무장애 접근성 필터가 없다. 트렌딩 1위 장소인 북촌 한옥마을을 클릭해보니 경사로·엘리베이터·휠체어 진입 가능 여부 정보가 전혀 없다. 트렌딩 데이터가 아무리 좋아도 "내가 실제로 갈 수 있는 곳인지" 알 수 없으면 의미가 없다. 결국 별도로 에이블링 앱을 열어 무장애 여행지를 확인하는 이중 작업이 발생한다.

---

## 사용자 여정

### 1단계: 진입
- 앱 실행 → 홈 레이더 화면 확인
- 무장애 필터 탐색 → 부재 확인
- 트렌딩 TOP 10 리스트 열람 (기술적 관심으로 계속 사용)

### 2단계: 행동
- 북촌 한옥마을 (트렌딩 1위) 상세 화면 확인
- 이동통신·카드·내비 지표 분석 (개발자 시각에서 API 구조 추정)
- 무장애 정보(경사로, 엘리베이터, 주차) 탐색 → 정보 없음
- 에이블링 앱 병행 실행으로 북촌 무장애 여부 별도 확인
- 북촌이 휠체어 진입 어려운 지형임 확인 → 목적지 변경
- 서울숲 트렌드 데이터 확인 → 에이블링에서 무장애 확인 병행

### 3단계: 결과
- 서울숲 방문 결정 (두 앱 교차 확인 후)
- KoreaTrend Radar 트렌드 데이터 자체는 유용하다고 평가
- 앱 피드백: "무장애 필터 없으면 장애인 사용자에게는 반쪽짜리"

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **6** | 트렌드 데이터 자체의 품질과 신뢰성은 높이 평가. 그러나 무장애 정보 부재로 실제 여행 결정에 단독 활용 불가 — 이중 앱 사용 강제 |
| 사용 편의 | **7** | IT 개발자로서 데이터 대시보드·지표 해석에 어려움 없음. UI/UX 자체는 긍정 평가. 무장애 접근성 필터 부재가 기능적 결함 |
| 구현 타당성 | **6** | area-tourism-resource-demand API에 무장애 시설 정보 연계 가능성 있으나 현재 미구현. 외부 무장애 DB와 연동 필요 |

## 개선 제안

1. **무장애 오버레이 필터**: area-tourism-resource-demand API 또는 외부 무장애 DB와 연동해 "휠체어 접근 가능" 토글 필터 추가. 트렌딩 장소에 무장애 배지 표시
2. **접근성 정보 통합 카드**: 각 장소 상세 페이지에 경사로·엘리베이터·장애인 화장실·주차 여부를 아이콘으로 요약한 '접근성 정보' 섹션 추가

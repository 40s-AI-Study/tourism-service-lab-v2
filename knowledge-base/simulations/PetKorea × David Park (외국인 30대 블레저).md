# SIM-35: PetKorea × David Park (외국인 30대 블레저)

## 시나리오

2026년 9월, David Park(35세, 미국 PM, 한국계 2세)은 서울 출장 7박 중 주말 이틀을 관광에 활용하는 블레저(Bleisure) 여행 중이다. 반려동물은 미국에 있으며 한국 여행 중 동반할 계획이 전혀 없다. 강남 호텔 체크인 후 동료 한국인이 주말 나들이 앱으로 PetKorea를 언급한다. David는 영어 사용자로, 앱이 영어를 지원하는지 먼저 확인한다.

앱을 설치해보니 한국어 전용임을 확인한다. 한국어를 어느 정도 읽을 수 있지만 유창하지 않아 메뉴 탐색에 시간이 걸린다. 반려동물 동반 니즈가 없으므로 "반려동물 없이 방문하기 좋은 곳"을 찾으려 하지만 앱의 모든 검색·추천이 펫 프렌들리 필터 기반이라 일반 관광 정보로는 활용이 어렵다. 결국 앱을 닫고 Google Maps와 Naver로 대체한다.

---

## 사용자 여정

### 1단계: 진입
- 한국인 동료 구두 추천으로 앱 설치
- 언어 설정 확인 → 영어 미지원
- 한국어 메뉴 탐색 시도 (기초 한국어 보유)
- 반려동물 프로필 등록 건너뜀

### 2단계: 행동
- 메인 화면에서 관광지 카테고리 탐색 시도
- `tourism-info-korean` API 기반 일반 관광 정보 존재하나, 반려동물 동반 필터가 기본 활성화되어 있어 일반 여행자 관점에서 결과 해석 혼선
- `visitor-concentration-forecast` API → 혼잡도 위젯 발견, 한국어 라벨이지만 색상 코드로 의미 파악 가능
- 경복궁·북촌 한옥마을 검색 시도 → 반려동물 동반 불가 장소라 검색 결과에서 누락 또는 하위 노출
- 앱 내 영어 콘텐츠 없음 확인 후 이탈

### 3단계: 결과
- PetKorea 앱 이탈, Google Maps·Naver 지도로 주말 일정 독립 계획
- 반려동물 동반 니즈 없는 외국인에게 앱 핵심 가치 전달 실패
- 경복궁·이태원 방문은 성공했으나 PetKorea와 무관

## 개선 제안

1. **영어 인터페이스 최소 지원**: 한국계 해외 교포 및 외국인 관광객 유입을 위해 영어 UI 우선 추가 (일본어·중국어 순차 확장)
2. **반려동물 없는 방문자 모드**: "펫 카페 체험" 또는 "반려동물 문화 탐방" 카테고리를 일반 관광 탭으로 분리해 비펫 사용자에게도 진입점 제공
3. **타겟 사용자 명확화**: 앱스토어 소개 및 초기 온보딩에서 "반려동물 동반 여행 전용 앱"임을 명확히 안내해 비타겟 사용자의 기대 불일치 방지

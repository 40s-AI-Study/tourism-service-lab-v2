# SIM-46: KoreaTrend Radar × 김영자 (한국 시니어)

## 시나리오

2026년 4월 17일 토요일 오전 11시. 은퇴 교사 김영자(67)는 친구들과 5월 여행을 계획 중이다. 딸이 "요즘 핫한 여행지 알려주는 앱이 있대"라며 KoreaTrend Radar를 설치해줬다. 평소 스마트폰은 카카오톡과 유튜브만 쓰는 그녀에게 새 앱은 낯선 도전이다.

앱을 열자 화면 가득 색깔별 히트맵과 수치 그래프가 펼쳐진다. "이게 뭔 소리야" — 이동통신 유동인구 지수, 카드 소비 지수, 내비게이션 목적지 설정 건수 등 데이터 용어가 즐비하다. 글씨도 작고, 지도 위에 겹쳐진 레이어들이 어디를 눌러야 할지 모르게 만든다. 30분 동안 이것저것 눌러봤지만 결국 "어디가 좋은지"를 알 수 없었다. 딸에게 전화해 "이 앱 너무 어렵다, 그냥 블로그 볼게"라고 한다.

트렌드 빅데이터 대시보드 UI는 디지털 친화적인 젊은 세대를 전제로 설계되어 있어, 저기술 시니어 사용자에게는 근본적인 접근성 장벽이 존재한다.

---

## 사용자 여정

### 1단계: 진입
- 앱 실행 → 홈 히트맵 화면 진입
- 화면 구성 파악 시도 → 레이어 복잡도로 혼란
- 글씨 크기 작아 가독성 저하

### 2단계: 행동
- 지도 이곳저곳 탭 → 원하는 정보 도달 실패
- 카테고리 필터 버튼 발견했으나 용어 불명확
- 특정 장소 상세 화면 진입 → 수치 그래프만 있고 "좋은지 나쁜지" 판단 불가
- 뒤로가기 반복 후 앱 이탈

### 3단계: 결과
- 앱 사용 포기, 삭제 검토
- 여행 정보는 네이버 블로그로 탐색
- 딸에게 "이 앱 시니어용 아님" 피드백 전달

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **2** | 트렌드 데이터 자체는 가치 있으나 시니어 페르소나가 해석·활용할 수 없어 실질 유용성 거의 없음 |
| 사용 편의 | **2** | 히트맵 레이어, 소형 텍스트, 데이터 용어 등이 저기술 사용자에게 극심한 장벽. 접근성 설계 부재 |
| 구현 타당성 | **6** | 앱 기능 자체는 구현 가능하나, 시니어 접근성 모드(큰 글씨, 단순 UI) 미구현이 이 페르소나에서의 타당성을 제한 |

**종합 점수: 10/30**

## Pass/Fail 판정

> **❌ FAIL**

유용성 2점, 종합 10/30으로 기준 대폭 미달. 빅데이터 대시보드 중심의 UI가 저기술 시니어 사용자에게 근본적으로 접근 불가능한 수준. 이 페르소나는 앱의 주요 타겟이 아니나, 시니어 접근성 모드 없이는 해당 세그먼트 완전 이탈.

## 개선 제안

1. **시니어 모드 / 쉬운 화면**: 히트맵 대신 "이번 주 추천 여행지 TOP 5" 카드형 리스트로 전환하는 단순 모드 제공. 큰 글씨(18pt 이상), 고대비 색상, "지금 뜨는 곳이에요" 같은 자연어 설명 표시
2. **음성 안내 + 챗봇 진입점**: "어디 가고 싶으세요?" 라는 음성/텍스트 질문으로 시작해 트렌드 데이터를 대화형으로 안내하는 진입 경로 추가

# SIM-38: PetKorea × 타나카 하루카 (일본 직장인)

## 시나리오

2026년 11월, 타나카 하루카(27세, 도쿄 마케터)는 3박4일 한국 여행 중이다. K-드라마 성지 순례가 주목적으로 반려동물 동반 계획은 전혀 없다. 도쿄에서 기르는 고양이 '미케'는 집에 두고 왔다. 인천공항 입국 후 숙소 이동 중 한국인 여행 블로거의 SNS에서 PetKorea 앱이 "귀여운 동물 카페 찾기 좋다"는 언급을 발견한다.

하루카는 일본어와 기초 영어를 구사하며 한국어는 드라마로 배운 수준이다. 앱을 설치하자마자 일본어 미지원을 확인하고 영어 설정을 시도하지만 영어도 없다. 한국어 인터페이스에서 사진 위주로 탐색하며 강남 근처 고양이 카페를 찾는다. 한국의 펫 카페 문화가 일본에도 있지만 규모와 분위기가 달라 콘텐츠 소재로 흥미롭다고 생각한다.

검색 결과에서 사진이 예쁜 카페 2곳을 발견하지만 영업시간·위치·예약 방법을 읽을 수 없어 외부 앱으로 이탈한다.

---

## 사용자 여정

### 1단계: 진입
- 한국 여행 블로거 SNS 게시물에서 앱 언급 발견 후 설치
- 언어 설정 탐색 → 일본어·영어 모두 미지원 확인
- 한국어 인터페이스에서 사진·아이콘 위주로 탐색 시도
- 반려동물 프로필 등록 없이 탐색

### 2단계: 행동
- `pet-friendly-travel` API (카테고리: 카페) → 강남 고양이 카페 검색 (한국어 검색어 복붙)
- 카드 UI에서 사진·별점·위치 아이콘 식별 가능, 텍스트 내용은 이해 불가
- `visitor-concentration-forecast` API 혼잡도 색상 코드(초록/노랑/빨강) 직관적으로 파악
- 카페 상세 페이지 클릭 → 영업시간·예약 방법·입장료 텍스트 이해 불가 → 앱 이탈
- 네이버 지도 영어 버전으로 대체 검색

### 3단계: 결과
- PetKorea 앱에서 카페 2곳 사진 확인 후 네이버 지도로 이탈
- 결국 강남 고양이 카페 1곳 방문 성공 (네이버 지도로 찾은 곳)
- PetKorea 기여도: 카페 후보 발견 역할만 수행 (사진 탐색), 실제 방문 결정은 타 앱
- 반려동물 동반 니즈 없어 앱 핵심 기능 미활용
- 재설치 의향 없음

## 개선 제안

1. **일본어 지원 우선 추가**: 일본인 한국 방문객은 연 300만 명 이상으로 펫 카페 체험 수요가 높음. 일본어 UI 추가 시 신규 사용자층 확보 가능
2. **비주얼 퍼스트 카드 UI 강화**: 언어 장벽이 있는 외국인도 사진·아이콘·색상만으로 핵심 정보(영업 여부, 혼잡도, 가격대)를 파악할 수 있는 픽토그램 기반 UI 설계
3. **펫 카페 체험 전용 외국인 탭**: "Visit Pet Cafe" 영문 탭으로 외국인 관광객의 펫 문화 체험 수요를 별도 수용하여 앱 사용 범위 확장

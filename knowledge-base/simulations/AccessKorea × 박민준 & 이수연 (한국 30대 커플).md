# SIM-12: AccessKorea × 박민준 & 이수연 (한국 30대 커플)

## 시나리오

박민준(33세)의 어머니가 갑자기 무릎 수술 후 휠체어를 사용하게 됐다. 마침 부모님과 함께 제주도 여행 계획이 있는데, 어머니 동선을 고려해야 해서 AccessKorea를 설치한다. 이수연은 "로맨틱한 커플 코스"보다 이번만큼은 "어머니가 편히 다닐 수 있는 코스"를 우선순위로 둔다.

---

## 사용자 여정

### 1단계: 진입
- 앱 실행 → 온보딩: "휠체어 사용자 동행" 선택
- "제주도, 2박 3일, 휠체어 동반" 입력

### 2단계: 행동
- AI 무장애 코스 생성:
  ```
  [1일차]
  10:00 성산일출봉 (입구 전망대까지 휠체어 접근 가능, 정상 제외)
  13:00 무장애 인증 한식당 (엘리베이터 완비)
  15:00 섭지코지 해안 산책로 (휠체어 통행 가능 구간 안내)

  [2일차]
  09:00 제주 민속촌 (평지 코스 안내)
  12:00 협재해수욕장 (휠체어용 모래 경사로 유무 표시)
  ```
- 각 장소에 "휠체어 접근 가능 구간", "엘리베이터 위무", "장애인 화장실 위치" 상세 표시
- 오디오 가이드 자동 재생 → 어머니도 이어폰으로 해설 청취
- 민준이 어머니 대신 앱 탐색 → 코스 링크 공유

### 3단계: 결과
- 어머니가 "이런 앱이 있으면 예전에 포기했던 여행도 갔을 텐데" 감동
- 이수연: "나중에 우리 부모님 모실 때도 이걸 써야겠다"
- 커플 본인들에게는 직접적 유용성 낮지만 가족 동반 여행 맥락에서 필수 도구

---

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **8** | 휠체어 동반 상황에서 대체 불가한 정보 제공. 단, 이 커플의 일상적 여행에는 해당 안 됨 |
| 사용 편의 | **8** | 동반자 조건 입력 후 코스 생성까지 직관적 |
| 구현 타당성 | **8** | `barrier-free-travel` API 실제 제공, 무장애 코스 생성 가능 |

**종합 점수: 24/30**

---

## Pass/Fail 판정

> **⚠️ CONDITIONAL PASS**

P2 커플은 AccessKorea의 일상적 타겟은 아니지만, "거동 불편한 가족 동반" 상황에서 강력한 유즈케이스가 발생한다. 이 상황에서는 PASS. 단, 커플 여행 일반 맥락에서는 타겟 외 사용자.

---

## 개선 제안

1. **"동행자 중 거동 불편하신 분" 진입 경로 추가**: 일반 여행자도 "부모님/조부모님 동반" 상황에서 진입 가능하도록 온보딩 개선
2. **임시 조건 입력**: "이번 여행만 휠체어 동반" 옵션 → 다음 여행에서 일반 모드 전환 가능
3. **커플+가족 코스 분리**: 무장애 코스 내에서도 "로맨틱 포인트" 태그 → P2처럼 미적 기준도 포기 안 하는 사용자 만족

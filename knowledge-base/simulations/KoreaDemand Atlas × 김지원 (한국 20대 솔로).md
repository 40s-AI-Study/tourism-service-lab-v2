# SIM-181: KoreaDemand Atlas × 김지원 (한국 20대 솔로)

## 시나리오

금요일 밤 11시, 지원은 인스타그램에서 "강원도 어딘가"라는 모호한 릴스를 보고 다음날 당일치기를 결심한다. 그런데 이번엔 평소와 달리 검색 대신 친구가 공유해준 KoreaDemand Atlas를 열어본다. 앱 첫 화면에 전국 지도 위로 수요 강도 히트맵이 펼쳐지고, 강릉 주변은 이미 붉게 달아오른 반면 양양 인근 소도시 하나가 노란색(상승 중, 아직 붐비지 않음)으로 표시된다. "뜨기 전에 가세요" 알림이 달린 그 지역을 눌러보니 잠재력 점수 72점, 방문자 집중률 예측 그래프가 뜬다. 지원은 데이터가 보장하는 "선점 여행지"라는 프레이밍에 흥미를 느끼고, 인스타에서 찾아봤던 곳보다 더 설레는 느낌을 받는다.

## 사용자 여정

### 1단계: 진입
- 친구 카카오톡 공유 링크로 앱스토어 진입 → 설치 (약 2분)
- 첫 실행 시 전국 히트맵 화면이 바로 등장 — 별도 온보딩 없이 지도가 핵심임을 직관적으로 인식
- "뜨기 전에 가세요" 뱃지가 붙은 지역들이 눈에 띄어 즉시 탐색 시작

### 2단계: 행동
- 양양 인근 소도시 탭 → 관광 잠재력 종합 점수 72/100, 체류 강도 상승 중(+34% 전월 대비) 확인
- 방문자 집중률 예측 그래프: 내일 오전 9-11시 혼잡도 낮음 → 출발 시각 결정
- 지역 핵심 명소 탭으로 이동 → 카페 3곳, 해변 포토스팟 2곳, 로컬 식당 정보 확인
- "코스 생성" 버튼 탭 → 자동 생성된 반나절 코스 저장
- 그러나 지역 명소 상세 페이지에서 실제 영업 여부·후기가 없고, 데이터 지표 설명이 어려워 두 번 읽어야 이해됨

### 3단계: 결과
- 다음날 오전 실제 방문 → 예측대로 한산, 포토스팟 독점 촬영 성공
- 인스타 게시 후 "이 앱 어디서 알았어?" 댓글 7개 → 앱 자발적 공유
- 단, 코스에 포함된 로컬 식당 한 곳이 현재 휴업 중이라 현장에서 당황 → 신뢰도 소폭 하락

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **7** | 혼잡도 회피 + 선점 여행지 발굴은 김지원의 핵심 니즈에 정확히 부합. 단, 실제 운영 정보(영업 여부) 부재로 완전한 유용성 미달 |
| 사용 편의 | **6** | 히트맵 직관적이나 수요 강도 지표 용어(체류 강도, 소비 강도)가 일반 소비자에게 낯설어 학습 곡선 존재 |
| 구현 타당성 | **8** | 빅데이터 API 4종 통합 기반으로 실현 가능하며, 방문자 집중 예측 기능은 실제 차별점으로 작동 |

**종합 점수: 21/30**

## Pass/Fail 판정

> **✅ PASS**

KoreaDemand Atlas는 김지원의 "사람 적은 곳 선점" 니즈와 히트맵 기반 데이터 탐색 방식이 잘 맞아 핵심 가치를 전달한다. 유용성 7점으로 기준(4점 이상)을 충족하고, 실제 방문 성공 경험이 자발적 바이럴로 이어지는 전환을 확인했다. 다만 실시간 영업 정보 연동이 없으면 현장 실망 사례가 반복되어 리텐션이 약해질 수 있다.

## 개선 제안

1. 수요 지표 설명을 "이 지역, 지금 뜨는 중 (방문자 +34%)" 같은 자연어 요약으로 전환해 데이터 리터러시 진입 장벽 낮추기
2. 지역 명소 데이터에 영업 상태 크라우드소싱 레이어 추가 — 최근 방문자 "정상 영업 확인" 뱃지로 신뢰도 보완

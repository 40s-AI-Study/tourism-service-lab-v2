# SIM-22: KoreaPath AI × 이재호 (전동휠체어 개발자)

## 시나리오

38세 IT 개발자 이재호는 척수 장애로 전동휠체어를 사용하며, 제주 2박 3일 여행을 혼자 계획하고 있다. 경사도·바닥 재질·충전소 위치 같은 세부 접근성 조건을 일일이 검색하는 기존 방식에 지쳐 있던 그는 KoreaPath AI에 상세 휠체어 조건을 한 번에 입력하고 최적 동선을 자동 생성받는다. INTJ 성향답게 데이터 근거를 꼼꼼히 검증하며 최종 코스를 확정한다.

---

## 사용자 여정

### 1단계: 진입

이재호는 개발자답게 앱 스토어 리뷰와 GitHub 이슈 트래커를 훑어본 뒤 KoreaPath AI를 설치한다. 첫 실행 시 "이동 약자 맞춤 모드"를 선택하고, 프로필에 "전동휠체어 사용자"를 등록한다. 앱이 "상세 접근성 조건을 입력하면 AI가 최적 경로를 계산합니다"라는 온보딩 안내를 표시하며, 이재호는 곧바로 고급 필터 화면으로 진입한다.

### 2단계: 행동 (핵심 기능 3회 이상 사용 장면)

**인터랙션 1 — 상세 접근성 조건 입력**
이재호가 고급 조건 입력 패널에 다음 값을 정밀하게 입력한다.
- 목적지: 제주도
- 여행 기간: 2박 3일
- 인원: 1명
- 예산: 1인 25만 원/일
- 이동 장비: 전동휠체어 (폭 68cm, 길이 110cm)
- 접근성 조건: "경사도 15° 이하만", "바닥 포장 구간 우선", "비포장·모래사장 제외", "전동휠체어 충전소 반경 3km 이내"
- 혼잡도: "동시 방문객 200명 이하 시간대 선호"
- 관광 유형: 자연 경관, 오름, 해안도로

"AI 코스 생성" 버튼을 누르면 barrier-free-travel API, area-tourism-demand-density API, related-attractions API가 병렬 호출되며 "상세 조건 분석 중 (경사도 데이터 검증 포함)…" 메시지와 함께 진행 바가 표시된다. 12초 후 "접근성 조건을 만족하는 제주 코스 3개를 생성했습니다"라는 결과가 나타난다.

**인터랙션 2 — 코스 비교 및 데이터 검증**
AI가 생성한 3개 코스(A: 서부 해안 중심, B: 동부 오름 중심, C: 시내 문화 중심)가 카드 형태로 나열된다. 이재호가 코스 A를 탭하면 각 장소별 세부 데이터가 표시된다.

- 협재해변: "포장 산책로 700m, 경사도 최대 3°, 모래 구간 우회 경로 있음, 충전소 1.2km"
- 한림공원: "전동휠체어 전용 입구 있음, 내부 포장로 100%, 경사도 평균 5°"
- 오설록 티뮤지엄: "주차장~전시관 경사로 완비, 단차 없음, 충전소 구내 설치"

이재호가 "데이터 출처 확인" 버튼을 탭하면 각 수치의 API 소스(공공 barrier-free 데이터베이스 최종 업데이트 날짜 포함)와 크라우드소싱 리뷰 링크가 표시된다. "협재해변 최신 리뷰" 섹션에서 다른 휠체어 사용자가 한 달 전 방문한 후기를 확인하고 신뢰도를 높인다.

**인터랙션 3 — 혼잡도 기반 방문 시간 조정**
코스 A의 2일차에 배치된 성산일출봉 일정을 탭하면 72시간 예측 혼잡도 그래프가 표시된다. 오전 7시 혼잡도 지수 12(매우 낮음), 오전 10시 45(보통), 오후 2시 89(매우 높음)로 시각화된다. 이재호가 "전동휠체어 접근 불가 구간 있음" 경고 배지를 발견하고 탭하자 "정상 등반로는 계단 구간 포함, 성산일출봉 전망 데크(지상 레벨)로 대체 경로 제안" 팝업이 나타난다. 이재호가 "대체 경로로 교체" 버튼을 누르면 일정이 자동 수정되고 예상 혼잡도 최적 방문 시간이 오전 7:30으로 재설정된다.

**인터랙션 4 — 충전소 경로 통합**
최종 확정된 코스 화면 하단에 "전동휠체어 충전소 지도" 탭이 제공된다. 각 숙소, 관광지 주변 충전소 위치와 충전 타입(220V 일반/급속), 영업시간이 지도 핀으로 표시된다. 숙소(제주시 장애인 편의 호텔)에는 "객실 내 220V 콘센트 2구, 욕실 손잡이, 전동휠체어 회전 반경 확보" 정보가 상세히 표시된다.

### 3단계: 결과

이재호는 제주 여행 내내 사전에 검증된 접근성 정보 덕분에 즉흥적인 경로 변경 없이 계획대로 이동했다. 협재해변 포장 산책로에서 일몰을 감상하고, 오설록에서 충전하며 여유롭게 티타임을 즐겼다. 귀가 후 앱 내 리뷰 기능으로 성산일출봉 대체 경로의 실제 경사도를 업데이트하여 다음 사용자를 위한 크라우드소싱 데이터에 기여했다.

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **10** | 경사도·바닥 재질·충전소 등 휠체어 사용자 복합 조건을 단일 입력으로 처리. 기존에 수 시간 걸리던 사전 조사를 10분으로 단축 |
| 사용 편의 | **9** | 고급 조건 입력 패널이 개발자 친화적으로 상세함. 데이터 출처 투명성이 INTJ 이재호의 신뢰 요구를 충족 |
| 구현 타당성 | **8** | barrier-free-travel API의 경사도 데이터 커버리지가 제주 주요 관광지에 한해 현실적. 오프로드 구간 데이터 공백은 크라우드소싱으로 보완 가능 |

**종합 점수: 27/30**

## Pass/Fail 판정

> **✅ PASS**

전동휠체어 사용자의 복잡한 접근성 조건(경사도, 바닥 재질, 충전소, 회전 반경)을 AI 단일 입력으로 통합 처리하는 것은 이재호가 기존에 여러 플랫폼을 오가며 수행하던 사전 조사 작업을 획기적으로 단축한다. 데이터 출처 투명성과 크라우드소싱 리뷰 연동이 기술 친화적 INTJ 사용자의 신뢰 요구를 충족시켜 실사용으로 이어진다.

## 개선 제안

1. **실시간 현장 피드백 루프**: 방문 중 "이 경로 접근성 평가" 마이크로 인터랙션을 추가하여 크라우드소싱 데이터 갱신 주기를 단축
2. **전동휠체어 배터리 소모량 예측**: 코스 거리·경사도 데이터를 기반으로 배터리 소모량을 예측하고 충전 타임을 일정에 자동 삽입
3. **API 데이터 공백 알림**: 특정 구간의 접근성 데이터가 6개월 이상 미갱신인 경우 "현장 확인 권장" 배지를 표시하여 신뢰도 관리

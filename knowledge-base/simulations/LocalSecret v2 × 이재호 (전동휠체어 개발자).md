# SIM-R3-038: LocalSecret v2 × 이재호

## 시나리오

이재호(38세)는 서울 마포구에 사는 IT 개발자다. 전동휠체어 사용자다. 혼잡한 장소는 그에게 두 가지 의미에서 문제다. 하나는 감각적 불편함 — 사람이 많으면 이동 경로가 막히고, 좁은 공간에서 전동휠체어와 인파가 뒤섞이면 아찔한 상황이 생긴다. 다른 하나는 정보의 신뢰성 — 무장애 여부가 블로그에는 "가능하다"고 적혀 있지만 실제 가보면 계단 3개, 턱 5cm, 화장실 없음인 경우가 비일비재하다.

재호에게 여행 앱의 가장 큰 문제는 무장애 정보의 부정확도다. 그는 방문 전에 항상 직접 전화로 확인한다. 앱이나 블로그를 믿지 않는다. 그런 그가 LocalSecret v2에 관심을 가진 건 GitHub 개발자 커뮤니티의 리뷰 때문이었다. "이 앱은 데이터 출처를 명시한다" — 개발자로서 그 한 줄이 달랐다.

앱을 열었을 때, 그는 감정 선택보다 먼저 설정을 열었다. 무장애 필터가 있는지 확인했다. 있었다. 그것도 "계단 없음", "경사로 있음", "전동휠체어 진입 가능", "장애인 화장실 여부"로 세분화되어 있었다. 재호가 처음으로 스마트폰 화면 앞에서 잠시 멈춘 순간이었다.

---

## 사용자 여정

### 1단계: 진입
- GitHub 개발자 커뮤니티 리뷰 ("데이터 출처 명시")에서 앱 신뢰성 첫 인지
- 앱 설치 후 감정 선택보다 설정 메뉴 먼저 진입 — 무장애 필터 세분화 확인
- "전동휠체어 진입 가능" 항목 존재 자체가 결정적 신호 — 처음으로 앱을 진지하게 탐색하기 시작
- 데이터 출처 탭에서 공공 무장애 인증 데이터 + 방문자 실측 후기 조합 확인, 개발자로서 아키텍처 이해

### 2단계: 행동
- 🌿고요 선택 — 혼잡도 낮은 자연, 이동 공간 확보 목적
- 추천 결과: 서울 하늘공원 비주요 구간 (전동휠체어 전 구간 진입 가능, 방문자 집중률 하위 7%)
- 각 장소의 무장애 정보: 데이터 수집 날짜, 출처 기관, 실측 후기 작성자 정보 투명하게 표시
- 혼잡도 예측 그래프: 평일 오전 이른 시간대가 최저, 주말 오후는 가족 단위 방문 급등
- 방문 전 앱에 표시된 번호로 전화 확인(습관) — 앱 정보와 일치 확인, "이 앱은 믿어도 되겠다" 결론
- 하늘공원 비주요 구간 방문: 예측대로 조용함, 전동휠체어 이동 경로 여유로움
- 앱에 실측 후기 작성: "2026년 4월 방문, 경사로 상태 양호, 화장실 휠체어 진입 가능 확인 (직접 측정: 출입구 폭 85cm)"

### 3단계: 결과
- "이 앱 정보는 믿을 수 있다"는 결론 — 재호가 앱에 내리는 가장 높은 평가
- 이후 방문한 3개 장소 모두 앱 정보와 현장 일치, 신뢰 강화
- 장애인 여행자 온라인 커뮤니티에 앱 소개 글 작성, "무장애 데이터 출처 명시 앱"으로 소개

| 사용 편의 | **8** | 설정 메뉴의 무장애 필터 접근성 우수. 다만 첫 화면 감정 선택 전 무장애 설정이 바로 보이면 더 좋음 |
| 구현 타당성 | **7** | 공공 무장애 인증 데이터 API 존재. 실측 후기 수집 시스템 구축이 데이터 정확도의 핵심 |

## Pass/Fail 판정

> **✅ PASS**

재호에게 여행 앱의 PASS 기준은 단 하나 — "데이터가 맞는가"다. 세 번 방문에서 세 번 모두 앱 정보와 현장이 일치한 결과가 이 판정의 근거다. 무장애 필터 세분화와 데이터 출처 명시가 만성적 불신을 깨뜨렸고, 재호가 장애인 여행자 커뮤니티에 앱을 소개한 것은 가장 강력한 PASS 신호다.

## 🔄 리텐션 평가: 이 페르소나가 앱을 다시 열까?

**재사용 확률**: ⬆️높음

**감정적 이유**:
> "드디어 전화 안 해도 되는 앱을 찾은 것 같다. 정보가 맞았다. 개발자로서 데이터 출처가 보이는 앱은 처음이었다. 이 신뢰를 깨는 순간까지는 계속 쓸 것이다."

**재사용 트리거**:
- 다음 방문 계획 시 무장애 필터로 첫 검색
- 장애인 여행자 커뮤니티에서 앱 추천 반응 확인 후 재접속
- 새로 오픈한 장소의 무장애 정보 업데이트 확인

**이탈 리스크**:
- 한 번이라도 무장애 정보가 현장과 다르면 신뢰 즉각 붕괴, 복구 불가
- 데이터 업데이트 주기가 느려지면 "오래된 정보"로 인식하고 다시 전화 확인 습관으로 회귀

## 개선 제안

1. **무장애 정보 첫 화면 접근**: 앱 진입 시 감정 선택 이전에 "무장애 여행자 모드"를 선택할 수 있는 경로 제공 — 재호처럼 설정 메뉴를 먼저 찾는 사용자가 첫 화면에서 자신에게 맞는 모드를 바로 선택할 수 있어야 한다.
2. **실측 후기 인센티브 시스템**: 장애인 방문자가 무장애 실측 정보(출입구 폭, 경사도, 화장실 유무)를 후기로 남기면 "현장 검증 기여자" 배지 부여 — 재호가 자발적으로 후기를 남긴 패턴을 공식 기여 시스템으로 만들어 데이터 정확도를 커뮤니티가 유지하는 구조.

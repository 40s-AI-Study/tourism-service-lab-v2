# SIM-40: PetKorea × 레이첼 김 (교포 2세)

## 시나리오

2026년 7월 말, 레이첼 김(42세, LA 건축가)은 딸 소피(10세)와 함께 10박11일 한국 여행 중이다. 한국계 2세로 한국어는 기초 수준이며 주로 영어로 소통한다. 소피는 동물을 매우 좋아해 한국 여행의 버킷리스트 중 하나가 "한국 펫 카페 방문"이다. 집에 반려동물은 없지만 소피가 입양을 원하고 있어 레이첼은 펫 카페를 통해 딸이 다양한 동물과 교감하게 해주고 싶다.

서울 한남동 호텔에서 소피가 "엄마, 강아지 카페 가고 싶어!"라고 조르자 레이첼이 앱스토어에서 "pet cafe korea"를 검색해 PetKorea를 발견한다. 앱을 열었을 때 한국어 전용 인터페이스가 나오지만, 레이첼은 구글 번역 앱을 병행해 핵심 메뉴를 파악한다. 건축가로서 UI/UX에 예민해 한국어 인터페이스의 불편함을 금방 인지하지만, 딸의 열의에 끌려 계속 사용해본다.

"펫 카페" 카테고리를 발견하고 서울 강남·이태원 지역 펫 카페 목록을 탐색한다. 소피가 화면을 들여다보며 강아지 사진이 가장 많은 카페를 직접 고른다. 혼잡도 색상 코드(초록=여유)를 소피가 직관적으로 이해하고 "이거 초록색이야, 사람 없대!"라고 외친다.

## 사용자 여정

### 1단계: 진입
- 딸 소피의 펫 카페 방문 요청이 트리거
- 앱스토어 "pet cafe korea" 검색으로 PetKorea 발견 및 설치
- 언어 설정 탐색 → 영어 미지원 확인, 구글 번역 병행 사용 결정
- 반려동물 프로필 없이 탐색 모드 진입 (집에 반려동물 없음)

### 2단계: 행동
- `pet-friendly-travel` API (카테고리: 카페) → "강남 펫 카페" 검색, 강아지 카페·고양이 카페·토끼 카페 등 다양한 유형 결과 수신
- 소피가 직접 사진 보며 카페 선택 참여 (강아지 사진 가장 많은 카페 1순위 지목)
- `visitor-concentration-forecast` API → 실시간 혼잡도 색상 코드 확인, 소피가 "초록색=한산" 직관적으로 이해
- 카페 상세 페이지 → 구글 번역으로 입장료·연령 제한·이용 규정 확인 (10세 어린이 입장 가능 여부)
- `related-attractions` API → 카페 인근 어린이 친화 관광지(코엑스 아쿠아리움, 봉은사) 추가 추천 수신
- 입장료 사전 확인 (1인 15,000원), 예약 외부 링크로 이동

### 3단계: 결과
- 강남 강아지 카페 방문 성공. 소피가 포메라니안·말티즈와 교감, 인생 최고의 경험이라고 표현
- 레이첼도 카페 분위기·청결도에 만족 (건축가로서 인테리어 감탄)
- 이후 앱 추천으로 고양이 카페도 추가 방문 (10박11일 중 펫 카페 총 3곳)
- 불만족: 영어 미지원으로 레이첼이 모든 텍스트를 번역해야 하는 피로감. 예약이 외부 링크로 연결되어 UX 단절
- 만족: 소피가 혼잡도 색상 코드를 직관적으로 이해하고 여행 계획에 참여한 경험이 긍정적. 다양한 펫 카페 유형 발견이 유용

## 개선 제안

1. **영어 UI 우선 구현**: 교포 2세 및 영어권 외국인 관광객의 펫 카페 체험 수요는 검증된 니즈. 영어 지원만으로도 레이첼 같은 사용자층 전환율 대폭 개선 가능
2. **어린이 동반 필터**: 펫 카페·관광지 상세 페이지에 "어린이 입장 가능 연령" 정보를 명시하고 "어린이 동반" 필터를 추가해 가족 여행자 경험 개선
3. **펫 카페 체험 전용 탭 분리**: 반려동물 없이 방문하는 "펫 카페 체험" 수요를 위한 별도 탭을 만들어 비펫 가족·외국인 관광객에게도 명확한 진입 경로 제공

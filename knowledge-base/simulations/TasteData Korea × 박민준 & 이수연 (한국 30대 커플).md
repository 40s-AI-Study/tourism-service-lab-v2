# SIM-192: TasteData Korea × 박민준 & 이수연 (한국 30대 커플)

## 시나리오

결혼 1주년 여행 코스를 짜는 수연이 TasteData Korea를 발견한다. "파인다이닝 수준의 미식 여행"을 원하는 수연에게 "신용카드 데이터가 보증하는 로컬 맛집"이라는 설명은 설득력 있다. 민준은 "이번엔 블로그 후기 믿었다가 폐업한 식당 또 가는 거 없게 하자"는 입장이었는데, 실제 소비 데이터 기반이라는 점에서 신뢰성을 인정한다. 두 사람이 함께 앱을 열고 전주 미식 여행 코스를 탐색한다.

---

## 사용자 여정

### 1단계: 진입
- 수연이 인스타그램에서 앱 소개 게시물 발견 → 민준에게 공유 → 함께 설치
- 전주 검색 → 소비 강도 히트맵에서 현지인 고소비 지역(객사길·삼천동)과 관광 지역(한옥마을) 분리 시각화
- "현지인이 실제로 가는 곳과 관광객이 가는 곳이 다르다"는 인사이트 → 즉각 흥미

### 2단계: 행동
- 음식 테마 선택: 전통 한정식 + 파인다이닝 필터 → 데이터 기반 상위 지역 확인
- 지역 특산 음식 코스: 저녁 한정식 → 다음날 아침 전주 콩나물국밥 → 점심 비빔밥 자동 생성
- 음식 포토 큐레이션: 수연이 음식 비주얼 확인 → "이거 진짜 예쁘다" 반응 → 방문 의향 상승
- 연관 관광지: 맛집 지역 주변 한옥 카페·공예 체험 연결 → 전체 여행 코스 완성
- 소비 강도 높은 지역의 특정 식당 정보 요청 → 개별 식당 이름·주소 데이터 없음 (지역 단위 정보만 제공)

### 3단계: 결과
- 객사길 현지인 밀집 지역 방문 → 실제로 로컬 분위기 압도적, 음식 품질 높음
- 그러나 "어느 식당으로 가야 하나"는 현장에서 직접 결정해야 했음 → 약간의 불확실성
- 민준: "여행지 선택과 지역 선택에는 최고, 식당 선택은 아직 카카오맵 필요"

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **8** | 현지인 소비 지역 발굴 + 음식 코스 자동 생성이 커플 미식 여행 기획 핵심 니즈 충족. 개별 식당 데이터 부재가 아쉬움 |
| 사용 편의 | **8** | 포토 큐레이션이 수연의 비주얼 탐색 방식에 잘 맞음. 코스 흐름이 자연스러워 기획 부담 감소 |
| 구현 타당성 | **8** | 소비 빅데이터 + 관광 정보 + 포토 API 조합 실현 가능. 지역 단위 정보의 한계는 데이터 소스 특성상 불가피 |

**종합 점수: 24/30**

## Pass/Fail 판정

> **✅ PASS**

TasteData Korea는 박민준 & 이수연 커플의 미식 여행 기획에서 "어디 지역으로 갈지"와 "어떤 음식 테마로 짤지"를 명확히 해결한다. 유용성 8점, 종합 24점으로 높은 점수를 기록했다. 개별 식당 단위 정보 부재라는 한계가 있으나, 지역 선택 단계에서의 가치는 충분하다.

## 개선 제안

1. 소비 밀집 지역 내 개별 식당 데이터 연동 — 카카오맵·네이버 플레이스 API 연결로 "지역 선택 → 식당 선택"까지 원스톱 완결
2. 커플 특화 "데이트 코스" 프리셋 — 분위기 좋은 식당 + 와인바 + 카페 조합을 소비 데이터 기반으로 자동 생성하는 로맨틱 코스 옵션

# SIM-R3-018: K-Camp Finder v2 × 이재호

## 시나리오

이재호(38세)는 서울 마포구에 사는 시니어 백엔드 개발자다. 척수손상으로 전동휠체어를 사용한다. 기술 친숙도는 업계 최고 수준 — API 문서를 직접 읽고, 앱 버그를 발견하면 개발자 포럼에 제보하는 타입이다. 캠핑은 예전부터 좋아했지만 휠체어 이후 갈 수 있는 곳이 극히 제한됐다. "접근 가능하다"고 써놓은 캠핑장도 실제 가보면 데크 입구에 5cm 턱이 있거나, 화장실 문이 좁거나, 모래 바닥이라 바퀴가 빠지는 경우가 비일비재했다.

GitHub 트렌딩에서 K-Camp Finder v2의 오픈소스 기여 이슈를 본다. "접근성 데이터 크라우드소싱 모듈" PR이 올라와 있고, 앱 개발팀이 장애인 사용자 피드백을 적극 수집 중이라는 README 내용이 있다. "이 팀, 진심이구나." 앱을 설치하고 GitHub Stars도 눌렀다.

앱을 열면 풀스크린 캠핑 영상이 나온다. 아름답지만 그는 1초만에 "저 데크 경사도가 얼마지?"를 생각한다. 개발자 본능과 장애인 현실 감각이 동시에 작동한다. 🌊물멍을 선택한다 — 계곡 옆 평지 캠핑이 휠체어에 가장 안전하다는 경험칙에서 나온 선택이다.

## 사용자 여정

### 1단계: 진입
- GitHub 이슈 발견 → 앱스토어 다운로드 → 개발자 시각으로 UI 구조 분석 (3분)
- 🌊물멍 선택 후 "무장애 접근" 필터 활성화 시도
- 필터 내 세부 항목 확인: "데크 평지 여부", "화장실 출입구 폭(cm)", "주차장~사이트 거리", "바닥 재질(포장/모래/흙)", "경사도(%)" — "이 정도 데이터면 믿을 수 있다"

### 2단계: 행동
- **접근성 데이터 상세 검증**: 충청북도 괴산 계곡 캠핑장 선택. 화장실 출입구 85cm(휠체어 통과 기준 80cm 이상 충족), 데크~주차장 포장도로 연결, 경사도 2% — 기준치 모두 통과
- **크라우드소싱 접근성 후기 탭**: 이전 방문한 휠체어 사용자 3명의 후기 확인. "전동휠체어로 사이트까지 진입 가능했음", "화장실 핸드레일 있음" 등 구체적 경험 기술. "이게 진짜 정보다"
- **데이터 정확도 의심 테스트**: 개발자 본능으로 "이 데이터 언제 업데이트됐지?" 확인. 각 접근성 항목에 "최근 검증일: 2026-02-14" 타임스탬프 표시. "크라우드소싱이지만 오래된 데이터는 아니구나"
- **버그 발견**: 경사도 필터 슬라이더가 5% 초과 값으로 설정해도 3% 초과 캠핑장이 결과에 포함됨. 스크린샷 캡처
- **예약 + 버그 제보**: 예약 완료 후 앱 내 "개발자에게 피드백" 탭으로 버그 리포트 전송. 재현 단계, 기대 동작, 실제 동작 포함한 표준 버그 리포트 형식으로 작성

### 3단계: 결과
- 괴산 계곡 캠핑장 1박. 앱 정보대로 진입 가능. "드디어 믿을 수 있는 접근성 정보였다"
- 계곡 소리 들으며 노트북으로 사이드 프로젝트 코딩 — "이게 내 불멍이야"
- 버그 제보 3일 후 개발팀으로부터 GitHub 이슈 링크와 함께 "수정 완료 예정" 답장 수신. 개발자로서 팀에 대한 신뢰 극상향
- 장애인 캠핑 커뮤니티 카페에 앱 소개글 게시. "접근성 데이터가 실제로 맞는 앱"
- 불만: 접근성 데이터가 있는 캠핑장이 전국 등록 캠핑장 대비 약 15% 수준. 선택지가 아직 부족

## 평가

| 항목 | 점수 (1-10) | 근거 |
|---|---|---|
| 유용성 | **9** | 경사도·바닥재질·화장실폭 등 정량적 접근성 데이터가 기존 앱과 완전히 다른 수준. 단, 커버리지 15%는 실질 선택지 제한 |
| 사용 편의 | **8** | 개발자 페르소나에게 데이터 타임스탬프·상세 수치 제공이 신뢰 획득. 일반 사용자 대비 정보 밀도 선호 |
| 구현 타당성 | **7** | 크라우드소싱 접근성 DB + 타임스탬프 관리는 구현 가능. 전국 캠핑장 접근성 데이터 커버리지 확대가 장기 과제 |

## Pass/Fail 판정

> **✅ PASS** (커버리지 확대 조건부)

접근성 데이터의 질은 합격이다. 정량 수치 + 크라우드소싱 후기 + 타임스탬프의 조합이 "진짜 정보"에 굶주린 장애인 여행자의 신뢰를 획득했다. 다만 15% 커버리지는 실사용 시 "갈 수 있는 곳이 없다" 경험으로 이어질 수 있어 확대가 급선무다. 개발자 페르소나와의 오픈소스 협업 구조는 앱 신뢰도와 커뮤니티 확산 모두에 탁월한 전략이다.

## 리텐션 평가: 이 페르소나가 앱을 다시 열까?

**재사용 확률**: ⬆️높음

**감정적 이유**:
> "캠핑 가기 전에 접근성 데이터 때문에 몇 시간씩 전화하고 직접 확인하던 게 없어졌어. 그 시간에 코딩 더 할 수 있게 됐다는 게 진짜 크다. 그리고 버그 제보했더니 실제로 고쳐줬어 — 이 팀 믿을 수 있다."

**재사용 트리거**:
- 주말 캠핑 계획 시 가장 먼저 여는 앱으로 자리잡음
- 접근성 데이터가 업데이트됐다는 알림 수신 시
- 장애인 캠핑 커뮤니티에서 "어디 가봤어요?" 질문 올라올 때 앱 켜서 답변

**이탈 리스크**:
- 접근성 데이터 커버리지가 늘지 않으면 "볼 곳이 없다" 포기
- 현장 데이터와 앱 데이터 불일치 경험 누적 시 이탈 확실

## 개선 제안

1. **접근성 데이터 커버리지 확대 크라우드소싱 캠페인**: 앱 내에서 "접근성 정보 직접 등록하기" 기능 강화. 방문 후 접근성 항목 체크리스트(15개 항목) 제출 시 포인트 지급. 특히 장애인 당사자 커뮤니티(지체장애인 협회, 장애인 여행 카페)와 파트너십으로 데이터 수집 속도 가속화
2. **실시간 접근성 이슈 알림**: 특정 캠핑장의 데크 보수 공사, 화장실 리모델링 등 접근성 관련 변경사항을 캠핑장 측이 앱에 직접 업데이트하는 파트너 채널 구축. "정보가 살아있다"는 신뢰 제공으로 이재호 같은 페르소나의 불안을 근본적으로 해소

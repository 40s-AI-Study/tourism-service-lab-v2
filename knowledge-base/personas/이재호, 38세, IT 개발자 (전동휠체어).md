# P7: 이재호 — 한국 장애인 여행자 (전동휠체어 사용자)

## 기본 정보

- **이름**: 이재호
- **나이**: 38세
- **직업**: 스타트업 백엔드 개발자 (재택근무)
- **거주지**: 서울 마포구 연남동
- **소득**: 월 420만원

## 배경 (서사)

재호는 28세 때 교통사고로 척수 장애를 입어 10년째 전동휠체어를 사용한다. 기술에 밝고 여행 정보를 직접 조사하는 스타일이지만, 막상 여행지 무장애 정보를 찾으려면 공공기관 PDF를 뒤지거나 장애인 카페 후기에 의존해야 한다. 데이터를 다루는 개발자로서 "왜 이 정보가 앱으로 나오지 않냐"는 불만이 누구보다 크다. 연 3~4회 혼자 또는 파트너와 국내 여행을 즐기며, 최근 제주·경주·부산 완전 자립 여행에 성공했다.

## 목표 (Goals)

- 사전에 정확한 무장애 동선 데이터를 확인하고 여행 계획 수립
- 전동휠체어 진입 가능 여부, 주차 공간, 충전 포인트 사전 확인
- 현장에서 "계단 있어서 못 들어간다"는 당황 없애기
- 장애인도 일반 여행자와 동일한 정보 품질 확보

## 고민 (Pain Points)

- **정보 신뢰성 낮음**: 공공기관 무장애 인증 데이터가 2~3년 전 정보라 현장과 불일치
- **세부 정보 부족**: "휠체어 접근 가능"이라고 되어 있어도 경사도·바닥 재질·단차 높이 미기재
- **실시간 크라우드소싱 없음**: 최근 방문자 리뷰에서 "경사로 보수 중"이라는 정보 얻을 방법 없음
- **충전 정보**: 전동휠체어 충전소 위치 안내 서비스 전무

## 관광 패턴

- **여행 빈도**: 연 3~4회 (1박 2일~3박 4일)
- **선호 유형**: 자연 경관, 현대 미술관, 로컬 카페·레스토랑 (접근성 확인 후)
- **예산**: 1박 기준 15~20만원
- **동반자**: 파트너(60%), 혼자(40%)
- **정보 탐색 채널**: 장애인 여행 카페(네이버), 직접 전화 확인, 개인 블로그

## 기술 친숙도

very high — 개발자 배경. 앱 UX 품질에 민감. 데이터 정확도 최우선. API 직접 호출해서 확인할 수 있는 수준.

## MBTI / 성향

INTJ — 철저한 사전 계획, 데이터 기반 의사결정. 감성보다 정보 정확도 우선. 여행 중 불확실성 최소화 선호.

## 대표 시나리오

AccessKorea 앱에서 "제주 동부 2박" 검색 → 전동휠체어 통과 가능한 경사도(15° 이하) 필터 → 각 장소의 바닥 재질·단차 높이 상세 데이터 확인 → 크라우드소싱 최신 리뷰("3주 전 방문, 경사로 상태 양호") 확인 → 출발 전 동선 완전 확정. 현장에서 예기치 않은 장벽 없음.

## 관련 API 시사점

- `barrier-free-travel` — 무장애 시설 상세 데이터 (경사도·단차·바닥재질까지)
- `visitor-concentration-forecast` — 혼잡 시간 회피 (전동휠체어 이동 공간 확보)
- `accommodation-info` — 장애인 객실·충전 시설 필터
- `related-attractions` — 접근 가능한 주변 장소 연결 추천

# 치유여정 Korea — 사업모델 구체화

> 작성: 💰 분석님 (BusinessAnalyst, claude-sonnet-4-6) | 2026-05-01

## 1. 서비스 개요

**치유여정 Korea**는 번아웃·디지털 피로·회복을 키워드로, 한국의 웰니스 관광지를 감성 기반으로 큐레이션하는 여행 앱이다. 단순 관광지 나열이 아닌 "지금 당신의 상태에 맞는 치유 루트"를 AI가 제안하며, 내국인·외국인·접근성 약자 3개 세그먼트를 하나의 플랫폼으로 커버한다.

## 2. 페르소나 3종

### Persona A — 번아웃 내국인 직장인 (KR 20-40대)

| 항목 | 내용 |
|---|---|
| 이름/나이 | 이민지, 32세 |
| 직업 | 서울 IT회사 마케터 |
| 핵심 고통 | 주 60시간 근무, 번아웃 상태, 짧은 주말에 "제대로 쉬고 싶다" |
| 여행 패턴 | 1박 2일 당일치기, 혼자 또는 친구 1-2명, 대중교통 선호 |
| 정보 탐색 | 인스타그램, 유튜브 쇼츠, 네이버 블로그 |
| 핵심 니즈 | 복잡한 계획 없이 "지금 내 기분"에 맞는 코스 바로 받기 |
| 결제 의향 | 월 6,900원 구독 또는 프리미엄 코스 1만원/회 |

### Persona B — 한국 웰니스 체험 외국인 (20-40대 인바운드)

| 항목 | 내용 |
|---|---|
| 이름/나이 | Sarah Chen, 29세 (싱가포르 출신) |
| 직업 | 프리랜서 디자이너, 워케이션 중 |
| 핵심 고통 | K-뷰티·K-웰니스에 관심 있으나 한국어 장벽, 정보 과부하 |
| 여행 패턴 | 2-3주 체류, 서울 베이스로 1-2일 지방 투어 |
| 정보 탐색 | TikTok, YouTube, Reddit r/korea |
| 핵심 니즈 | 영어 지원, 전통 치유(한방·산림욕·온천) 체험 예약 연계 |
| 결제 의향 | 프리미엄 코스 $12/회, 현지 체험 예약 수수료 수용 |

### Persona C — 접근성 약자 여행자 (장애인·고령자·영유아 동반)

|---|---|

| 결제 의향 | 무료 서비스 기대, B2G 사업 연계 가능 |

## 3. 핵심 사용자 시나리오 — 5단계 플로우

> Persona A (이민지) 기준 골든 패스

```
[Stage 1] 감정 진단
  └─ 앱 오픈 → "지금 어떤 상태?" 감정 카드 3장 선택
     (예: 지침, 소음 피하고 싶음, 자연이 그리움)
  └─ API: 없음 (로컬 감정 분류 모델)

[Stage 2] 코스 추천
  └─ AI가 감정 → 웰니스 유형 매핑 (산림욕/온천/명상/요가)
  └─ 현재 위치 + 이동 시간 설정 (1시간 이내 / 2시간 / 3시간+)
  └─ API: 웰니스관광정보.locationBasedList
         관광지 집중률 방문자 추이 예측.visitorTrendPredict
         (혼잡 낮은 시간대 추천)

[Stage 3] 루트 확인 & 커스터마이징
  └─ 치유 코스 지도 표시 (웰니스 거점 + 연관 관광지 3개)
  └─ 무장애 여부 토글 (Persona C 활성화 시)
  └─ API: 관광지별 연관 관광지.areaBasedList1
         두루누비.courseList (걷기 코스)
         무장애 여행.locationBasedList2 (접근성 필터 시)

[Stage 4] 체험 예약 & 정보 확인
  └─ 선택 장소 상세: 사진·오디오 가이드·리뷰
  └─ 예약 연결 (네이버 예약 / 외부 예약 딥링크)
  └─ API: 국문 관광정보.detailCommon + detailImage
         관광지 오디오 가이드.audioGuide
         생태 관광.areaBasedList (산림치유원)

[Stage 5] 여정 기록 & 공유
  └─ 치유 일기 작성 (감정 before/after 기록)
  └─ SNS 공유 카드 자동 생성
  └─ 다음 추천 코스 씨앗 데이터 수집
  └─ API: 없음 (앱 내부 저장)
```

## 4. Visit Korea OpenAPI 매핑

### 4-1. 사용 API 명세

| # | API명 | 핵심 오퍼레이션 | 치유여정 활용 목적 |
|---|---|---|---|
| 1 | **웰니스관광정보** | `locationBasedList`, `areaBasedList`, `detailInfo` | 웰니스 거점 목록 · 상세정보 핵심 데이터소스 |
| 2 | **관광지 집중률 방문자 추이 예측** | `visitorTrendPredict` | 혼잡 피크 회피 → "고요한 시간대" 추천 |
| 3 | **관광지별 연관 관광지** | `areaBasedList1`, `searchKeyword1` | 웰니스 거점 중심 연계 코스 자동 구성 |
| 4 | **두루누비 정보** | `courseList`, `crsTourInfo`, `routeList` | 도보/자전거 치유 트레일 코스 데이터 |
| 5 | **무장애 여행 정보** | `locationBasedList2`, `detailInfo2` | Persona C 접근성 필터링 |
| 6 | **생태 관광 정보** | `areaBasedList`, `areaBasedList1` | 산림치유원·공정관광지 연계 |
| 7 | **국문 관광정보 서비스** | `detailCommon`, `detailImage`, `searchKeyword` | 기본 관광지 상세·이미지 |
| 8 | **관광빅데이터 정보** | `locgoRegnVisitrDDList` | 지역별 방문 수요 분석 → 비성수기 치유여행 추천 |

### 4-2. 호출 시퀀스 다이어그램

```
클라이언트
  │
  ├─[감정선택 완료]──▶ 내부 감정→웰니스유형 매핑 엔진
  │                        │
  │                        └──▶ API ①  웰니스관광정보.locationBasedList
  │                                    (mapX,mapY,radius + 웰니스유형 필터)
  │                                        │
  │                        ┌──────────────┘
  │                        │
  ├─[장소 목록 수신]─────▶ API ②  방문자 추이 예측
  │                                (contentId 리스트 → 혼잡도 스코어)
  │                                        │
  │                        ┌──────────────┘
  │                        │
  ├─[코스 후보 정렬]─────▶ API ③  연관 관광지
  │                                (대표 웰니스지 contentId → 연관지 Top5)
  │                                        │
  │                        ┌──────────────┘ (병렬)
  │                        ├──▶ API ④  두루누비 courseList (도보 코스)
  │                        └──▶ API ⑤  무장애 여행 (접근성 필터 ON 시)
  │
  ├─[코스 확정]──────────▶ API ⑦  국문관광정보.detailCommon + detailImage
  │                                (각 장소 상세 · 사진)
  │
  └─[예약 페이지]────────▶ 외부 예약 시스템 딥링크 (네이버 예약 등)
```

## 6. 차별화 포인트 (경쟁 서비스 대비)

| # | 경쟁 서비스 | 치유여정 차별점 |
|---|---|---|
| **①** | **네이버 여행** (여행지 정보 포털) | 단순 정보 나열 vs. **감정 기반 AI 큐레이션** — "장소를 보여주는 것"이 아니라 "지금 내 상태에 맞는 치유를 처방" |
| **②** | **야놀자·여기어때** (숙박 예약 플랫폼) | 가격 경쟁·숙박 중심 vs. **치유 여정 설계** — 숙박이 아닌 "회복 경험" 중심, 예약은 딥링크 연계로 보완 |
| **③** | **힐링 앱 (명상: 코끼리, Calm 등)** | 실내 명상 콘텐츠 vs. **오프라인 공간 연결** — 앱 안에 머무는 것이 아닌 실제 자연·공간으로 데려다주는 연결 |
| **④** | **Visit Korea 공식 앱** | 망라형 정보 vs. **감정 필터링 + 혼잡도 회피** — Visit Korea OpenAPI를 활용하면서도 감성화된 UX 레이어 |
| **⑤** | **무장애여행 정보 앱** | 접근성 정보만 vs. **접근성 + 치유 경험 통합** — 장애인·고령자도 "치유여행"이라는 감성 프레임으로 접근 가능 |

**핵심 USP 요약**: "Visit Korea OpenAPI 공식 데이터 × 감정 AI 큐레이션 × 혼잡도 회피 × 무장애 통합"

## 7. MVP 기능 우선순위 (Must/Should/Could)

### Must Have (4주 내 구현 필수)

| 기능 | 설명 | 연계 API |
|---|---|---|
| 감정 진단 온보딩 | 감정 카드 선택 → 웰니스 유형 매핑 | 없음 (로컬) |
| 웰니스 장소 목록 | 위치 기반 웰니스 관광지 조회 | 웰니스관광정보 |
| 코스 추천 지도 | 추천 코스 지도 표시 | 연관 관광지, 두루누비 |
| 장소 상세 페이지 | 사진, 기본정보, 운영시간 | 국문관광정보 |
| 무장애 필터 | 접근성 토글 필터 | 무장애 여행 정보 |

### Should Have (완성도 향상)

| 기능 | 설명 | 연계 API |
|---|---|---|
| 혼잡도 표시 | 실시간 방문자 예측 뱃지 | 방문자 추이 예측 |
| 오디오 가이드 | 장소별 오디오 해설 | 관광지 오디오 가이드 |
| 치유 일기 | 감정 before/after 기록 | 없음 (로컬 저장) |
| 영어 지원 | Persona B 외국인 대응 | 영문관광정보서비스 |
| SNS 공유 카드 | 치유점수 공유 이미지 생성 | 없음 (Canvas API) |

### Could Have (v2 로드맵)

| 기능 | 설명 |
|---|---|
| 예약 연동 | 네이버 예약 파트너 API 직접 연동 |

### 9-2. 바텀업 BEP 계산

```
[Y1 목표]
MAU 목표: 50,000 (다운로드 150,000 × 전환율 33%)

  └─ 구독 수익: 4,000 × 6,900 × 12개월 = 3.3억

체험 예약:
  └─ MAU 50K × 예약 전환 2% × 평균 예약금 30,000원 × 수수료 5%
     = 50,000 × 0.02 × 30,000 × 0.05 × 12 = 1.8억

B2B: 10개사 × 200만 = 0.2억
광고: 0.7억
────────────────────
Y1 총 매출: ≈ 6억 (보수적) ~ 10억 (낙관적)

[Y2 BEP]
고정비: 인건비 4명 × 6,000만 = 2.4억/년
       서버·API비용: 0.3억/년

BEP: Y2 초 달성 (매출 > 비용 교차점)
```

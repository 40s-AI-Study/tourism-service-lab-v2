## 1. 서비스 개요

**Hidden Restaurant Finder**는 Visit Korea OpenAPI 관광빅데이터와 AI를 결합해  
카카오맵·네이버에 숨어있는 진짜 맛집을 발굴하는 서비스다.  
핵심 차별화: **SNS 협찬 비율 수치화** + **빅데이터 군중 패턴 역이용**.

## 3. SWOT 분석

### Strengths (강점)
- **SNS 협찬 비율 AI 수치화**: 인스타그램/유튜브 협찬 여부를 수치로 공개 — 국내 유일
- **빅데이터 군중 역이용**: 관광빅데이터 방문자 트래픽 낮은 시간대 + 구글 인기도 낮은 식당을 역발산으로 추천
- **B2B SaaS + B2C 구독 듀얼 MRR**: 광고 RPM 의존 탈피, 경기 변동 저항성
- **Visit Korea OpenAPI 독점 활용**: 관광빅데이터·지역별 수요강도·연관관광지 API 조합

### Weaknesses (약점)
- **카카오맵·네이버 무료 대체재**: 이미 방대한 맛집 DB 보유, 신규 사용자 습관 전환 비용 높음
- **콘텐츠 초기 밀도**: 론칭 시 빅데이터가 적으면 "숨은 맛집" 카테고리 빈약
- **SNS API 의존성**: 인스타그램·유튜브 API 정책 변경 시 핵심 기능 타격

### Opportunities (기회)
- **맛집 정보 신뢰도 위기**: 네이버·카카오 체험단 마케팅 과다로 소비자 불신 심화 (2025 언론 보도 반복)
- **K-Food 글로벌 수요**: 방한 외래객 1,730만명 중 음식 목적 방문 41% (KTO 2024)
- **식당 B2B SaaS 시장**: 국내 음식점 75만개, 데이터 기반 마케팅 솔루션 도입률 5% 미만
- **외국인 맛집 신뢰 니즈**: 구글 리뷰 조작 의심 증가 → 투명 지표 수요

### Threats (위협)
- **카카오·네이버 따라잡기**: 대형 플랫폼이 "협찬 라벨" 의무화 시 차별화 약화
- **규제 리스크**: 식품위생법·광고법상 협찬 표시 의무 강화 시 경쟁사도 개선 가능
- **소비자 관성**: "이미 카카오맵 쓰는데"의 전환 마찰

## 4. 수익 모델 검증 — B2B 식당 + B2C 구독

### 4-1. B2B 식당 SaaS

| 플랜 | 월 요금 | 포함 기능 | 전환 목표 |
|------|--------|----------|----------|
| Basic | 49,000원 | 방문자 분석 대시보드, 협찬 비율 리포트 | Y1 3,000개 식당 |
| Pro | 129,000원 | 경쟁 식당 벤치마크, SNS 언급량 추이, 최적 영업시간 추천 | Y2 8,000개 식당 |
| Enterprise | 월 500,000원~ | 데이터 API 제공, 전담 CS | Y3 500개 프랜차이즈 |

**바텀업 매출 산출:**
```
Y1: Basic 2,500개 × 4.9만 × 12 = 14.7억
    Pro   500개 × 12.9만 × 12 = 7.7억
    B2B 소계: 22.4억

Y2: Basic 5,000 × 4.9만 × 12 = 29.4억
    Pro  3,000 × 12.9만 × 12 = 46.4억
    Ent.  200 × 50만 × 12 = 12억
    B2B 소계: 87.8억
```

> **B2B 전환 근거**: 국내 음식점 75만개 × SNS 마케팅 활용 식당 30% = 22.5만개가 잠재 고객.  
> Y1 목표 3,000개 = 잠재 시장 1.3% — 보수적.

### 4-2. B2C 구독

| 플랜 | 월 요금 | 포함 기능 |
|------|--------|----------|
| Free | 0원 | 월 10회 AI 맛집 추천, 기본 협찬 스코어 |
| Premium | 3,900원 | 무제한 추천, 방문 이력 분석, 맞춤 알림, 지도 히트맵 |
| Annual | 29,900원 | Premium × 12개월 (36% 할인) |

**바텀업 매출 산출:**
```
Y1: MAU 80만명 × 전환율 3% = 2.4만 유료 구독
    월 평균 ARPU 3,500원 (Premium + Annual 혼합)
    연 매출: 2.4만 × 3,500 × 12 = 10.1억

Y2: MAU 200만명 × 전환율 4.5% = 9만 유료 구독
    연 매출: 9만 × 3,500 × 12 = 37.8억
```

## 6. 페르소나 3종

### P1. KR 내국인 — 이지혜, 28세, 콘텐츠 크리에이터
- **Pain**: 인스타 맛집 갔더니 체험단 리뷰 천지. "진짜 맛집이 어딘지 모르겠어요"
- **Gain**: 협찬 비율 0% 필터 → 진짜 현지인 맛집 발견
- **사용 시나리오**: 홍대 주변 협찬없는 식당 → AI 추천 3개 → 방문 → 후기 업로드

### P2. 외국인 — Tanaka Haruka, 27세, 일본 직장인 (주 1회 방한)
- **Pain**: 구글 맵 리뷰 신뢰도 불확실. "어디가 진짜 한국 음식인지 모름"
- **Gain**: 외국인 방문 비율 낮은 식당 = 현지인만 아는 진짜 맛집
- **사용 시나리오**: 명동 → 관광객 비율 필터 낮음 → 골목 한식 발견 → 영문 메뉴 가이드

### P3. 접근성 약자 — 이재호, 38세, IT 개발자 (전동 휠체어)
- **Pain**: 맛집 정보에 접근성 필터 없음. 방문해보면 계단 있는 경우 다반사
- **Gain**: 무장애 여행 API + 식당 접근성 데이터 통합 → 사전 확인
- **사용 시나리오**: 이태원 → 휠체어 접근 가능 필터 ON → 협찬없는 맛집 → 예약

## 7. 핵심 사용자 시나리오 (5단계)

```
1. 발견 (Discover)
   → 앱 홈에서 "오늘 어디?" 감정 버튼 탭
   → AI가 현재 위치 + 시간대 + 군중 밀집도 분석

2. 탐색 (Explore)
   → 협찬 스코어 낮은 식당 지도 오버레이
   → 필터: 거리 / 음식 카테고리 / 접근성 / 현지인 비율

3. 신뢰 확인 (Trust)
   → 식당 카드: 협찬 비율 %, SNS 언급 중 유료 비율, 실방문자 후기 비율
   → Visit Korea 빅데이터 방문자 추이 그래프

4. 행동 (Act)
   → 길안내 (카카오맵 딥링크) + 예약 (외부 링크)
   → B2C Premium: 방문 이력 자동 저장 + 재방문 알림

5. 공유 (Share)
   → 방문 후기 카드 생성 (협찬 없음 뱃지 자동 부여)
   → SNS 공유 → 바이럴 루프
```

## 8. Visit Korea OpenAPI 매핑

| API | 활용 목적 | 호출 빈도 |
|-----|---------|----------|
| **관광빅데이터 정보서비스** | 식당 방문자 수 추이, 시간대별 혼잡도 | 일 1회 배치 |
| **지역별 관광 수요 강도** | 지역 인기도 점수 → 역추천 (낮은 곳 발굴) | 주 1회 배치 |
| **지역별 관광 다양성** | 특정 지역 음식 카테고리 다양성 지수 | 주 1회 배치 |
| **관광지별 연관 관광지 정보** | 식당 주변 관광지 연계 추천 | 실시간 |
| **국문 관광정보 서비스** | 식당 기본 정보 (영업시간, 주소, 전화) | 일 1회 배치 |
| **무장애 여행 정보** | 휠체어 접근 가능 식당 필터 (P3 페르소나) | 주 1회 배치 |
| **영문 관광정보서비스** | 외국인 대상 식당 정보 다국어 제공 | 일 1회 배치 |

**호출 시퀀스 (식당 탐색 플로우):**
```
사용자 위치 입력
  → 관광빅데이터 API: 주변 식당 혼잡도 조회
  → 지역별 수요강도 API: 해당 지역 인기도 점수
  → 국문 관광정보 API: 식당 상세 정보
  → [내부 AI] SNS 협찬 비율 분석 결과 병합
  → 최종 추천 리스트 생성 (협찬 스코어 오름차순)
```

## 10. MVP 기능 우선순위 (MoSCoW)

### Must (출시 필수)
- 협찬 스코어 AI 분석 + 지도 오버레이
- Visit Korea 빅데이터 기반 혼잡도 표시
- 식당 기본 정보 (국문 관광정보 API)
- 협찬 없음 필터
- B2C 프리미엄 구독 결제

### Should (초기 출시 포함 권장)
- 외국인 다국어 지원 (영·일·중)
- 무장애 접근성 필터
- 방문 이력 저장 (Premium)
- B2B 식당 Basic 대시보드

### Could (2차 릴리스)
- SNS 공유 카드 + 바이럴 루프
- 식당 예약 외부 연동
- B2B Pro 경쟁사 벤치마크
- 개인화 알림 (재방문 추천)

### Won't (이번 MVP 제외)
- 배달 연동
- 직접 예약 시스템 구축
- 글로벌 확장 (해외 맛집)

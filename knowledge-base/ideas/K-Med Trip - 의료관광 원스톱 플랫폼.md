# 아이디어 11: K-Med Trip

## 한 줄 정의
성형·피부·건강검진 등 한국 의료관광 목적 외국인이 **시술 검색 → 병원 예약 → 회복기 관광 → 쇼핑**까지 한 앱에서 해결하는 의료관광 원스톱 가이드.

## 타겟 사용자
- **1차**: 방한 외국인 의료관광객 (중국·동남아·미국·러시아) — 연 30만명 내외, 평균 체류 9일
- **2차**: 회복기 보호자 동반객 — 관광·쇼핑 수요 높음

## 문제 정의 / 근거
- 한국 의료관광 시장 규모: 2024년 약 4.5조원 (보건복지부 추산)
- 주요 Pain Point:
  - 공식 병원 정보가 분산되어 있어 **신뢰 가능한 후기** 접근 어려움
  - **회복기 5-7일 동안 뭘 해야 할지** 몰라 시간 낭비
  - 수술 직후 무리한 동선으로 **회복 지연** 사례 다수
- Visit Korea에는 `medical-tourism` API가 있으나 현재 활용 서비스 사실상 전무

## 핵심 기능
1. **공식 의료기관 검색**: 분야(성형·피부·치과·건강검진·한방) + 지역 + 언어 지원 필터
2. **회복기 맞춤 동선**: 시술 유형별 "가능한 활동 수준"에 맞춰 관광지 자동 추천 (누워 있어야 하는 1-3일 → 호텔 근처 카페·스파만 / 4-7일 → 도보 관광)
3. **다국어 통역 연결**: 병원 방문 시 실시간 통역 또는 예약 통역사 매칭
4. **쇼핑 가이드**: K-뷰티·K-약국·면세점 루트 (의료관광객 평균 쇼핑 지출 370만원)
5. **보호자 관광 프로그램**: 수술 대기 중 보호자가 즐길 수 있는 근거리 코스

## 활용 API
- `medical-tourism` — 의료관광 공식 정보
- `tourism-info-chinese-simplified/english/japanese/russian` — 4개 언어 상세
- `barrier-free-travel` — 수술 후 무장애 동선 (휠체어·엘리베이터)
- `related-attractions` — 병원 주변 편의시설 연계
- `visitor-concentration-forecast` — 회복기 한산한 시간대 추천

## 차별화 포인트
- 국내 의료관광 앱 사실상 전무 (플랫폼 몇 개는 한국어 전용)
- **"회복기 활동 수준"** 이라는 유니크한 필터 — 경쟁 서비스 없음
- Visit Korea 공식 데이터 기반 = 신뢰성 확보
- B2B 모델: 병원·호텔·면세점 제휴

## 예상 사용 시나리오
```
중국인 여성 (35세): 강남 피부과 4박 5일 패키지
D-30: 앱에서 병원 비교 검색 → 한국인 후기 + 공식 인증 정보 확인
D-7:  시술 후 회복기 루트 저장 (1-2일 호텔, 3-4일 명동 쇼핑, 5일 강남 카페 투어)
D-Day: 통역 예약 완료 상태로 입국 → 병원 안내
D+3:  앱이 "오늘부터 가벼운 산책 가능" 알림 + 근처 카페 3곳 추천
D+5:  귀국 전 쇼핑 루트 + 면세점 픽업 스팟 제공
```

## 예상 사업성 (1-10)
- 시장 규모: 8 (연 4.5조원, 성장세 뚜렷)
- 차별화: 9 (국내외 경쟁 서비스 전무)
- 구현 난이도: 6 (다국어 4개 + 병원 API 협업 필요)
- 공모전 적합성: 8 (데이터 활용 + 사회적 가치 + 외국인 관광 활성화)
- **총점: 31/40**

## 리스크
- 의료광고법 — 후기·병원 비교는 규제 대상, 공식 정보만 노출 전략 필요
- 병원 제휴 초기 부담 — 2-3개 주요 병원부터 MVP로 시작

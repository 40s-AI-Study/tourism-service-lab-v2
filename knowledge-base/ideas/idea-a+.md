## 1. 문제 정의

- 인스타그램 바이럴 → 특정 지역에 방문자 급증 → 현지인·상권 모두 피해
- 북촌한옥마을 거주 인구 27.6% 감소(2018~2023), 레드존 출입 금지 조치(2024) — 구조적 해결책 부재
- "비슷한데 덜 붐비는 곳"을 안내하는 서비스가 현재 없음
- 한국관광공사의 공식 정책 방향 = **지역 균형 관광** → 심사 방향과 직결

## 2. 목표 사용자 (페르소나)

| 페르소나      | 특성                                                                                                     |
| ------------- | -------------------------------------------------------------------------------------------------------- |
| **주 타깃**   | 20~30대 인스타 여행자. SNS에서 바이럴 장소를 발견하고 검색하지만, 막상 가보면 줄·혼잡에 실망한 경험 보유 |
| **부 타깃**   | 조용한 분위기를 선호하는 30~40대 커플·소규모 가족. 주말 당일치기 위주                                    |
| **공통 니즈** | "분위기는 비슷하되 사람이 덜한 곳" — 트렌드 포기 없이 경험 품질 확보                                     |

## 3. 서비스 컨셉

서비스가 한 곳을 지정 추천 → 그곳으로 또 몰림 → 문제 재발 + 서비스 책임.

**17개 시도별 유사 장소 3개 후보를 제시하고, 최종 선택은 사용자에게 맡긴다.**
→ 자연스러운 분산 + 책임 주체 명확화

## 4. 서비스 흐름

```
[바이럴 감지]
TourAPI 빅데이터 통계 API → 방문자 급증 장소 자동 탐지
(또는 사용자가 직접 장소 검색)
        ↓
[특성 벡터화]
  - catcode 소분류 (장소 유형)
  - CLIP 이미지 임베딩 (분위기·뷰)
  - overview 텍스트 임베딩 (설명 유사도)
        ↓
[17시도 매핑]
바이럴 발생 시도 제외 → 나머지 16개 시도별
유사도 상위 3개 후보 추출 (Chroma DB 벡터 검색)
        ↓
[지도 UI 출력]
한국 지도에 16개 시도 핀 표시
각 핀 클릭 → 해당 시도 후보 3개 카드 펼쳐짐
        ↓
[사용자 선택]
3개 카드 중 1개 선택 → 카카오맵 경로 실행
```

## 6. 바이럴 감지 기준

"방문자 급증"의 모호함을 해소하기 위해 명확한 수치 기준을 설정.

| 기준            | 정의                                                                   |
| --------------- | ---------------------------------------------------------------------- |
| **집중률 등급** | TourAPI `tatsCnctrRatedList` 기준 상위 10% 이내 장소                   |
| **급증 판정**   | 최근 4주 평균 방문자가 직전 13~52주 평균 대비 **+50% 이상** 증가       |
| **수동 입력**   | 사용자가 직접 장소명 검색 → 바이럴 여부와 무관하게 유사 장소 탐색 가능 |

---

## 7. 유사도 계산 (3축 가중 평균)

| 축          | 방법                             | 비중 |
| ----------- | -------------------------------- | ---- |
| 장소 타입   | catcode 소분류 일치              | 40%  |
| 분위기·뷰   | CLIP 이미지 임베딩 코사인 유사도 | 40%  |
| 설명 텍스트 | overview 텍스트 임베딩 유사도    | 20%  |

**유사도 수치 산출 근거**: 코사인 유사도(0~1)를 백분율로 변환. 3축 가중 합산 후 정규화. 발표 시 샘플 장소 쌍(예: 안목해변 ↔ 오이도)의 실제 계산 과정을 시연으로 보완.

## 8. "여유로움 지수" 설계

실시간 혼잡도 API는 유료 통신사 전용으로 무료 구현 불가.
**빅데이터 통계 API 기반 상대 지표**로 표현하되, 데이터 시제를 명확히 고지.

```
여유로움 지수 = 바이럴 장소 최근 N개월 평균 방문자 / 해당 장소 최근 N개월 평균 방문자
```

- UI 표현: `"최근 6개월 통계 기준 여유로운 편"` — 과거 통계임을 명시
- 실시간이 아님을 주석 처리하여 과장 없는 정보 제공
- 로드맵: 향후 통신사 데이터 연동으로 실시간화 예정으로 기획서 명시

## 9. 대안 선택 동기 설계

"바이럴 장소를 원해서 온 사람이 왜 대안을 선택하는가"에 대한 유인.

| 유인                 | 방식                                                                                         |
| -------------------- | -------------------------------------------------------------------------------------------- |
| **즉각적 실망 유발** | 바이럴 장소의 현재 집중률 등급과 예상 대기 체감을 카드 상단에 표시 ("주말 기준 혼잡")        |
| **감성 동등성 강조** | "이 장소와 분위기 유사도 91%" + 대표 이미지 나란히 배치 → 분위기 포기 없음을 시각적으로 납득 |
| **희소성 어필**      | "현재 이 장소를 추천받은 사람 N명" (세션 기반 카운터) → 오히려 여기가 남들 모르는 곳         |
| **카카오톡 공유**    | "이 장소 친구에게 공유" → 소셜 바이럴 역이용                                                 |

## 10. 데이터 파이프라인

**오프라인 (1회 사전 작업)**

```
TourAPI 전체 26만건 수집
→ CLIP 이미지 임베딩 + 텍스트 임베딩 계산
→ areaCode별 분류
→ Chroma DB 저장 (로컬, 무료)
```

**임베딩 처리 현실성**

| 항목             | 추정값                                                                          |
| ---------------- | ------------------------------------------------------------------------------- |
| 이미지 보유 장소 | 약 10~12만건 (전체 중 이미지 URL 존재 비율 약 45%)                              |
| CLIP 처리 속도   | CPU 기준 약 2~5초/장 → 12만건 약 60~170시간. GPU(Colab 무료) 사용 시 약 5~8시간 |
| 텍스트 임베딩    | sentence-transformers 사용, 12만건 약 1~2시간                                   |
| **결론**         | Google Colab GPU로 1~2일 내 완료 가능. 현실적 범위 내                           |

**온라인 (실시간 서비스)**

```
바이럴 장소 벡터 조회 → 시도별 유사도 상위 3개 추출 → 지도 렌더링
```

| ------------------- | ------------------------------------- | ----------------------- | -------------------------------------------- |
| 바이럴 감지         | `visitor-concentration-forecast`      | `tatsCnctrRateList`     | 관광지 집중률 목록으로 방문자 급증 장소 탐지 |

| 여유로움 지수       | `area-tourism-demand-density`         | `areaTarExpDsList`      | 지역별 관광 지출 수요 강도 (상대 여유 지표)  |
| 장소 수집·벡터화    | `tourism-info-korean`                 | `areaBasedList2`        | areaCode별 전체 장소 목록 수집 (26만건)      |
| 장소 수집·벡터화    | `tourism-info-korean`                 | `detailCommon2`         | overview 텍스트 임베딩용 상세 공통정보       |
| 장소 수집·벡터화    | `tourism-info-korean`                 | `detailImage2`          | CLIP 이미지 임베딩용 대표 이미지 URL         |
| 유사 장소 후보 보완 | `related-attractions`                 | `searchKeyword1`        | 연관 관광지 정보로 벡터 검색 결과 보완       |
| 지도 UI 핀          | `central-attractions-by-municipality` | `areaBasedList1`        | 시도별 중심 관광지 좌표 (핀 초기값)          |
| 소외 지역 가중      | `area-tourism-diversity`              | `areaTouDivList`        | 관광 다양성 지수로 소외 시도 식별            |
| 경로 실행           | 카카오맵 API                          | 길찾기                  | 선택 장소 길찾기                             |
| 소셜 공유           | 카카오톡 공유 API                     | 메시지 공유             | 선택한 대안 장소를 친구에게 공유             |
| 사용자 인증         | 카카오 로그인 API                     | OAuth 2.0               | 간편 로그인, 재방문 이력 관리                |

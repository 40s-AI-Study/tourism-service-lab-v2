# 아이디어 B+ — 지역 여행 원페이지 필수 투두리스트

> **"ChatGPT는 여행 계획을 말해주고, 이 서비스는 여행 계획을 실행시켜줍니다."**

## 1. 문제 정의

- 여행 계획 시 정보 파편화: 인스타·블로그·지도·예약 앱을 따로 써야 함
- ChatGPT로 "여행 계획 짜줘" 해도 실행 불가능한 텍스트만 나옴
- 1박2일 단기 여행자에게 "핵심만" 보여주는 서비스 부재

---

## 2. 목표 사용자 (페르소나)

| 페르소나      | 특성                                                                                                                                                  |
| ------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| **주 타깃**   | 20~30대 직장인. 주말 1박2일 여행을 자주 하지만 계획 세우는 시간이 부족. 인스타·블로그로 장소 리서치 후 지도 앱, 예약 앱을 따로 켜는 과정을 번거로워함 |
| **부 타깃**   | 지방 여행 경험이 적은 수도권 거주자. "거기 뭐가 있는지 모르겠다"는 막막함이 출발 장벽                                                                 |
| **공통 니즈** | 계획에 쓰는 시간 최소화 + 도착하면 바로 실행 가능한 정보                                                                                              |

---

## 3. 서비스 컨셉

소셜 트렌드 기반으로 특정 지역 여행의 핵심만 담은 **원페이지 플랜**을 자동 생성하고,
예약·동선·경비까지 **원스톱**으로 제공한다.

단, 소셜 트렌드 장소만 추천하면 인기 장소 쏠림이 심화될 수 있으므로
**소외 지역 가중 보정**을 적용해 한국관광공사의 지역 균형 관광 정책과 연계한다.

---

## 4. 서비스 흐름

```
[입력] 지역 선택 (예: 대전) + 여행 기간 (예: 1박2일)
        ↓
[소셜 분석 + 소외 지역 보정]
  네이버 블로그 API → "대전여행" 최다 언급 장소 추출
  + area-tourism-diversity API로 소외 시도일 경우 노출 가중치 1.5배 적용
        ↓
[큐레이션] TourAPI catcode 분류
  - 🍽️ 먹방 코스 2곳
  - 📸 포토스팟 1곳
  - 🏞️ 관광지 2곳
  - 🏨 숙박 추천 1곳
  (총 6곳으로 정보 과부하 방지)
        ↓
[원페이지 출력] Day1 → Day2 타임라인 카드
        ↓
[항목 클릭 시]
  → 카카오맵 경로 바로 실행
  → 예약 페이지 바로 연결 (TourAPI 예약 URL 또는 카카오맵 플레이스 예약)
  → 예상 경비 자동 합산
  → 카카오톡으로 플랜 공유
```

## 5. 경비 계산

| 항목   | 데이터 출처                               | 기준                                      | 비고                                              |
| ------ | ----------------------------------------- | ----------------------------------------- | ------------------------------------------------- |
| 입장료 | TourAPI `usefee`                          | 1인 기준                                  | 무료인 경우 0원 표시                              |
| 숙박비 | TourAPI `roomrate`                        | 2인 기준 1박 평균                         | 데이터 없는 경우 "확인 필요" 표시                 |
| 이동비 | 카카오 모빌리티 거리 × 단가               | 자가용 기준 (연비 12km/L, 유가 1,700원/L) | 대중교통 모드 토글 제공                           |
| 식비   | 지역 평균 단가 (한국소비자원 외식비 기준) | 1인 1식 1만 5천 원 기본값                 | 블로그 파싱 대신 고정 기준값 사용으로 신뢰성 확보 |

## 11. 확장 레이어 — 팔도 스탬프 시스템

아이디어 B 위에 얹는 장기 여정 엔진.
**단기(오늘 여행) → 중기(지역 스탬프) → 장기(전국 완주)** 의 3단 구조로 재방문 동기 부여.

### 서비스 흐름

```
[지역 선택] 17시도 지도에서 목적지 선택 (팔도 감성 UI)
        ↓
[원페이지 플랜 생성] 아이디어 B 그대로 실행
        ↓
[인증]
  - 대표 관광지: 인증 사진 (위치 메타데이터 + TourAPI 장소 매칭)
  - 지역 식당: 영수증 등록 (OCR → 사업자번호 → 지역 확인)
        ↓
[스탬프 적립] 해당 시도 스탬프 획득
        ↓
[리워드 지급]
  - 지역상품권
  - 숙박 할인 쿠폰
  - 다음 여행지 큐레이션
        ↓
[17개 시도 완주] 전국 여행 달성 배지
```

### 인증 부정 수급 방지

| 위협             | 방지책                                                                         |
| ---------------- | ------------------------------------------------------------------------------ |
| GPS 조작         | 사진 EXIF 위치 + 촬영 시점 검증. 최근 24시간 내 촬영분만 유효                  |
| 동일 사진 재사용 | 이미지 해시 중복 체크                                                          |
| 영수증 조작      | 사업자번호를 국세청 오픈API로 실시간 조회 + 결제일이 방문일 ±1일 이내인지 확인 |
| 단기 도용        | 동일 장소 인증은 30일 내 1회만 적립                                            |

### 리워드 재원 및 파트너십 계획

| 리워드            | 재원                                                                 | 파트너십 대상 |
| ----------------- | -------------------------------------------------------------------- | ------------- |
| 지역상품권        | 한국관광공사 지역관광 예산 연계 (디지털 관광주민증 사업과 동일 채널) | 한국관광공사  |
| 숙박 할인 쿠폰    | 야놀자·여기어때 제휴 할인코드                                        | 숙박 플랫폼   |
| 카카오페이 포인트 | 카카오 공동 주최 연계 마케팅 예산                                    | 카카오        |

> 공모전 단계에서는 리워드 시뮬레이션으로 제시하고, 실제 파트너십은 수상 후 협의 예정임을 명시.

### 소외 지역 가중치 보상

```
area-tourism-diversity > areaTouDivList → 관광 다양성 지수로 방문자 하위 N% 지역 산출
area-tourism-demand-density > areaTarSjrnDsList → 체류 수요 강도로 소외 지역 2차 검증
        ↓
소외 지역 방문 시 리워드 1.5~2배 지급
        ↓
사용자는 더 많은 혜택을 위해 소외 지역 선택
        ↓
지역 분산 효과 자연스럽게 달성
```

### 지역 기준

- **기준:** 17시도 (서울·부산·대구·인천·광주·대전·울산·세종 / 경기·강원·충북·충남·전북·전남·경북·경남·제주)
- **UI:** 전통 팔도 지도 감성 유지 (스토리텔링 + 현실적 완주 가능성 동시 충족)

### 스탬프 시스템 추가 강점

| 항목      | 내용                                     |
| --------- | ---------------------------------------- |
| 정책 부합 | 지역 균형 관광 = KTO 공식 정책 방향 일치 |
| 재방문율  | 인증 기반 실물 리워드로 지속 사용 유도   |
| 약점 보완 | B의 1회성 사용 약점 → 서비스 지속성 입증 |

# 아이디어 15: Blejure Half-Day

## 한 줄 정의
**"Business + Leisure = Blejure"** — 출장 중 갑자기 비는 반나절(3-6시간)을 GPS 현재 위치 기반으로 자동 생성된 코스로 활용하게 해주는 출장자 전용 미니 투어 앱.

## 타겟 사용자
- **1차**: 외국인 블레저 여행자 (미국·일본·중국 IT·금융 30-50대) — 연 방한 출장객 240만명
- **2차**: 국내 지방 출장자 — 대전·대구·부산 출장 중 반나절 활용 수요

## 문제 정의 / 근거
- 방한 외국인 비즈니스 방문 실태 조사: 출장자 62%가 "일정 중 1회 이상 관광" 희망, 그러나 54%는 "시간 부족·정보 부족"으로 포기
- 기존 관광 앱은 "종일/숙박형" 코스 중심 — **반나절 3-6시간 코스는 공백**
- David Park (페르소나 P5) 실제 Pain Point: "수요일 오후 3시간 비는데 어디 가야 할지 모른다"
- 현재 위치 기반 즉시 실행이 안 됨

## 핵심 기능
1. **"지금 여기부터 N시간" 버튼**: GPS 읽기 → 3/4/6시간 선택 → 즉시 코스 생성
2. **이동 시간 정밀 계산**: 호텔/회의실 복귀까지 포함, 지하철·택시 예상 시간 포함
3. **저녁 미팅 모드**: 복귀 시각 입력 시 여유 15분 앞당겨 종료
4. **다국어 요약**: 장소당 "30초 소개" 영상·텍스트 (영·중·일·한)
5. **출장자 프리셋**: "K-역사 30분 체험 / K-푸드 점심 / K-뷰티 쇼핑 / 인생사진 포토스팟" 4개 프리셋

## 활용 API
- `visitor-concentration-forecast` — 현재 시간대 혼잡도로 최적지 선택
- `related-attractions` — 호텔/회의실 근처 클러스터
- `tourism-info-english/chinese-simplified/japanese` — 다국어 콘텐츠
- `area-tourism-demand-density` — 지역별 밀도 고려
- `central-attractions-by-municipality` — 지자체별 필수 방문지

## 차별화 포인트
- 블레저 전용 앱 글로벌에서도 소수 (미국 TripIt 수준)
- "지금 위치 + N시간" 즉시 실행 = 출장자 UX 완벽 적중
- 방한 외국인 P5 시뮬레이션 FAIL → 개선 반영의 킬러 케이스
- B2B: 호텔 컨시어지·기업 출장 서비스 제휴 가능

## 예상 사용 시나리오
```
35세 미국 IT기업 PM (서울 출장):
  수요일 오후 2시, 갑자기 3시간 공백
  앱 실행 → "지금 여기부터 3시간 · K-역사" 선택
  GPS 읽기 (강남역) → 코스 자동 생성:
    14:15 지하철 3호선 이동 (18분)
    14:35 경복궁 입장 (60분 관람, 혼잡도 보통)
    15:40 북촌 한옥마을 빠른 산책 (40분)
    16:25 인사동 전통차 체험 (30분)
    17:00 택시 호텔 복귀 (20분)
    17:20 호텔 도착 (17:30 저녁 미팅 전 여유)
  영어 나레이션 자동 제공, 호텔 주소까지 경로 표시
```

## 예상 사업성 (1-10)
- 시장 규모: 7 (방한 출장자 연 240만, 국내 지방 출장자 포함)
- 차별화: 9 (반나절 특화 + 위치 기반 즉시 실행 앱 전무)
- 구현 난이도: 7 (다국어 + GPS + 교통 시간 API 복잡)
- 공모전 적합성: 8 (외국인 관광 활성화 + 심사 기준 UX 혁신)
- **총점: 31/40**

## 리스크
- 현재 위치 연동 오차 — GPS 보정 로직 중요
- 교통 시간 예측 정확도 — 카카오·네이버 API 외부 연동 필요
- 경쟁: 여행 종합 앱 기능 추가 대응 — "출장자 전용"이라는 포지셔닝으로 방어

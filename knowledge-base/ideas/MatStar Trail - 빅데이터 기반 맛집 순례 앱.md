# 아이디어 13: MatStar Trail

## 한 줄 정의
관광 빅데이터에서 **실시간 급상승 중인 지역 맛집**을 추출하고 도보 30분 이내 순례 코스로 묶어주는 미식 전문 여행 앱.

## 타겟 사용자
- **1차**: 20-30대 미식 덕후 (혼밥 OK 솔로 여행자 포함)
- **2차**: 방한 외국인 중 K-푸드 체험 목적 — 설문 조사 시 방문 동기 1위(42%)

## 문제 정의 / 근거
- K-푸드 글로벌 트렌드: 2024년 해외 한식당 9,923개 (농림부), 한식 관심도 급증
- 현재 맛집 정보는 네이버·인스타에 **광고·뒷광고 다수**, 신뢰 하락
- 여행자가 "이 지역의 진짜 맛집"을 찾으려면 **검색 시간 30분+**
- 기존 서비스는 "평점 중심" — 갑자기 뜨는 '로컬 핫' 포착 못함

## 핵심 기능
1. **급상승 맛집 탐지**: `tourism-big-data` + `area-tourism-demand-density` → "지난 2주 검색·방문 증가율 TOP 10"
2. **순례 코스 자동 묶기**: 도보 30분 원 안에서 메인 + 디저트 + 카페 3-4곳 자동 매칭
3. **혼잡도 기반 방문 순서**: 오후 피크 시간대 예측해 최적 방문 시각 제안
4. **수상작 포토스팟**: `photo-contest-winners` 연계로 식당+포토 연결
5. **수기 체크인 블록체인** (Phase 2): 광고·뒷광고 차단 위해 GPS+사진 기반 체크인 필수

## 활용 API
- `tourism-big-data` — 지역·시기별 관광 빅데이터
- `area-tourism-demand-density` — 지역별 수요 강도
- `visitor-concentration-forecast` — 식당 혼잡도 예측
- `photo-contest-winners` — 포토스팟 연계
- `tourism-info-korean/english` — 맛집 기본 정보

## 차별화 포인트
- 맛집 앱 많지만 **빅데이터 기반 급상승 탐지는 국내 전무**
- KoreaTrend Radar가 전반적 핫플 대상이라면, MatStar Trail은 **미식 특화**
- 도보 30분 순례 묶음 = 트립어드바이저/캐치테이블도 없는 각도
- B2B: 급상승 식당에 노출권 판매

## 예상 사용 시나리오
```
27세 솔로 여행자 (전주 1박):
  앱 실행 → "전주 미식 순례 · 반나절" 선택
  생성 코스:
    12:30 한옥마을 입구 2주간 검색 +340% 콩나물국밥
    13:30 도보 8분 이동 급상승 전통차 카페
    14:00 광장 디저트 팝업 신상 호떡 (2주 전 신규 등록)
    15:30 포토 수상작 장소 — 경기전 앞 돌계단
  순례 완료 → 앱 스탬프 적립 → 지역 맛집 패스포트 진화
```

## 예상 사업성 (1-10)
- 시장 규모: 8 (미식 관광 여행자 대다수가 수요)
- 차별화: 7 (빅데이터 급상승 지표 차별적)
- 구현 난이도: 6 (빅데이터 분석 + 도보 원 묶음 알고리즘)
- 공모전 적합성: 7 (빅데이터 활용 + K-푸드 정책 연계)
- **총점: 28/40**

## 리스크
- 광고·뒷광고 검증 — 초기에는 추천 로직 투명하게 공개 필요
- 로컬 맛집 DB 정확성 — 지자체 협업으로 오염 방지
- 경쟁: 네이버·망고플레이트 대응 — 큐레이션 품질로 차별화

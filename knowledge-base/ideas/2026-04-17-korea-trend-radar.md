## 한 줄 정의

이동통신·카드소비·내비게이션 데이터를 실시간 시각화해 '지금 이 순간 뜨는 여행지'를 발굴하고 추천하는 관광 빅데이터 레이더 앱.

## 타겟 사용자

- **주 타겟**: 30-40대 트렌드 세터, 데이터 기반 의사결정을 선호하는 여행자
- **세부 세그먼트**: 주말 즉흥 여행족, 여행 크리에이터·블로거, 소상공인·지역 관광 사업자
- **부 타겟**: 관광 정책 담당자, 지역 관광공사 실무자 (B2B 확장 가능)

## 핵심 기능

- **실시간 트렌드 레이더**: 이동통신·카드소비·내비 데이터를 통합 분석해 급부상 여행지를 실시간 지도에 시각화
- **숨은 핫플 발굴 알고리즘**: 관광 다양성 지수와 수요 밀도를 교차 분석해 아직 과포화되지 않은 신흥 명소 선제 발굴
- **트렌드 히트맵 대시보드**: 주간·월간·계절별 여행 트렌드 변화를 인터랙티브 히트맵으로 제공
- **자원 수요 매칭**: 관광 자원 수요 데이터 기반으로 특정 지역의 어떤 콘텐츠(음식·자연·문화)가 뜨는지 카테고리별 분석
- **트렌드 알림 구독**: 관심 지역 설정 시 트렌드 급상승 감지 즉시 푸시 알림 발송

## 활용 API

| API | 활용 목적 |
|-----|-----------|
| `tourism-big-data` | 이동통신·카드소비·내비 통합 빅데이터 기반 여행 트렌드 분석 |
| `area-tourism-demand-density` | 지역별 관광 수요 밀도 — 핫플 집중도 측정 |
| `area-tourism-diversity` | 관광 다양성 지수 — 특정 지역의 관광 콘텐츠 다양성 분석 |
| `area-tourism-resource-demand` | 관광 자원별 수요 — 어떤 유형의 명소가 뜨는지 카테고리 분석 |

## 차별화 포인트

- **빅데이터 4개 API 동시 활용**: 단일 데이터 소스가 아닌 4종 빅데이터를 교차 분석해 정확도·신뢰도 극대화
- **"지금"에 집중**: 기존 여행 앱은 과거 리뷰 기반, KoreaTrend Radar는 실시간 이동·소비 데이터 기반 — 시의성 차별화
- **발굴형 추천**: 이미 유명한 곳이 아닌 막 뜨기 시작하는 신흥 명소를 선제 추천 — 얼리어답터 욕구 자극
- **B2B 확장성**: 지역 관광공사·소상공인 대상 트렌드 리포트 유료 구독 모델로 수익화 가능

## 공모전 강점

- **빅데이터 API 집중 전략**: 관광 빅데이터 계열 API 4종을 유기적으로 엮은 설계 — 공모전 심사 기준 '데이터 활용도' 최고점 기대
- **"관광 트렌드 레이더" 네이밍**: 직관적이고 임팩트 있는 컨셉으로 심사위원 인상에 남는 피칭 가능
- **정책 연계 가능성**: 지역 균형 발전·비수도권 관광 분산 정책과 직접 연결되는 앱 목적 — 공익성 가점
- **데이터 시각화 완성도**: 히트맵·레이더 차트 UI로 시연 시 시각적 임팩트 극대화

## 예상 사용 시나리오

> **"이번 주말 어디가 요즘 뜨는 곳인지 모르겠다"**
>
> 사용자가 금요일 퇴근 후 앱 실행. 레이더 화면에서 강원도 양양군이 트렌드 급상승 중임을 확인 (서핑 + 감성 카페 수요 동시 상승, 카드소비 전주 대비 +34%).
>
> 양양 탭 진입 시 "지금 뜨는 이유: 서피비치 주변 신규 카페 3곳 오픈, 인스타 태그 폭증" 요약과 함께 관광 자원 수요 카테고리 분석("음식 42% · 자연 35% · 숙박 23%") 제공.
> 알림 구독 설정 시 다음 주에도 양양 트렌드 변화를 자동으로 받아볼 수 있음.

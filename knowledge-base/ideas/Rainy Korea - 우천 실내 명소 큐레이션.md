# 아이디어 16: Rainy Korea

## 한 줄 정의
여행 중 **갑자기 비가 오면** 현재 위치 반경의 실내 명소·박물관·카페·공연을 혼잡도까지 고려해 즉시 재조합 코스로 제안하는 우천·우기 특화 여행 앱.

## 타겟 사용자
- **1차**: 여행 중 날씨 변동 겪는 모든 여행자 (특히 6-9월 장마철 외래객)
- **2차**: 실내 명소 선호 가족 여행자 (영유아 동반)

## 문제 정의 / 근거
- 한국 6-8월 장마철 평균 강수일 40일 이상, 방한 외래객 월평균 150만명이 영향
- 관광객 조사: 우천 시 "대체 코스 못 찾아서 숙소에서 시간만 보냄" 47%
- 기존 여행 앱은 날씨 연동 X — 외부 검색에 의존해 30분+ 소모
- 박물관·미술관·쇼핑몰·PC방·찜질방 등 실내 POI가 통합 큐레이션된 앱 없음

## 핵심 기능
1. **우천 알림 자동 트리거**: 기상청 단기예보 + 현재 여행지 결합 → 비 30분 전 push "지금 코스 대체 제안 보시겠어요?"
2. **실내 POI 자동 필터**: 현재 위치 반경 3km / 대중교통 20분 / 도보 10분 기준으로 실내만 추출
3. **혼잡도 역활용**: 비 오면 실내 POI에 사람 몰림 → `visitor-concentration-forecast`로 덜 붐비는 곳 우선
4. **"방향별" 큐레이션**: 박물관 애호 / 카페 투어 / 쇼핑몰 원데이 / 공연·영화 4개 프리셋
5. **어린이 동반 모드**: 유모차 가능 + 수유실 있는 실내 POI만 노출

## 활용 API
- `tourism-info-korean/english` — 실내 POI 기본 정보
- `visitor-concentration-forecast` — 우천 시 혼잡도 역추적
- `barrier-free-travel` — 유모차·휠체어 진입 가능 필터
- `related-attractions` — 근거리 클러스터
- `area-tourism-diversity` — 지역별 실내 자원 다양성

## 차별화 포인트
- 날씨 기반 관광 앱 **국내 전무** (해외도 드문 블루오션)
- 혼잡도 역활용 = 비 오는 박물관 몰림 대응 = 실질적 가치
- 연중 상시 수요(우기 40일+) + 겨울 한파일(연 20일+) 대체 활용 가능
- B2B: 실내 POI(박물관·쇼핑몰·공연장) 우천일 송객 수수료

## 예상 사용 시나리오
```
28세 일본 여행자 (6월 제주 2박):
  Day 2 오전, 호텔에서 "오후 한라산 예정" 일정
  앱 자동 알림: "12:30부터 3시간 비 예보, 코스 변경 제안"
  제안:
    12:00 호텔 체크아웃
    12:30 이마셰트로 이동 (유모차 OK, 혼잡도 낮음)
    14:00 제주 도립미술관 (실내 관람, 한산)
    16:00 제주 카페거리 (실내 카페 3곳 추천)
    18:00 저녁 해녀 식당 (실내)
  원래 한라산 예정 일정은 Day 3 오전으로 자동 재조정
```

## 예상 사업성 (1-10)
- 시장 규모: 7 (우천·한파 연 60일+ × 방문객)
- 차별화: 9 (날씨 기반 관광 앱 전무)
- 구현 난이도: 6 (기상청 API + 실내 POI 메타데이터 구축)
- 공모전 적합성: 7 (데이터 조합 창의성 + 실용성)
- **총점: 29/40**

## 리스크
- 기상청 API 예보 정확도 — 단기예보(6시간) 신뢰도 활용, 중기예보는 참고용
- 실내 POI 메타 업데이트 — 지자체 협업 필수
- 계절 편중 매출 — 연중 실내 추천 기능으로 상시화

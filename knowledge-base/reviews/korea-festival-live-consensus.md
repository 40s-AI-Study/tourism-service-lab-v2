## 합의 결론

- **추천도**: 상 (공모전 유력 출품작)
- **핵심 강점**:
  - "최적 방문일 자동 추천" — 네이버/카카오도 미제공하는 혼잡도 예측 특화 기능으로 핵심 차별화
  - `visitor-concentration-forecast` + `tourism-info-english` + `area-tourism-diversity` 3종 빅데이터 동시 활용으로 공모전 데이터 활용(20점) 고득점 유력
  - 발견→확인→결정→코스 4단계 여정이 API와 1:1 매핑 — 기획력(30점) 항목 강점
  - 국내 20-40대 + 외국인 이중 타겟으로 공모전 요건 완전 충족
  - `area-tourism-diversity`로 축제 이후 주변 체류 전환 유도 → 지역 관광 활성화 심사 포인트
  - SNS 바이럴 루프(한국인·외국인 페르소나 전반)로 유기적 성장 채널 검증
- **핵심 약점**:
  - 수익 모델 미명시 — 광고·프리미엄 알림 구독 등 구체적 BM 필요
  - 계절성 리스크 — 축제 비시즌 콘텐츠 전략(지역 상시 이벤트 포함) 부재
  - `tourism-info-english` API 품질이 핵심 변수 — 지방 소규모 축제 영문 데이터 공백 리스크
  - 페르소나 세분화 필요 — "주말 여행 계획자" 등 구체화 권장
- **공모전 제출 전 필수 조치**:
  1. 수익 모델 명시 (광고·프리미엄 알림 구독 등) → 사업성 배점 보완
  2. 축제 비시즌 콘텐츠 전략 추가 — 계절성 리스크 완화
  3. 포토그래퍼·크리에이터 타겟 "골든아워 × 혼잡도 최저" 교차 배지 기능 추가 검토
- **사용자 선택 시 고려사항**:
  - 혼잡도 예측 기반 축제 최적화 앱으로 공모전 심사위원 시연 임팩트 높음
  - 공모전 수상 후 지역 관광 활성화 사업으로 확장 가능성 있음
  - 계절성을 극복하는 콘텐츠 전략 보강 시 연중 사용 앱으로 발전 가능

## 소수 의견 (의견 충돌 기록)

- **리서치·기획 9점 만장일치**: 두 평가자 모두 동일 점수로 의견 충돌 없음. Pain Point 해결력과 API 활용 전략에 대한 높은 확신을 공유.
- **공통 보완 권고**: 수익 모델 미명시와 계절성 리스크는 두 평가자 공통 지적. 공모전 제출 전 기획서에 반드시 반영 필요.
- **리서치님 추가 제안**: 포토그래퍼·크리에이터 타겟 골든아워 배지 기능 추가 시 인플루언서 자발적 홍보 루프 락인 가능 — 시장 진입 초기 바이럴 비용 절감 효과 기대.

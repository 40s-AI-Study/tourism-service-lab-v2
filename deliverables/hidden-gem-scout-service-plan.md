## 2. 시장 규모 분석 (스토리텔링 + 발견 결합 모델)

### 스토리텔링 + 발굴 결합 시장 특성

- **오디오 콘텐츠 시장 성장률**: 글로벌 오디오 투어 앱 시장 CAGR 14.2% (2024-2028, Statista)
- **숨은명소 여행 수요**: 국내 여행자 56%가 "새로운 장소 발굴"을 1순위 동기 (KTO 2025)
- **게임화 관광 효과**: 포켓몬GO 사례 — 방문자 없던 지역 관광 방문 280% 증가 (닌텐도 공식 보고 2018)
- **정부 정책 수혜**: 문체부 2026 비수도권 방문율 60% 목표 → 강소형 관광지 활성화 예산 확대

### 콘텐츠 제작 비용 vs 수익 분석

| 항목 | 비용 | 수익 기여 |
|---|---|---|
| AI 오디오 스토리 자동 생성 (GPT-4o + TTS) | 건당 약 ₩3,000 | 방문자 배지 획득 → 재방문률 +40% |
| 에디터 검수 (1인 파트타임) | 월 ₩1,200,000 | 품질 보증 → 유료 전환율 +15% |
| 사진 크라우드소싱 (UGC 인센티브) | 건당 ₩500 포인트 | 콘텐츠 생산 비용 80% 절감 |
| **월간 콘텐츠 총 비용** (100건 기준) | **₩1,650,000** | 구독 MAU 1,500명 → 월 수익 ₩18,000,000 |
| **콘텐츠 ROI** | | **10.9배** |

> **결론**: AI 자동 생성 오디오 스토리 + UGC 크라우드소싱 모델로 콘텐츠 제작 단가를 최소화하면서 게임화로 반복 소비를 유도 → 마진율 82% 달성 가능

## 3. 페르소나 3종

### 🇰🇷 페르소나 A — "탐험가 지수 (박지수, 34세)"
- **직업/거주**: 대전 거주 IT 개발자, 주말마다 전국 여행
- **상황**: 유명 관광지는 이미 다 가봄. 아무도 모르는 진짜 숨은 명소를 가장 먼저 발굴하고 SNS에 올리고 싶음
- **JTBD**: "내가 처음 간 곳, 내가 처음 찍은 사진 — 그 장소의 이야기를 내가 만들고 싶다"
- **페인 포인트**:
  - 인기 장소에 가면 항상 사람이 너무 많음
  - 블로그 후기가 있는 곳은 이미 "발견된" 곳
  - 새로운 곳을 찾아도 가는 방법, 주차, 안전 정보가 없음
- **기대 결과**: 앱이 방문자 0명 강소형 관광지를 추천 → AI 오디오 스토리로 역사·전설 제공 → 첫 방문자 배지 획득
- **사용 패턴**: 목요일 밤 탐색, 주말 당일치기 또는 1박 2일, 월 2-3회 방문

### 🌏 페르소나 B — "독립 여행자 Alex (미국 시카고, 29세)"
- **상황**: 3주 한국 여행, 서울·부산은 이미 방문. 현지인만 아는 진짜 한국을 경험하고 싶지만 영어 정보가 부족
- **JTBD**: "투어 버스가 안 가는 곳, 가이드북에 없는 한국을 직접 발굴하고 싶다"
- **페인 포인트**:
  - 영어 정보가 있는 관광지는 이미 외국인이 넘침
  - 현지 특색 없는 "외국인용" 경험에 실망
  - 대중교통 접근성 정보가 영어로 제공되지 않음
- **기대 결과**: 영문 앱으로 강소형 관광지 탐색 → 영어 오디오 가이드 + 접근 방법 → 개척자 배지로 "나만의 한국" 경험
- **사용 패턴**: 숙소에서 다음날 일정 계획, 영문 API 기반 오디오 가이드 필수

### ♿ 페르소나 C — "가족 여행 은혜 어머니 (김은혜, 58세)"
- **상황**: 은퇴 후 전국 여행 중. 무릎이 좋지 않아 경사가 심한 곳은 어려움. 붐비는 곳보다 조용하고 의미 있는 장소를 선호
- **JTBD**: "몸이 불편해도 갈 수 있는, 조용하고 이야기가 있는 숨은 명소를 찾고 싶다"
- **페인 포인트**:
  - 유명 관광지는 계단이 많고 사람이 많아 힘듦
  - 강소형 관광지의 무장애 접근성 정보가 없음
  - 혼자 또는 가족과 방문하기 안전한지 판단하기 어려움
- **기대 결과**: 무장애 필터 + 경사도 낮은 강소형 관광지 추천 → 한국어 오디오 스토리로 역사 감상 → "조용한 개척자" 배지
- **사용 패턴**: 자녀 추천으로 앱 설치, 출발 전날 코스 검색, 오디오 가이드 청취 중심 이용

## 4. 핵심 사용자 시나리오 — 5단계 플로우

> **시나리오 주인공**: 박지수 (페르소나 A) / 강원도 영월 주말 당일치기

### Step 1 — 발굴 의도 입력 (앱 진입)
```
[화면] 홈 → "지금 어디를 발굴할까요?" 버튼
[사용자 입력]
  - 현재 위치: 대전 (GPS 자동)
  - 이동 범위: 3시간 이내
  - 선호 유형: 자연 / 문화재 / 기이한 지형
  - 방문자 수 기준: "아무도 안 간 곳" (슬라이더 → 월 방문 50명 이하)
[API 호출] visitor-concentration-forecast → 혼잡도 최하위 필터링
```

### Step 2 — AI 발굴 리스트 제공
```
[화면] 지도 + 리스트 뷰 전환
[결과] "오늘의 잠재 발굴지 5곳"
  1. 🌿 영월 요선암 돌개구멍 — 방문자 0명 이번 달 | 강원 영월군
  2. 🏔️ 무주 나제통문 비경 — 주간 방문 3명 | 전북 무주군
  3. 🌊 태안 신두리 사구 숨은 구역 — 방문자 12명 | 충남 태안군
  4. 🍃 봉화 청량산 숨은 폭포 — 방문자 8명 | 경북 봉화군
  5. 🗿 단양 고수동굴 외곽 — 방문자 5명 | 충북 단양군
[API 호출] tourism-info-korean (기본 정보), tourism-photos (대표 사진)
[표시] 각 장소: 거리 · 예상 방문자 수 · 개척자 배지 희귀도
```

### Step 3 — 장소 상세 + 오디오 스토리 청취
```
[화면] 장소 상세 페이지 (영월 요선암 선택)
[콘텐츠]
  - 사진 갤러리 (관광사진 API)
  - 🎧 AI 오디오 스토리 (3-5분): "요선암에 숨겨진 1,200년 전 신선 전설..."
    → 오디오가이드 API 기반 + AI GPT 스토리 보강
  - 접근 정보: 주차장 · 화장실 · 경사도
  - 최근 방문자: "이번 달 아직 0명 — 첫 방문자가 될 수 있습니다!"
[API 호출] audio-guide (오디(odii) 기존 해설), related-attractions (주변 연계지)
```

### Step 4 — 현장 방문 + 인증
```
[화면] AR 카메라 / 인증 모드 진입 (현장 도착 GPS 인증)
[사용자 액션]
  - 사진 촬영 → 앱 업로드 (UGC 콘텐츠 생성)
  - AI가 사진 분석 → 방문 자동 인증
[결과]
  🥇 "요선암 첫 번째 개척자 배지 획득!"
  → SNS 공유 카드 자동 생성 (바이럴 유도)
  → 방문 로그 기록 (나만의 탐험 지도)
[API 호출] tourism-big-data (방문자 수 업데이트)
```

### Step 5 — 커뮤니티 + 다음 탐험 추천
```
[화면] 방문 완료 → "다음 발굴 미션" 제안
[콘텐츠]
  - 내가 올린 사진 → 공식 장소 사진으로 등록 (포인트 적립)
  - 연관 미발굴지 3곳 자동 추천: "요선암 근처 미발굴 2곳 발견!"
  - 주간 개척자 랭킹 업데이트
[API 호출] related-attractions, area-tourism-diversity (지역 탐험 다양성 점수)
```

## 5. Visit Korea OpenAPI 매핑

### 핵심 API 명세

| # | API 이름 | API ID | 주요 활용 | 호출 빈도 |
|---|---|---|---|---|
| 1 | 관광지 집중률·방문자 추이 예측 | `visitor-concentration-forecast` | **핵심**: 방문자 최소 장소 필터링, 발굴 가치 산정 | 검색 시 1회 |
| 2 | 국문 관광정보 서비스 | `tourism-info-korean` | 장소 기본 정보 (명칭·위치·소개·운영시간) | 상세 진입 시 1회 |
| 3 | 영문 관광정보서비스 | `tourism-info-english` | 외국인 타겟 영어 장소 정보 | 언어 설정 영어 시 |
| 4 | 관광지 오디오 가이드정보 | `audio-guide` | **스토리텔링 핵심**: 오디(odii) 오디오 해설 + 대본 | 상세 진입 시 1회 |
| 5 | 관광빅데이터 정보서비스 | `tourism-big-data` | 지자체별 방문자 통계, 방문 인증 후 업데이트 | 인증 시 1회 |
| 6 | 관광사진 정보 | `tourism-photos` | 장소 대표 사진 (사용자 업로드 전 기본 이미지) | 상세 진입 시 1회 |
| 7 | 관광지별 연관 관광지 정보 | `related-attractions` | 방문 완료 후 주변 미발굴지 추천 체인 | 인증 완료 시 1회 |
| 8 | 지역별 관광 다양성 | `area-tourism-diversity` | 지역별 탐험 다양성 점수 → 개척자 랭킹 계산 | 일 1회 갱신 |

### 호출 시퀀스 다이어그램

```
[사용자 앱 진입]
       │
       ▼
[1] visitor-concentration-forecast
    → 위치 반경 내 관광지 집중률 최하위 50곳 조회
    → 파라미터: areaCode(지역코드), forecastDate(오늘), limit=50
       │
       ▼
[2] tourism-info-korean / tourism-info-english
    → 필터된 50곳의 기본 정보 배치 조회 (이름·좌표·유형)
    → 캐시 적용: TTL 24시간
       │
       ▼ (사용자가 장소 선택 시)
[3] audio-guide
    → 해당 관광지 오디오 스크립트·음성파일 URL 조회
    → 없으면: AI 자동 생성 스크립트 fallback
       │
[4] tourism-photos
    → 관광지 대표 사진 최대 5장 조회
       │ (병렬 호출)
[5] related-attractions
    → 해당 관광지 연관 관광지 조회 (미발굴 체인 준비)
       │
       ▼ (현장 방문 인증 완료 시)
[6] tourism-big-data
    → 방문자 수 카운트 기록 (PUT)
[7] area-tourism-diversity
    → 지역 탐험 점수 업데이트 → 개척자 랭킹 재계산
```

### 캐싱 전략

| 데이터 유형 | TTL | 캐시 레이어 | 이유 |
|---|---|---|---|
| 장소 기본 정보 (국문/영문 관광정보) | **24시간** | Redis | 변경 빈도 낮음 |
| 집중률 예측 데이터 | **1시간** | Redis | 실시간성 중요하나 API 호출 비용 절감 |
| 오디오 가이드 파일 URL | **7일** | CDN | 파일 자체는 불변 |
| 관광사진 | **7일** | CDN | 이미지 변경 없음 |
| 방문자 수 (실시간) | **5분** | In-memory | 개척자 배지 실시간 반영 |
| 연관 관광지 네트워크 | **12시간** | Redis | 일 2회 갱신 충분 |

> **총 API 호출 절감 목표**: 캐싱 적용 시 일평균 API 호출 수 82% 절감 (콜드 캐시 시 8콜 → 웜 캐시 시 1-2콜/세션)

## 7. 차별화 포인트 (경쟁 서비스 대비)

### 차별화 1 — "강소형 잠재관광지 DB 독점" vs 네이버/카카오맵

| 구분 | 네이버/카카오맵 | **Hidden Gem Scout** |
|---|---|---|
| 데이터 소스 | 사용자 리뷰 기반 인기 장소 | **KTO 강소형 잠재관광지 2,400개소 독점** |
| 추천 방향 | 방문자 多 = 좋은 곳 | **방문자 少 = 발굴 가치 높음** |
| 경쟁 | 수억 명 사용, 포화 시장 | **강소형 잠재관광지 특화 독점 포지션** |

### 차별화 2 — "개척자 게임화" vs Audio Story V2 / 오디(odii)

| 구분 | 오디(odii) | **Hidden Gem Scout** |
|---|---|---|
| 콘텐츠 방향 | 완성된 이야기 수신 | **내가 첫 번째 이야기 창조자** |
| 재방문 동기 | 콘텐츠 소비 완료 시 종료 | **배지·랭킹으로 무한 재방문 루프** |
| 사회적 가치 | 개인 학습 | **지역 분산 관광 직접 기여** |
| 차별화 지표 | BA 6/10 (보통) | **BA 지역분산 10/10, AI차별화 9/10** |

### 차별화 3 — "정부 정책 정합성" vs 일반 여행 앱

| 구분 | 일반 여행 앱 | **Hidden Gem Scout** |
|---|---|---|
| 정책 연계 | 없음 | **문체부 2026 비수도권 60% 정책 직결** |
| B2B 수익원 | 광고 의존 | **지자체 지역 관광 활성화 계약 가능** |
| 공모전 부합 | 낮음 | **공모전 심사 기준 "사회적 가치" 23/25 최강** |

### 차별화 4 — "AI 오디오 스토리 자동 생성" vs 인스타그램/블로그

| 구분 | SNS 정보 검색 | **Hidden Gem Scout** |
|---|---|---|
| 미발굴지 정보 | 없음 (방문자 없으니 후기 없음) | **AI가 역사·지리·전설 조합해 자동 생성** |
| 언어 지원 | 한국어 위주 | **국문·영문 동시 지원 (API 다국어)** |
| 접근성 | 없음 | **무장애 필터 + 경사도 정보 제공** |

## 8. MVP 기능 우선순위 (MoSCoW)

### Must Have (반드시 출시)
- [M1] 강소형 잠재관광지 목록 표시 (집중률 예측 API 기반 필터)
- [M2] 장소 기본 정보 제공 (국문/영문 관광정보 API)
- [M3] AI 오디오 스토리 재생 (오디오가이드 API + AI 보강)
- [M4] GPS 기반 현장 방문 인증 시스템
- [M5] 개척자 배지 부여 (첫/희귀/전설 등급)
- [M6] 탐험 기록 지도 (나의 발굴 히스토리)

### Should Have (2차 스프린트)
- [S1] SNS 공유 카드 자동 생성 (바이럴 핵심)
- [S2] 개척자 랭킹 시스템 (주간/월간/누적)
- [S3] 무장애 접근성 필터 (경사도·주차·화장실 정보)
- [S4] 연관 미발굴지 자동 추천 체인 (related-attractions API)
- [S5] UGC 사진 업로드 + 포인트 리워드

### Could Have (3차 이후)
- [C1] AR 카메라 인증 (GPS 위치 + AR 오버레이)
- [C2] 지자체 파트너 공식 미션 (지역 특산품 연계)
- [C3] 다국어 오디오 (영어·일본어·중국어 자동 번역 TTS)
- [C4] 오프라인 다운로드 (오디오 + 지도)
- [C5] AI 여행 동반자 챗봇 (현장 질문 응답)

### Won't Have (이번 MVP 제외)
- 숙박·식당 예약 통합 (기존 플랫폼과 경쟁 불필요)
- 실시간 CCTV 혼잡도 (인프라 비용 과다)
- 자체 오디오 콘텐츠 수동 제작 (AI 자동화로 대체)

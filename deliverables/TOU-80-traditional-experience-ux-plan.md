## 1. 서비스 개요

**핵심 가치 제안**:  
"김치부터 한복까지 — 장인과 함께하는 한국 전통문화 체험을 5개국어로, 원클릭으로"

## 3. 핵심 사용자 시나리오 — 5단계 플로우

### 시나리오 A: 이서연의 "주말 즉석 도예 예약" (KR 내국인)

```
[1단계: 발견]
인스타 릴스 → "인사동 도예 체험" 릴스 발견
→ 댓글에 Traditional Experience 링크 → 앱 첫 실행

[2단계: 탐색]
홈 화면 → "도예/도자기" 카테고리 탭
→ 위치: 서울 전체 → 날짜: 이번 주 토요일
→ 정렬: 별점 순 → 결과 12개 표시
→ 2개 공방 비교 (슬라이드 뷰)

[3단계: 결정]
"인사동 백자 공방" 선택
→ 사진 갤러리 (체험 공간, 결과물 10장)
→ "당일 수령 가능" 뱃지 확인
→ "영어 가능" 뱃지 (외국인 친구 고려)
→ 토요일 오후 2시 슬롯 선택 (잔여 3석)

[4단계: 예약]
참가자 1명 → 가격 확인 (45,000원)
→ 카카오페이 결제 (원터치)
→ 예약 확인 알림 수신 (카카오톡)
→ 캘린더 저장 옵션

[5단계: 체험 후]
체험 완료 → 앱 푸시 "후기를 남겨주세요"
→ 별점 + 사진 후기 작성
→ 인스타 공유 버튼 (해시태그 자동 생성)
→ 다음 체험 추천 (성향 기반)
```

## 4. Visit Korea OpenAPI 매핑

### 4.1 사용 API 명세

| API | 용도 | 엔드포인트 | 주요 파라미터 |
|---|---|---|---|
| 국문 관광정보 서비스 | 체험 장소 기본 정보 | `areaBasedList2` | `contentTypeId=28` (체험) |
| 국문 관광정보 서비스 | 키워드 검색 | `searchKeyword2` | `keyword=도예\|한복\|서예` |
| 국문 관광정보 서비스 | 위치 기반 탐색 | `locationBasedList2` | `mapX`, `mapY`, `radius` |
| 국문 관광정보 서비스 | 상세 정보 | `detailInfo2`, `detailCommon2` | `contentId` |
| 국문 관광정보 서비스 | 이미지 조회 | `detailImage2` | `contentId`, `imageYN=Y` |
| 영문 관광정보 서비스 | 외국인용 영문 정보 | `areaBasedList2` | 동일 contentId |
| 일문 관광정보 서비스 | 일본어 지원 | `areaBasedList2` | 동일 contentId |
| 중문(간체) 관광정보 서비스 | 중국어(간체) 지원 | `areaBasedList2` | 동일 contentId |
| 관광사진 정보 | 고품질 체험 사진 | `searchPhotoList` | `keyword=전통체험` |
| 지역별 관광 수요 강도 | 혼잡도 예측 | 수요 강도 API | 지역코드 + 날짜 |

### 4.2 API 호출 시퀀스

```
[앱 초기 로드]
1. searchKeyword2("전통체험") → 전체 체험 목록 캐시
2. areaBasedList2(contentTypeId=28) → 카테고리별 분류

[사용자 위치 허용 시]
3. locationBasedList2(mapX, mapY, radius=5000) → 주변 체험 목록

[체험 상세 진입]
4. detailCommon2(contentId) → 기본 정보 (주소, 전화, 운영시간)
5. detailInfo2(contentId) → 체험 상세 (소요시간, 체험 가격, 예약방법)
6. detailImage2(contentId) → 이미지 목록 (최대 10장)

[다국어 전환]
7. 선택 언어에 따라: 영문/일문/중문 API 동일 contentId로 호출
   → 언어별 응답 캐시 (TTL 24시간)

[혼잡도 표시]
8. 지역별 관광 수요 강도 API → 해당 날짜 혼잡도 레이블 표시
```

### 4.3 캐싱 전략

```
[레이어 1 — 정적 데이터 (TTL: 24시간)]
- 체험 장소 기본 정보 (주소, 연락처, 카테고리)
- 이미지 URL 목록
- 다국어 텍스트 콘텐츠

[레이어 2 — 준정적 데이터 (TTL: 1시간)]
- 검색 결과 목록 (키워드별)
- 위치 기반 탐색 결과

[레이어 3 — 실시간 데이터 (TTL: 5분)]
- 혼잡도/수요 강도
- 예약 가능 슬롯 (자체 DB + API 조합)

[구현 방식]
- Redis 캐시 레이어
- SWR(stale-while-revalidate) 패턴
- 오프라인 모드: 마지막 캐시 데이터 표시 + 안내 배너
```

## 6. 차별화 포인트 (경쟁 서비스 대비)

### 차별점 1: 접근성 인증 시스템 ♿

**기존 서비스(Klook, 야놀자 체험)**: 접근성 정보 없음 또는 단순 "장애인 이용 가능" 표시  
**Traditional Experience**: 
- 공방 직접 실사 후 측정값 DB화 (입구 폭 cm, 작업대 높이, 경사로 각도)
- 사진 증거 포함 (입구, 동선, 화장실)
- 사용자 리뷰에서 접근성 태그 별도 수집
- 국내 유일 전통 체험 접근성 인증 뱃지

**임팩트**: 장애인 여행 시장 (국내 약 265만명) 공략, 차별화 PR 포인트

### 차별점 2: 문화 맥락 레이어 (Cultural Context) 🏛️

**기존 서비스**: 체험 상품 나열 (가격, 시간, 별점)  
**Traditional Experience**:
- 각 체험 카테고리별 역사/문화 설명 (2-3단락)
- 장인 프로필 — 경력, 수상 이력, 철학
- "이 체험이 왜 특별한가" 스토리텔링
- 관련 Visit Korea OpenAPI 문화재 연결 링크

**임팩트**: 외국인 관광객 차별화 (Airbnb Experiences 대비 깊이), 앱 체류 시간 증가

### 차별점 3: 당일 즉석 예약 + 실시간 가용성 ⚡

**기존 서비스**: 최소 2-3일 전 예약 필요, 전화 예약 비중 높음  
**Traditional Experience**:
- 당일 오전 9시 기준 당일 예약 오픈 (참여자 최소 1명만 있어도 진행)
- 잔여 좌석 실시간 표시 (0→1→2→full 카운터)
- 1인 예약 가능 (그룹 미달 취소 없음 — 장인이 1:1도 진행)
- 취소 정책: 2시간 전까지 100% 환불

**임팩트**: 즉흥 여행자 (20대 혼행, 당일 투어 외국인) 흡수

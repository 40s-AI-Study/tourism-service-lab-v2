# 🎧 Audio Story Korea V2 — 기획 고도화 문서

> 위치 기반 AI 오디오 스토리텔링 서비스  
> **카테고리**: 문화·전통 체험 | **BA 점수**: 131/160 | **팀 합의**: 8.7/10  
> 슬로건: *"당신이 서 있는 바로 그 자리에서, 이야기가 시작됩니다"*

---

## 1. 서비스 개요

### 핵심 가치 제안

문화유산 현장에서 **지오펜싱이 자동으로 이야기를 시작**하는 몰입형 오디오 스토리텔링 앱. 사용자의 5가지 성향(탐험가·감성가·학자·가족·접근성)과 4계절 컨텍스트를 교차해 **20가지 스토리 변형**을 제공한다. AI가 오디오 가이드 원문을 재해석해 단순 정보 전달을 넘어 '살아있는 역사 체험'을 만든다.

### 핵심 수치

| 항목 | 값 |
|------|-----|
| 활용 API | 7개 |
| 스토리 변형 | 5성향 × 4계절 = 20종 |
| 지원 언어 | 한/영/중(간체)/일 |
| 지오펜싱 반경 | 50–300m (문화재 규모별 조정) |
| 구독 가격 | ₩9,900/월 (연간 ₩89,000) |

## 3. 핵심 사용자 시나리오 (5단계 플로우)

**시나리오**: 가을 오후, 김지수가 창덕궁을 혼자 방문한다.

### Step 1 — 발견 & 준비 (앱 진입)

```
[화면: 홈 / 위치 감지]
- 앱 실행 → 현재 위치 기반 "근처 스토리 3개" 자동 표시
- 창덕궁 250m 접근 감지 → "🍂 가을 감성 스토리 준비됨" 배너 표시
- 성향 설정: 감성가 선택 (최초 1회, 언제든 변경 가능)
- 오프라인 캐시 다운로드 제안 (Wi-Fi 환경에서 자동)
```

**핵심 API 호출**: `storyLocationBasedList` (반경 500m 스토리 목록)

---

### Step 2 — 입장 & 지오펜싱 트리거 (자동 재생)

```
[화면: 지도 오버레이 + 파형 애니메이션]
- 정문 진입 (지오펜스 50m) → 진동 + 오디오 자동 시작
- "🎧 창덕궁 입니다. 가을 감성 스토리를 시작합니다."
- 배경음악: 장단(長短) 기반 국악 앰비언트
- 현재 위치 표시 + 다음 포인트까지 거리 안내
- 언제든 일시정지 / 속도 조절 (0.75× – 1.5×)
```

**핵심 API 호출**: `storyBasedList` (스토리 ID → 대본·음성 URL)

---

### Step 3 — 이동 중 연속 스토리 (순서 보장)

```
[화면: 이동 경로 + 진행률 바]
- 인정전 접근 (100m) → 현재 스토리 페이드아웃 → 인정전 스토리 시작
- 우측 하단: 미니 플레이어 (접기/펼치기)
- "이 길을 걸었던 왕들의 이야기" — 감성가 버전 내레이션
- 사진 명소 알림: "지금 위치에서 최고의 사진 명소입니다 📸"
```

**핵심 API 호출**: `detailImage2` (관광지 사진), `related-attractions` (주변 추천)

---

### Step 4 — 스토리 완료 & 상호작용

```
[화면: 스토리 카드 + 반응 버튼]
- 스토리 종료 → 감상 카드 표시
  - ❤️ 저장 | 🔁 다시 듣기 | 📤 공유 | 📖 대본 보기
- "오늘 탐험한 스토리 3/8 완료" 진행률 표시
- AI 추천: "함께 들으면 좋은 다음 스토리 →"
- 계절 한정 스탬프 획득: 🍂 가을 창덕궁 마스터
```

---

### Step 5 — 퇴장 & 리텐션 훅

```
[화면: 여행 다이어리 요약]
- 방문 종료 감지 (지오펜스 이탈) → 다이어리 자동 생성
  - 방문 경로 지도, 청취 스토리 목록, 명소 사진
- 푸시 알림 동의 시: "다음 방문 전 새 스토리 업데이트 알림"
- 구독 유도: "프리미엄으로 20개 스토리 변형 모두 해제 →"
```

## 4. Visit Korea OpenAPI 매핑

### 사용 API 7종

| # | API ID | 서비스명 | 활용 목적 |
|---|--------|---------|---------|
| 1 | `audio-guide` | 관광지 오디오 가이드정보 | 핵심: 스토리 음성·대본·사진 |
| 2 | `tourism-info-korean` | 국문 관광정보 서비스 | 관광지 상세정보, 위치기반 조회 |
| 3 | `tourism-info-english` | 영문 관광정보서비스 | 외국인 대상 콘텐츠 |
| 4 | `tourism-info-chinese-simplified` | 중문 간체 관광정보서비스 | 중국어권 관광객 |
| 5 | `tourism-info-japanese` | 일문 관광정보서비스 | 일본어권 관광객 |
| 6 | `related-attractions` | 관광지별 연관 관광지 정보 | 다음 방문지 AI 추천 |
| 7 | `visitor-concentration-forecast` | 관광지 집중률 방문자 추이 예측 | 혼잡도 회피 추천 |

### 호출 시퀀스

```
[앱 진입 시]
1. locationBasedList2 (반경 500m, 현재 위치)
   → 주변 관광지 ID 목록 반환

[지오펜스 진입 시]
2. storyLocationBasedList (위도/경도, 반경 50–300m)
   → 해당 지점 스토리 ID 목록

3. storyBasedList (storyId)
   → 스토리 대본 텍스트 + 음성 URL (한/영/중/일)

4. detailImage2 (contentId)
   → 관광지 이미지 (사진 명소 UI용)

[스토리 완료 후]
5. related-attractions (contentId)
   → 연관 관광지 최대 3개 추천

6. visitor-concentration-forecast (지역코드, 일자)
   → 혼잡도 예측 → "지금 가기 좋은 시간대" 안내
```

### 캐싱 전략

| 데이터 유형 | 캐시 위치 | TTL | 이유 |
|-----------|---------|-----|------|
| 스토리 대본·음성 URL | 로컬 (SQLite) | 30일 | 변경 빈도 낮음, 오프라인 필수 |
| 관광지 기본 정보 | 로컬 (SQLite) | 7일 | 안정적 데이터 |
| 이미지 파일 | 로컬 캐시 (LRU, 500MB) | 방문 완료 시 삭제 | 용량 관리 |
| 혼잡도 예측 | 메모리 캐시 | 6시간 | 실시간성 필요, 자주 변경 |
| 연관 관광지 | 메모리 캐시 | 24시간 | 중간 빈도 변경 |

**오프라인 모드**: 앱 진입 시 현재 위치 반경 2km 내 스토리를 백그라운드 프리페치. Wi-Fi 환경에서만 실행.

## 6. 차별화 포인트 (경쟁 서비스 대비)

### 경쟁 서비스 분석

| 서비스 | 강점 | 한계 |
|--------|------|------|
| 오디(odii) | 한국관광공사 공식 오디오 가이드 | 단일 버전, 정적 콘텐츠 |
| 카카오맵 오디오 | 대규모 사용자 기반 | 깊이 있는 스토리텔링 부재 |
| 구글 AR 가이드 | AR 기술 | 한국 문화유산 콘텐츠 빈약 |
| 큐레이터 앱 | 박물관 특화 | 야외 지오펜싱 없음 |

### 차별화 포인트 3가지

**① 성향×계절 20종 스토리 변형 (콘텐츠 혁신)**

기존 오디오 가이드는 1개의 정해진 스토리만 존재한다. Audio Story Korea V2는 방문자의 성향(5가지)과 방문 계절(4가지)을 교차해 최대 20가지 다른 스토리 버전을 제공한다. 같은 경복궁을 방문해도 탐험가·봄 버전과 감성가·가을 버전은 완전히 다른 내러티브를 경험한다. **재방문 유인을 내재화**한 구조다.

**② 지오펜싱 자동 트리거 (UX 혁신)**

사용자가 "재생" 버튼을 누를 필요가 없다. 문화유산에 발걸음을 내딛는 순간 스토리가 시작된다. 다음 포인트로 이동하면 자연스럽게 다음 챕터로 넘어간다. 앱 조작 없이 완전한 몰입 경험을 제공하는 서비스는 국내에 없다. Visit Korea `storyLocationBasedList` API를 실시간 활용한다.

**③ 접근성 퍼스트 설계 (시장 혁신)**

국내 관광 앱 중 장애인·노인을 핵심 타깃으로 설계한 서비스는 거의 없다. 무장애 여행 정보 API + 보청기 블루투스 연동 + 큰 글씨 모드 + 느린 내레이션(0.75×)으로 접근성 약자를 위한 프리미엄 경험을 제공한다. **공모전 심사 기준인 사회적 가치 항목에서 높은 점수**를 기대할 수 있다.

## 7. MVP 기능 우선순위

### Must (출시 필수)

| 기능 | 설명 | API 의존성 |
|------|------|-----------|
| 지오펜싱 자동 재생 | 반경 감지 → 스토리 자동 시작 | `storyLocationBasedList` |
| 위치기반 스토리 목록 | 근처 문화유산 스토리 표시 | `locationBasedList2` |
| 오디오 플레이어 | 재생/정지/속도조절/대본 | `storyBasedList` |
| 성향 선택 (5종) | 최초 1회, 언제든 변경 | 로컬 저장 |
| 계절 자동 감지 | 현재 날짜 기반 계절 판단 | 시스템 날짜 |
| 오프라인 캐시 | 방문 전 스토리 다운로드 | 로컬 SQLite |
| 4개 언어 지원 | 한/영/중간/일 | 다국어 API |

### Should (2차 출시)

| 기능 | 설명 |
|------|------|
| 여행 다이어리 자동 생성 | 방문 경로 + 청취 목록 + 사진 |
| 혼잡도 예측 | 방문 최적 시간대 안내 |
| 연관 관광지 추천 | 스토리 완료 후 다음 방문지 |
| 스탬프/배지 시스템 | 계절별 방문 달성 보상 |
| 접근성 전용 모드 | 큰 글씨 + 블루투스 보청기 연동 |

### Could (MVP 이후 로드맵)

| 기능 | 설명 |
|------|------|
| AI 스토리 생성 | GPT 기반 실시간 스토리 커스터마이징 |
| AR 포인트 오버레이 | 카메라로 문화재 비추면 스토리 표시 |
| 문화기관 파트너십 콘텐츠 | 박물관·미술관 독점 스토리 |
| 소셜 청취 | 친구와 같은 스토리 동시 청취 |
| 크리에이터 모드 | 사용자 직접 스토리 제작 |

# [고도화] Audio Story v2 — 기술 구현 검토 및 제품 기획

## 1. 서비스 개요

**Audio Story Korea v2** — 전국 주요 관광지를 몰입형 오디오 스토리텔링으로 경험하는 앱.

한국관광공사의 '오디(odii)' 오디오 가이드 API를 핵심 데이터로 활용하며, 단순 안내를 넘어 **내러티브 기반 스토리텔링 + 접근성 최적화 + 다국어 지원**으로 차별화한다.

## 2. 페르소나 3종

### P1. 내국인 — "문화재 덕후" 지수진 (28세, 서울)

| 속성 | 내용 |
|---|---|
| 직업 | 대기업 마케터 |
| 여행 빈도 | 월 1-2회 국내 당일치기 |
| 관심사 | K-역사, 문화재, 드라마 촬영지 |
| Pain Point | 문화재 설명 현판이 딱딱함, 오디오 가이드 기기 대여 번거로움 |
| 목표 | 스마트폰으로 몰입감 있는 스토리 들으며 혼자 여행하고 싶다 |
| 주요 기기 | iPhone 15, AirPods |

**핵심 니즈**: "경복궁의 역사가 드라마처럼 들려오면 좋겠다"

| 속성 | 내용 |
|---|---|

**핵심 니즈**: "창덕궁이 어느 드라마에 나왔는지, 왜 유명한지 일본어로 듣고 싶다"

| 속성 | 내용 |
|---|---|
| 특성 | 시력 저하(노안), 청력 일부 감소 |

**핵심 니즈**: "큰 글씨 + 느린 속도로 경주 불국사 이야기를 편하게 들었으면"

## 3. 핵심 사용자 시나리오 — 5단계 플로우

### 시나리오 A: 내국인 — 경복궁 오디오 스토리 투어

```
[Step 1] 앱 실행 · 위치 허용
  └─ 사용자가 경복궁 앞 도착
  └─ 앱이 GPS 기반으로 "경복궁 오디오 투어 시작하시겠어요?" 알림

[Step 2] 스토리 선택
  └─ "역사 스토리" / "전설 스토리" / "왕조 드라마" 3가지 테마 선택
  └─ 각 테마별 예상 시간 (15분 / 20분 / 30분) 표시

[Step 3] 오디오 재생 시작
  └─ 내러티브 오디오 재생 (배경 음악 + 성우 해설)
  └─ 현재 위치에 맞는 챕터 자동 활성화
  └─ 화면에 관련 역사 사진 + 현장 지도 표시

[Step 4] 관련 장소 탐색
  └─ "이 근처에 연관된 다른 이야기" 추천 카드
  └─ 연관 관광지 (근정전 → 경회루 → 향원정) 순서 가이드

[Step 5] 스토리 저장 · 공유
  └─ 투어 완료 후 "나의 스토리 카드" 자동 생성
  └─ 방문한 장소 + 들은 스토리 요약 저장
  └─ SNS 공유 (인스타그램, 카카오톡)
```

### 시나리오 B: 외국인 — 다국어 K-컬처 투어

```
[Step 1] 언어 설정 + 관심사 선택
  └─ 앱 첫 실행 → 언어 자동 감지 (일본어) 또는 수동 선택
  └─ "K-드라마 촬영지" 관심 태그 선택

[Step 2] 맞춤 스팟 추천
  └─ 현재 위치 기반 "드라마 촬영지 오디오 스팟" 지도 표시
  └─ 각 스팟에 연관 드라마 제목 + 장면 썸네일

[Step 3] 오디오 해설 (일본어)
  └─ 창덕궁 선정릉 → "이 장소는 ドラマ〇〇〇의 촬영지..." 일본어 해설
  └─ 한국어 대본도 동시 표시 (언어 학습 모드 옵션)

[Step 4] 문화 맥락 설명
  └─ 단순 사실 + "왜 이 장소가 특별한가" 문화 배경 설명
  └─ 조선왕조 왕계도 인터랙티브 뷰 제공

[Step 5] 여행 기록
  └─ 방문 인증 + 일본어 코멘트 남기기
  └─ 일본인 사용자 커뮤니티 피드에 공유 옵션
```

### 시나리오 C: 접근성 약자 — 배리어프리 느린 투어

```
[Step 1] 접근성 모드 자동 제안
  └─ 시스템 글꼴 크기 감지 → "큰 글씨 / 느린 속도 모드" 자동 제안
  └─ 원터치 활성화

[Step 2] 간편 경로 안내
  └─ 계단 없는 경로만 필터 (무장애 여행 API 활용)
  └─ 휠체어/유모차 접근 가능 여부 아이콘 표시

[Step 3] 느린 오디오 해설
  └─ 0.75x 속도 기본 설정, 큰 글씨 자막
  └─ 중요 내용 반복 재생 버튼 (30초 되감기)

[Step 4] 쉬어가기 안내
  └─ "근처 벤치" / "화장실" / "편의시설" 위치 자동 표시
  └─ 이동 거리 예상 (걸어서 3분, 약 200m)

[Step 5] 가족 공유
  └─ 동행 자녀에게 "지금 이 이야기 들어봐" 링크 전송
  └─ 오늘 투어 요약 자동 앨범 생성
```

## 4. Visit Korea OpenAPI 매핑

### 4.1 핵심 API — 관광지 오디오 가이드정보

| 오퍼레이션 | 용도 | 파라미터 |
|---|---|---|
| `storyLocationBasedList` | GPS 기반 주변 오디오 스팟 조회 | mapX, mapY, radius, lang |
| `storySearchList` | 키워드로 오디오 스토리 검색 | keyword, lang |
| `storyBasedList` | 스토리 목록 조회 (전체) | pageNo, numOfRows |
| `themeLocationBasedList` | 위치 기반 테마 조회 | mapX, mapY, radius |
| `themeCategory` | 테마 카테고리 코드 조회 | - |
| `themeBasedList` | 테마별 스팟 목록 | themeCode |

**지원 언어**: 한국어(ko), 영어(en), 중국어(cn), 일본어(jp)

---

### 4.2 보조 API 매핑

| API | 용도 | 연동 포인트 |
|---|---|---|
| `국문 관광정보 서비스` | 관광지 기본 정보 (사진, 주소, 전화) | `detailCommon2`, `locationBasedList2` |
| `무장애 여행 정보` | 배리어프리 접근성 정보 | P3 페르소나, 접근성 모드 |
| `관광지별 연관 관광지 정보` | "이 근처 연관 스팟" 추천 | Step 4 연관 장소 탐색 |
| `관광지 집중률 방문자 추이 예측` | 혼잡도 기반 방문 시간 추천 | "지금 방문하기 좋은 시간" |
| `영문 관광정보서비스` | 외국인 영어 상세 정보 | P2 페르소나 |
| `일문 관광정보서비스` | 일본어 관광 정보 | P2 일본인 페르소나 |
| `관광빅데이터 정보서비스` | 지역별 인기 스팟 트렌드 | 홈 화면 추천 |

---

### 4.3 API 호출 시퀀스

#### 앱 시작 시 (위치 기반 초기화)

```
사용자 앱 실행
      │
      ├─► GPS 위치 획득 (mapX, mapY)
      │
      ├─► [병렬 호출]
      │      ├─ storyLocationBasedList(mapX, mapY, radius=5km, lang=ko)
      │      │        → 주변 오디오 스팟 목록
      │      ├─ visitor-concentration-forecast(contentId)
      │      │        → 현재/예측 혼잡도
      │      └─ related-attractions(contentId)
      │               → 연관 관광지
      │
      └─► 통합 지도 렌더링 + 스팟 카드 표시
```

#### 오디오 스팟 상세 진입 시

```
사용자 스팟 선택
      │
      ├─► [순차 호출]
      │      ├─ detailCommon2(contentId) → 기본 정보, 사진
      │      ├─ storyBasedList(contentId, lang=ko|en|jp|cn) → 오디오 URL, 대본
      │      └─ 무장애여행정보(contentId) → 접근성 정보 (접근성 모드 ON 시)
      │
      └─► 오디오 플레이어 화면 렌더링
```

#### 외국인 다국어 플로우

```
사용자 언어 = JP
      │
      ├─► storyLocationBasedList(lang=jp) → 일본어 오디오 스크립트
      ├─► 일문 관광정보서비스 → 추가 일본어 컨텐츠
      └─► 화면 UI도 일본어 전환
```

## 6. 차별화 포인트 (경쟁 서비스 대비 3개+)

### 경쟁 서비스 분석

| 서비스 | 특징 | 한계 |
|---|---|---|
| 오디(odii) 앱 | 한국관광공사 공식 | UI 구식, 스토리텔링 부족 |
| 구글 맵 오디오 가이드 | 영어 중심, 범용 | 한국 문화 맥락 부족 |
| 리디북스 오디오 | 도서 기반 | 현장 위치 연동 없음 |
| 문화재청 앱 | 공식 자료 | 딱딱한 설명, 접근성 약함 |

### 차별화 포인트 4가지

**DP1. 내러티브 오디오 스토리텔링**
- 기존: "경복궁은 1395년 건립된 조선의 정궁입니다" (백과사전식)
- 차별화: "1392년 이성계가 새 왕조를 세우던 그날 밤..." (드라마틱 내러티브)
- 성우 + 배경음악으로 몰입감 극대화

**DP2. 위치 감응형 챕터 자동 전환**
- 경복궁 안에서 이동하면 챕터가 자동으로 현재 위치에 맞게 전환
- GPS + 가속도 센서 결합 → "걷는 속도"에 맞춰 해설 속도 조절

**DP3. 3중 접근성 최적화**
- 배리어프리 API 연동으로 휠체어 경로 실시간 제공
- 느린 속도(0.75x) + 큰 자막 + 고대비 UI 패키지 제공
- 기존 어떤 경쟁 앱도 이 3가지를 동시에 제공하지 않음

**DP4. K-컬처 콘텍스트 레이어**
- "이 장소가 어느 K-드라마에 나왔는가" 연결 정보
- 외국인이 이미 아는 콘텐츠(드라마)와 현장을 연결하여 몰입도 3배↑
- 일본어/영어 전용 "K-컬처 맥락 설명" 컨텐츠 큐레이션

## 9. 기술 아키텍처 요약

```
[클라이언트]
  React Native (iOS / Android)
  ├── 지도: react-native-maps + GPS
  ├── 오디오: react-native-track-player
  ├── i18n: react-i18next (KO/EN/JP/CN)
  └── 접근성: @react-native-community/accessibility

        │ HTTPS
        ▼

[BFF — Node.js / Express]
  ├── /api/spots?lat=&lng=&lang=  ← storyLocationBasedList
  ├── /api/audio/:contentId       ← storyBasedList
  ├── /api/detail/:contentId      ← detailCommon2 + detailImage2
  ├── /api/related/:contentId     ← related-attractions
  └── /api/crowd/:contentId       ← visitor-concentration-forecast

        │ REST
        ▼

[Visit Korea OpenAPI]
  data.go.kr ServiceKey 인증
  ├── 관광지 오디오 가이드정보  (핵심)
  ├── 국문/영문/일문 관광정보   (다국어)
  ├── 연관 관광지 정보          (추천)
  ├── 혼잡도 예측               (방문 최적화)
  └── 무장애 여행 정보          (접근성)

[DB — PostgreSQL]
  ├── users (언어 설정, 접근성 모드)
  ├── tour_history (방문 기록)
  └── cached_spots (API 응답 캐시, TTL 24h)
```

## 2. 핵심 사용자 시나리오 — 5단계 플로우

**시나리오**: Tanaka Kenji가 경주 방문 첫날

```
┌─────────────────────────────────────────────────────────────────────┐
│  STEP 1          STEP 2           STEP 3        STEP 4    STEP 5   │
│  입국 전 준비  → AI 탐색·질문 → 현장 실시간  → SOS 대응 → 여행 후  │
│                               번역·안내                    루트 저장 │
└─────────────────────────────────────────────────────────────────────┘
```

### Step 1 — 입국 전 준비 (T-3일)
- 앱 설치 → 언어 선택 (日本語)
- 여행 일정 입력 → AI가 경주 추천 코스 3개 자동 생성
- 오프라인 핵심 데이터 캐시 다운로드 (관광지 정보, 긴급연락처)
- **KTO API**: 일문 관광정보서비스 `areaBasedList2` (경주 관광지 전체)

### Step 2 — 도착 후 AI 탐색 (현장)
- "石窟庵の近くで車椅子も入れる食堂はありますか？" 자연어 질문
- AI → 무장애 여행 API 조회 + 일문 관광정보 병합 → 日本語 응답
- 연관관광지 API로 "석굴암 방문자가 함께 가는 Top 3" 자동 추가 제안
- **KTO API**: `locationBasedList2` (위치기반), `related-attractions`, `barrier-free-travel`

### Step 3 — 현장 실시간 번역·안내
- 식당 메뉴판 카메라 번역 (OCR + LLM 문화 해설)
- 지하철 노선도 AR 오버레이 — 일본어 환승 안내
- "今日の慶州は混んでいますか？" → 집중률 예측 API → "오후 3시 이후 한산"
- **KTO API**: `visitor-concentration-forecast`, 오디오가이드 (석굴암 한/영/일/중)

### Step 4 — 긴급 SOS 대응
- 대화 중 "財布を盗まれました" 감지 → SOS 모드 자동 전환
- 현재 위치 기반 가장 가까운 외국어 가능 경찰서 + 일본 영사관 즉시 표시
- AI가 한국어로 상황 요약 → 112에 텍스트 전달 준비
- **KTO API**: (긴급안전정보 연계 — 추후 통합)

### Step 5 — 여행 후 루트 저장·공유
- 방문 경로 자동 기록 → "나의 경주 2일 코스" 생성
- SNS 공유 / 친구에게 루트 전송
- 국적별 익명 집계 → 서비스 추천 데이터 품질 향상
- **KTO API**: 관광사진 정보 (방문지 공식 사진 첨부)

## 3. Visit Korea OpenAPI 매핑

### 3-1. 핵심 사용 API (10개)

| API ID | 이름 | 역할 | 우선순위 |
|--------|------|------|----------|
| `tourism-info-english` | 영문 관광정보 | 영어권 기본 데이터 | Must |
| `tourism-info-japanese` | 일문 관광정보 | 일본인 최대 관광객 | Must |
| `tourism-info-chinese-simplified` | 중문 간체 | 중국 본토 관광객 | Must |
| `tourism-info-chinese-traditional` | 중문 번체 | 대만/홍콩 관광객 | Must |
| `tourism-info-spanish` | 서어 관광정보 | 스페인어권 (Sofia 페르소나) | Should |
| `tourism-info-french` | 불어 관광정보 | 유럽 관광객 | Should |
| `barrier-free-travel` | 무장애 여행 정보 | 접근성 약자 필터 | Must |
| `related-attractions` | 연관 관광지 | AI 추천 고도화 | Must |
| `visitor-concentration-forecast` | 집중률 예측 | 혼잡 회피 안내 | Should |
| `audio-guide` | 오디오 가이드 | 한/영/일/중 음성 안내 | Should |

### 3-2. 오퍼레이션별 활용 매핑

```
사용자 액션                   호출 API                      오퍼레이션
────────────────────────────────────────────────────────────────────
언어 선택 후 지역 탐색    → tourism-info-{lang}          areaBasedList2
GPS 위치 기반 검색        → tourism-info-{lang}          locationBasedList2
관광지 상세 보기          → tourism-info-{lang}          detailCommon2
                           tourism-info-{lang}          detailImage2
무장애 필터 ON            → barrier-free-travel          locationBasedList2
                                                         searchKeyword2
"함께 가는 곳" 추천       → related-attractions          (contentId 기반)
오늘 혼잡도 조회          → visitor-concentration        (날짜+지역코드)
오디오 가이드 실행        → audio-guide                  (contentId)
키워드 검색               → tourism-info-{lang}          searchKeyword2
```

### 3-3. API 호출 시퀀스 — 메인 탐색 플로우

```
Client (앱)
    │
    ▼
[1] 사용자: "경복궁 근처 무장애 식당 추천해줘" (일본어 입력)
    │
    ▼
[2] LLM Intent 파싱
    ├── 언어: JA
    ├── 위치: 경복궁 (위도 37.5796, 경도 126.9770)
    ├── 카테고리: 음식점 (contentTypeId=39)
    └── 필터: 무장애=true
    │
    ▼ (병렬 호출)
[3a] tourism-info-japanese.locationBasedList2
     params: mapX=126.9770, mapY=37.5796, radius=1000,
             contentTypeId=39
     → 일본어 음식점 목록 (최대 20개)

[3b] barrier-free-travel.locationBasedList2
     params: mapX=126.9770, mapY=37.5796, radius=1000,
             contentTypeId=39
     → 무장애 시설 목록 (무장애 정보 포함)
    │
    ▼
[4] 교집합 필터링 (contentId 기준)
    → 일본어 정보 있는 무장애 음식점
    │
    ▼
[5] 상위 3개 → detailCommon2 (상세 정보 병렬 조회)
    │
    ▼
[6] LLM Context 주입
    "다음 장소 정보를 바탕으로 일본어로 친절하게 추천 문장 생성:
     {장소명, 주소, 무장애 시설, 영업시간}"
    │
    ▼
[7] 사용자에게 일본어 응답 반환 + 지도 핀 표시
```

### 3-4. 캐싱 전략

```
┌──────────────────────────────────────────────────────┐
│                 Redis Cache 계층                       │
├──────────────────┬──────────┬────────────────────────┤
│ 데이터 유형       │ TTL      │ 캐시 키 패턴            │
├──────────────────┼──────────┼────────────────────────┤
│ 다국어 관광정보   │ 24h      │ info:{lang}:{areaCode} │
│ 위치기반 결과     │ 1h       │ loc:{lang}:{geohash}   │
│ 무장애 목록       │ 6h       │ bf:{areaCode}          │
│ 연관 관광지       │ 12h      │ rel:{contentId}        │
│ 집중률 예측       │ 2h       │ crowd:{date}:{id}      │
│ 오디오 가이드 URL │ 48h      │ audio:{contentId}      │
│ 상세 정보         │ 6h       │ detail:{contentId}     │
├──────────────────┼──────────┼────────────────────────┤
│ 오프라인 캐시     │ 다운로드  │ SQLite (앱 내장)       │
│ (여행 전 저장)    │ 시 고정   │ offline:{region}       │
└──────────────────┴──────────┴────────────────────────┘

전략:
- 다국어 API → 언어별 분리 캐시 (메모리 최적화)
- 위치기반 → Geohash 기반 캐시 키 (반경 ~1km 단위)
- 무장애 데이터 → 변경 빈도 낮아 TTL 길게
- 집중률 예측 → 2h (예측값 신뢰도 유지)
- 오프라인 모드 → 여행 전 SQLite 다운로드 (지역별 ~50MB)
```

- AI 통역: 사용자 설명 → 한국어 요약 → 112 전달
- 현재 위치 자동 GPS 공유

## 5. 차별화 포인트 (경쟁 서비스 대비)

**핵심 차별화**: 단순 번역 도구 → 여행 전반을 케어하는 AI 가이드

## 6. MVP 기능 우선순위 (MoSCoW)

### Must Have (공모전 데모 필수)
- [ ] 5개국어 AI 채팅 (EN, JA, ZH-S, ZH-T, ES) + KTO API 연동
- [ ] 위치기반 관광지 추천 (GPS + `locationBasedList2`)
- [ ] 무장애 필터 (`barrier-free-travel` API)
- [ ] 상세 페이지 (다국어 정보 + 무장애 시설 표시)
- [ ] SOS 긴급 화면 (기관 연결 UI)

### Should Have (완성도 향상)
- [ ] 오디오 가이드 재생 (`audio-guide` API)
- [ ] 연관 관광지 추천 (`related-attractions` API)
- [ ] 오늘 혼잡도 표시 (`visitor-concentration-forecast`)
- [ ] 루트 저장 + 공유
- [ ] 오프라인 캐시 (SQLite 다운로드)

### Could Have (향후 버전)
- [ ] 카메라 메뉴판 OCR 번역
- [ ] 그룹 여행 공동 루트 플래닝
- [ ] 국적별 개인화 추천 (빅데이터 기반)
- [ ] 프리미엄 구독 (무광고 + 심화 가이드)
- [ ] B2G API 연동 (지자체 관광청 데이터)

# K-Culture Deep Dive — 몰입형 문화 체험 UX 종합 설계

> K-팝·드라마 성지순례 + AI 루트 생성 + 배리어프리 지원 통합 플랫폼

|---|---|---|---|
| P1 | 김민준, 28세 · KR 내국인 드라마 팬 | 드라마 촬영지 실시간 정보 + 최적 동선 | 정보 파편화, 현장 폐업 낭패 |

## 2. 핵심 사용자 시나리오 (5단계 플로우)

### 2-A. 대표 시나리오: P2 Yuki의 뉴진스 성지순례 하루

```
[STEP 1] 탐색 (Discovery)
  입력: "뉴진스" 검색
  시스템: 관련 성지 12곳 로드 (tourism-info-japanese API)
          + 각 장소 운영 여부 실시간 표시
          + 팬 커뮤니티 인기순 정렬
  출력: 지도 + 리스트 뷰 (일본어 UI)

    ↓

[STEP 2] 루트 생성 (Route Planning)
  입력: "오늘 반나절 루트 만들기" 탭
  시스템: AI가 현재 위치 + 혼잡도(visitor-concentration-forecast)
          + 운영시간 고려해 최적 4개 장소 선별
          + 지하철 동선 포함 총 소요 시간 산출
  출력: 루트 카드 (지도 + 시간표 + 이동 방법)

    ↓

[STEP 3] 몰입 체험 (Immersive Experience)
  입력: 첫 번째 성지 도착 (GPS 감지)
  시스템: 자동 오디오 가이드 시작 (audio-guide API)
          일본어로 "멤버 민지가 이 카페에서..." 스토리 재생
          AR 오버레이로 멤버 사진 원래 위치에 표시
  출력: 오디오 + 화면 스토리텔링 + 포토스팟 가이드

    ↓

[STEP 4] 공유 & 기록 (Share & Archive)
  입력: 방문 완료 체크인
  시스템: 방문 인증 배지 생성 ("뉴진스 버니 레벨 3")
          X / Instagram 공유 카드 자동 생성
          나의 성지순례 지도에 기록 저장
  출력: 공유 카드 + 개인 순례 지도 업데이트

    ↓

[STEP 5] 발견 & 재참여 (Discovery Loop)
  시스템: 방문 기록 분석 → 미방문 관련 성지 3곳 추천
          커뮤니티에 같은 날 방문한 글로벌 팬 표시
          다음 방한 시 완성할 "성지 컬렉션" 제안
  출력: 개인화 추천 피드 + 글로벌 팬 지도
```

### 2-B. 배리어프리 시나리오: P3 박정호의 접근성 성지 방문

```
[STEP 1] 배리어프리 모드 활성화
  설정 → "휠체어 접근성 모드" ON
  자동 적용: 텍스트 16px+, 고대비, 큰 터치 타깃

[STEP 2] 접근성 필터 검색
  "도깨비 촬영지" + 배리어프리 A등급 필터
  → 접근 가능 장소만 표시 (barrier-free-travel API)

[STEP 3] 배리어프리 루트 생성
  엘리베이터 + 경사로 포함 경로
  + 중간 장애인 화장실 위치 표시
  + 전동 휠체어 이동 거리 최소화 옵션

[STEP 4] 현장 접근성 후기 등록
  방문 후 접근성 실제 상태 리뷰 (경사로 각도, 엘리베이터 크기 등)
  → 다른 장애인 여행자에게 실시간 공유

[STEP 5] 배지 & 커뮤니티
  "배리어프리 개척자" 뱃지 → 장애인 여행 커뮤니티 포스팅
```

## 3. Visit Korea OpenAPI 매핑

### 3-A. 사용 API 명세 (6개)

| # | API ID | 서비스명 | 활용 목적 | 호출 시점 |
|---|---|---|---|---|
| 1 | `tourism-info-korean` | 국문 관광정보 서비스 | 드라마/K-팝 성지 기본 정보 (운영시간, 주소, 상태) | 검색·상세보기 |
| 2 | `tourism-info-japanese` | 일문 관광정보서비스 | 일본어 사용자 성지 정보 제공 | 언어=JA 설정 시 |
| 3 | `audio-guide` | 관광지 오디오 가이드정보 | 성지 도착 시 자동 오디오 스토리텔링 | GPS 도착 감지 |
| 4 | `visitor-concentration-forecast` | 관광지 집중률 방문자 추이 예측 | 혼잡 시간대 회피 → 최적 방문 시각 추천 | 루트 생성 |
| 5 | `related-attractions` | 관광지별 연관 관광지 정보 | 성지 주변 연관 명소 자동 연결 | 상세보기·루트 |
| 6 | `barrier-free-travel` | 무장애 여행 정보 | 배리어프리 접근성 등급·경로 | 배리어프리 모드 |

### 3-B. API 호출 시퀀스 다이어그램

```
사용자 액션                  앱 클라이언트              Visit Korea API
────────────                 ─────────────              ───────────────

1. "뉴진스" 검색
                →  GET tourism-info-japanese
                    ?keyword=뉴진스&contentTypeId=12
                                               →  관광지 목록 반환
                ←  리스트 + 지도 표시

2. 혼잡도 확인
                →  GET visitor-concentration-forecast
                    ?contentId={각 장소 ID}
                                               →  시간대별 집중률 반환
                ←  혼잡도 배지 표시 (낮음/보통/혼잡)

3. 성지 도착 (GPS)
                →  GET audio-guide
                    ?contentId={장소 ID}&lang=ja
                                               →  일본어 오디오 URL 반환
                ←  오디오 자동 재생 + 스크립트 표시

4. 연관 성지 추천
                →  GET related-attractions
                    ?contentId={현재 장소 ID}
                                               →  연관 관광지 리스트
                ←  "이 성지를 방문한 팬이 다음에 간 곳" 추천

5. 배리어프리 모드 (P3)
                →  GET barrier-free-travel
                    ?contentId={장소 ID}
                                               →  접근성 상세 (경사로, 엘리베이터, 장애인 화장실)
                ←  배리어프리 등급 + 상세 접근 정보
```

### 3-C. 언어별 API 라우팅

```
사용자 언어 설정 → API 선택
  KO  →  tourism-info-korean
  JA  →  tourism-info-japanese
  ZH-CN → tourism-info-chinese-simplified
  ZH-TW → tourism-info-chinese-traditional
  EN  →  tourism-info-english
```

## 5. 차별화 포인트 (경쟁 서비스 대비 3개+)

| # | 차별화 포인트 | 기존 경쟁 서비스 | K-Culture Deep Dive |
|---|---|---|---|
| 1 | **실시간 성지 상태 + AI 루트** | 네이버맵: 위치만 제공, 운영 여부 불명확 / Visit Korea 앱: 정적 정보 | Visit Korea API + 팬 커뮤니티 실시간 데이터 통합 → 운영 여부·혼잡도·최적 시각 포함 AI 루트 자동 생성 |
| 2 | **팬덤 맥락 오디오 스토리텔링** | 기존 오디오 가이드: 역사·문화 중심, K-culture 성지 특화 없음 | audio-guide API + 아티스트·드라마 스토리 레이어 추가 → "민지가 이 카페에서..." 팬 감성 몰입 |
| 3 | **배리어프리 성지순례 통합** | 장애인 여행 정보: 별도 앱 분산 / K-culture 앱: 접근성 미고려 | barrier-free-travel API 통합 → K-문화 성지 배리어프리 등급 + 휠체어 경로 원스톱 제공 |
| 4 | **글로벌 팬 연결 (소셜 레이어)** | 해외 팬이 한국 성지 방문 시 고립감 | 같은 성지 방문 글로벌 팬 실시간 연결 + 8개국어 공유 카드 자동 생성 → 팬덤 커뮤니티 현지 확장 |
| 5 | **팬덤 배지 게임화** | 일반 여행 앱: 방문 체크인 기능 없거나 단순 스탬프 | 아티스트별 성지 컬렉션 + 레벨 시스템 → 재방문 동기부여 + 팬덤 정체성 강화 |

## 6. MVP 기능 우선순위 (Must / Should / Could)

### Must (출시 필수)

| 기능 | 연관 API | 이유 |
|---|---|---|
| K-팝/드라마별 성지 검색 + 지도 | `tourism-info-korean/japanese` | 핵심 가치 제안 |
| 성지 운영 상태 실시간 표시 | `tourism-info-korean` | P1 핵심 페인포인트 해결 |
| 다국어 UI (KO/JA/EN/ZH) | 다국어 API 라우팅 | P2 핵심 페인포인트 해결 |
| AI 루트 생성 (3~5개 장소) | `visitor-concentration-forecast` + `related-attractions` | 핵심 차별화 |
| 성지 오디오 가이드 | `audio-guide` | 몰입형 체험 핵심 |
| 배리어프리 필터 + 접근성 정보 | `barrier-free-travel` | P3 페인포인트 + 공모전 가산점 |
| 방문 인증 체크인 | - | 재참여 루프 기본 |

### Should (우선 후속 버전)

| 기능 | 이유 |
|---|---|
| 팬덤 배지 & 레벨 시스템 | 재방문·공유 동기 강화 |
| 현장 팬 실시간 채팅 | 글로벌 팬 연결 차별화 |
| AR 포토스팟 가이드 | 몰입 체험 강화 |
| 성지 사용자 리뷰 & 접근성 후기 | 정보 신선도 크라우드소싱 |
| 혼잡도 그래프 (시간대별) | 방문 시각 최적화 |

### Could (장기 로드맵)

| 기능 | 이유 |
|---|---|
| K-콘텐츠 IP 공식 파트너십 (HYBE·SM 등) | B2B 수익화, 독점 콘텐츠 |
| 성지 AR 필터 (멤버 포토카드 오버레이) | TikTok 바이럴 잠재력 |
| 팬덤 투어 상품 예약 연동 | 수익화 채널 확대 |
| 오프라인 성지 굿즈 QR 연동 | 커머스 레이어 |
| AI 챗봇 성지 큐레이터 | 개인화 심화 |

```
Week 1 (5/5~5/9): 기반 인프라 + Must 기능 1/2

□ 지도 컴포넌트 (React Native Maps / 카카오맵)
□ 다국어 i18n 설정 (KO/JA/EN/ZH-CN 4개국어)
마일스톤: 검색 → 지도 핀 표시 동작 확인

Week 2 (5/12~5/16): Must 기능 2/2
────────────────────────────────────────────────

□ AI 루트 생성 로직
  - visitor-concentration-forecast 혼잡도 가중치 적용
  - related-attractions 연관 성지 그래프 탐색
  - 지하철 동선 계산 (Kakao Mobility API)

□ 방문 체크인 + 배지 지급 로직

────────────────────────────────────────────────
□ 팬덤 배지 & 레벨 UI (화면 6)
```

## 1. 서비스 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | 치유여정 (Healing Journey) |
| 카테고리 | 웰니스/감성여행 |

## 2. 페르소나 3종

### 🇰🇷 페르소나 A — "번아웃 직장인 지수"
- **나이/직업**: 34세, 서울 IT 기업 PM
- **상황**: 프로젝트 마감 연속 3개월, 주말에 어디든 가고 싶지만 계획 짜기조차 지침
- **JTBD (해결 과제)**: "지금 내 감정에 맞는 곳에 가서 아무 생각 없이 쉬고 싶다"
- **페인 포인트**:
  - 여행 계획 세우는 에너지가 없음
  - SNS에서 본 곳은 항상 붐빔
  - '좋다는 곳' 가도 정작 힐링이 안 됨
- **기대 결과**: 앱이 내 상태를 보고 조용한 곳, 자연 소리, 명상 가이드를 알아서 준비해줌
- **사용 패턴**: 목요일 밤에 앱 열어 주말 코스 설정, 1박 2일 이내

### 🌏 페르소나 B — "힐링 한국 방문 외국인 Emma"
- **나이/직업**: 29세, 캐나다 프리랜서 디자이너
- **상황**: 6개월 한국 체류 중, K-웰니스 트렌드(사찰음식·명상·온천)에 관심
- **JTBD**: "진짜 한국 치유 문화를 깊이 경험하고 싶다. 번역된 관광지 말고"
- **페인 포인트**:
  - 웰니스 특화 영어 정보 부족
  - 한국의 힐링 장소를 문화적으로 이해하기 어려움
  - 예약/접근성 정보가 단편적
- **기대 결과**: 영어로 전통 사찰 오디오 가이드 + 조용한 시간대 추천 + 사찰 예약까지
- **사용 패턴**: 앱에서 영어 코스 탐색 → 1주일 전 계획

### ♿ 페르소나 C — "접근성 약자 은혜 어머니"
- **나이/직업**: 62세, 무릎 수술 회복 중, 서울 거주
- **상황**: 자녀가 선물한 치유 여행, 휠체어 없이도 걷기 힘든 상태
- **JTBD**: "몸이 불편해도 아름다운 자연에서 마음이 편안해지고 싶다"
- **페인 포인트**:
  - 경사로/계단 정보가 없어 갔다가 낭패
  - '무장애' 여행지 정보가 흩어져 있음
  - 체력에 맞는 코스 조합이 안 됨
- **기대 결과**: 평지 위주 힐링 경로 + 휴게시설 간격 + 동반 보호자 편의시설 안내
- **사용 패턴**: 가족과 함께 앱 탐색, 출발 2주 전 계획

## 3. 핵심 사용자 시나리오 — 5단계 플로우

> 페르소나 A (번아웃 직장인 지수) 기준 대표 시나리오

### 단계 1: 감정 체크인 (0-30초)
```
사용자가 앱을 열면:
  → "지금 어떤 느낌이에요?" 3가지 감정 선택기 표시
  → [🔥 지쳐있어요] [😶 멍하니 있고 싶어요] [🌱 뭔가 발견하고 싶어요]
  → 지수: "지쳐있어요" 선택
  → 부가 질문: "얼마나 멀리 갈 수 있어요?" → [당일] [1박] [2박+]
```

### 단계 2: 힐링 모드 자동 배정 (30초-1분)
```
감정 입력 → AI 매핑 로직:
  "지쳐있어요" + "1박" → 🎵 LISTEN 모드 (듣기: 자연음 + 오디오 명상)
  
  시스템 호출:
  1. wellness-tourism API → 강원/충남 웰니스 장소 조회
  2. visitor-concentration-forecast → 이번 주말 혼잡도 낮은 장소 필터
  3. area-tourism-demand-density → 체류 만족도 높은 지역 선별
```

### 단계 3: 코스 제안 (1-2분)
```
화면: "지수님을 위한 치유 코스 3가지"

[코스 A] 🌊 강원 정선 — 조용한 산사 + 별빛 숲
  예상 혼잡도: ●○○ (낮음) | 왕복 3시간 | 1박 추천

[코스 B] 🌿 충남 태안 — 해변 명상 + 석양 오디오 가이드
  예상 혼잡도: ●●○ (보통) | 왕복 2시간 | 당일 가능

[코스 C] 🍃 경기 양평 — 강변 걷기 + 사운드배스 웰니스센터
  예상 혼잡도: ●○○ (낮음) | 왕복 1.5시간 | 당일/1박

  → 지수: 코스 A 선택
```

### 단계 4: 코스 상세 및 오디오 준비 (2-5분)
```
코스 A 상세:
  - 이동 경로 + 대중교통 안내
  - audio-guide API → 정선 산사 한국어 오디오 스크립트 다운로드
  - 관련 웰니스 장소 5곳 상세 정보 (wellness-tourism API)
  - 연관 관광지: 주변 식당/숙소 (related-attractions API)
  - 두루누비 트레일: 산사 인근 걷기 코스 (durunubi-trails API)

  오프라인 저장 옵션: 오디오 + 지도 캐싱 (신호 약한 산간 대비)
```

### 단계 5: 여정 중 & 이후 (여행 당일)
```
당일:
  → 위치 기반: 현재 장소 인근 힐링 포인트 실시간 안내
  → 오디오 자동 재생: 장소 도착 시 명상 가이드 시작
  → 체크인 알림: "이 장소에서 5분 명상 해볼까요?"

여정 후:
  → 감정 체크아웃: "지금 느낌은 어때요?"
  → 힐링 지수 트래킹: 방문 전/후 감정 변화 기록
  → 다음 여정 추천: "다음엔 DISCOVER 모드 어때요?"
```

## 4. Visit Korea OpenAPI 매핑

### 4.1 사용 API 명세

| # | API ID | API명 | 힐링 모드 | 주요 활용 |
|---|---|---|---|---|
| 1 | `wellness-tourism` | 웰니스관광정보 | ALL | 웰니스 장소 기반 풀 (핵심) |
| 2 | `tourism-info-korean` | 국문 관광정보 | ALL | 관광지 상세/검색/위치기반 |
| 3 | `tourism-info-english` | 영문 관광정보 | ALL | 외국인 페르소나 B 전용 |
| 4 | `audio-guide` | 오디오 가이드 | LISTEN | 명상·해설 오디오 콘텐츠 |
| 5 | `visitor-concentration-forecast` | 집중률 예측 | ALL | 혼잡도 회피 (고독한 힐링 보장) |
| 6 | `related-attractions` | 연관 관광지 | DISCOVER | 코스 확장 (식당·숙소 자동 연결) |
| 7 | `eco-tourism` | 생태관광정보 | REST/DISCOVER | 자연 힐링 장소 |
| 8 | `durunubi-trails` | 두루누비 정보 | REST | 걷기·자전거 힐링 코스 |

> 페르소나 C (접근성 약자) 추가 API: `barrier-free-travel` (무장애 여행 정보)

### 4.2 호출 시퀀스 다이어그램

```
[사용자 감정 입력]
       │
       ▼
[Step 1: 후보 장소 풀 구성]
wellness-tourism.wellnessTursmSyncList(areaCode)
tourism-info-korean.areaBasedList2(contentTypeId=wellness)
eco-tourism.areaBasedList(areaCode)
       │ 병렬 호출 (Promise.all)
       ▼
[Step 2: 혼잡도 필터링]
visitor-concentration-forecast.tatsCnctrRateList(contentId[])
       → 집중률 < 40% 장소만 통과 ("조용한 힐링" 보장)
       │
       ▼
[Step 3: 힐링 모드별 상세 로드]
  LISTEN 모드:
    audio-guide.storyLocationBasedList(mapX, mapY, radius=5km)
    audio-guide.themeBasedList(themeCategory=nature)
  REST 모드:
    durunubi-trails.locationBasedList(mapX, mapY)
    wellness-tourism.detailInfo(contentId)
  DISCOVER 모드:
    related-attractions.detailList(contentId)
    tourism-info-korean.locationBasedList2(mapX, mapY)
       │
       ▼
[Step 4: 코스 조합 + 출력]
tourism-info-korean.detailCommon2(contentId[]) — 상세 정보
tourism-info-korean.detailImage2(contentId[]) — 이미지
```

### 4.3 캐싱 전략

| 데이터 유형 | TTL | 전략 | 이유 |
|---|---|---|---|
| 웰니스 장소 목록 | 24시간 | Redis 서버 캐시 | 변경 빈도 낮음 |
| 혼잡도 예측 | 6시간 | Redis 서버 캐시 | 30일 예측이므로 일중 최신화 충분 |
| 오디오 가이드 파일 | 영구 | CDN + 클라이언트 로컬 저장 | 정적 콘텐츠, 오프라인 필수 |
| 연관 관광지 | 12시간 | Redis 서버 캐시 | 반정적 데이터 |
| 위치기반 검색 | 5분 | 메모리 캐시 | 실시간성 중요 |
| 관광지 이미지 | 7일 | CDN | 대용량, 정적 |

## 6. 차별화 포인트 (경쟁 서비스 대비)

### 차별화 1: 감정→코스 자동 매핑 (업계 최초)
| 비교 | 네이버 여행 | 트립어드바이저 | 치유여정 |
|---|---|---|---|
| 추천 기반 | 인기도/별점 | 리뷰 수 | **현재 감정 상태** |
| 혼잡도 정보 | 없음 | 부분적 | **KTO 예측 데이터로 30일 예측** |
| 웰니스 특화 | 없음 | 없음 | **KTO 웰니스 API 전체 활용** |

**근거**: 기존 여행 앱은 "어디가 좋아요?" 방식. 치유여정은 "지금 나는 어떤 상태인가?"에서 출발 — 심리적 접근이 차별점.

### 차별화 2: 오디오 힐링 레이어 (KTO 공공 오디오 × 웰니스 결합)
- 기존 오디오 가이드: 역사/문화 해설 중심
- 치유여정: 동일 API 데이터를 **명상 가이드 + 힐링 내레이션**으로 재편집
- 자연 사운드스케이프 + KTO 오디오를 레이어드 → 이동 중 힐링 시작
- **오프라인 저장**: 산간/오지 힐링 장소 신호 불안 해소

### 차별화 3: "혼잡도 0" 보장 큐레이션
- KTO 집중률 예측 API (향후 30일) 활용
- 집중률 40% 미만 장소만 힐링 코스에 포함
- "남들 없는 곳에서 진짜 혼자만의 시간" — 번아웃 회복의 핵심 조건
- 경쟁 서비스에는 이 기능 없음

### 차별화 4: 힐링 지수 트래킹 (감정 변화 데이터화)
- 방문 전/중/후 감정 체크인 → 힐링 효과 수치화
- 개인 힐링 패턴 학습: "지수님은 자연음에서 회복 속도 2.3배"
- 기업 B2B: 직원 복지 프로그램에 리포트 제공 (번아웃 예방)

# LocalSecret v2 — 감정 발견 UX 기획 디테일

## 1. 서비스 개요

**서비스명**: LocalSecret v2  
**슬로건**: "오늘, 어떤 발견을 원해?"  
**한 줄 포지션**: 인기 순위 대신 감정으로 찾는, 아직 아무도 모르는 한국

**핵심 JTBD**:
> "SNS에서 본 관광지 말고, 아직 아무도 모르는 나만의 장소를 처음 발견하는 순간이 좋아서 앱을 연다."

## 2. 페르소나 3종

### P1 — 김지원 (25세, 한국 내국인 솔로 여행자)

| 항목 | 내용 |
|---|---|
| **직업** | 프리랜서 그래픽 디자이너, 서울 마포구 |
| **여행 패턴** | 월 1–2회 당일치기, 혼자 70% |
| **예산** | 당일 2–4만원 |
| **핵심 고통** | SNS 포토스팟 막상 가면 혼잡 / 믿을 수 있는 큐레이션 없음 |
| **감정 JTBD** | "아직 아무도 올리지 않은 곳을 내가 먼저 발견해서 SNS에 올리고 싶다" |
| **선택 감정** | 비밀 ✨, 계절 🌸 |
| **기술 친숙도** | High — 숏폼 네이티브, 신규 앱 거부감 없음 |

**대표 시나리오**: 금요일 밤 인스타 릴스에서 "강릉 숨은 카페"를 보고 즉흥 당일치기를 결심. 앱에서 "비밀" 선택 → 향후 30일 혼잡도 최하위 TOP 3 확인 → 토요일 오전 10시 출발 시각 최적화 → 아무도 없는 카페 첫 방문 인증샷 게시.

---

### P2 — 타나카 하루카 (27세, 일본 외국인 첫 방한 여행자)

| 항목 | 내용 |
|---|---|
| **직업** | 도쿄 IT 마케터 |
| **여행 패턴** | 3박 4일 연휴 여행, 친구 1–2명 동행 |
| **예산** | 총 50만 엔 (숙박·항공 포함) |
| **핵심 고통** | 언어 장벽 / 일본어 신뢰 정보 앱 없음 / K-drama 성지 위치 불확실 |
| **감정 JTBD** | "관광지도에 없는 진짜 한국을 발견해서 일본 팔로워에게 먼저 공유하고 싶다" |
| **선택 감정** | 정취 🏮, 비밀 ✨ |
| **기술 친숙도** | High — LINE·Instagram·X 헤비 유저, 일본어 UI 없으면 영어 대체 |

**대표 시나리오**: 앱 언어를 일본어로 전환 → "정취" 선택 → 군산 신흥동 일본식 가옥 골목 추천 → "現在の訪問者: 3人" 혼잡도 확인 → 일본어 오디오 가이드 재생하며 방문 → X에 "韓国の隠れた名所発見！" 바이럴 게시.

---

### P3 — 이재호 (38세, 전동휠체어 사용 접근성 약자)

| 항목 | 내용 |
|---|---|
| **직업** | 스타트업 백엔드 개발자 (재택) |
| **여행 패턴** | 연 3–4회, 1박 2일–3박 4일, 파트너 60%·혼자 40% |
| **예산** | 1박 기준 15–20만원 |
| **핵심 고통** | 무장애 정보 신뢰성 낮음 / 경사도·단차 등 세부 데이터 없음 / 충전소 정보 전무 |
| **감정 JTBD** | "예기치 않은 계단으로 당황하는 일 없이, 나도 일반 여행자처럼 숨은 명소를 자립 발견하고 싶다" |
| **선택 감정** | 고요 🌙, 정취 🏮 |
| **기술 친숙도** | Very High — 개발자, 데이터 정확도 최우선 |

**대표 시나리오**: "고요" + 무장애 필터 활성화 → 전동휠체어 진입 가능(경사도 15° 이하) 장소만 표시 → 크라우드소싱 최신 리뷰("3주 전 방문, 경사로 양호") 확인 → 동선 사전 확정 후 현장 방문 → 예기치 않은 장벽 없이 완전 자립 여행 완수.

## 3. 핵심 사용자 시나리오 (5단계 플로우)

### 플로우 명칭: "비밀 발견 → 방문 → 공유"

```
[단계 1: 감정 진입]
앱 실행 → Hero Screen
"오늘, 어떤 발견을 원해?"
  ├── ✨ 비밀  — 아직 아무도 모르는 곳
  ├── 🌙 고요  — 사람 없는 자연
  ├── 🏮 정취  — 오래된 골목과 시간
  └── 🌸 계절  — 지금만 볼 수 있는 풍경
           ↓
[단계 2: 개인화 스코어링]
선택 감정 + 현재 위치 + 계절 + 현재 시각 →
  API 호출:
    - visitor-concentration-forecast (향후 30일 혼잡도)
    - area-tourism-demand-density (수요 밀도 하위 구간)
    - area-tourism-diversity (다양성 지수 낮은 지역 보완)
  → 혼잡도 최하위 + 감정 매핑 TOP 5 추천 리스트 생성
           ↓
[단계 3: 발견 카드 탐색]
숨은 명소 카드 스와이프 뷰
  - 실시간 방문자 수 ("지금 3명")
  - 30일 예측 최적 방문 타이밍
  - 한국인/외국인 이중 렌즈 설명 탭 전환
  - 계절별 매력 포인트 (현재 계절 강조)
  - 무장애 필터 (P3 전용 세부 데이터)
           ↓
[단계 4: 방문 준비 & 알림]
카드 저장 → "방문 타이밍 알림" 설정
  - API: related-attractions → 주변 연관 명소 연결
  - 출발 당일 아침 혼잡도 재확인 푸시 알림
  - 오디오 가이드 다운로드 (오프라인 대응)
           ↓
[단계 5: 방문 후 공유]
방문 완료 체크인 →
  - 실제 방문자 수 크라우드소싱 업데이트 (다음 사용자를 위해)
  - SNS 공유 카드 생성 ("내가 발견한 비밀 — [장소명]")
  - 발견 포인트 누적 → 더 숨은 명소 잠금 해제 게이미피케이션
```

## 4. Visit Korea OpenAPI 매핑

### 4-1. 사용 API 명세 (5종)

| # | API 파일명 | 역할 | 핵심 파라미터 | 호출 빈도 |
|---|---|---|---|---|
| 1 | `한국관광공사_관광지 집중률 방문자 추이 예측 정보` | 향후 30일 혼잡도 예측 — 핵심 추천 필터 | `contentId`, 예측기간 | 실시간 (추천 시) |
| 2 | `한국관광공사_지역별 관광 수요 강도` | 수요 밀도 하위 구간 선별 (덜 알려진 곳 발굴) | `areaCode`, 기간 | 일 1회 캐시 |
| 3 | `한국관광공사_지역별 관광 다양성` | 다양성 지수 낮은 지역 보완 추천 | `areaCode` | 일 1회 캐시 |
| 4 | `한국관광공사_관광지별 연관 관광지 정보` | 혼잡 명소의 한적한 대체지 추천 | `contentId` | 추천 시 |
| 5 | `한국관광공사_관광빅데이터 정보서비스` | "아직 덜 알려진 곳" 트렌드 감지 | `areaCode`, 방문자수 | 일 1회 캐시 |

**보조 API (다국어·무장애)**

| API | 용도 |
|---|---|
| `한국관광공사_무장애 여행 정보` | P3 접근성 약자 무장애 필터 |
| `한국관광공사_관광지 오디오 가이드정보` | P2 외국인 오디오 가이드 (일/영) |
| `한국관광공사_일문 관광정보서비스` | P2 일본어 UI 전환 시 관광정보 제공 |
| `한국관광공사_영문 관광정보서비스` | P2 영어 UI 전환 시 관광정보 제공 |

### 4-2. API 호출 시퀀스

```
사용자 감정 선택 (예: "비밀")
        │
        ▼
① area-tourism-demand-density
   → 수요 밀도 하위 20% 지역 코드 목록 추출
        │
        ▼
② area-tourism-diversity
   → 관광 다양성 지수 보정 (동일 지역 쏠림 방지)
        │
        ▼
③ visitor-concentration-forecast
   → 필터된 contentId 목록에 대해 향후 30일 혼잡도 예측
   → 감정 키워드 매핑 스코어 × 혼잡도 역가중치
   → TOP 5 추천 정렬
        │
        ▼
④ related-attractions (선택한 명소에 대해)
   → 주변 연관 명소 3개 추가 제안
        │
        ▼
⑤ tourism-big-data (배경 배치)
   → 일 1회 실행, 신흥 트렌드 명소 DB 업데이트
```

**캐시 전략**: ②③⑤는 일 1회 CDN 캐시. ①④는 사용자 요청 시 실시간 호출.

## 6. 차별화 포인트 (경쟁 서비스 대비 4개)

### 차별화 1: 감정 선택 첫 화면 (vs. 목적지 검색 첫 화면)

| 경쟁 서비스 | LocalSecret v2 |
|---|---|
| 트리플·야놀자·Klook: "어디 갈래요?" 검색창 | "오늘, 어떤 발견을 원해?" 감정 4종 선택 |
| 인기도·별점·예약수 기반 순위 표시 | 혼잡도 최하위 × 다양성 지수 역순위 정렬 |
| 이미 알려진 명소 재발견 | 아직 아무도 모르는 곳을 처음 발견하는 경험 |

### 차별화 2: 예측 혼잡도 역활용 (vs. 주간 평균 혼잡도 표시)

경쟁사가 "지금 혼잡도"를 회피 정보로만 쓸 때, LocalSecret v2는 **향후 30일 예측 혼잡도를 적극적으로 역활용**한다.
- "이 장소는 언제 가장 한산한가?"를 자동 계산
- 최적 방문 타이밍 알림으로 발견 성공률 보장
- SNS 바이럴 전에 먼저 방문하는 "선점 경험" 설계

### 차별화 3: 한국인/외국인 이중 렌즈 (vs. 단일 관점 설명)

같은 장소를 보는 한국인과 외국인의 의미 차이를 명시적으로 병기:
- 한국인: "어릴 때 외갓집 가던 길 느낌" (향수, 정서적 연결)
- 외국인: "What rural Korea looked like 30 years ago" (진정성, 문화 발견)
- 동일 POI가 두 타겟 모두에게 최적 콘텐츠로 기능 → 국내·외국인 이중 타겟 달성

### 차별화 4: 접근성 약자를 위한 무장애 데이터 정밀화

기존 앱의 "휠체어 접근 가능" 한 줄 표시 대신:
- 경사도(°) · 바닥 재질 · 단차 높이 · 전동휠체어 충전소 위치
- 크라우드소싱 최신 현장 리뷰 (날짜 + 구체적 상태)
- 무장애 API + 사용자 생성 데이터 결합으로 신뢰도 향상

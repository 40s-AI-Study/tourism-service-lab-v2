# 🧘 Mood Travel Matcher — 기획고도화 산출물

> **카테고리**: 웰니스/감성여행 | **순위**: ★2 | **이슈**: TOU-86

## 1. 페르소나 3종

### 페르소나 A — 🇰🇷 KR 내국인: 번아웃 MZ 직장인

| 항목 | 내용 |
|------|------|
| **이름** | 김하은, 26세 |
| **거주** | 서울 마포구 |
| **직업** | 스타트업 마케터 (3년차) |
| **키워드** | 번아웃 · 즉흥 여행 · SNS 감성 |
| **디지털** | 앱 네이티브, 평균 스마트폰 7시간/일 |

**Pain Points**
- 매일 야근 후 주말 새벽에 "어디든 가고 싶다"는 충동이 오지만 검색조차 에너지가 없음
- 네이버 여행·카카오맵 검색 시 정보 과다로 결정 마비 (choice overload)
- "기분"을 입력해서 여행지를 받아본 경험이 없어 기대감 낮음

**Goals**
- 30초 안에 "지금 내 기분에 맞는 여행지" 1~3개 추천
- 교통편·숙박 포함 원스톱 패키지 제안
- 혼자 또는 친구 1명과 당일치기/1박 위주

**시나리오 트리거**
> 금요일 오후 11시, 야근 퇴근 후 지하철 안. 인스타그램 피드에서 온천 사진을 보고 갑자기 힐링이 고팠다. 앱 열어서 "지금 지쳐서 아무것도 하기 싫어" 입력.

- "내 취향(감성, 힐링, 조용한 곳)"에 맞는 한국 여행지를 발견하기 어려움
- 예약·이동 정보가 한국어 중심이라 실행 장벽이 높음

**Goals**
- 자국어(일본어) 또는 영어로 감정 입력 → 한국 여행지 추천

**시나리오 트리거**
> 도쿄에서 한국 여행을 2주 뒤로 계획. "relaxed, want to escape from city noise, love nature and quiet cafes" 영어로 입력. AI가 남해·통영·전주 한옥마을 조합을 추천.

## 2. 핵심 사용자 시나리오 — 5단계 플로우

> **대표 시나리오**: 김하은 (페르소나 A) · 금요일 밤 즉흥 힐링 여행

### Step 1 — 감정 입력 (Emotion Input)

**화면**: 감정 선택 화면
**시간**: 0~15초

사용자가 앱을 열면 "지금 기분이 어때요?" 한 문장이 화면을 채운다. 선택 방법은 두 가지:

- **터치 선택**: 😴지쳐요 / 😰불안해요 / 😢슬퍼요 / 😤화나요 / 😌차분해요 / 🥳신나요 / 😐그냥그래요 중 하나 탭
- **자연어 입력**: "지금 지쳐서 아무것도 하기 싫어" 자유 텍스트 입력

자연어 → 감정 카테고리 AI 파싱 (번아웃/피로 → `exhausted` 태그 할당).

---

### Step 2 — AI 분석 & 여행지 매칭 (Matching)

**화면**: 로딩 + 분석 결과 카드
**시간**: 15~30초

AI가 감정 태그(`exhausted`, `nature_craving`, `solo_preferred`)를 기반으로:

1. 웰니스관광정보 API + 국문 관광정보 API 호출
2. 감정-여행지 매핑 모델 적용 (번아웃 → 자연/온천/사찰 우선순위 부여)
3. 결과 3개 카드 생성:
   - **1순위**: 충주 수안보온천 — "완전 방전 리셋 코스"
   - **2순위**: 강원 홍천 자연휴양림 — "숲멍 힐링"
   - **3순위**: 전남 구례 화엄사 — "사찰 명상"

---

### Step 3 — 상세 탐색 & 커스터마이징 (Explore)

**화면**: 여행지 상세 + 필터
**시간**: 30초~3분

선택한 카드(예: 수안보온천) 탭 시:
- 대표 이미지 갤러리 (관광사진 API)
- 감정-여행지 매칭 이유: "번아웃 해소에 온천 효과 연구 근거 + 주변 조용한 숲길"
- 교통 정보: 서울 청량리 → 충주 (ITX 1시간 30분)
- 주변 관광지 3곳 (관련 관광지 API)
- 숙소 유형 선택: 온천 리조트 / 독채 펜션 / 한옥 민박

사용자 필터 조정:
- 이동 시간 (2시간 이내 / 당일치기 / 1박)
- 동행 유형 (혼자 / 친구 / 커플)
- 예산 (3만원 이하 / 5만원 / 10만원 이상)

---

### Step 4 — 코스 생성 & 저장 (Plan)

**화면**: 여행 코스 플래너
**시간**: 3~5분

AI가 선택 조건 기반으로 당일 또는 1박 일정 자동 생성:

```
[당일 코스 예시 — 수안보온천]
10:00  청량리 출발 (ITX-청춘)
11:30  충주 도착 → 수안보 이동 (버스 30분)
12:00  점심: 충주 닭갈비 or 수안보 온천두부
13:30  수안보온천 스파 입장 (2시간)
15:30  월악산 자락 산책로 (30분 코스 — 무리 없는 거리)
17:00  카페에서 디저트 & 충전
18:30  충주 출발
20:00  서울 귀환
```

저장 옵션: 마이페이지 저장 / 카카오톡 공유 / 캘린더 추가

---

### Step 5 — 피드백 & 러닝 (Feedback Loop)

**화면**: 여행 후 리뷰 (귀가 시점 푸시)
**시간**: 여행 후 알림

여행 귀가 예상 시간에 푸시 알림 발송:
- "수안보 여행 어땠어요? 기분이 나아졌나요?" 별점 + 짧은 감정 기록
- 긍정 피드백 → 유사 감정-여행지 패턴 강화
- 부정 피드백 → 다음 매칭에서 해당 카테고리 가중치 조정
- 여행 사진 업로드 → UGC 콘텐츠로 전환 (동의 시)

## 3. Visit Korea OpenAPI 매핑

### 3.1 사용 API 목록

| # | API명 | 용도 | 핵심 오퍼레이션 |
|---|-------|------|----------------|
| 1 | 웰니스관광정보 | 감정 → 웰니스 여행지 매칭 핵심 데이터 | `areaBasedList`, `locationBasedList`, `detailInfo`, `wellnessTursmSyncList` |
| 2 | 국문 관광정보 서비스 | 여행지 기본 정보, 이미지, 주변 관광지 | `areaBasedList2`, `detailCommon2`, `detailImage2`, `locationBasedList2` |
| 3 | 관광지별 연관 관광지 정보 | "이런 곳도 좋아요" 추천 | `relatedTourSpotList` |
| 4 | 관광사진 정보 | 여행지 감성 이미지 제공 | `photoList`, `photoDetail` |
| 5 | 관광지 집중률 방문자 추이 예측 정보 | 혼잡도 예측 (힐링 여행자 니즈) | `visitorForecast`, `congestionRate` |
| 6 | 무장애 여행 정보 | 접근성 약자(페르소나 C) 필터 | `barrierFreeList`, `barrierFreeDetail` |
| 7 | 지역별 관광 수요 강도 | AI 매칭 정확도 보완 데이터 | `regionDemandIndex` |

### 3.2 API 호출 시퀀스

```
[사용자 감정 입력]
       │
       ▼
[1단계 — 감정 파싱 (서버사이드 LLM)]
  감정 태그 생성: exhausted | nature | solo | budget_mid
       │
       ▼
[2단계 — 웰니스 여행지 1차 후보 (캐시 우선)]
  GET /wellness/areaBasedList?contentTypeId=&areaCode=...
  → 웰니스관광정보 API (캐시 TTL: 6시간)
       │
       ├── [병렬] GET /tourism/locationBasedList2 (국문 관광정보)
       ├── [병렬] GET /visitor/visitorForecast (혼잡도 예측)
       └── [조건부] GET /barrierfree/barrierFreeList (페르소나C만)
       │
       ▼
[3단계 — 1차 필터 + 스코어링]
  감정-여행지 매핑 알고리즘 적용
  (피로/번아웃 → 온천·산림·해안가 가중치 상향)
       │
       ▼
[4단계 — 상위 5개 상세 조회 (병렬)]
  GET /tourism/detailCommon2?contentId={id}  (5개 병렬)
  GET /tourism/detailImage2?contentId={id}   (5개 병렬)
  GET /related/relatedTourSpotList?contentId={id}
       │
       ▼
[5단계 — 응답 조합 → 카드 3개 생성]
  최종 추천 결과 반환 (2~3초 목표)
```

### 3.3 캐싱 전략

| 데이터 유형 | TTL | 캐시 레이어 | 이유 |
|------------|-----|------------|------|
| 웰니스 여행지 목록 | 6시간 | Redis | 데이터 변경 빈도 낮음 |
| 관광지 기본 정보 | 24시간 | Redis + CDN | 정적 데이터 |
| 관광 사진 | 7일 | CDN (S3) | 이미지는 거의 불변 |
| 혼잡도 예측 | 1시간 | Redis | 시간대별 갱신 필요 |
| 연관 관광지 | 12시간 | Redis | 준정적 데이터 |
| 무장애 정보 | 24시간 | Redis | 변경 빈도 낮음 |
| 지역별 수요 강도 | 3시간 | Redis | 주기적 업데이트 |

**캐시 키 설계**:
```
wellness:{areaCode}:{contentTypeId}:{sort}  → 웰니스 목록
detail:{contentId}:common                   → 기본 정보
photo:{contentId}:list                      → 사진 목록
barrierfree:{areaCode}:list                 → 무장애 정보
```

**Cache Miss 처리**: API 호출 타임아웃 3초 설정, 실패 시 스태일 캐시 반환 (Stale-while-revalidate 패턴).

| 힐링 관련 앱 | 명상·마음챙김 | 여행 연계 없음 |

### 차별화 포인트 3가지

**① 감정 → 여행지 실시간 AI 매칭 (업계 최초)**

기존 서비스는 모두 "목적지 → 정보 검색" 방향이다. Mood Travel Matcher는 "감정 → 여행지 발견"의 역방향 흐름을 도입. 사용자가 목적지를 몰라도 됨. 자연어 감정 입력 하나로 맞춤 추천 제공.

- 경쟁 우위: 네이버/카카오는 감정 기반 추천 없음
- KPI: 감정 입력 → 추천까지 30초 이내
- 기술 기반: Visit Korea 웰니스관광정보 API + LLM 감정 파싱

**② Visit Korea 공공 데이터 × AI 융합 신뢰성**

민간 앱들이 접근하기 어려운 한국관광공사 공식 27개 API를 풀 활용. 웰니스관광정보, 무장애여행정보, 관광지 혼잡도 예측 등 공공 데이터로 신뢰 기반 추천. 상업적 광고성 추천이 아닌 데이터 기반 중립 추천.

- 경쟁 우위: 공공 데이터 활용으로 광고 편향 없음
- KPI: 사용자 신뢰도 NPS 70 이상 목표
- 기술 기반: 7개 OpenAPI 통합 + 실시간 캐싱

**③ 3개 언어 × 3개 페르소나 통합 (다국적 웰니스 플랫폼)**

국내 MZ + 외국인 (영어/일본어 UI) + 접근성 약자를 단일 플랫폼에서 지원. 영문관광정보서비스 + 일문관광정보서비스 + 무장애여행정보 API를 활용해 진정한 포용적 서비스 구현. 경쟁 앱들은 내국인 또는 외국인 중 하나에만 집중.

- 경쟁 우위: 한·영·일 3개 언어 + 무장애 정보 통합
- KPI: 외국인 사용자 비율 30% 이상
- 기술 기반: i18n 다국어 API + 무장애여행정보 API

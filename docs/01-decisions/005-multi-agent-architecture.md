# ADR-005: Paperclip 기반 7-에이전트 병렬 협업 아키텍처 채택

## 상태

`approved` — 프로젝트 시작 시 결정 (2026-04-17), 사후 문서화 (2026-05-13)

## 맥락

2026 관광데이터 활용 공모전 준비를 위해 20일 이내에 59개 아이디어 발산 → 262건 시뮬레이션 → 27개 프로토타입 → 제안서 PDF까지 완성해야 했다. 단일 에이전트로 순차 처리하면 컨텍스트 한계와 처리 시간이 병목이 될 것으로 판단했다.

## 결정

**Paperclip 멀티에이전트 프레임워크 기반, 역할 분리된 7개 에이전트를 `maxConcurrentRuns=3`으로 병렬 운영**한다.

### 에이전트 구성

| 에이전트 | 역할 | 주요 산출물 |
|---|---|---|
| 🎯 CEO (전략님) | 오케스트레이션, 최종 결정, Roll Call | 이슈 배분, 통합 매트릭스, 최종 선정 |
| 📋 PM (기획님) | 서비스 기획, 기능 정의, 문서 조율 | 서비스 기획서, 아이디어 발산, 위키 |
| 🔍 MR (리서치님) | 시장 조사, 경쟁 분석 | 관광 트렌드, 경쟁 서비스 매트릭스 |
| 🎨 UX (디자인님) | 페르소나, 와이어프레임, 프로토타입 | 페르소나 10명, HTML 프로토타입 27개 |
| 💰 BA (분석님) | 사업성 점수화, 수익 모델 | 150점 매트릭스, TAM/SAM/SOM |
| 🔌 API (엔지니어님) | API 연동 설계, 기술 아키텍처 | API 카탈로그, 기술 스펙, 프로토타입 구현 |
| 🎭 Sim (시뮬님) | 페르소나 시뮬레이션 | 262건 Pass/Fail 판정 |

## 근거

### 채택 이유

1. **컨텍스트 분리**: 역할별 에이전트가 각자 전문 컨텍스트만 유지 → 단일 에이전트의 컨텍스트 오염 방지

2. **병렬 처리**: `maxConcurrentRuns=3`으로 동시 3개 태스크 → 20일 안에 77개 이슈 완료 가능

3. **전문성 시뮬레이션**: 실제 스타트업 팀(PM·UX·BA·개발자)처럼 역할 분리 → 편향 없는 다각도 평가

4. **추적 가능성**: Paperclip 이슈 시스템으로 모든 결정·산출물이 GitHub에 연결된 형태로 기록

5. **재현성**: AGENTS.md 명세로 동일 구성을 다른 프로젝트에 재사용 가능

### 대안 검토

| 대안 | 기각 이유 |
|---|---|
| 단일 Claude 세션 | 컨텍스트 한계(200K 토큰), 역할 혼동, 병렬 처리 불가 |
| 수동 프롬프트 체인 | 매 단계 사람 개입 필요, 20일 스케줄 불가 |
| AutoGen/CrewAI | 로컬 인증·이슈 추적·Git 통합 지원 약함 |

## 결과

- 프로젝트 기간: **20일** (2026-04-17 ~ 2026-05-06)
- 완료 이슈: **77개**
- 생성 아이디어: **59개**
- 시뮬레이션: **262건**
- 프로토타입: **27개**
- 총 비용: **~$285** (claude-sonnet-4-6 × 7 에이전트)
- 결과물: 공모전 제출 가능한 5페이지 PDF + 인터랙티브 프로토타입

## 교훈

- `maxConcurrentRuns=3` 설정이 동시 체크아웃 충돌을 일부 유발 → 다음 프로젝트에서는 이슈별 단일 에이전트 잠금 고려
- 에이전트 간 직접 소통(CEO 경유 없이)이 속도를 높였으나 일부 산출물 상태 불일치 발생
- git push 인증 자동화가 필요 — 에이전트 환경에 GitHub PAT 사전 설정 필수

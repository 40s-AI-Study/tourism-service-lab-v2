Section 11 — OpenAPI 매핑 & 호출 시퀀스

Visit Korea OpenAPI 통합 아키텍처

5개 공공 OpenAPI를 사용자 여정 단계별로 연동. 실시간성이 필요한 데이터는 서버 캐시 5분, 정적 접근성 정보는 24시간 캐시 정책 적용.

단계API오퍼레이션핵심 파라미터반환 데이터
① 프로파일 설정 무장애 여행 정보 areaBasedList2 contentTypeId=장애유형코드, areaCode 장애 유형별 관광지 목록, 편의시설 현황
② 경로 탐색 위치기반관광정보 locationBasedList2 mapX, mapY, radius=5000, contentTypeId 반경 5km 내 무장애 관광지·편의시설 좌표
② 경로 탐색 국문 관광정보 detailInfo2 contentId (관광지 ID) 경사로·엘리베이터·장애인화장실 상세 유무
③ 실시간 알림 무장애 여행 정보 detailInfo2 contentId, addrinfoYN=Y 시설 운영 상태, 임시 폐쇄 여부
④ 목적지 상세 무장애 여행 정보 detailImage2 contentId, imageYN=Y 접근성 시설 사진 (경사로·화장실·주차)
⑤ 긴급 지원 위치기반관광정보 locationBasedList2 mapX, mapY, radius=1000, contentTypeId=복지시설 반경 1km 내 복지관·응급의료 시설
호출 시퀀스 다이어그램 (텍스트)
앱 시작 → [무장애 여행 API] areaBasedList2 (장애유형코드)
         ↓ 관광지 목록 반환
경로 탐색 → [위치기반 API] locationBasedList2 (GPS좌표, 반경)
          → [국문 관광정보 API] detailInfo2 (contentId) × N개 병렬호출
          ↓ 접근성 시설 유무 + 좌표 데이터
AI 경로 최적화 (자체 알고리즘: 장애유형 × 경사도 × 엘리베이터 경로 그래프)
          ↓
경로 확정 후 → 백그라운드 폴링 [무장애 여행 API] detailInfo2 (5분 주기)
             ↓ 시설 상태 변경 감지 시 → 푸시 알림 + 우회 경로 재계산
목적지 도착 → [무장애 여행 API] detailImage2 (contentId)
            ↓ 내부 접근성 사진 표시

Section 12 — MVP 기능 우선순위

Must / Should / Could

4주 개발 기간 기준 우선순위. 시뮬레이션 결과를 반영해 Must에 시각장애 TalkBack 대응을 포함.

레벨기능근거개발 난이도
Must 장애 유형 프로파일 설정 (6종) 모든 기능의 기반. 없으면 서비스 불성립 낮음
Must 무장애 관광지 목록 + 접근성 시설 상세 Visit Korea OpenAPI 직접 연동. 핵심 데이터 레이어 낮음
Must AI 접근성 경로 최적화 (장애유형 × 경사도 × 엘리베이터) 핵심 차별화 기능. 박재현 시뮬 핵심 가치 확인 높음
Must 실시간 엘리베이터/시설 상태 알림 + 우회 재계산 시뮬에서 가장 높은 만족도. 공모전 데이터 활용 강점 중간
Must TalkBack/VoiceOver 완전 호환 (WCAG 2.1 AA) 최선영 시뮬 조건부 PASS 원인. 미해결 시 시각장애 페르소나 FAIL 중간
Must 긴급 지원 버튼 (119 + 보호자 GPS 동시 발송) 안전망. B2G 납품 요건 필수 항목 낮음
Should 영문 UI 완전 지원 (EN 관광정보 API 연동) Margaret 시뮬 22/30. 외국인 시니어 타겟 확장 중간
Should 시니어 모드 (큰 글씨·고대비·간소화 UI) Margaret 조건부 PASS → PASS 전환 조건 낮음
Should 관광지 접근성 사진 갤러리 (경사로·화장실·주차) detailImage2 API 연동. 시뮬에서 "보여줘야 믿는다" 확인 낮음
Could 음성 명령 인터페이스 ("경복궁 접근성 어때?") 최선영 시뮬 개선 제안 1순위. V2 로드맵 높음
Could 오프라인 경로 캐싱 (주요 관광지 구역) 박재현 개선 제안. 통신 불안 지역 안전 이슈 높음
Could 숙소 접근성 정보 (침대 높이·롤인 샤워·출입문 폭) 박재현 개선 제안. 숙박 데이터 파트너십 선행 필요 높음
Could 가족 동반 모니터링 (Companion View) Margaret 시뮬 개선 제안. 보호자 안심 기능 중간

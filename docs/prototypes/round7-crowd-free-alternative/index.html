API 호출 시퀀스 다이어그램

사용자가 "경복궁" 입력 후 대안 3곳을 받기까지의 백엔드 호출 흐름.

/* ① 사용자 입력 → 관광지 ID 확보 */ GET /KorService1/searchKeyword2 params: { keyword: "경복궁", contentTypeId: "12", numOfRows: 1 } → contentId: "126508" /* ② 현재 혼잡도 예측 조회 (핵심 API) */ GET /tatsCnctrRateList params: { contentId: "126508", baseYmd: "20260501" } → cnctrRate: 0.87, predVisitrCnt: 23000 // 혼잡 판정 /* ③ 연관 관광지 후보 수집 (최대 50개) */ GET /areaBasedList1 params: { contentId: "126508", relateType: "all", numOfRows: 50 } → [ 창덕궁(id:264337), 덕수궁(id:126501), 경희궁(id:126457), ... ] /* ④ 후보 각각 혼잡도 병렬 조회 (Promise.all) */ PARALLEL /tatsCnctrRateList × N → 창덕궁: 0.31 (여유) | 덕수궁: 0.18 (한산) | 경희궁: 0.11 (매우 한산) /* ⑤ 지역 방문자 통계 — 비수도권 우선 가중치 */ GET /locgoRegnVisitrDDList params: { areaCode: "서울외", baseYmd: "202605" } → 비수도권 후보 점수 +0.1 가산 /* ⑥ 최종 랭킹 계산 후 Top 3 반환 */ score = (1 - cnctrRate) × 0.5 + similarityScore × 0.3 + (1 / distanceKm) × 0.1 + regionalBonus × 0.1 → [ 창덕궁, 덕수궁, 경희궁 ] // 최종 추천 3곳
①
관광지 ID 확보
searchKeyword2
사용자 입력 키워드로 관광지 contentId 획득. 이후 모든 API 호출의 기준 식별자.
②
혼잡도 예측 조회 — CORE
tatsCnctrRateList
입력 관광지 집중률(cnctrRate 0-1)과 예측 방문자 수 반환. 임계값(0.7 이상) 초과 시 혼잡 판정 + 경고 UI 표시.
③
연관 관광지 후보 수집
areaBasedList1
테마·분위기 유사 관광지 최대 50곳 반환. 관광지/음식/숙박 유형별 분리 조회 가능. 이 후보군에서 혼잡도 낮은 대안 탐색.
④
후보 병렬 혼잡도 조회
tatsCnctrRateList × N (Promise.all)
후보 50개를 병렬 API 호출로 혼잡도 조회. 응답 지연 최소화. 최대 50개 → 혼잡도 기준 필터링 → Top 10 → 최종 3곳 선정.
⑤
지역 다양성 가중치 보정
areaTouDivList + locgoRegnVisitrDDList
비수도권 대안 명소에 가중치 +0.1 부여 → 서울 외 지역 관광 분산 유도. 공모전 "지역분산 관광" 심사 기준 직접 대응.

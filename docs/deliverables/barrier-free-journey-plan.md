# Barrier-Free Journey — 기획 고도화 문서

> **한 줄 정의**: 장애인·시니어·외국인 이동 약자 모두를 위한 무장애 통합 관광 플랫폼 — Visit Korea OpenAPI 7종을 조합해 실시간 경사도·단차·엘리베이터 정보를 제공하고, AI가 최적 무장애 경로를 자동 생성한다.

---

## 1. 문제 정의 (Why)

### 1-1. 핵심 Pain Point

| # | 문제 | 근거 |
|---|---|---|
| P1 | **무장애 정보 공백** — "휠체어 접근 가능"이라 적혔지만 현장 계단 발견, 경사도·단차·바닥재질 미기재 | 이재호 페르소나: 사전조사 6시간 소요, 매 여행마다 현장 낭패 경험 |
| P2 | **시니어 디지털 접근성 취약** — 기존 앱 글씨 작고 단계 복잡, 60세 이상 스스로 사용 불가 | 김영자 페르소나: 딸이 설치·조작 대행, 혼자선 사용 포기 |
| P3 | **외국인 장애인 정보 언어 장벽** — 무장애 관광 안내 한국어 전용, 영어·일어 무장애 앱 전무 | 2025년 방한 외래객 1,620만명 중 장애·시니어 비율 추산 8% → 약 130만명 정보 사각 |
| P4 | **실시간 장벽 감지 없음** — 엘리베이터 점검, 공사 중 경사로 등 당일 정보를 앱에서 확인 불가 | 크라우드소싱 후기 의존 → 최신 정보 평균 60일 지연 |
| P5 | **전동휠체어 충전 인프라 정보 없음** — 충전소 위치·규격·예약 여부 안내 서비스 전무 | 배터리 25km/일 한계 → 미확인 충전소로 인한 여행 중단 사례 |

### 1-2. 시장 공백

| 기존 서비스 | 한계 |
|---|---|
| 무장애 관광 공식 사이트 (한국관광공사) | 정보 2~3년 지연, 앱 없음, 경로 자동화 없음 |
| 네이버 지도 | "장애인 주차" 필터 있지만 경사도·단차 데이터 없음 |
| Google Maps | 무장애 데이터 한국 미비, 한국어 전용 |
| 트리플 | 무장애 필터 없음, 수동 일정 배치 |
| Wheelmap (해외) | 한국 데이터 커버리지 5% 미만, 한국어·UI 미지원 |

→ **"실시간 무장애 데이터 × AI 경로 자동 생성 × 다국어 × 시니어 UI"** 조합은 국내외 전무.

## 3. 핵심 사용자 시나리오 — 5단계 플로우

### 시나리오: "이재호의 제주 2박 자립 여행"

```
[진입] → [설정] → [탐색] → [실행] → [기여]
```

**1단계 — 진입 (앱 설치 & 프로필 등록)**
- 앱스토어 리뷰 847개 4.6점 확인 → 설치
- 장애 유형 선택: 전동휠체어 / 최대 경사도 15° / 포장 노면만 / 단차 2cm 이하
- 배터리 프로필: 1일 25km → 충전소 자동 알림 설정 ON

**2단계 — 설정 (여행 조건 입력)**
- 지역: 제주 동부 / 기간: 2박 3일 / 동반: 파트너(비장애)
- AI 무장애 경로 생성 시작 → 5초 내 3개 코스 옵션 반환
- 코스별 접근성 스코어 (A/B/C등급) + 총 배터리 소모 예상치 표시

**3단계 — 탐색 (상세 정보 검증)**
- 성산일출봉 상세: 구간별 경사도 표 (1구간 8° ✅ / 2구간 22° ❌)
- 크라우드소싱 최신 리뷰 (90일 이내) 확인 → "3주 전 방문, 경사로 상태 양호"
- 충전소 레이어: 제주 23개소 + C타입/DC 규격 + 운영시간 + 예약여부
- 숙소 필터: 출입문 90cm+ / 롤인 샤워 / 침대 45~55cm → 4개소 반환

**4단계 — 실행 (현장 탐방)**
- 경복궁역 2번 엘리베이터 점검 알림 → 우회 경로 자동 업데이트
- 음성 내비게이션: "50m 직진 후 경사로 진입, 경사 7도, 노면 콘크리트"
- 긴급 지원 버튼: 위치 공유 + 주변 장애인 콜택시 연결

**5단계 — 기여 (크라우드소싱 리뷰)**
- 방문 후 리뷰 작성 권유 (리뷰 3개 → 포인트 적립)
- "경사로 상태", "단차 변경", "충전소 운영 여부" 체크리스트형 입력
- 개발자 커뮤니티 슬랙에 공유 → "휠체어 필수 앱" 자발적 확산

## 4. Visit Korea OpenAPI 매핑

### 4-1. 핵심 API 7종 활용 명세

| # | API 이름 | 주요 오퍼레이션 | Barrier-Free Journey 활용 목적 |
|---|---|---|---|
| **1** | 무장애 여행 정보 | `areaBasedList2`, `detailInfo2`, `locationBasedList2` | 경사도·단차·노면 상세 데이터 핵심 |
| **2** | 국문 관광정보 서비스 | `areaBasedList2`, `searchKeyword2`, `detailCommon2` | 관광지 기본 정보·이미지 |
| **3** | 영문 관광정보서비스 | `areaBasedList2`, `detailCommon2` | 외국인 페르소나(James) 영어 콘텐츠 |
| **4** | 관광지 집중률 방문자 추이 예측 | `getPredictVisitorList` | 혼잡 시간 회피 → 이동 약자 공간 확보 |
| **5** | 관광지별 연관 관광지 정보 | `getRelatedTourismList` | 접근성 통과 장소 기반 연관 추천 |
| **6** | 기초지자체 중심 관광지 정보 | `getTourismResourceList` | 시니어 맞춤 지역별 관광 자원 필터 |
| **7** | 관광지 오디오 가이드 정보 | `getAudioGuideList` | 시니어 느린 속도 오디오 + 시각장애 지원 |

### 4-2. API 호출 시퀀스 (무장애 경로 생성 플로우)

```
[사용자: 여행 조건 입력]
        ↓
[Step 1] 무장애 여행 정보 API
  · locationBasedList2 (반경 10km, 장애유형 필터)
  · detailInfo2 (경사도·단차·노면 상세)
        ↓
[Step 2] 관광지 집중률 예측 API
  · getPredictVisitorList (방문 예정일 혼잡도)
  · 결과: 혼잡 RED 장소 제외 → 후보 필터링
        ↓
[Step 3] 연관 관광지 API
  · getRelatedTourismList (접근성 통과 장소 기준)
  · 결과: 인접 무장애 장소 연결
        ↓
[Step 4] 국문/영문 관광정보 API
  · detailCommon2 (이미지·설명·운영시간)
  · 결과: UI 카드 렌더링용 메타데이터
        ↓
[Step 5] AI 경로 최적화
  · 배터리 소모 계산 (총 거리 + 경사도 가중치)
  · 일정 배치 (혼잡도 낮은 시간대 우선)
  · 3개 코스 옵션 반환
        ↓
[사용자: 코스 선택 → 탐방 시작]
```

### 4-3. 캐싱 전략

| 데이터 유형 | TTL | 저장소 | 이유 |
|---|---|---|---|
| 무장애 시설 기본 정보 (경사도·단차) | 7일 | Redis | 물리 시설은 자주 변경 없음. 주 1회 새로고침 |
| 혼잡도 예측 데이터 | 1시간 | Redis | 예측 모델 입력값 변경 주기에 맞춤 |
| 관광지 이미지·설명 | 30일 | CDN | 정적 콘텐츠, 장기 캐시 유효 |
| 크라우드소싱 리뷰 | 실시간 (캐시 없음) | DB | 최신성이 핵심 가치, 캐싱 금지 |
| 충전소 운영 상태 | 30분 | Redis | 운영 상태 변경 빈도 반영 |
| 엘리베이터·경사로 실시간 알림 | 5분 | SSE/WebSocket | 실시간 장벽 감지 핵심 기능 |

## 6. 차별화 포인트 — 경쟁 서비스 대비 5개

### D1. 구간별 경사도·단차 정밀 데이터 (기존 서비스 전무)

| 기존 | Barrier-Free Journey |
|---|---|
| "휠체어 접근 가능" 한 줄 표기 | 매표소→A구간: 경사 8° / B구간: 경사 22° 불가 / 바닥: 콘크리트·자갈 구분 |

Visit Korea 무장애 여행 정보 API의 `detailInfo2`를 활용해 관광지 내 **구간별** 접근성 데이터를 제공. 기존 앱들은 관광지 단위 "접근 가능 여부"만 표시하나, Barrier-Free Journey는 입구~전망대까지 구간을 쪼개 사용자가 자신의 이동 능력에 맞게 판단 가능.

### D2. 실시간 장벽 감지 + 우회 경로 자동 재계산

엘리베이터 점검, 경사로 공사 등 당일 발생 장벽을 크라우드소싱 + 공공 데이터로 수집해 5분 내 SSE 알림 발송. 경로 재계산 알고리즘이 즉시 대안 경로 생성. **기존 앱은 정적 정보만 제공 — 당일 장벽 대응 기능 전무**.

### D3. 전동휠체어 전용 배터리 인프라 통합

전동휠체어 충전소 위치·규격(C타입/DC)·운영시간·예약여부를 지도 레이어로 통합. 배터리 잔량 추적 및 경로 내 충전소 자동 포함 기능. **국내 어떤 여행 앱도 전동휠체어 충전 인프라 정보를 통합 제공하지 않음**.

### D4. 시니어 전용 극단적 단순 UI (3탭 이내 완료)

시니어 모드 선택 시 모든 화면이 재구성. 글씨 1.5배, 터치 영역 44px→80px, 색상 대비 WCAG AAA, 모든 기능 3단계 이내 완료. 자녀가 설치 후 부모가 혼자 사용 가능. **트리플·카카오맵 등은 시니어 전용 UI 모드 없음**.

### D5. 다국어 무장애 정보 (영어·일어·중문 지원)

Visit Korea 영문·일문·중문 API를 무장애 정보와 결합. 외국인 장애인/시니어 여행자에게 영어로 경사도·단차·접근성 데이터 제공. **한국 무장애 앱 중 외국어 지원은 전무 — 방한 외국인 접근성 여행자 시장 독점 가능**.

## 1. 한국관광공사 OpenAPI 활용 매트릭스

**Base URL**: `http://apis.data.go.kr/B551011/KorService1/`  
**인증**: 공공데이터포털(data.go.kr) ServiceKey 발급 후 모든 요청에 포함

> 총 12종 API, 23개 오퍼레이션 활용. 공모전 데이터 활용도 배점 20점 만점 공략.

  |                           |<-----------------------------------------------+                 |

## 5. 데이터 활용도 점수 공략 근거

### 공모전 배점 구조 (데이터 활용도 20점)

| 평가 항목 | 만점 | TripCraft Korea 달성 내용 |
|-----------|------|--------------------------|
| TourAPI 활용 종류 | 약 10점 | 12종 API, 23개 오퍼레이션 → 단순 나열이 아닌 각 API가 서비스 로직에 직접 연산 입력 |
| 카카오 OpenAPI 활용 | 약 5점 | 카카오맵·로그인·카카오톡 공유·모빌리티 4종 완전 연동 |
| 데이터 결합·가공 창의성 | 약 5점 | KTO 빅데이터 3종(집중률·다양성·수요강도) + CLIP 벡터 임베딩 = 새로운 지표(여유로움 지수) 생성 |

### KTO API 12종 활용 근거

```
[국문 관광정보] 6개 오퍼레이션 — 서비스의 전체 POI 데이터 골격
[오디오 가이드] 2개 오퍼레이션 — 문화유산 카테고리 핵심 차별화
[집중률 예측]   2개 오퍼레이션 — 바이럴 감지 + 방문 시간 추천
[관광빅데이터]  1개 오퍼레이션 — 지역별 방문자 추이 통계
[연관 관광지]   1개 오퍼레이션 — AI 동선 구성 연결고리
[기초지자체 중심관광지] 1개 — 지도 핀 초기값
[관광 다양성]   1개 오퍼레이션 — 소외 지역 가중치 근거
[관광 수요 강도] 2개 오퍼레이션 — 여유로움 지수 보완 지표
[관광 자원 수요] 2개 오퍼레이션 — 카테고리 큐레이션 가중치
[관광사진]      1개 오퍼레이션 — 카드 UI 비주얼
[공모전 수상사진] 1개 오퍼레이션 — 오디오 화면 고품질 비주얼
[영문/일문/중문] 각 2개 오퍼레이션 — 외국인 다국어 완전 지원
```

### 차별화 포인트 — 단순 나열 vs. 연산 결합

| 방식 | 내용 |
|------|------|
| 단순 나열 (감점 요인) | API를 호출해 데이터를 표시만 함 |
| **TripCraft 방식 (가점 요인)** | `tatsCnctrRatedList` + `locgoRegnVisitrDDList` → **여유로움 지수** 신규 산출 |
| | `areaTouDivList` + `areaTarExpDsList` → **소외 지역 가중치** 자동 결정 |
| | `detailImage2` (CLIP 임베딩) + `detailCommon2` (텍스트 임베딩) → **유사도 벡터** 생성 |
| | `areaTarSvcDemList` + 네이버 블로그 → **소셜-공식 복합 트렌드** 지수 |

**결론**: KTO API 12종 × 카카오 4종 = 16종 동시 활용, 데이터 상호 연산으로 새로운 지표 3종 생성 → 데이터 활용도 20점 만점 공략 가능.

## 6. API 캐시·비용 전략

### 캐시 TTL 정책

| 데이터 유형 | TTL | 캐시 키 | 근거 |
|------------|-----|---------|------|
| 장소 목록 (searchKeyword2, locationBasedList2) | 6시간 | `{apiId}:{keyword}:{areaCode}:{catCode}` | 관광 정보는 하루 단위로 거의 변하지 않음. 6시간은 안전 마진 |
| 장소 상세 (detailCommon2, detailInfo2) | 24시간 | `contentId:{id}` | 입장료·숙박료 등 정보 변경 주기 최소 1일 |
| 이미지 URL (detailImage2, galSearchKeyword) | 7일 | `contentId:{id}:images` | 이미지는 거의 변하지 않음. CDN 레이어 추가 적용 |
| 오디오 스토리 (storyBasedList) | 7일 | `storyId:{id}:{lang}` | 오디오 대본/음성은 정기 업데이트 없음 |
| 위치 기반 오디오 목록 (storyLocationBasedList) | 10분 | `geo:{lat_grid}:{lng_grid}` | 위치 이동 감지 주기 반영 |
| 집중률 예측 (tatsCnctrRateList/Rated) | 24시간 | `areaCode:{code}:date:{yyyyMMdd}` | 예측 데이터는 일 1회 갱신 기준 |
| 빅데이터 통계 (locgoRegnVisitrDDList) | 7일 | `areaCode:{code}:week:{yyyyWW}` | 주간 집계 데이터로 변동 없음 |
| 수요 지수 3종 (다양성·강도·자원) | 24시간 | `areaCode:{code}:{indexType}` | 지수 계산 기준 일 1회 갱신 |
| 네이버 블로그 검색 | 1시간 | `naver:blog:{keyword}` | 소셜 트렌드는 빠르게 변함. 짧은 TTL 유지 |

|------|------|

### 무료 한도 내 운영 가능 근거

| API | 무료 한도 | 예상 일 호출 (캐시 후) | 여유율 |
|-----|-----------|----------------------|--------|
| 한국관광공사 KTO OpenAPI 전체 | 일 1,000건/키 × 복수 키 발급 가능 (공공데이터포털 정책상 서비스키 복수 신청 가능) | 약 5,000~8,000건 | 키 5~8개로 무료 커버 |
| 카카오맵 API | 일 300,000건 | 약 1,000~2,000건 | 충분한 여유 (사용량 0.7%) |
| 카카오 로그인 | 무제한 | — | 무제한 |
| 카카오톡 공유 | 무제한 | — | 무제한 |
| 카카오 모빌리티 | 일 300,000건 | 약 500~1,000건 | 충분한 여유 (사용량 0.3%) |
| 네이버 블로그 검색 | 일 25,000건 | 약 300~500건 | 충분한 여유 (사용량 2%) |

### 비용 절감 추가 전략

1. **배치 사전 수집**: KTO `areaBasedList2`로 26만건 장소 데이터를 오프라인 배치로 1회 수집 후 Chroma DB에 저장. 사용자 요청 시 KTO API를 호출하지 않고 로컬 DB에서 즉시 응답.
2. **좌표 격자 캐시 공유**: 위치 기반 조회는 1km × 1km 격자로 좌표를 정규화해 인접 사용자의 API 결과를 공유 캐시로 재활용.
3. **비활성 지역 스킵**: `areaTouDivList` 기준 방문자 하위 5% 지역의 집중률 예측은 호출 생략 (사실상 혼잡 없음).
4. **언어별 지연 로딩**: 다국어 API(영문·일문·중문)는 사용자가 해당 언어를 선택한 시점에만 호출. 기본값(한국어)은 국문 API만 사용.
5. **공모전 단계 KTO 키 전략**: 공공데이터포털 서비스 신청 시 "관광 앱 서비스" 목적으로 신청하면 일 1,000건 이상의 확장 한도 요청 가능. 복수 엔드포인트별 별도 키 발급으로 한도 분산.

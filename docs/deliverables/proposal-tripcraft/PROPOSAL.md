## 1) 서비스 기획배경 및 필요성

### 1-1 서비스 기획 배경

국내 관광 시장은 네 가지 구조적 문제가 하나의 악순환을 이루고 있습니다.

| 문제 | 현황 | 출처 |
|------|------|------|
| **오버투어리즘** | 북촌한옥마을 거주 인구 5년간 **27.6% 감소**, 주말 방문자 하루 1.5만 명 초과 | 서울시 북촌 관광관리 실태조사 2023 |
| **지역 관광 불균형** | 서울·경기·제주가 전체 관광 지출의 **55% 이상** 독점 | 문체부 국내관광 통계조사 2024 |
| **여행 정보 파편화** | 여행자 1인당 평균 **5개 앱** 사용, 준비 시간의 40% 소모 | 한국소비자원 여행 행태 조사 2024 |
| **외국인 FIT 공백** | 방한 **1,750만 명** 중 개별여행 70%+, 통합 다국어 가이드 전무. 일본·중국 FIT 커뮤니티(小红书, X 여행 태그) 조사 결과 "한국 지방 여행 정보 부족"이 1위 불만 사항 | KTO 외래관광객 실태조사 2024 / 자체 SNS 모니터링 2025 |

> 상기 통계는 각 기관의 2023~2024년 공식 보고서 기준이며, 시뮬레이션 수치는 TourAPI 과거 데이터 기반 추정입니다.

이 네 문제는 서로 연결되어 있으며, 현재 어떤 서비스도 이를 통합적으로 해결하지 못하고 있습니다.

### 1-2 서비스 필요성

**(가칭) TripCraft Korea**는 한국관광공사 공공데이터(혼잡도·다양성·수요강도)와 카카오 서비스 4종을 알고리즘으로 결합해 위 순환을 직접 끊어냅니다.

- 한국관광공사 2025~2027 핵심 정책 **'지역 균형 관광'** 과 직접 정합
- 소외 지역 가중치 1.5배 + 팔도 스탬프 리워드 2배로 **정책 목표를 사용자 동기로 전환**
- 분산 추천 적용 시 상위 10% 과부하 관광지 집중도 **20~30% 완화** 목표
- 소외 지역 선택 비율 현행 15% → **33% 이상** 달성 목표

## 2) 서비스 개요

### 2-1 기획 서비스 소개명

> **(가칭) TripCraft Korea — 카테고리 테마 기반 개인화 여행 플래너**

여행 카테고리(문화유산·미식·자연·축제·힐링·포토스팟·K-컬처)와 지역을 선택하면, 한국관광공사 혼잡도·다양성 데이터를 실시간 결합해 동선·경비·예약·오디오·스탬프가 통합된 **원페이지 실행 플랜**을 5초 만에 자동 생성합니다.

기존 서비스는 단순 검색·예약에 머물지만, (가칭) TripCraft Korea는 공공데이터를 결합한 알고리즘으로 **5가지를 동시 제공**하는 유일한 서비스입니다.

| 비교 항목 | 트리플 | 야놀자 | 마이리얼트립 | ChatGPT | **(가칭) TripCraft Korea** |
|:---|:-:|:-:|:-:|:-:|:-:|
| AI 코스 자동 생성 | △ | △ | △ | ✗ | **✓** |
| 위치 기반 오디오 자동 재생 | ✗ | ✗ | ✗ | ✗ | **✓** |
| 혼잡도 자동 회피 + 분산 | ✗ | ✗ | ✗ | ✗ | **✓** |
| 경비 자동 합산 (4항목) | ✗ | △ | △ | ✗ | **✓** |

**3대 핵심 차별점:**

1. **분위기 보존 분산** — 바이럴 장소를 포기하지 않고 AI 유사도 91%+ 대안을 같은 카테고리 맥락에서 제시
2. **원페이지 실행 플랜** — 지도 길찾기·예약·경비·오디오가 단일 화면에 통합 (텍스트 코스가 아닌 즉시 실행)
3. **걸으면 따라오는 이야기** — 문화유산 50m 진입 시 5개국어 오디오 자동 재생, QR 없이 외국인까지 포괄

> **왜 소규모 팀이 가능한가?** — 한국관광공사 TourAPI(무료)와 카카오 SDK(무료 티어)를 조합하면 초기 인프라 비용 0원으로 핵심 기능 구현이 가능합니다. 대형 플랫폼은 자체 데이터에 갇혀 공공 혼잡도·다양성 지표를 활용하지 못하지만, 본 서비스는 공공데이터 결합에 특화된 경량 구조로 차별점을 확보합니다.

### 2-4 서비스 내 지역 특화 관련 사항 (지역 특화 서비스 — 작성 필수)

| 지역특화 구현 | 설명 | 적용 시도 | 활용 공공데이터 |
|---|---|---|---|
| 소외 지역 노출 강화 | 관광 다양성 지수 하위 33% 시도 노출·리워드 **1.5~2배** 자동 가중 | 강원·충남·전북·전남·경북 등 | 지역별 관광다양성 서비스 |
| RTO 협업 콘텐츠 우선 | 8개 RTO 추천 코스·축제 카드 우선 노출 | 부산·대전·광주·세종·충남·경북·강원·제주 | 기초지자체 중심관광지정보 서비스 |

| KTO 정책 연계 | 디지털 관광주민증 인증 데이터 양방향 연동 | 전 시도 | KTO 협약 채널 |

> **동작 사례**: '강릉(강원)' 선택 → 다양성 지수 하위 가중 1.5배 적용, 안목해변 바이럴 장소는 강문해변·정동진으로 분산 추천, 강원 RTO 추천 축제 우선 노출, 강원도 팔도 스탬프 적립

### 3-2 데이터 활용 방식 (용도)

단순히 공공데이터를 화면에 나열하는 것이 아닌, **데이터를 결합하는 알고리즘**으로 설계했습니다. 한국관광공사 빅데이터 3종을 AI 분석과 교차 연산해 기존에 없는 **새로운 기준 3가지**를 만들어냅니다.

**기술 스택**: Next.js(프론트엔드) + Python FastAPI(백엔드) + OpenAI CLIP(이미지 임베딩) + PostgreSQL + 카카오 SDK 4종. TourAPI 호출 → 벡터 DB 캐싱 → 실시간 유사도 연산 → 원페이지 플랜 렌더링의 파이프라인으로 구성됩니다.

| 새로운 기준 | 만드는 방법 | 사용자에게 보이는 모습 |
|---|---|---|
| **여유로움 지수** | 관광빅데이터(인기 장소 방문자) ÷ 관광빅데이터(후보지 방문자) | "이 장소는 인기 장소 대비 3배 여유로워요" |
| **소외 지역 가산점** | 관광다양성 지수 하위 33% 판정 → 노출·리워드 ×1.5 적용 | "이 지역 방문 시 스탬프·상품권 1.5배!" |
| **분위기 유사도** | OpenAI CLIP 이미지 임베딩 코사인 유사도 40% + 카테고리 일치 40% + TF-IDF 텍스트 유사도 20% → 가중합산 | "안목해변과 분위기 91% 유사한 곳: 강문해변" |

**데이터 결합 시나리오 — 문화유산 카테고리 (경주)**

```
① 관광자원수요 → 경주 문화자원 수요 지수 확인
② 관광다양성   → 소외 지역 가산점 결정
③ 국문관광정보 → 문화유산 후보 수집
④ 집중률예측   → 혼잡 시간대 회피 배치
⑤ 관광사진     → 카드 비주얼 표시
⑥ 공모전수상작 → 촬영 각도 가이드
⑦ 국문관광정보 → 입장료·예약·좌표 획득
⑧ 오디오가이드 → 50m 반경 오디오 자동 연결
⑨ 연관관광지   → 다음 코스 추천
⑩ 오디오가이드 → 언어·계절·성향별 재생
⑪ 카카오맵     → 길찾기 즉시 실행
```

| 구분 | 설명 |
|---|---|
| 단순 나열 방식 | API 결과를 화면에 그대로 보여주기만 함 |
| **TripCraft 결합 방식** | 빅데이터 3종 + AI 벡터 → 신규 기준 3가지 생성 → 원페이지 플랜에 통합 |

## 4) 서비스 발전 방향

### 4-1 개발 서비스 향후 발전방향

**단계별 발전 로드맵**

| 단계 | 핵심 목표 | 이용자 | 어떻게 달성하나 |
|---|---|---|---|
| **제안서 접수 (현재)** | 프로토타입 6화면 시연, 분산 추천 알고리즘 검증 | 심사위원 | TourAPI 26만건 연동 + 카카오 4종 연동 완료 |
| **개발 기간 (5~9월)** | 내국인용 정식 서비스 완성, 코스 저장·공유·경비 계산 | 베타 테스터 **200~500명** | 공모전 컨설팅 지원 활용, 네이버 여행 카페(여행에 미치다·국내여행) + 인스타그램 여행 해시태그 타겟 베타 모집 |
| **수상 후 6개월** | 다국어 확장(영·중·일), 팔도 스탬프 정식 운영 | **3,000~5,000명** | KTO 수상작 홍보 채널 노출, 창업 지원 프로그램 연계 |
| **1~2년 (투자 유치 시)** | 지자체 시범 탑재, 외국인 FIT 채널 확장 | **1~3만 명** | 지자체 MOU(무상 시범 → 유료 전환), KTO 해외사무소 공동 홍보 |

> 상기 이용자 수는 각 단계의 전제 조건(수상·KTO 지원 연계·투자 유치)이 충족될 경우의 달성 목표이며, 전제 미충족 시 이전 단계에서 서비스를 안정화한 뒤 재도전합니다.

**수익 모델 (수상 후 1~2년차, 투자 유치 전제)**

| 수익원 | 구조 | 연간 규모 |
|---|---|---|
| 지자체 시범 계약 | 지역 관광 채널에 TripCraft 엔진 무상 탑재 → 성과 검증 후 유료 전환 | 2~3곳 × 1,500~2,000만원 = **3,000~6,000만** |
| 예약 전환 수수료 | 플랜 내 숙박·체험 예약 링크 클릭 시 수수료 5~8% | MAU 1만 × 전환 10% × 건당 3,000원 = **월 300만 → 연 3,600만** |
| 스탬프 스폰서십 | 지역 소상공인·상공회의소가 리워드 재원 제공 | 월 200~300만 × 12 = **2,400~3,600만** |
| **합계** | | **약 1~1.3억** |

> 공모전~수상 직후에는 수익 없음. 위 모델은 시드 투자 + 지자체 MOU를 전제로 하며, 서비스 무료 운영으로 사용자 기반 확보가 우선. 수상 전까지 운영 비용은 클라우드 프리 티어(Vercel + Supabase) + TourAPI 무료 호출 한도 내에서 충당하며 월 실비용 5만 원 이내로 유지합니다.

**핵심 운영 목표 (정식 출시 1년 기준)**

| 지표 | 정의 | 목표 | 측정 방법 |
|---|---|---|---|
| 분산률 | 소외 시도(다양성 하위 33%) 선택 비율 | **30% 이상** | 플랜 생성 로그 중 소외 시도 선택 비율 집계 |
| 코스 실행률 | 생성된 플랜 중 실제 방문 인증 1건 이상 | **20% 이상** | 플랜 생성 → 스탬프 인증 이벤트 매칭 |
| 재사용율 | 플랜 1회 생성자의 60일 내 재방문 | **25% 이상** | 카카오 로그인 세션 기반 코호트 분석 |

> 분산률 30%: 가중치 1.5배 적용 시 소외 6개 시도 합산이 균등 수준 도달 가능 (17시도 균등 = 시도당 5.9%). 나머지 지표는 베타 기간 실측 후 상향 조정.

**장기 비전**

내국인 직장인의 주말 여행 실행 도구로 시작해, 수상 후 다국어 확장을 통해 방한 외국인 FIT까지 포괄합니다. 각 단계 전환은 이전 단계 지표 달성(분산률 30%+, 코스 실행률 20%+)이 검증된 후 진행하며, KTO 창업 지원·지자체 MOU·시드 투자의 순서로 사업화를 추진합니다.

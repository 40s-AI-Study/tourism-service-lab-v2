## 1. 양식 제약 체크리스트

제출 전 모든 항목을 순서대로 확인한다.

- [ ] **분량 5페이지 이내** — 페이지 번호 기준 5페이지를 초과하지 않는다. 내용 압축이 필요하면 표·불릿 형식으로 대체하고 서술 단락을 줄인다.
- [ ] **글자 크기 12pt 이상** — 본문 전체 12pt. 표 내부 셀, 캡션, 각주 포함 모두 12pt 이상 유지. PDF 출력 전 단락 스타일을 일괄 확인한다.
- [ ] **작성 양식 수정 금지** — 제안서.png 원본 양식의 항목 번호·제목·순서를 변경하지 않는다. 행·열 추가 및 셀 병합은 허용 범위 내 자율이나 항목 삭제·이동은 금지.
- [ ] **모든 항목 필수 작성** — 1-1, 1-2, 2-1, 2-2, 2-3, 2-4, 3-1, 3-2, 4-1 총 9개 세부항목 전부 작성 완료 여부 확인.
- [ ] **2-4 지역 특화 항목 작성** — 괄호 내 조건 "(지역 특화 서비스의 경우 작성 필수)"에 해당하므로 반드시 작성. 공백 또는 "해당 없음" 기입 시 감점.
- [ ] **3-1 API 한글 정식명 기재** — 한국관광공사 OpenAPI의 정식 서비스 명칭을 공공데이터포털 등록명 기준으로 기재. 예: "한국관광공사_국문관광정보_서비스" (영문 코드명 `tourism-info-korean` 단독 사용 금지).
- [ ] **제출 파일 PDF, 10KB 미만** — 내보내기 시 이미지 압축 설정을 확인. 텍스트만으로 10KB 미달 가능성이 낮으나, 이미지 제거 상태로 10KB 미달 시 SVG 1개 이상 삽입으로 보완.
- [ ] **이미지·표 크기 자율 범위 내 배치** — 삽입 비주얼이 텍스트를 가리거나 양식 경계를 침범하지 않도록 여백 확보.
- [ ] **PDF 내보내기 폰트 임베딩** — 한글 폰트(예: 나눔고딕, 맑은 고딕) 임베딩 옵션 활성화. 미임베딩 시 심사 환경에서 폰트 깨짐 발생.

## 5. 본문 작성 시 톤·문체 가이드

### 언어 및 문체

- **언어**: 한국어 전용. 영어 표현은 다음 규칙에 따라 병기한다.
  - KTO OpenAPI 서비스명: 한글 정식명 우선, 괄호 안에 API ID 병기. 예: "국문관광정보서비스(`tourism-info-korean`)"
  - 일반 영어 약어(AI, API, GPS, URL, SDK, MAU, MVP 등): 최초 1회 한글 풀어쓰기 불필요. 단, 심사위원 비기술 독자를 고려해 AI는 "인공지능(AI)", API는 "공개 API"로 첫 등장 시 기재.
  - 고유 서비스명(TripCraft Korea, ChatGPT, CLIP 등): 영문 그대로 사용.

### 문장 스타일

- **단정문** 사용. "~할 것으로 기대됩니다" 대신 "~를 달성한다", "~로 전환된다" 형식.
- **수치 강조**: 핵심 수치는 굵게 처리. 예: "방문자 집중도 최대 **20~30%** 완화".
- **표 권장**: 비교·목록·단계별 설명은 산문 대신 표로 정리. 공모전 심사자가 스캐닝하기 용이한 형식.
- **단락 길이**: 1단락 최대 5문장. 각 항목 도입 문장은 1문장으로 핵심 주장을 선언하고 이후 근거·수치를 제시.

### 12pt 준수

- 모든 본문·표 셀·불릿 포인트: 12pt.
- 항목 제목(1-1, 2-3 등): 12pt Bold 또는 14pt Bold.
- 이미지 캡션: 12pt (감소 불가).
- 머리글·바닥글(페이지 번호): 10pt 허용(양식 고정 요소).

## 6. 체크 후 보완 항목 사전 경고

제안서 작성 중 놓치기 쉬운 항목을 미리 식별해 두고, 최종 검토 전에 각 항목을 점검한다.

---

### 경고 1. 2-4 지역 특화 항목 — 공백 처리 절대 금지

제안서.png 양식에 "(지역 특화 서비스의 경우 작성 필수)"라는 조건이 명시되어 있다. TripCraft Korea는 소외 지역 가중치·팔도 스탬프·KTO 정책 연계를 핵심 설계로 삼는 지역 특화 서비스이므로, 이 항목은 **필수 작성 대상**이다. "해당 없음", "추후 작성" 등으로 공백을 두면 채점 항목에서 직접 감점된다.

대응: 03-features-and-differentiation.md § 4. 지역 특화 섹션의 4-1~4-4 내용을 압축해 반드시 기재한다.

---

### 경고 2. 3-1 API명 — 영문 코드명 단독 기재 금지

"ex) 한국관광공사\_국문관광정보\_서비스"라는 양식 예시가 인쇄되어 있다. 이 예시는 공공데이터포털 등록 한글 정식명 형식을 요구하는 것이다. 사전자료에서 `tourism-info-korean`, `visitor-concentration-forecast` 등 영문 API ID가 주로 사용되었으나, 제안서에는 섹션 3의 정식명 대조표(본 문서 § 3 참조)를 기준으로 한글명으로 변환해 기재해야 한다.

대응: 04-api-utilization.md의 "한글명" 컬럼을 참조해 12종 전부 한글 정식명으로 변환한다. 최초 기재 후 "(이하 국문관광정보서비스)" 축약 표현 사용 가능.

---

### 경고 3. 3-2 데이터 활용 용도 — "활용 예정"과 구분되는 실제 용도 서술 필요

3-1이 "무슨 API를 쓰는가"라면, 3-2는 "왜·어떻게 쓰는가"를 서술하는 항목이다. 두 항목을 동일 내용으로 채우면 감점 요인이 된다. 3-2에는 API 단순 호출이 아닌 **데이터 연산 결합** 방식, 즉 여유로움 지수·소외 지역 가중치·유사도 벡터라는 신규 지표 생성 과정을 중심으로 서술해야 한다.

대응: 04-api-utilization.md § 5. 데이터 활용도 점수 공략 근거의 "차별화 포인트 — 단순 나열 vs. 연산 결합" 표를 제안서 3-2 항목의 핵심 설명 구조로 활용한다.

---

### 경고 4. 페이지 2 분량 초과 위험 — 4개 항목(2-1~2-4) 집중

페이지 2는 2-1부터 2-4까지 4개 항목을 1페이지에 수용해야 한다. 각 항목의 서술을 압축하지 않으면 페이지가 넘친다.

대응:
- 2-1: 서비스명 + 한 줄 정의만 기재, 별도 설명 단락 생략.
- 2-2: 기능 6개를 2열 표 또는 2줄 이내 불릿으로 압축. 기능 상세 흐름은 생략.
- 2-3: 경쟁사 비교표는 행 수를 최대 6개사로 제한.
- 2-4: 알고리즘 설명 대신 정책 정합성 항목 표 4행으로 압축.

### 경고 5. SVG 삽입 — PDF 변환 시 벡터 품질 확인

assets/ 폴더의 4개 SVG 파일은 텍스트 편집기·InDesign·Word 각각에서 PDF로 변환 시 렌더링 결과가 다를 수 있다. 특히 한글 폰트가 포함된 SVG는 변환 환경에 따라 깨질 수 있다.

대응: SVG를 PDF 삽입 전에 고해상도 PNG(300dpi, 최소 너비 1,000px)로 사전 변환한다. SVG 원본은 보관, 실제 제안서 삽입은 PNG로 진행한다.

### 경고 6. 양식 수정 금지 — 행 추가 방식 주의

표 또는 항목을 추가할 때 양식 원본의 항목 제목 행(1-1, 2-3 등)을 삭제하거나 위치를 이동하면 양식 수정으로 간주될 수 있다. 행 추가(내용 삽입 목적)는 허용되지만, 기존 항목 행의 레이아웃을 변경하는 것은 금지 범위에 해당한다.

대응: 양식 파일의 잠금 레이어를 해제하지 않고 내용 입력 셀만 편집한다. 표 행 추가 시 항목 번호 행 아래에만 행을 삽입하며, 항목 번호 셀은 병합하지 않는다.

### 경고 7. 2-2 기능명 표기 — 양식 예시 기준 준수

양식에 "(검색 기능, 맞춤형 코스 추천 기능 등)"이라는 예시가 기재되어 있다. 기능 명칭을 자유롭게 쓰되, 최소한 "검색 기능"과 "맞춤형 코스 추천 기능"이라는 표현을 포함하거나 그에 상응하는 항목을 명시해야 양식 의도에 부합한다.

대응: 6개 기능 목록에 "장소 검색 및 카테고리 큐레이션 기능"과 "개인화 맞춤형 코스 자동 생성 기능"을 첫 번째·두 번째 항목으로 배치한다.

## 1. 카테고리별 상세 기능 흐름

TripCraft Korea는 여행 목적에 따라 7개 카테고리를 제공한다. 사용자가 카테고리를 선택하면, 아래 표에 정의된 흐름에 따라 AI 동선이 자동 생성된다.

---

### 1-A. 문화유산 (Heritage)

| 단계 | 내용 |
|---|---|
| **입력** | 지역 선택 (예: 경주) / 일수 (예: 1일) / 인원 (예: 2명 성인) |
| **후보 추출 API** | `tourism-info-korean` → `areaBasedList2` (contentTypeId=14 문화시설, 76 문화유산) + `audio-guide` API로 오디오 콘텐츠 보유 장소 필터링 |
| **추천 알고리즘** | `related-attractions` 연관 관광지 데이터 기반 클러스터링 + `visitor-concentration-forecast` 혼잡도 낮은 시간대 우선 배치 + 관광 다양성 지수(`area-tourism-diversity`) 하위 30% 장소 노출 가중치 1.3배 |
| **동선 최적화 규칙** | 이동 거리 최소화 (카카오 모빌리티 경로 API) + 오전 개장 직후 혼잡 명소 → 오후 한적 명소 순 배치 + 연속 문화시설 3곳 초과 시 식사·휴식 장소 자동 삽입 |
| **부가기능** | **위치기반 오디오 자동재생**: 사용자가 해당 문화유산 반경 **50m** 진입 시 진동 알림 + 오디오 자동 재생. 동일 장소 오디오 재생은 **24시간 내 1회**로 제한(중복 재생 방지). 오프라인 캐시 지원(저장 후 Wi-Fi 없이 재생). 5성향별 테마 분기(전통감성/계절사진/조용/활달/역사분석). 국내/외국인 이중 렌즈: 동일 POI에 대해 한국인 앵글·외국인 앵글 오디오 별도 제공. 4계절 스토리 분기(봄·여름·가을·겨울 동일 장소 다른 이야기). |

> **오디오 자동재생 트리거 상세**
> - 감지 반경: **50m** (GPS + 블루투스 비콘 보조)
> - 재생 제한: 동일 장소 **24시간 1회** (쿨다운 타이머 기기 로컬 저장)
> - 진동 패턴: 1회 짧은 진동 → 알림 배너 "N미터 앞, [장소명] — 탭해서 이야기 듣기"
> - 자동재생 거부: 알림 배너 닫기 시 재생하지 않음 (사용자 동의 원칙)
> - 배터리 최적화: 활성 감지는 도보 이동 감지 시에만 작동, 차량 이동 중 비활성화

### 1-B. 미식 (Gourmet)

| 단계 | 내용 |
|---|---|
| **입력** | 지역 / 일수 / 인원 + 식이 제한 옵션(채식·할랄·알레르기) |
| **후보 추출 API** | `tourism-info-korean` → `searchKeyword2` (contentTypeId=38 음식점) + 네이버 블로그 검색 API 최다 언급 맛집 교차 검증 |
| **추천 알고리즘** | 소셜 언급 빈도 × TourAPI 등록 공식 정보 신뢰도 가중 합산 + 동일 식당 연속 2끼 추천 금지 룰 + 지역 향토음식 우선 배치(catcode 소분류 한식·향토음식 가중 1.2배) |
| **동선 최적화 규칙** | 점심·저녁 시간대 기준 반경 500m 내 후보 우선 + 줄서기 예상 시간(혼잡도 등급) 높은 식당은 오픈런 시간대로 자동 조정 |
| **부가기능** | 메뉴별 예상 식비 자동 합산(1인 기준, 한국소비자원 외식비 기준 기본값 적용) + 카카오맵 플레이스 예약 원터치 연결 + 지역 식재료 스토리 카드(해당 식재료 산지·제철 정보) |

### 1-C. 자연·에코 (Nature & Eco)

| 단계 | 내용 |
|---|---|
| **입력** | 지역 / 일수 / 인원 + 체력 수준(가족·아동 동반 여부) |
| **후보 추출 API** | `tourism-info-korean` → `areaBasedList2` (contentTypeId=12 관광지, 39 레포츠) + `area-tourism-demand-density` → `areaTarSjrnDsList` 체류 수요로 자연 명소 우선 필터 |
| **추천 알고리즘** | 계절 적합도 점수(봄-꽃, 여름-해수욕, 가을-단풍, 겨울-설경) × 혼잡도 예측 낮은 시간대 우선 + 아동 동반 시 난이도 하(평지·완만 코스) 자동 필터 |
| **동선 최적화 규칙** | 오전 이동 거리 최소화(체력 보존) + 두 자연 명소 사이 이동 30분 이내 원칙 + 날씨 API 연동 시 강수 확률 70% 이상 야외 명소 자동 실내 대체 알림 |
| **부가기능** | 탄소 발자국 예상 표시(이동 거리 × 이동 수단별 CO₂ 계수) + 에코 인증 장소 뱃지(국립공원·환경부 인증 생태탐방로) + 실시간 입장 예약 링크(국립공원 예약 시스템 연결) |

### 1-D. 축제·이벤트 (Festival)

| 단계 | 내용 |
|---|---|
| **입력** | 지역 / 여행 날짜 / 인원 |
| **후보 추출 API** | `tourism-info-korean` → `areaBasedList2` (contentTypeId=15 행사/공연/축제) + `tourism-big-data` → `locgoRegnVisitrDDList` 최근 방문자 급증 축제 지역 탐지 |
| **추천 알고리즘** | 여행 날짜 × 축제 개최 기간 교집합 자동 계산 + 규모(참가 인원 추정) 중간 이상 우선(너무 작으면 현장감 부족, 너무 크면 혼잡) + 지역 고유 축제 가중(체인 행사 대비 1.3배) |
| **동선 최적화 규칙** | 주 축제 → 인근 관광지 → 숙박 순 배치 + 축제장 혼잡 피크 시간대(개막식·폐막식 전후 1시간) 도착 전 주변 관광지 먼저 방문 패턴 제안 |
| **부가기능** | 축제 공식 일정 카드(타임라인 형식) + 준비물 체크리스트 자동 생성(야외/실내 구분) + 카카오톡 축제 일정 공유 |

### 1-E. 힐링·웰니스 (Healing & Wellness)

| 단계 | 내용 |
|---|---|
| **입력** | 지역 / 일수 / 인원 + 힐링 성향(온천·명상·한방·산림욕) |
| **후보 추출 API** | `tourism-info-korean` → `searchKeyword2` (힐링·웰니스·스파·한방 키워드) + `related-attractions` 웰니스 연관 클러스터 추출 |
| **추천 알고리즘** | 인파 최소 시간대 우선(평일 오전, `visitor-concentration-forecast` 활용) + 힐링 성향별 장소 매칭(온천→수온 정보 포함 / 산림욕→국립산림치유원 인증 여부) + 소외 지역 웰니스 명소 가중 노출 |
| **동선 최적화 규칙** | 일정 내 이동 3회 이하 제한(피로 최소화) + 오전 활동적 / 오후 정적인 순 배치 원칙 + 숙박은 웰니스 테마 숙소(펜션·한옥·글램핑) 우선 |
| **부가기능** | 웰니스 지수 카드(해당 장소 방문 후 기대 효과 설명) + 한방 관광지 연계 정보(한의원 예약 URL) + 산림욕 코스 거리·소요시간·난이도 표시 |

### 1-F. 포토스팟·K-컬처 (Photo & K-Culture)

| 단계 | 내용 |
|---|---|
| **입력** | 지역 / 일수 / 인원 + K-컬처 세부 관심사(드라마·K-POP·한복·거리미술) |
| **후보 추출 API** | `tourism-info-korean` → `areaBasedList2` + `photo-contest-winners` → `koKeyword` 공모전 수상 사진 촬영지 우선 추출 + `tourism-photos` → `galSearchKeyword` 고품질 관광 사진 보유 장소 필터 |
| **추천 알고리즘** | SNS 언급 빈도 × 수상사진 보유 수 × 혼잡도 역수(덜 알려진 포토스팟 가중) 3축 합산 + 바이럴 포화 장소(idea-A+ 기준 방문자 +50% 초과) 자동 감지 후 유사 대안 분산 추천 |
| **동선 최적화 규칙** | 황금시간대(일출 후 1시간·일몰 전 1시간) 야외 포토스팟 배치 + 실내 K-컬처 스팟은 날씨 무관 오후 배치 + 이동 중 포토스팟 연계(카카오맵 경유지로 자동 추가) |
| **부가기능** | 수상작 각도 가이드 오버레이(공모전 수상사진 참고 각도 AR 가이드) + K-드라마·K-POP 촬영지 인증 정보 + 카카오톡/인스타 그리드 형식 공유 최적화 카드 |

### 기능 1. AI 개인화 동선 생성 (KoreaPath AI 기반)

여행 조건(출발지·일수·테마·인원·동반 형태·예산)을 입력하면, 연관 관광지 데이터와 혼잡도 예측을 조합해 최적 동선을 자동 생성한다.

**작동 방식**

```
[입력] 지역 + 일수 + 테마 + 인원 구성 (예: 제주 2박3일, 가족 4명, 7세 아이 포함)
        ↓
[연관 관광지 클러스터링]
  related-attractions API → 함께 방문하는 연관 관광지 50위 데이터 기반
  + central-attractions-by-municipality → 지자체 핵심 관광지 100위 보완
        ↓
[혼잡도 타이밍 최적화]
  visitor-concentration-forecast → 향후 30일 혼잡도 예측
  → 명소별 최적 방문 시간대 자동 배정
  → "경복궁 오전 8시 집중률 35%(한산) / 오후 2시 85%(붐빔)" 알림
        ↓
[동선 완성]
  카카오 모빌리티 API → 이동 거리 최적화
  → Day별 타임라인 자동 생성 (소요시간·이동수단 포함)
        ↓
[실시간 재조정]
  현장 혼잡도 급증 감지 시 대체 동선 자동 제안
```

**차별점**: 트리플·야놀자는 사용자가 직접 코스를 조합해야 하지만, TripCraft Korea는 입력 5초 → 완성 코스. LLM을 서비스 내부에 내장하여 자연어 입력도 지원(예: "조용한 곳 위주로 경주 당일치기").

### 기능 2. 위치기반 오디오 자동재생 (Audio Story Korea v2 기반)

걸으면서 이야기가 따라오는 경험. 사용자가 문화유산 반경 50m에 진입하면 자동으로 오디오 스토리가 시작된다.

**핵심 사양**

| 항목 | 사양 |
|---|---|
| 감지 반경 | 50m (GPS 기본, 블루투스 비콘 보조) |
| 재생 제한 | 동일 장소 24시간 1회 쿨다운 |
| 콘텐츠 원천 | 한국관광공사 오디오 가이드 API + TTS 보완 |
| 성향 분기 | 5개 테마(전통감성·계절사진·조용·활달·역사분석) |
| 계절 분기 | 봄·여름·가을·겨울 동일 POI 다른 스토리 |
| 언어 지원 | 한국어·영어·일본어·중국어(간체)·중국어(번체) |
| 이중 렌즈 | 한국인 앵글 vs 외국인 앵글 오디오 별도 제공 |
| 오프라인 | 사전 다운로드 후 Wi-Fi 없이 재생 가능 |

**활용 API**: `audio-guide`, `photo-contest-winners`, `tourism-info-english/japanese/chinese`

### 기능 3. 혼잡도 최적화 + 유사장소 분산 추천 (idea-A+ 기반)

바이럴로 특정 장소가 포화 상태일 때, 비슷한 분위기의 여유로운 대안을 자동 제시한다.

**바이럴 감지 기준**

| 기준 | 정의 |
|---|---|
| 집중률 등급 | `visitor-concentration-forecast` → `tatsCnctrRatedList` 상위 10% 이내 |
| 급증 판정 | 최근 4주 평균 방문자가 직전 13~52주 평균 대비 +50% 이상 |
| 수동 입력 | 사용자가 직접 장소 검색 → 유사 장소 탐색 |

**유사도 3축 계산**

| 축 | 방법 | 비중 |
|---|---|---|
| 장소 타입 | catcode 소분류 일치 | 40% |
| 분위기·뷰 | CLIP 이미지 임베딩 코사인 유사도 | 40% |
| 설명 텍스트 | overview 텍스트 임베딩 유사도 | 20% |

**분산 추천 방식**: 바이럴 발생 시도 제외, 나머지 16개 시도별 유사도 상위 3개 후보를 지도 위에 핀으로 표시. 최종 선택은 사용자에게 위임 → 자연스러운 분산 + 책임 주체 명확화.

**여유로움 지수**: `tourism-big-data` 방문자 통계 기반 상대 지표. UI에 "최근 6개월 통계 기준 여유로운 편"으로 표시(실시간 아님 명시).

### 기능 4. 총경비 자동 합산 (idea-B+ 5장 표 활용)

여행 계획 생성 시, 예상 총경비를 4개 항목으로 자동 계산하여 원페이지에 표시한다.

| 항목 | 데이터 출처 | 계산 기준 | 비고 |
|---|---|---|---|
| 입장료 | TourAPI `usefee` 필드 | 1인 기준 | 무료 시 0원 표시 |
| 숙박비 | TourAPI `roomrate` 필드 (contentTypeId=32) | 2인 기준 1박 평균 | 데이터 없을 시 "확인 필요" |
| 이동비 | 카카오 모빌리티 거리 × 단가 | 자가용 기준 (연비 12km/L, 유가 1,700원/L) | 대중교통 모드 토글 제공 |
| 식비 | 한국소비자원 외식비 기준 | 1인 1식 15,000원 기본값 | 고정 기준값으로 신뢰성 확보 |

**합산 표시**: Day별 소계 + 여행 전체 총계 자동 계산. 인원수 입력 시 총경비 자동 곱산. 항목별 수정 가능(사용자 직접 입력 오버라이드).

### 기능 5. 원페이지 타임라인 + 카카오맵 원터치 실행

여행 계획 전체를 한 화면에서 파악하고, 모든 장소를 카카오맵으로 즉시 실행한다.

**원페이지 구조**

```
┌─────────────────────────────────────────┐
│ [지역명] [N박N일] [인원] — [카테고리]    │
├─────────────────────────────────────────┤
│ Day 1                                   │
│  09:00  [장소A]  ▶ 카카오맵  📍 입장료  │
│  11:30  [장소B]  ▶ 카카오맵  🍽️ 식비   │
│  14:00  [장소C]  ▶ 카카오맵  📸 무료   │
│  18:00  [숙소]   ▶ 카카오맵  🏨 예약   │
├─────────────────────────────────────────┤
│ Day 2  (동일 구조)                       │
├─────────────────────────────────────────┤
│ 예상 총경비: 입장료 N원 + 숙박 N원      │
│           + 이동 N원 + 식비 N원 = 합계  │
├─────────────────────────────────────────┤
│ [카카오톡으로 플랜 공유]                 │
└─────────────────────────────────────────┘
```

**카카오맵 원터치**: 각 장소 카드의 "카카오맵" 버튼 → 현재 위치에서 해당 장소까지 카카오맵 길찾기 즉시 실행(앱 전환 없이 딥링크). 전체 일정 멀티 경유지 경로도 한 번에 카카오맵으로 내보내기 지원.

### 기능 6. 팔도 스탬프 게이미피케이션 (idea-B+ 11장 기반)

**스탬프 적립 흐름**

```
[지역 여행 실행]
        ↓
[인증]
  대표 관광지: 인증 사진 (위치 EXIF 메타데이터 + TourAPI 장소 매칭)
  지역 식당: 영수증 등록 (OCR → 사업자번호 → 국세청 오픈API 조회 → 지역 확인)
        ↓
[스탬프 적립]

        ↓
[리워드 지급]
  지역상품권 / 숙박 할인 쿠폰 / 카카오페이 포인트 / 다음 여행지 큐레이션
        ↓
[17개 시도 완주]

```

**부정 수급 방지**

| 위협 | 방지책 |
|---|---|

## 4. 지역 특화 섹션

### 4-1. 소외 지역 가중 보정 알고리즘

TripCraft Korea는 한국관광공사의 지역 균형 관광 정책을 시스템 레벨에서 구현한다.

**작동 방식**

```
area-tourism-diversity → areaTouDivList
  관광 다양성 지수 산출 → 방문자 하위 33% 시도 = 소외 지역 분류
        ↓
area-tourism-demand-density → areaTarSjrnDsList
  체류 수요 강도로 소외 지역 2차 검증
        ↓
[노출 가중치 적용]
  일반 지역: 기본 가중치 1.0
  소외 지역 (하위 33%): 노출 가중치 1.5배
  소외 지역 방문 스탬프 리워드: 1.5~2배 지급
        ↓
사용자는 더 많은 리워드를 위해 소외 지역 자발 선택
→ 지역 분산 효과 자연 달성
```

**분산 효과 검증 계획**

| 단계 | 방법 |
|---|---|
| 출시 전 시뮬레이션 | TourAPI 과거 방문자 데이터로 "대안 추천 시" 16개 시도 분산 비율 추정 |
| 출시 후 지표 | 소외 시도(다양성 지수 하위 33%) 선택 비율이 33% 이상 달성 여부 모니터링 |
| 성공 기준 | 추천 후 선택 시 소외 시도 비율 ≥ 33% → 균등 분산 달성 |

### 4-2. 17시도 팔도 스탬프 지역 균형 설계

팔도 스탬프 시스템은 단순 게임화가 아니라, 지역 균형 관광을 사용자 동기로 전환하는 구조를 갖는다.

**시도별 스탬프 획득 조건**

| 구분 | 인증 방법 | 유효 조건 |
|---|---|---|
| 대표 관광지 | 인증 사진(위치 EXIF + TourAPI 매칭) | 24시간 내 촬영, 장소 반경 100m 이내 |
| 지역 식당 | 영수증 OCR(사업자번호 + 결제일) | 방문일 ±1일, 해당 시도 사업자 확인 |
| 지역 숙박 | 체크인 확인(예약 번호 입력) | 해당 시도 소재 숙소 |

**소외 지역 보정 보상 구조**

```
관광 다양성 지수 하위 33% 시도 방문 확인 시
  → 스탬프 × 2 지급 (빠른 완주 유도)
  → 지역상품권 리워드 1.5배
  → 다음 여행지 큐레이션 우선 소외 지역 제안
```

---

### 4-3. 지역상품권 리워드 체계

| 리워드 | 재원 | 파트너십 대상 |
|---|---|---|
| 지역상품권 | 한국관광공사 지역관광 예산 연계 (디지털 관광주민증 사업과 동일 채널) | 한국관광공사 |
| 숙박 할인 쿠폰 | 야놀자·여기어때 제휴 할인코드 | 숙박 플랫폼 |
| 카카오페이 포인트 | 카카오 공동 주최 연계 마케팅 예산 | 카카오 |

> 공모전 단계에서는 리워드 시뮬레이션으로 제시하고, 실제 파트너십은 수상 후 협의 예정.

### 4-4. 한국관광공사 지역 균형 관광 정책 부합 근거

| 정책 방향 | TripCraft Korea 대응 |
|---|---|
| 지역 관광 다양성 확대 | area-tourism-diversity API로 소외 지역 탐지 + 노출 가중치 1.5배 |

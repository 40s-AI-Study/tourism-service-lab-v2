# 06. 프로토타입 화면 사양 — TripCraft Korea

> **카테고리 테마 기반 개인화 여행 플래너**  
> 6개 핵심 화면 와이어프레임 + 화면별 사양표

---

## 화면 목록

| # | 화면명 | 진입 조건 | 다음 화면 |
|---|--------|-----------|-----------|
| S1 | 온보딩 / 카테고리 선택 | 앱 최초 실행 또는 홈 진입 | S2 (지역 선택) |
| S2 | 지역 선택 (17시도 지도) | S1에서 카테고리 선택 완료 | S3 (코스 결과) |
| S3 | 코스 결과 (Day1/2 타임라인) | S2에서 지역 확정 | S4 (오디오), S5 (경비) |

| S6 | 팔도 스탬프 | S3 하단 스탬프 탭 탭 | S3 (복귀) |

### 화면 사양표

| 항목 | 내용 |
|------|------|
| **목적** | 사용자가 여행 테마(카테고리)를 결정하고 기간·인원을 입력해 플랜 생성의 첫 조건을 설정한다 |
| **핵심 컴포넌트** | 앱 로고 + 태그라인 / 6개 카테고리 카드 그리드 (2열×3행) / 여행 기간 라디오 (당일·1박2일·2박3일) / 인원 드롭다운 / CTA 버튼 |
| **카테고리 카드** | 아이콘(이모지) + 한국어 명칭 + 한 줄 부제. 선택 시 카드에 primary 파란 테두리 + 체크 배지 표시. 복수 선택 허용(최대 2개) |
| **사용 API** | 없음 (로컬 입력 수집 단계) |
| **인터랙션** | 카드 탭 → 선택/해제 토글 + 진동 햅틱. 기간/인원 선택. "다음" 탭 → S2로 슬라이드 전환 |
| **다음 화면** | S2 (지역 선택) |
| **접근성** | 카드 aria-label 포함, 키보드 포커스 순서 보장 |

### 화면 사양표

| 항목 | 내용 |
|------|------|
| **목적** | 17개 시도를 지도 위에 시각화해 사용자가 목적지를 직관적으로 선택하도록 하고, 소외 지역을 ★로 강조해 자발적 분산을 유도한다 |

### 화면 사양표

| 항목 | 내용 |
|------|------|
| **목적** | 사용자가 문화유산 반경 50m에 진입했을 때 자동 트리거되는 오디오 가이드 화면. 지도 위 ⭐ 아이콘으로 숨겨진 유산을 표시하고 오디오 플레이어를 제공한다 |
| **핵심 컴포넌트** | 진동 알림 배너 (장소명 + 거리 표시) / 지도 (현재 위치 + ⭐ POI 아이콘) / 오디오 플레이어 (프로그레스바·재생 컨트롤·성향·언어 선택) / 주변 이야기 장소 수평 카드 |
| **트리거 조건** | GPS 위치가 POI 좌표 반경 50m 진입 AND 카테고리 = 문화유산 AND `audio-guide` API에 해당 contentId 오디오 URL 존재 AND 사용자 오디오 권한 허용 |
| **사용 API** | `audio-guide` → `getAudioUrl(contentId)` / HTML5 Geolocation API (반경 감지) / Web Audio API (재생) / `related-attractions` → `searchKeyword1` (주변 이야기 장소) / `tourism-info-korean` → `locationBasedList2` (주변 POI) |
| **인터랙션** | 자동 진동 1회 + 배너 팝업 → 탭 시 오디오 자동 재생. 성향 변경 드롭다운 → 스토리 분기 전환. 언어 변경 → 해당 언어 API 오디오 전환. ⭐ 지도 핀 탭 → 해당 POI 오디오 프리뷰. 주변 장소 카드 탭 → 해당 장소 오디오 로드 |
| **다음 화면** | S3 (코스로 복귀) |
| **오프라인** | 사전 다운로드된 오디오는 Wi-Fi 없이 재생 가능. 오프라인 상태 배지 표시 |

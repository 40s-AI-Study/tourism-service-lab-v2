# KoreaPath AI — 와이어프레임 & 사용자 흐름

> 7개 주요 화면 + 2개 사용자 흐름. 반응형 모바일 우선 (375×812 iPhone 기준). 디자인 토큰은 Android·iOS 모두 적용.

---

## 0. 디자인 원칙

1. **5초 이내 가치 전달** — 랜딩 → 입력 → 코스 제안까지 3탭 이내
2. **혼잡도는 색·배지로 직관** — 🟢한산 🟡보통 🔴혼잡 (색약 고려 + 아이콘 병행)
3. **다국어 1-탭 전환** — 언어 자동 감지 + 상단 항시 전환 버튼
4. **접근성 WCAG 2.1 AA** — 명도 대비 4.5:1 이상, 터치 영역 44×44pt
5. **한 손 조작** — 주요 CTA는 하단 Thumb-zone(화면 하단 1/3)

## 8. 사용자 흐름 (User Flow)

### Flow A — 신규 사용자 (5초 코스 생성)

```
앱 설치 → [S1] 언어 감지 → [S2] 조건 입력 (40초)
      → AI 생성 중 (3초 스피너)
      → [S3] 코스 결과 → [S7] 예산 확인
      → 💾 저장 → [S6] SNS 공유

      목표: 신규 진입 후 90초 이내 코스 완성 + 공유
```

### Flow B — 방문 당일 실시간 조정

```
[S3] 저장된 코스 열기 → 체크인 (GPS)
      → 다음 POI 혼잡도 체크
      → [Trigger] 임계치 70% 초과
      → Push 알림 [S5] 재조정 제안
      → [대안 선택] → 코스 자동 재계산
      → [S3] 갱신된 코스로 복귀

      목표: 현장 혼잡 회피로 대기 시간 -70%
```

## 9. 주요 인터랙션 & 피드백

| 상황 | 인터랙션 | 피드백 |
|---|---|---|
| 코스 생성 중 | 3초 스피너 + 진행 문구 | "경복궁 혼잡도 분석 중…" |
| 재생성 | 하단 🔁 탭 | 2.5초 내 새 코스, haptic tap |
| 저장 | 💾 탭 | 카카오톡/이메일 링크 복사 토스트 |
| 혼잡도 급증 | Push 알림 | 진동 1회 + [S5] 표시 |
| 접근성 경고 | 코스 내 ♿ 불가 POI | 자동 필터링 + "대안 1개 제안" |

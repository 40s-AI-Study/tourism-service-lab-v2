# KoreaPath AI — 서비스 기획서

> **한 줄 정의**: 여행 조건을 5초 입력하면 혼잡도 예측 + 연관 관광지 데이터를 조합해 **개인화 코스를 자동 생성**하고, 방문 당일에는 실시간으로 동선을 재조정해주는 AI 여행 플래너.

---

## 1. 문제 정의 (Why)

### 1-1. 핵심 Pain Point

| # | 문제 | 근거 |
|---|---|---|
| P1 | **코스 기획에 2~3시간 소요** — 네이버 블로그·카카오맵·인스타그램 3개 앱 병행 | 시장조사: 20-40대 82%가 "코스 추천" 니즈 |
| P2 | **혼잡도 예측 부재** — SNS 핫플에 도착하면 이미 3시간 대기 | 혼잡도 확인 니즈 76%, 오버투어리즘 이슈 확산 |
| P3 | **외국인 FIT 여행자 정보 단절** — Visit Korea 앱 UX 구식, 구글 번역 의존 | 2024 방한 외래객 1,620만명 중 FIT 비중 67% |
| P4 | **동반자별 제약 미반영** — 노부모·아이·반려동물 조건이 코스에 반영되지 않음 | 가족여행자 심층인터뷰에서 "맞춤 필터" 니즈 92% |

### 1-2. 시장 공백

| 기존 서비스 | 한계 |
|---|---|
| 네이버 여행·카카오맵 | 수동 검색, 코스 자동화 없음 |
| 트리플 | 사용자가 일정 직접 배치, 혼잡도 예측 없음 |
| Google Maps | 한국 혼잡도 데이터 없음, 로컬 큐레이션 없음 |
| Visit Korea 공식 앱 | 다국어 OK, UX 구식, AI 기능 없음 |

→ **"AI 자동 코스 생성 × 혼잡도 예측 × 다국어 × 접근성"** 조합은 국내외 전무.

|---|---|---|---|

| **P2 박민준·이수연** | 31/32, 직장인 커플 | "1주년 기념, 기획 시간 최소화" | ✅ PASS 25/30 |

- **SAM**: 20-40대 스마트폰 여행자 약 1,800만명
- **SOM (3년)**: 초기 3% 침투 목표 → 54만 MAU

## 3. 핵심 기능 (What)

### 3-1. MVP (공모전 제출 범위)

| # | 기능 | 설명 | 활용 API |
|---|---|---|---|
| F1 | **AI 코스 자동 생성** | 조건 입력(지역·일수·테마·동반·예산) → 5초 내 완성 코스 | `related-attractions`, `central-attractions-by-municipality` |
| F2 | **혼잡도 타이밍 추천** | 시간대별 혼잡도 배지(🟢🟡🔴) + 최적 방문 시각 제안 | `visitor-concentration-forecast` |
| F3 | **실시간 코스 재조정** | 현장 혼잡도 급증 시 대체 장소 push 알림 | `visitor-concentration-forecast` + `area-tourism-demand-density` |
| F4 | **4개 언어 지원** | 한·영·중(간체)·일 자동 감지 + 수동 전환 | `tourism-info-korean/english/chinese/japanese` |
| F5 | **접근성 필터** | "노부모 동반"/"휠체어"/"유모차" 선택 → 계단·경사 제외 | `barrier-free-travel` |
| F6 | **코스 공유·저장** | 링크·카카오톡·인스타 스토리 자동 레이아웃 | (내부) |

### 3-2. Phase 2 (공모전 이후 6개월)

- 예약 원스톱 연동 (숙박·식당·체험) — 파트너십 기반
- 기념일 모드 (100일/1주년/생일 자동 큐레이션)
- 오디오 가이드 다국어 TTS (`audio-guide` API 연동)
- 반려동물 동반 모드 (`pet-friendly-travel` API 연동)

### 3-3. Phase 3 (12개월+)

- 여행자 UGC 플랫폼 (코스 공유·평점)
- 글로벌 확장 (일본·대만 역방향)
- B2B API 판매 (지자체·호텔)

## 5. 예상 사용 시나리오

### 시나리오 A — 솔로 당일치기 (P1 김지원)

```
금요일 22:00 인스타 릴스로 강릉 감성카페 발견
      ↓
앱 실행 → "강릉, 당일, 혼자, 감성카페+포토스팟, 3만원" (40초)
      ↓
토요일 10:00 안목해변 (혼잡 28%, 줄 없이 촬영)
     11:30 강릉중앙시장 (로컬 먹거리)
     13:00 경포호 인생사진 → 14:30 정동진
     16:00 KTX 귀가
      ↓
인스타 릴스 업로드 + 코스 링크 공유 → 바이럴 유입
```

### 시나리오 B — 외국인 솔로 (P4 Emma 개선 후)

```
Taipei → Incheon 직항 도착
      ↓
앱 자동 언어 감지 (中文繁體) → "Jeonju, 당일, 포토, 오디오 가이드"
      ↓
한옥마을 혼잡도 🟢(오전 9시) → 한적한 촬영
      ↓
각 장소 중국어 음성 해설 → 문화 맥락 이해
      ↓
Dcard에 코스 공유 → 대만 팬덤 유입
```

## 7. 리스크 & 대응

| 리스크 | 영향 | 대응 |
|---|---|---|
| API 응답 속도 | 코스 생성 지연 | Redis 캐시 + CDN, 코스 생성 백그라운드 잡 |
| AI 코스 품질 | 추천 불만족 | MVP는 rule-based → 이후 LLM 고도화 |
| 혼잡도 데이터 정확도 | 사용자 신뢰 저하 | 예측치 vs 실측치 AB 검증, 오차 범위 명시 |
| 외국인 콘텐츠 품질 | 낮은 리텐션 | 4개 언어부터 완성도 확보, 이후 확장 |
| LLM 연동 비용 | 운영비 부담 | 규칙 기반 하이브리드, 외부 LLM 호출 비율 제한 |

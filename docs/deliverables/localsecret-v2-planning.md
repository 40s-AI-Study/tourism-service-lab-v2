# 🗝️ LocalSecret V2 — 기획 고도화 문서

> **카테고리**: 로컬/숨은명소 발견 | **순위**: ★1 | **BA 점수**: 134/160 | **팀 합의**: 9.1/10

---

## 1. 서비스 개요

**LocalSecret V2**는 혼잡도 역발상(crowd-reversal) 알고리즘을 핵심으로, 아무도 모르는 숨은 명소만 추천하는 오버투어리즘 해소형 여행 플랫폼이다.

| 항목 | 내용 |
|---|---|
| 핵심 가치 제안 | "오늘, 어떤 발견을 원해?" — 인기 순위 반대의 정반대 추천 |
| 핵심 알고리즘 | 혼잡도 최하위 역정렬 + 감성 필터 이중 매칭 |
| 감성 진입점 | 비밀(Secret) / 고요(Quiet) / 정취(Atmosphere) / 계절(Seasonal) |
| KTO API 활용 | 5개 (아래 §3 상세) |
| 타겟 | 20-40대 한국인 + 방한 외국인 + 접근성 약자 |

## 3. Visit Korea OpenAPI 매핑

### 3.1 사용 API 5종 명세

| # | API ID | 서비스명 | 활용 목적 | 핵심 오퍼레이션 |
|---|---|---|---|---|
| 1 | `visitor-concentration-forecast` | 관광지 집중률 방문자 추이 예측 | 혼잡도 역정렬 알고리즘 핵심 데이터 | `tatsCnctrRateList` |
| 2 | `area-tourism-diversity` | 지역별 관광 다양성 | 숨은 명소 지수 산출 (다양성↑ + 혼잡도↓) | 지역별 다양성 지수 조회 |
| 3 | `tourism-big-data` | 관광빅데이터 정보서비스 | 급상승 트렌드 감지 → 아직 덜 알려진 명소 필터링 | 이동통신 + 신용카드 데이터 |
| 4 | `related-attractions` | 관광지별 연관 관광지 정보 | 동선 최적화 + 숨은 명소 클러스터링 | 관광지별 연관 50위 조회 |
| 5 | `tourism-info-korean` / `tourism-info-english` | 국문/영문 관광정보 | 장소 기본 정보 + 다국어 지원 | 위치기반 + 통합검색 |

### 3.2 혼잡도 역발상 알고리즘 로직 상세

```
[입력]
- 사용자 감성 선택: SECRET | QUIET | ATMOSPHERE | SEASONAL
- 위치: 현재 위치 or 목적지 지역
- 날짜: 오늘 or 특정 날짜

[Step 1] 후보 명소 풀 구성
  tourism-info-korean API → 지역별 관광지 목록 (contentTypeId로 필터)
  related-attractions API → 연관 관광지 포함 확장

[Step 2] 혼잡도 스코어 계산 (역정렬)
  visitor-concentration-forecast API
  → 향후 30일 집중률 데이터 조회
  → "혼잡도 역점수" = 100 - 집중률(%)
  → 역점수 높을수록 상위 노출

[Step 3] 숨은 명소 지수 (HGI: Hidden Gem Index)
  HGI = (혼잡도역점수 × 0.5) + (다양성지수 × 0.3) + (트렌드급등여부 × 0.2)
  area-tourism-diversity API → 다양성 지수
  tourism-big-data API → 최근 급등 여부 (급등이면 HGI 감점)

[Step 4] 감성 필터 매칭
  SECRET: contentTypeId = 12(관광지), 혼잡도 하위 10%, HGI 상위
  QUIET:  durunubi-trails + eco-tourism 연계, 인적 드문 트레일
  ATMOSPHERE: contentTypeId = 14(문화시설) + 사진 수상작 장소
  SEASONAL: 계절별 키워드 매핑 (봄:벚꽃/개나리, 여름:계곡/바다, 가을:단풍, 겨울:설경)

[출력]
  Top 3 숨은 명소 카드 (혼잡도 역순, 감성 매칭)
  + 최적 방문 시간대 (30일 예측 中 혼잡도 최저 구간)
```

### 3.3 API 호출 시퀀스 (사용자 앱 실행 기준)

```
앱 실행 → 감성 선택
    │
    ├─ [병렬] tourism-info-korean (후보 관광지 목록)
    │         related-attractions (연관지 확장)
    │
    ├─ visitor-concentration-forecast (30일 혼잡도 조회)
    │         → 혼잡도 역정렬 적용
    │
    ├─ area-tourism-diversity (다양성 지수)
    │         tourism-big-data (트렌드 급등 감지)
    │         → HGI 계산
    │
    └─ 감성 필터 → Top 3 카드 렌더링
                   타이밍 알림 스케줄 등록
```

### 3.4 캐싱 전략

| 데이터 | TTL | 캐시 이유 |
|---|---|---|
| 30일 혼잡도 예측 | 24시간 | 예측 데이터는 일 1회 업데이트 |
| 관광지 기본 정보 | 7일 | 변경 빈도 낮음 |
| 연관 관광지 | 3일 | 네트워크 데이터 안정적 |
| 다양성/빅데이터 지수 | 12시간 | 주간 단위 업데이트 |
| 감성별 추천 결과 | 1시간 (위치별) | 실시간성 vs 서버 부하 균형 |

**캐시 구조**: Redis (위치 그리드 기반 키) + CDN (정적 관광정보)

### 5.1 혼잡도 역발상 알고리즘 — "세상에서 유일한 반대 추천"

| | 트리플 | 네이버 블로그 | Airbnb Experiences | **LocalSecret V2** |
|---|---|---|---|---|
| 추천 기준 | 인기도·별점 | 후기·조회수 | 호스트 평점 | **혼잡도 최하위 역정렬** |
| 데이터 소스 | UGC 리뷰 | 텍스트 후기 | 플랫폼 내부 | **KTO 30일 예측 API** |
| 결과 | 모두가 아는 명소 | 이미 유행한 명소 | 비용 발생 | **진짜 아무도 없는 곳** |

**핵심**: "인기 있는 곳을 추천하지 않겠다"는 철학 자체가 차별화. 국내외 어떤 앱도 이 포지셔닝 없음.

---

### 5.2 감성 진입점 UX — 검색창 없는 앱

| | 일반 여행앱 | **LocalSecret V2** |
|---|---|---|
| 시작 화면 | 검색창 (어디 갈지 알아야 함) | "오늘, 어떤 발견을 원해?" (감성 먼저) |
| 사용자 부담 | 목적지를 알고 있어야 검색 가능 | 모르는 곳을 찾고 싶을 때 진입점 제공 |
| 알고리즘 기반 | 인기 + 최신 | 혼잡도 역순 + 감성 매칭 |

**시장 근거**: "어디 갈지 모르겠음"이 여행 계획의 최대 장벽 (한국문화관광연구원 2024 조사, 30대 58%). 감성 진입점은 이 장벽을 제거.

---

### 5.3 오버투어리즘 해소 사회적 가치

**현황**: 제주 3.5배 과잉 방문, 경복궁·명동 인산인해, 인기 명소 주민 갈등 증가.

**LocalSecret V2의 역할**:
- 방문자를 인기 명소 → 비인기 명소로 자연 분산
- 지역 관광 경제 균형 기여 (서울 집중 → 지방 분산)
- 공모전 "서비스 기획력" 항목 사회적 가치 어필 (예상 추가 득점 2-3점)

**B2B 확장**: 관광부처/지자체에 혼잡도 분산 효과 데이터 제공 → 정책 연동 가능성

---

### 5.4 30일 예측 기반 타이밍 알림 — 국내 유일

구글 Popular Times: 주간 평균 패턴 (실시간 아님, 예측 아님)
카카오맵: 현재 혼잡도 일부 (예측 없음)
**LocalSecret V2**: `visitor-concentration-forecast` → **향후 30일 예측** + 푸시 알림

"이번 주 토요일 오전 8시에 가면 집중률 9%입니다 — 지금 계획하세요" → 유일한 기능

---

### 5.5 독점 심리 UX — 발견 기록 아카이브

- 내 비밀 장소가 SNS에 퍼지는 순간 → 앱에서 새 장소 즉시 탐색
- 외부 공유 시 위치 흐림 옵션 → 독점 감정 보호
- "발견자 등급" 게이미피케이션 → 리텐션 극대화

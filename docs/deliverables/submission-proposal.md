# KoreaPath AI 서비스 기획 제안서 (구버전 — 보관용)

> **제출 부문**: 2026 관광데이터 활용 공모전 ① 웹·앱 개발 부문
> **서비스 한 줄 소개**: 여행 조건을 5초 입력하면 혼잡도 예측 + Visit Korea OpenAPI 데이터를 결합해 개인화 코스를 자동 생성하고, 방문 당일 실시간 동선을 재조정해주는 AI 여행 플래너

---

## 1. 서비스 기획 배경 및 필요성

### 기획 배경

한국 관광 시장은 연 3.2억 회의 국내 이동과 1,620만 명(2024년)의 방한 외래객으로 구성되는 대규모 시장입니다. 그러나 현재 여행자들은 세 가지 구조적 문제에 직면해 있습니다.

첫째, **코스 기획 시간 낭비**: 20-40대 여행자의 82%가 네이버 블로그·카카오맵·인스타그램 등 3개 이상 플랫폼을 병행하며 평균 2-3시간을 코스 기획에 소비합니다.

둘째, **혼잡도 정보 부재**: SNS에서 발견한 핫플레이스에 도착하면 이미 3시간 이상의 대기가 발생하는 사례가 반복됩니다. 혼잡도 사전 확인 니즈가 76%에 달하나 실질적인 예측 서비스가 없습니다.

셋째, **외국인 FIT 여행자 정보 단절**: 방한 외래객 중 FIT(자유여행) 비중이 67%에 달하나, Visit Korea 공식 앱은 UX가 구식이고 실시간 AI 기능이 없습니다.

### 서비스 필요성

| 문제 | 기존 서비스 한계 | KoreaPath AI 해결 |
|---|---|---|
| 코스 기획 2-3시간 | 수동 검색 의존 | AI 자동 생성, 40초 내 완성 |
| 혼잡도 예측 불가 | 실시간 데이터 없음 | 시간대별 혼잡도 배지 + 최적 시각 제안 |
| 외국인 정보 단절 | 한국어 전용 UI | 4개 언어 자동 감지 (한·영·중·일) |
| 접근성 코스 없음 | 필터 기능 부재 | 무장애 여행 자동 필터 |

KoreaPath AI는 이 네 가지 문제를 **단일 앱**에서 해결하는 국내 유일의 서비스입니다.

## 2. 서비스 개요

### 2-1. 주요 기능 (MVP)

| # | 기능 | 설명 |
|---|---|---|
| **F1** | AI 코스 자동 생성 | 지역·일수·테마·동반자·예산 입력 → 5초 내 완성 코스 |
| **F2** | 혼잡도 타이밍 추천 | 시간대별 배지 🟢🟡🔴 + 최적 방문 시각 |
| **F3** | 실시간 코스 재조정 | 현장 혼잡도 급증 시 대체 장소 push 알림 |
| **F4** | 4개 언어 지원 | 한·영·중(간체)·일 자동 감지 및 전환 |
| **F5** | 접근성 필터 | 노부모·휠체어·유모차 조건 → 계단·경사 자동 제외 |
| **F6** | 코스 공유·저장 | 카카오톡·인스타 스토리 자동 레이아웃 |

### 2-2. 사용자 시나리오

**시나리오 A — 한국인 솔로 여행자 (25세, 프리랜서)**
```
금요일 22:00 인스타로 강릉 카페 발견
→ 앱 실행 "강릉, 당일, 혼자, 감성카페+포토, 3만원" (40초)
→ 토10:00 안목해변 🟢 → 11:30 강릉시장 → 13:00 경포호 → 16:00 귀가
→ 인스타 코스 카드 공유 → 바이럴 유입
```

**시나리오 B — 외국인 여행자 (대만 23세, 사진작가)**
```
인천공항 도착 → 앱 자동 언어 감지 (中文繁體)
→ "전주, 당일, 포토+문화, 오디오 가이드" 입력
→ 한옥마을 오전 9시 🟢 한산 → 중국어 음성 해설 → Dcard 공유
```

### 2-3. 서비스 차별성

| | KoreaPath AI | 트리플 | 네이버 여행 | Visit Korea 앱 | Google Maps |
|---|:---:|:---:|:---:|:---:|:---:|
| AI 코스 자동 생성 | ✅ | ❌ | ❌ | ❌ | ❌ |
| 혼잡도 예측 | ✅ | ❌ | ❌ | ❌ | 부분 |
| 다국어 (4개+) | ✅ | 부분 | ❌ | ✅ | ✅ |
| 무장애 접근성 필터 | ✅ | ❌ | ❌ | 부분 | ❌ |
| 한국 특화 데이터 | ✅ | ✅ | ✅ | ✅ | ❌ |

**KoreaPath AI는 국내·외국인 여행자 모두를 위한 "AI 자동화 × 혼잡도 × 다국어 × 접근성" 4종 결합 서비스로, 국내 유사 사례가 없습니다.**

## 3. 데이터 활용 방안

### 3-1. 활용 Visit Korea OpenAPI (15개)

| # | API명 | 활용 기능 |
|---|---|---|
| 1 | 국문관광정보 서비스 | 관광지 기본정보 (국내 사용자) |
| 2 | 영문관광정보 서비스 | 영어 사용자 콘텐츠 |
| 3 | 중문관광정보 서비스 | 중국어 사용자 콘텐츠 |
| 4 | 일문관광정보 서비스 | 일본어 사용자 콘텐츠 |
| 5 | 연관관광지 정보 서비스 | AI 코스 자동 생성 핵심 (연관도 기반 순서 결정) |
| 6 | 시군구별 인기관광지 서비스 | 지역별 추천 초기 리스트 |
| 7 | 방문자 집중도 예보 서비스 | 혼잡도 🟢🟡🔴 배지 + 최적 방문 시각 |
| 8 | 관광지역 관광수요밀도 서비스 | 실시간 코스 재조정 트리거 |
| 9 | 무장애여행 서비스 | 접근성 필터 (휠체어·경사도) |
| 10 | 오디오 가이드 서비스 | 다국어 TTS 현장 해설 |
| 11 | 반려동물 동반 여행 서비스 | 펫 동반 코스 옵션 |
| 12 | 캠핑 정보 서비스 (고캠핑) | 야외 숙박 코스 연계 |
| 13 | 숙박 정보 서비스 | 코스 내 숙박 연계 |
| 14 | 음식 서비스 | 맛집 코스 포인트 |
| 15 | 관광지 기본정보 서비스 | 운영시간·입장료·주차 정보 |

### 3-2. 데이터 활용 방식

```
사용자 입력 (지역·테마·동반·예산)
         ↓
[API 5번] 연관관광지 → 후보 관광지 풀 생성
         ↓
[API 7번] 혼잡도 예보 → 시간대별 배지 부여
         ↓
[API 6번] 인기관광지 → 가중치 보정
         ↓
[API 9번] 무장애 필터 (접근성 조건 시)
         ↓
AI 코스 정렬 알고리즘 → 최적 순서 결정
         ↓
[API 1/2/3/4번] 언어별 콘텐츠 조회 → 다국어 화면 렌더링
         ↓
[API 8번] 실시간 재조정 → 현장 혼잡도 push
```

**Visit Korea OpenAPI 15개 활용으로 심사 기준 "데이터 활용 적절성(20점)"에서 경쟁 우위 확보 전략.**

## 4. 서비스 발전 방향

### 4-1. 개발 로드맵

| 단계 | 기간 | 목표 |
|---|---|---|
| **MVP** | 2026 Q2-Q3 | 5개 핵심 API + 4개 언어 + 코스 자동 생성 |
| **Beta** | 2026 Q4 | 예약 연동 + 오디오 가이드 + 1만 MAU |
| **V1.0** | 2027 Q1 | 9개 언어 + 지역 확대 + 10만 MAU |
| **B2B** | 2027 Q4 | 지자체·호텔 API 판매 + ARR 5억 |

### 4-2. 기대 효과

**사용자 관점**
- 코스 기획 시간: 2-3시간 → **40초** (95% 절감)
- 혼잡도 사전 예측으로 현장 대기 시간 50% 감소 (시뮬레이션 검증)
- 외국인 FIT 여행자 언어 장벽 완전 해소

**사회·정책적 관점**
- **오버투어리즘 해소**: 혼잡도 분산 알고리즘으로 유명 명소 집중 완화
- **지방 관광 활성화**: 숨은 명소 연관 추천으로 수도권 외 지역 송객 증대
- **무장애 관광 접근성**: 장애인·고령자 코스 자동화로 관광 격차 해소
- **방한 외국인 만족도 제고**: 한국관광공사 데이터 기반 정확한 다국어 정보

**사업성**
- 3년 MAU 80만, 연매출 30억 목표
- Freemium(35%) + 커머스 커미션(40%) + B2B API(18%) 복합 수익 모델
- LTV/CAC 9.1× (Y2 기준), BEP Month 18 달성 경로

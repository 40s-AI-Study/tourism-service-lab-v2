### [02] 문제 제기 — 왜 지금 이 서비스인가?

> **"북촌에 거주 인구가 줄고 있다"**

| 문제 | 수치 |
|---|---|
| 오버투어리즘 | 북촌 거주 인구 △27.6% (2018→2023) |
| 수도권 쏠림 | 관광 지출 55%+ 수도권·제주 집중 |
| 정보 파편화 | 여행자 평균 5개 앱 사용 |
| 외국인 FIT 공백 | 방한 1,750만 명 중 개별 여행 70%, 통합 가이드 없음 |

**→ 이 4가지는 하나의 순환 고리. TripCraft Korea가 이 순환을 끊는다.**

### [04] DEMO — 30초 시연 (라이브 또는 영상)

### [05] 데이터 활용 — KTO OpenAPI의 심장

> **"공공데이터를 조합한 새로운 알고리즘"**

| API | 기능 연결 |
|---|---|
| 국문관광정보 | 26만 장소 검색 기반 |
| 연관관광지정보 | 코스 동선 클러스터링 |
| 관광지집중률·방문자추이 | 혼잡도 회피 알고리즘 |
| 오디오가이드 | 위치 트리거 자동재생 |
| 지역별 관광다양성 | 소외 지역 가중치 1.5배 |
| + 카카오 4종 | 지도·경로·로그인·간편결제 |

→ **KTO OpenAPI 12종 + 카카오 4종 = 총 16종 실제 연동**

### [07] 핵심 기능 상세 (2) — 오디오 가이드 & 스탬프

### [14] Q&A 대비 — 예상 질문 핵심 답변

**Q1. 실시간 혼잡도 데이터 정확도는?**
→ 방문자 추이 예측 API(빅데이터 통계 기반) 사용. 실시간이 아님을 명시하고 예측 오차 범위 표시.

**Q2. 수익화까지 시간이 얼마나 걸리나?**
→ 구독 전환율 3% 가정 시 Y2(2027년)에 BEP 달성 시뮬레이션 완료.

**Q3. KTO API가 서비스 종료되면?**
→ 자체 데이터 캐시 + 카카오 등 대체 API 병행. 의존도 분산 설계.

**Q4. 기존 트리플/야놀자가 동일 기능을 만들면?**
→ KTO 공공데이터 기반 혼잡도 알고리즘 + 팔도 스탬프 정책 연계는 단기 복제 불가. 데이터 누적 효과가 진입 장벽.

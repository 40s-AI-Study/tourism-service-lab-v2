### 4-2. 최종 제출: TripCraft Korea (4개 통합)

최종 제출 서비스는 단일 아이디어 대신 **4개 아이디어의 핵심 기능을 통합**한 TripCraft Korea를 선택:

| 통합 소스 | 기여 기능 |

| KoreaPath AI | 카테고리 테마 코스 자동 생성, 혼잡도 예측 |
| Audio Story Korea v2 | 위치 기반 오디오 가이드 자동재생, 4계절×5성향 분기 |
| 아이디어 A+ (치유여정 류) | CLIP 유사도 대안 추천, 소외 지역 균형 분산 |

→ **선택 근거**: 개별 아이디어의 강점을 결합해 심사 5개 항목(데이터 활용·기획력·사업성·혁신성·사회적 가치) 전체를 균형있게 커버.

## 5. 잘 된 점 (Keep)

### 5-1. 체계적인 다단계 평가 프로세스
7개 라운드를 통해 아이디어를 지속적으로 검증·고도화. 초기 아이디어부터 최종 통합까지 일관된 평가 기준(150점 매트릭스)을 적용해 주관성을 최소화.

### 5-2. 시뮬레이션 기반 검증
262건의 페르소나 시뮬레이션으로 아이디어의 실제 사용 시나리오를 수치화. "Pass/Fail" 판정으로 공모전 설득력 강화.

### 5-3. 병렬 에이전트 협업
CEO·PM·MR·BA·UX·Sim·API 7개 에이전트가 `maxConcurrentRuns=3`으로 병렬 작업. 20일 만에 27개 프로토타입 + 77개 이슈 완료.

### 5-4. 인터랙티브 프로토타입 완성도
단순 목업이 아닌 실제 클릭 가능한 HTML 프로토타입(27개). 심사위원이 `http://127.0.0.1:3300/docs/prototypes/`에서 즉시 체험 가능.

## 6. 개선할 점 (Improve)

### 6-1. 제출 전략 결정 지연
공모전 마감 D-2(2026-05-04)까지 제출 아이디어가 미확정(KoreaPath AI 초안 vs. LocalSecret v2 1위). Board 의사결정이 늦어져 최종 TripCraft Korea 통합은 D-1~D-Day에 집중됨. **개선안**: Round 5 완료 시점에 제출 전략 결정 체크포인트 설정.

### 6-2. git push 인증 미비
에이전트 환경에 GitHub 인증 수단이 없어 커밋은 로컬에만 존재. 실제 원격 push는 Board가 수동으로 수행해야 했음. **개선안**: 프로젝트 시작 시 에이전트 환경에 GitHub PAT 또는 SSH 키 설정.

### 6-3. PM 산출물 실시간 동기화 부족
service-plan.md, submission-proposal.md 등 PM 핵심 문서가 Round 7 이후 TripCraft Korea로 전환될 때 즉시 업데이트되지 않았음. **개선안**: 제출 서비스 결정 시 PM 문서 업데이트를 CEO 오케스트레이션 STOP POINT에 포함.

### 6-4. 감정·UX 관점 초기 반영 부족
Round 1~2는 API 활용·시장 규모 중심 평가여서 감정 JTBD, UI/UX 차별화 점수가 낮았음. Round 3 이후 6축이 추가됐으나 초기 아이디어들은 소급 환산 방식 사용. **개선안**: Round 1부터 감정 JTBD 관점을 포함한 평가 기준 적용.

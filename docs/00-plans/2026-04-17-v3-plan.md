# 실행 계획 v3 (최종) — 2026-04-17

> v2 계획에 **LLM 중립성(GPT/Claude/로컬 모두 호환)** 요구사항 추가.

## v3 추가 요구사항

Paperclip 장기 운영 전략 변경:
- **주 모델**: OpenAI GPT + 로컬 LLM (opencode)
- **부 모델**: Claude (중요 부분만 선택적)

→ 모든 llm-wiki / AGENTS.md / 지침 / 프롬프트가 **특정 LLM에 종속되지 않도록** 설계 필요.

## LLM 중립 설계 원칙

### 1. 프롬프트 포맷 중립
**금지**:
- Claude 전용 `<thinking>`, `<answer>` XML 태그
- GPT 전용 function calling 스키마 가정
- 특정 모델의 시스템 메시지 형식 가정

**채택**:
- 순수 Markdown 지시문
- 명확한 역할 정의 (role, goal, constraints, output format)
- 예시는 Few-shot 형태로 (자연어)

### 2. 지식베이스 포맷 중립
**모든 llm-wiki / GitHub wiki 페이지는**:
- Markdown + YAML frontmatter
- 표, 리스트, 코드블록 표준 문법만 사용
- 모델별 특수 기능 의존 금지

**frontmatter 예시**:
```yaml
---
title: 페이지 제목
category: api-spec | decision | persona | idea | simulation
tags: [tag1, tag2]
created: 2026-04-17
llm_compatibility: universal  # claude | gpt | local | universal
---
```

### 3. AGENTS.md 중립
각 에이전트 지침은:
- 역할, 작업 절차, 제약을 **LLM-agnostic**하게 기술
- 모델별 최적화가 필요하면 별도 부록으로 분리
- 기본 동작은 GPT-4급 / Claude Sonnet급 / 26B 로컬 LLM 모두 소화 가능한 수준

**AGENTS.md 구조 템플릿**:
```markdown
# {Agent Name}

## Role
명확한 한 문장

## Responsibilities
- 작업 1
- 작업 2

## Workflow (LLM-agnostic)
1. Input 수신
2. 처리 절차
3. Output 생성

## Output Format
JSON/Markdown 스키마 명시

## Constraints
- 제약 1
- 제약 2

## Tools Available
- tool 1 (bash)
- tool 2 (read)

## Model-Specific Notes (optional, 부록)
- Claude 사용 시: ...
- GPT 사용 시: ...
- 로컬 LLM 사용 시: ...
```

### 4. 어댑터 호환성
Paperclip 어댑터별 지원 확인:
- **claude_local**: Claude CLI
- **opencode_local**: OpenAI-compatible (OpenAI API, LM Studio, Ollama)
- **cursor_local**: Cursor (multi-model)
- **codex_local**: Codex
- **gemini_local**: Gemini

각 에이전트 config에 `model` 필드만 바꾸면 즉시 전환 가능하도록 구조화.

### 5. 공통 메타데이터 스키마
모든 산출물(아이디어, 페르소나, 시뮬레이션)은 공통 JSON 스키마:
```json
{
  "type": "idea | persona | simulation | decision",
  "id": "uuid",
  "title": "...",
  "author_agent": "agent-name",
  "author_model": "gpt-4o | claude-opus-4-7 | supergemma4-26b | ...",
  "created_at": "ISO8601",
  "content": { /* 타입별 구조 */ },
  "llm_compatibility": "universal"
}
```

이렇게 하면 **모델 교체 후에도 과거 산출물 해석 가능**.

## OpenAI API 연동 준비 (신규)

### 0-7. OpenAI API 설정
- API 키 저장: `~/.openai/api_key` 또는 env var (`OPENAI_API_KEY`)
- opencode에 OpenAI provider 추가 (`~/.config/opencode/opencode.jsonc`):
  ```jsonc
  "openai": {
    "npm": "@ai-sdk/openai",
    "name": "OpenAI",
    "options": { "apiKey": "{env:OPENAI_API_KEY}" },
    "models": {
      "gpt-4o": { "name": "GPT-4o" },
      "gpt-4o-mini": { "name": "GPT-4o Mini" }
    }
  }
  ```
- LM Studio provider는 **로컬 모델 fallback**으로 유지

### 0-8. 모델 교체 테스트 플랜
회사 창립 전에 3가지 어댑터/모델 조합 핵심 검증:
1. **opencode_local + GPT-4o**: API 호출 + tool use 동작 확인
2. **opencode_local + supergemma4**: 기존 로컬 동작 유지 확인
3. **claude_local + Sonnet**: 기존 Claude 동작 유지 확인

모든 조합에서 같은 AGENTS.md로 동일한 기본 행동이 나오는지 확인.

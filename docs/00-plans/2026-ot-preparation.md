# TripCraft Korea — 예비심사 통과 후 OT 준비 가이드

> **대상**: 2026 관광데이터 활용 공모전 예비심사 OT (2026년 5월 중)  
> **전제**: 예비심사 통과 통보 수신 후 활성화

---

## 1. OT에서 확인해야 할 사항

### 1-1. 개발 기간 관련

- [ ] 서비스 개발 공식 시작일 확인
- [ ] 1차 심사(기능 심사) 정확한 날짜·장소·형식
- [ ] 최종심사(PT) 시간 배정 방식 (발표 순서 추첨 여부)
- [ ] 중간 점검/멘토링 일정 여부

### 1-2. 제출물 관련

- [ ] 1차 심사 제출 형식 (URL 제출? APK? 동영상?)
- [ ] 소스코드 공개 여부 (GitHub public 필수인지)
- [ ] 서비스 운영 서버 요구 사항 (심사 기간 중 접근 가능해야 하는지)
- [ ] 팀 구성 변경 가능 여부

### 1-3. KTO API 관련

- [ ] 공모전 참가팀 전용 API Key 발급 여부
- [ ] API 호출 한도 증량 지원 여부 (기본 일일 한도 제한)
- [ ] `관광지집중률/방문자추이예측` API 실시간 데이터 접근 방법

### 1-4. 멘토링/지원

- [ ] 기술 멘토 배정 여부 (KTO 담당자 or 외부 전문가)
- [ ] 멘토링 채널 (Slack? 이메일? 대면?)
- [ ] 중간 발표/데모 세션 일정

## 3. OT 이후 즉시 할 일 (D+0 ~ D+3)

### D+0 (OT 당일)

- [ ] OT 내용 팀 공유 (주요 일정·규칙·유의사항)
- [ ] KTO API Key 발급 신청 (공모전 참가팀 전용이 있다면)
- [ ] GitHub 레포 `tripcraft-korea-app` 새로 생성
- [ ] ADR-006 상태 `proposed` → `approved` 변경

### D+1 (다음날)

- [ ] Phase 2 Sprint 1 이슈 생성 (Paperclip에 등록)
- [ ] 카카오 개발자 앱 신청 (카카오맵·로그인·모빌리티)
- [ ] 개발 환경 셋업 문서 작성 (`docs/00-plans/dev-setup.md`)

### D+3

- [ ] Sprint 1 킥오프 (5월 말 목표: F1 검색 + F2 코스 생성 기초)
- [ ] 첫 번째 실제 동작하는 코스 생성 화면 데모

## 4. 예비심사 미통과 시 시나리오

OT 없이 탈락 통보를 받는 경우:

1. **결과 수신**: 이메일 또는 접수 사이트 공지 확인
2. **피드백 요청**: gongmo@stunning.kr 에 탈락 사유 문의 (있을 경우)
3. **다음 라운드 검토**:
   - 내년 공모전 출품 준비 (현재 기획 자산 보존)
   - 독립 서비스 출시 가능성 검토 (TripCraft Korea 자체 론칭)
4. **자산 보존**: 현재 레포 private 유지, 아이디어 59개 + 프로토타입 27개 보존

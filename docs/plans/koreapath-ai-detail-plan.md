## 2. 핵심 사용자 시나리오 (5단계 플로우)

> 대표 시나리오: 박지수, 제주도 2박3일 여행 계획

```
단계 1 ── 조건 입력 (30초)
  └─ "제주도" / 출발 2026-05-15(금) / 2박3일 / 커플 2명 / 테마: 자연+맛집
  └─ 선택 옵션: 렌트카 이용 / 예산 1인 15만원/일

단계 2 ── AI 코스 생성 (3-5초)
  └─ 연관 관광지 API → 방문 빈도 높은 스팟 풀 추출
  └─ 혼잡도 예측 API → 날짜별·시간대별 집중률 계산
  └─ 최적 순서 계산 (이동거리 최소화 + 혼잡 회피)
  └─ Day1/2/3 완성 코스 카드 표시

단계 3 ── 코스 확인 & 커스터마이징 (2-5분)
  └─ 각 스팟 카드: 혼잡도 게이지(낮음/보통/높음) + 예상 소요 시간
  └─ "교체" 버튼으로 같은 테마 대안 스팟 제안
  └─ 드래그로 순서 변경

단계 4 ── 저장 & 공유 (1분)
  └─ 코스 저장 → 링크 생성
  └─ 카카오맵 / 네이버지도 내보내기
  └─ 인스타그램 스토리 카드 생성

단계 5 ── 현장 실시간 알림
  └─ 방문 당일 "지금 경복궁 혼잡도 급상승 → 대체 스팟: 창덕궁 제안"
  └─ 다음 스팟 이동 시간 안내 (현재 위치 기반)
```

## 3. Visit Korea OpenAPI 매핑

### 사용 API 목록 (5개)

| # | API명 | 엔드포인트 | 역할 |
|---|-------|-----------|------|
| 1 | 관광지 집중률·방문자 추이 예측 | `tatsCnctrRateList` | 향후 30일 혼잡도 예측 → 최적 방문 시간 계산 |
| 2 | 관광지별 연관 관광지 | `areaBasedList1` | 함께 방문하는 스팟 네트워크 → 코스 후보 생성 |
| 3 | 국문 관광정보 서비스 | `areaBasedList2`, `detailCommon2`, `locationBasedList2` | 관광지 상세 정보, 이미지, 운영시간 |
| 4 | 무장애 여행 정보 | `areaBasedList` (barrier-free) | 페르소나 C 접근성 필터 데이터 |
| 5 | 지역별 관광 수요 강도 | `demandDensityList` | 지역 단위 수요 밀집도 → 지역 추천 우선순위 |

### API 호출 시퀀스

```
[사용자 입력: 지역, 일수, 테마]
        │
        ▼
Step 1. 관광 수요 강도 API
  → 입력 지역의 수요 밀집 지구 파악
  → 고수요 지구 = 코스 앵커 스팟 후보
        │
        ▼
Step 2. 연관 관광지 API
  → 앵커 스팟 기준 연관 스팟 50위 조회
  → 테마 필터 적용 (자연/문화/맛집/액티비티)
  → 코스 후보 풀 30-50개 구성
        │
        ▼
Step 3. 혼잡도 예측 API
  → 후보 스팟별 여행 날짜 혼잡도 조회
  → 시간대별 집중률 계산
  → "낮은 혼잡도" 방문 시간대 도출
        │
        ▼
Step 4. 국문 관광정보 API
  → 최종 선정 스팟 상세 정보 로드
  → 이미지, 주소, 운영시간, 전화번호
  → [페르소나 C] 무장애 API로 접근성 태그 추가
        │
        ▼
Step 5. 코스 최적화 (서버 로직)
  → 이동 거리 최소화 (TSP 근사)
  → 하루 보행 거리 8km 이내 제한
  → Day별 시간표 생성
        │
        ▼
[완성 코스 반환]
```

### 핵심 차별화 3가지

**① AI 완전 자동 코스 생성 (기존 서비스에 없음)**
- 트리플, 마이리얼트립 등 모든 기존 앱은 사용자가 직접 코스를 구성해야 함
- KoreaPath AI는 조건 입력 30초 → 완성 코스 5초 반환
- 연관 관광지 데이터 + 혼잡도 + 이동 거리 3개 변수를 동시에 최적화

**② 공공 빅데이터 기반 혼잡도 예측 (30일 선행)**
- 기존 서비스는 실시간 혼잡도가 없거나 구글 Popular Times 수준
- 한국관광공사 집중률 예측 API → 30일 앞 시간대별 혼잡도 제공
- "언제 가야 덜 붐빌까"를 계획 단계에서 해결 → 현장 실망 방지

**③ 접근성 약자 특화 코스 (무장애 API 연동)**
- 국내 관광 앱 중 무장애 데이터를 코스 생성에 통합한 서비스 전무
- 한국관광공사 무장애 여행 정보 API 연동 → 휠체어·노약자 최적 코스

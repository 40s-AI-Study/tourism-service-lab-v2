# 07. Ideas and Simulations

> **Phase 3 Stage 3-4에서 채워질 예정** — 핵심 섹션

## 구조

### Part A: 아이디어 발산 (Stage 3)
- ProductManager + MarketResearcher + UXDesigner + BusinessAnalyst가 협업
- 10개 서비스 아이디어 도출
- 각 아이디어에 대해 사업성 점수화 (BusinessAnalyst)
- 상위 3개 CEO 선정

### Part B: 시뮬레이션 루프 (Stage 4) ⭐

```
각 후보 서비스 × 각 페르소나
    ↓ Simulator (Claude Sonnet) 수행
"실제 사용자로서 이 앱을 어떻게 사용할 것인가?"
    ↓ 유용성/사용성/구현타당성 평가 (1-10)
    ↓
Pass (4/5 이상 유용) → 다음 페르소나
Fail → PM이 개선 or 다른 아이디어로 이동
    ↓
모든 페르소나 Pass + 사업성 7점 이상 → 최종 후보 확정
```

## 최종 선정 기준

1. 5개 페르소나 중 4개 이상 Pass
2. BusinessAnalyst 사업성 7점 이상
3. APISpecialist 기술 구현 가능성 확인
4. CEO 최종 승인

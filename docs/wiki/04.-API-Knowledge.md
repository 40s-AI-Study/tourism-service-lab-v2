| ID | 언어 | 플래그 |

| `tourism-info-english` | 영문 | 🇬🇧 |

| `tourism-info-chinese-simplified` | 중문 간체 | 🇨🇳 |
| `tourism-info-chinese-traditional` | 중문 번체 | 🇹🇼 |
| `tourism-info-german` | 독어 | 🇩🇪 |

|---|---|---|

## 🎯 공모전 관점 API 우선순위

### 1순위: 필수 기반 (모든 앱)
- `tourism-info-korean` (국문)
- `tourism-info-english` (영문)

### 2순위: 차별화 조합
- `visitor-concentration-forecast` + `related-attractions` = 스마트 코스 생성
- `tourism-big-data` + `area-tourism-demand-density` = 트렌드 대시보드

### 3순위: 니치 타겟
- `pet-friendly-travel` (반려인 20-40대)
- `wellness-tourism` (20-40대 라이프스타일)
- `eco-tourism` (환경 민감 세대)
- `barrier-free-travel` (접근성 특화)

### 가점 전략: 지역 특화
- `central-attractions-by-municipality` + 1개 지자체 데이터
- 대상 지역: 부산, 대전, 광주, 세종, 충남, 경북, 강원, 제주 (+2점, RTO 특별상)

### 외국인 타겟 전략
- 다국어 API 다수 활용 (영/중(간/번)/일 필수)
- `audio-guide` 4개 언어 지원 활용

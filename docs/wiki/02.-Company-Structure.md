## 운영 원칙

### Heartbeat 전략
- **CEO만 1-2시간 주기로 heartbeat ON**
- 나머지 전원 `heartbeat: disabled, wakeOnDemand: true`
- CEO가 필요한 에이전트만 선택적으로 깨움

### 모델 혼합 (Option B)
| 레벨 | 사용 모델 | 역할 |
|---|---|---|
| **핵심** | Claude Sonnet 4-6 | CEO, Simulator (전략적 판단) |
| **일반** | GPT-4o | PM, MarketResearcher, UXDesigner, BusinessAnalyst |
| **저비용** | GPT-4o-mini 또는 로컬 LLM | APISpecialist (단순 데이터 매핑) |

### 예산 관리
- CEO: $80/월 (Claude)
- Simulator: $60/월 (Claude)
- 나머지 5명: $50-100/월 (GPT)
- **총 예산**: ~$150-200/월 상한

### 거버넌스
- 회사 단위 데이터 격리 (`companyId`)
- Chain of command: 모든 에이전트는 CEO에 보고
- 주요 의사결정: ADR로 기록 → `docs/01-decisions/`

## GitHub 연동 규칙

### 레포 clone 위치
`/Users/sklee01/tourism-service-lab-v2` — 모든 에이전트가 이 경로에서 작업

### 커밋 규칙
```
[{agent-name}] {type}: {summary}

상세 설명...

Co-Authored-By: Paperclip <noreply@paperclip.ing>
```

### 라벨링
- `phase-N-xxx`: 단계
- `agent-{name}`: 담당 에이전트
- `api-spec` / `persona` / `idea` / `simulation`: 산출물 유형

## Paperclip ↔ GitHub 동기화

| Paperclip 이벤트 | GitHub 액션 |
|---|---|
| 이슈 생성 | Project 카드 생성 (Backlog 컬럼) |
| 이슈 in_progress | 카드 "In Progress" 이동 |
| 이슈 done | 카드 "Done" 이동 |
| 코멘트 추가 | repo commit (agent-outputs/) |
| 산출물 생성 | knowledge-base/ 또는 deliverables/ 커밋 |

## 데이터 흐름

```
외부 소스 (공모전 PDF, Visit Korea API)
    ↓
Human (저) — Phase 1 직접 수집
    ↓
llm-wiki (.omc/wiki/) + knowledge-base/ + GitHub Wiki (3중 동기화)
    ↓
Paperclip Agents — Phase 3 자율 실행
    ↓
knowledge-base/ (ideas, personas, simulations) + deliverables/
    ↓
최종 공모 제출물
```
